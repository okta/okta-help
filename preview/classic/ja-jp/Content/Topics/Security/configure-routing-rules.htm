ルーティング・ルールの構成

デフォルトのルーティング・ルールでは、OktaがIDプロバイダーとして指定されます。 プロバイダーごと、あるいはユーザー基準のさまざまな組み合わせごとにルールを作成できます。エンドユーザーがサインインを試みると、アクティブなルールが評価されます。そのユーザーと最初に一致したルールが適用されます。

開始する前に

ルーティング・ルールを追加する前に、Okta IWA Webエージェントと少なくとも1つの追加のIDプロバイダーを構成する必要があります（ソーシャルIDプロバイダーを利用できます）。インバウンドSAMLの構成と汎用OpenID Connectをご覧ください。

利用可能なIDプロバイダーのリストと同じページで、Oktaユーザー名とパスワードをエンドユーザーに要求する場合は、Oktaサインオンポリシーが［Password / IDP（パスワード/IDP）］で構成されていることを確認します。この組み合わせは、OktaをIDプロバイダーとして提供するルールに推奨され、デフォルトのルーティング・ルールを優先する場合にも推奨されます。

ChromeBookのIdP Discoveryは、Okta IDプロバイダーと、ChromeBookをサポートすると発表しているサードパーティのIDプロバイダーのみをサポートします。Okta Mobileは、IDプロバイダー検出での使用はサポートされません。

ルールを追加

  1. (missing or bad snippet) ［Identity Providers（IDプロバイダー）］に移動します。
  2. ［Routing Rules（ルーティング・ルール）］タブで、［Add Routing Rule（ルーティング・ルールを追加）］をクリックします。
  3. ルール名を入力します。
  4. ルーティング条件を構成します。
    IF User's IP isゾーンを指定します。少なくとも1つのネットワークゾーンが定義済みである必要があります。ゾーンについて詳しくは、IPゾーンをご覧ください。
    AND User's device platform isモバイルデバイスとデスクトップデバイスの任意の組み合わせを指定します。 iOS デバイスは、iOSルーティング・ルールを迂回する場合があります。 をご覧ください。
    AND User is accessingアプリケーションまたはアプリ・インスタンスを追加します。アプリケーションの名前を入力すると、一致するアプリがすべて表示されます。
    AND User matches

    ユーザーが一致する必要のあるログイン属性を選択します。

    • Anythingは、すべてのユーザーを含みます。
    • Regex on loginでは、ユーザーのログインに基づく有効な正規表現を入力して、これを照合に利用できます。これは、ドメインやユーザー属性を指定するだけでは照合に不十分である場合に便利です。たとえば、.*\+devtest@company.comはドメイン@company.comのログインに一致しますが、@記号の前に+devtestが含まれている場合に限ります。
    • Domain list on loginでは、照合するドメインの一覧を指定します（@ の記号は不要）。たとえば、mytest.comなどです。文字をエスケープする必要はありません（正規表現を使用する場合は必要）。
    • User attributesは、属性名、比較のタイプ、照合する値を指定します。比較タイプでRegexを選択した場合は、有効な正規表現を入力する必要があります。例：OktaユーザースキーマのDepartment属性の(Human Resources|Engineering|Marketing)。
    THEN Use this identity providerすべての条件が満たされた場合に使用するIDプロバイダーを指定します。
  5. ［Create Rule（ルールを作成）］をクリックして、ルールをすぐにアクティベートするかどうかを指定します。

macOS デバイス用のルーティング・ルールの構成

Safariがデバイスユーザーエージェントを報告する方法が変更されたため、Oktaは、macOSから来るアプリリクエストとiPadOSデバイスのSafariから来るアプリリクエストを区別できません。

iPadOSデバイスがiOSポリシーを迂回することを避けるために、macOSデバイスとiPadOSデバイスに適用されるDeny/Catch-Allルーティング・ルールを構成します。iPadOS デバイスのユーザーがmacOS アプリ用のルーティング・ルールの影響を受けないように、ユーザーに次のいずれかを行うように通知します：

  • オプション1：Safariからアクセスされるすべてのウェブサイト（iPadOS 13以降）。iPadの設定で、［Safari settings（Safari設定）］>［Request Desktop Website（デスクトップウェブサイトのリクエスト）］の順に移動して、［All Websites（すべてのウェブサイト）］の設定をオフにします。
  • オプション2：ウェブサイトごと。Safariを開き、検索フィールドの左側にあるAaをタップして、［Request Mobile Website（モバイルウェブサイトのリクエスト）］をタップします。
  • オプション3：ネイティブアプリまたはOkta Mobileを介してターゲットアプリにアクセスします。

ルーティング・ルールの管理

アクティブなルールは、［Routing Rules（ルーティング・ルール）］で表示されている順番で評価されます。そのユーザーと最初に一致したアクティブなルールが適用されます。

追加できるルールの数に制限はありません。ただし、一致を見つけるまでに評価する必要のあるルールの数はルーティングのパフォーマンスに影響します。

  1. ルールの優先度を変更するには、ルール番号の左側の領域にカーソルを合わせます。ドラッグして、目的の順番まで移動します。
  2. ルールのアクティベート、ディアクティベート、編集、消去を行うには、ルール名をクリックしてから、右側のアクションボタンをクリックします。デフォルトのルールは変更できません。
