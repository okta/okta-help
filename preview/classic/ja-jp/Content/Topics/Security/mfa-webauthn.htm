WebAuthn (MFA)

FIDO2ウェブ認証（WebAuthn）は、さまざまなサイトやデバイスをカバーするウェブ上でユーザーをセキュアに認証するために使用されるウェブブラウザおよび関連ウェブプラットホームに組み込まれている標準ウェブAPIです。FIDO2 WebAuthn規格の詳細は、「FIDO2プロジェクト」を参照してください。

FIDO2 (WebAuthn)を多要素認証（MFA）オプションとして構成できます。WebAuthn規格は、WebAuthn専用に有効化および構成されたMFA 要素による認証方法をユーザーに提供します。WebAuthn要素を設定する場合、ユーザーはOktaにサインインするときに追加の確認が必要です。ユーザーは、同一WebAuthn要素を最大10インスタンスまで登録できます。ユーザーはエンドユーザーダッシュボード上のサインイン ウィジェットまたは設定からセットアップできます。

ユーザーがOktaにサインインしてセキュリティーキーまたは生体認証オーセンティケーターを選択すると、Oktaのサインインに成功するためにWebAuthnオーセンティケーターを登録するように求められます。ユーザーは、認証の成功を確実にするために追加の画面上のブラウザの指示またはOSの指示に従います。

注：エンドユーザーがWebAuthnオーセンティケータ－（MacintoshコンピュータおよびiPhonesのTouchID、iPhoneのFaceID、Windows Hello、Androidの指紋またはPIN、その他のデバイス依存のオーセンティケータ－）にのみ登録されている場合、WebAuthnオーセンティケータ－で問題が発生すると、エンドユーザーが自アカウントに対して認証できなくなるリスクがあります。このリスクを緩和するために、Oktaでは管理者がユーザーに非WebAuthn要素をセットアップできるようにし、バックアップとして追加の要素とオーセンティケーターをセットアップすることを勧めることを推奨しています。

  • エンドユーザー用のWebAuthnセキュリティキーの登録
  • エンドユーザーエクスペリエンス

WebAuthn認証のサポート

  • YubiKeysやGoogle Titanなどのセキュリティキーによる認証
  • Windows HelloやApple Touch IDなどの内蔵認証による認証

  • Trusted Origins APIを使用する際の信頼済みクロス オリジンおよびクロス証明書利用者ID（RP ID）検証。組織は、そのOktaまたはカスタムドメインURLと異なる信頼元でホストされているサインインページ用WebAuthnを使用できます。 を参照してください。

    WebAuthnはhttpsプロトコルを必要とします。このユースケースの信頼元を構成するとき、必ずhttps（httpではない）を指定してください。

WebAuthnウェブブラウザのサポート

WebAuthnは、Chrome、Firefox、Edgeブラウザでサポートされていますが、サポートの程度は異なります。これら3つのブラウザはすべてU2Fトークン（Yubico提供のトークンなど）を使用して資格情報の作成およびアサーションをサポートします。デスクトップとモバイルブラウザとの互換性のフルリストは、「ブラウザの互換性」を参照してください。

組み込みウェブブラウザは常にWebAuthnをサポートするとは限りません。

Firefox
  • Windows Hello for Windows 10 build 1903以降をサポート
  • Apple Touch IDは完全サポートではない
  • PIN使用CTAP2はサポートしない
Chrome
  • プラットホームオーセンティケータ－（Touch IDなど）とセキュリティ キーをサポート
  • プラットホームとローミングオーセンティケータ－が登録されてユーザーが使用できる場合、プラットホームオーセンティケータ－がデフォルトで表示される
  • CTAP2オーセンティケータ－でPINがオーセンティケータ－に登録されている場合、CromeはPIIN使用CTAP2をサポート

  • Apple Touch IDをリセットすると、Touch IDで登録された既存のWebAuthnは無効になる

  • Apple Touch IDをディアクティベートすると、Touc IDが再度セットアップされるまで今後Touc IDベースWebAuthn要素のインスタンスの登録が防止される

  • パスワードその他のサインインデータをクリアすると、WevAuthnプラットホームオーセンティケータ－がChromeプロファイルから削除されるOktaの登録は無効にされ、古くなり、有効なオーセンティケータ－に関連付けられなくなるクッキーその他のサイトデータのクリアも同じ結果となる

Safari
  • AppleはmacOS Big Sur でのTouch IDのサポートを追加したが、Oktaはまだこれをサポートしていない
  • PIN使用CTAP2はまだサポートされておらず（2019年10月25日現在）、ユーザー確認なしセキュリティのみ許可
  • WebAuthnダイアログプロンプトなしで、セキュリティキー入力をサイレントに待機
Edge
  • Windowsでプラットホームオーセンティケータ－（Windows Hello）とセキュリティキーをサポート
  • 顔またはPINでWebAuthnに登録すると、他のオーセンティケーター方法（指紋センサーなど）も登録する
  • Windows Helloは顔認証によるロック解除（使用可能な場合）の3分タイムアウト後PIN（使用可能な場合）に切り替える。 PIN用タイムアウトは約5分
Edge Chromium
  • ローミングオーセンティケーターのみサポートTouch IDとWindows Helloはサポートされない
  • Edgeの前の非Chromiumバージョンはローミングとプラットホーム認証を両方サポート
一般
  • セキュリティ キーまたは生体認証オーセンティケーター（WebAuthn）を登録するとき、エンドユーザーはOktoに特定の登録要素についての情報を持たせるようにプロンプトされる。これにより、各WebAuthn要素がエンドユーザーの［Settings（設定）］ページの［Extra Verification（追加の確認）］セクションに表示される
  • Windows 10 OS v 1809以降では、WebAuthnはWindows Hello +セキュリティ キーをEdgeブラウザのみでサポート
  • Windows 10 OS v 1903以降では、WebAuthnはWindows Hello +セキュリティ キーをEdge + Chrome + Firefoxブラウザでサポート
  • Windows全般では、デフォルトユーザー確認値「preferred（優先）」によりどのPIN対応CAP2オーセンティケーターでも何もデバイス上で設定されていなくてもPINを入力する（本質的にはセットアップを強制）
  • その他のオペレーティングシステムでは、「preferred（優先）」設定はオーセンティケーターで設定されている場合にのみPIN入力を強制適用する
  • Yubikey 5以降のみPIN使用CTAPをサポート

  • セキュリティ キーを消去すると、そのセキュリティデバイス、およびTouch ID、Windows HelloなどのプラットホームオーセンティケーターからのOktaでの既存のWebAuthn登録を無効にする

WebAuthn Windows 10のサポート

Windows 10 build 1903の時点で、Windows Hello用Official FIDO2証明書がMicrosoft Edge、Google Chrome、Mozilla Firefoxでのウェブブラウザ用にサポートされています。Windows 10の旧バージョンはWebAuthnの廃止となった実装を使用しており、それはOktaではサポートされていません。

エンドユーザー用のWebAuthnセキュリティキーの登録

お客様はエンドユーザーに代わってWebAuthnセキュリティ キーを登録できます。

  1. ［Profile（プロファイル）］をクリックして、［User attributes（ユーザー属性）］ページを表示します。
  2. ［More Actions（その他のアクション）］の下で［Enroll FIDO2 Security Key（FIDO2セキュリティキーを登録）］をクリックします。

  3. ［Register（登録）］をクリックして、そのキーを登録します。ブラウザまたはデバイスからキーを登録するようにプロンプトされます。

  4. 画面の指示に従います。登録に成功したら、確認メッセージが表示されます。

エンドユーザーエクスペリエンス

管理者がユーザー確認を［Discouraged（非推奨）］に設定して組織を構成すると、WebAuthn要素を登録するエンドユーザーには登録する要素のWebAuthn登録名が見えず、一般名「Authenticator（オーセンティケーター）」として表示され、要素の詳細は提供されません。
