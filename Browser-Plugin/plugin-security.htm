Security features of the

The provides several features to enhance the security of your end users' credentials.

SSL

When you start an -managed app integration that requires the plugin, a pop-up banner asks you if you want to have automatically fill in your credentials. After you accept the prompt request, the plugin uses an SSL connection to pass your credentials. If you have the automatic submission option selected, the plugin signs you in to the application without any further action.

Authentication takes place in the background. The plugin temporarily stores your credentials in a location that the application can't access. The plugin simulates the sign-in process by inserting and submitting your credentials to the sign-in page, but then deletes them after the page redirects. This connection uses either the HTTPS or HTTP protocol, depending on the target URL of the application.

For secure connections, always use the HTTPS protocol when configuring the location of an application.

SSL certificate pinning ()

The for supports SSL pinning to protect against man-in-the-middle (MITM) attacks. A MITM attack attempts to steal user credentials, session identifiers, or other sensitive information.

Using SSL pinning, the for maintains a list of previously validated and trusted server certificates.

When the user goes to an application, the plugin retrieves the site certificate and compares it against the list of trusted server certificates. If the comparison fails, denies the connection to *.okta.com and *.oktapreview.com orgs and prompts the user to contact .

Configure your environment to work with the plugin ()

If your enterprise uses web proxies to perform SSL interception, you need to configure your environment to work with the for .

  1. In the Windows registry editor, go to [HKEY_CURRENT_USER\Software\AppDataLow\Software\Okta\IE Plugin].
  2. Create a DWORD (32-bit) value called SkipCertPinning.
  3. Set the value to 1.

Disable certificate pinning only for scenarios where orgs are using web proxies to intercept SSL traffic. doesn't recommend turning it off for any other scenarios.

Anti-phishing protection

provides anti-phishing protection against non-trusted orgs. The plugin trusts the first org that you access. If you access an unrecognized org, a security warning appears. You're explicitly required to trust the org and give consent before accessing it.

Auto-generate strong passwords

When you create or change passwords for your SWA-based applications, auto-generates a strong password. This password is automatically saved to your account.
