Customize the Active Directory DSSO sign-out page

With Desktop Single Sign-on (DSSO) enabled, users are redirected to the Sign In page when they sign out of their account. When this occurs, DSSO recognizes that a user has landed on the Sign In page and the user is automatically signed back in to Okta. To avoid this, you can configure DSSO to send the user to a page that bypasses DSSO.

  1. In the , go to Settings Customization.
  2. In the Sign-Out Page area, click Edit.
  3. Select Use a custom sign out page.
  4. In the Sign-out page URL field, enter the URL of your tenant, followed by /login/default. For example: https://MyCompany.okta.com/login/default.
  5. Click Save.
