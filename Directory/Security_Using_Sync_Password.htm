Synchronize passwords from to Active Directory

You enable Sync Password to synchronize user passwords from to Active Directory (AD) and to provisioning enabled applications.

With to AD synchronization, the password is pushed to AD. This can be useful if you want to be the final authentication resource and you need to use your AD instance to authenticate access to legacy resources that you can't connect to . To allow to synchronize with AD, the delegated authentication setting for the AD domain must be off. The needs additional permissions to write the new password to AD. All password changes should be initiated in and propagated to AD and users should be prohibited from changing their passwords directly in AD.

These events activate an to AD synchronization:

  • A user updates their password
  • A user recovers their password
  • An administrator initiates an password reset

If an user is pushed to AD after they have activated their account, the AD user object is in a "User must change password at next logon" state. In this scenario, the user must first log onto in order for the password to be pushed from to AD.

Before you begin

To synchronize passwords from to AD and to provisioning enabled applications:

  • You have an AD instance integrated with
  • Users imported or assigned to the AD instance are sourced
  • The service account allows users to reset passwords and forces password change permissions
  • Delegated Authentication is disabled and the is not installed

Synchronize passwords to Active Directory

Push a user's password to AD during initial set up, or whenever the user's password changes.

  1. In the , go to Directory Directory Integrations Active Directory Provisioning .
  2. In the Settings list, click Integration.
  3. Scroll down and clear the Enable delegated authentication to Active Directory check box. This transfers password sourcing from AD to .
  4. Click Save.

  5. Select Create password (recommended).

    Note: This will reset existing passwords and send automated emails to all users assigned to AD to create a new password.

  6. Click Disable AD Authentication.
  7. In the Settings list, click To App, click Edit, scroll to the Sync Password section and select Enable.
  8. Click Save.
