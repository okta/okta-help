Configure to LDAP provisioning settings

After installing and configuring the , you can use this procedure to update your to LDAP provisioning settings as the needs of your org change. The to LDAP provisioning settings define how shares and updates user data on your LDAP instance.

  1. In the , go to Directory Directory Integrations .
  2. Select the LDAP agent from the list of directories.
  3. Click the Provisioning tab and select To App in the Settings list.
  4. Click Edit, and complete the following settings:
    • Create Users: Select Enable to create or link a user in LDAP for members of groups in where LDAP is assigned. If LDAP is the highest priority Profile Source, user profiles are automatically updated based on changes made to user profiles in LDAP.
    • Activation email recipient: Enter the email address to which new LDAP account credentials are sent. The recipient is responsible for distributing the credential information to the appropriate user.
    • RDN attribute name: Select the attribute type to be used for user Relative Distinguished Name (the leftmost portion of the user Distinguished Name). You can customize the attribute value on the Profile Editor page.

      If you set the RDN attribute to UID, you must map the attribute to the userName attribute. The attribute type you select must be mapped correctly in Profile Editor for this LDAP instance, in the same direction for provisioning.

    • Update User Attributes: Select Enable to allow to update a user's attributes in LDAP when the app is assigned. Future attribute changes made to the user profile automatically overwrite the corresponding attribute value in LDAP.
    • Deactivate Users: Select Enable if you want to deactivate a user's LDAP account when it is unassigned in or their account is deactivated. Accounts can be reactivated if the app is reassigned to the user in .
    • Sync Password: Select Enable if you want each user's LDAP password to be synced to their password. Any subsequent changes to a user's password are pushed to the on-premise LDAP server. To enable this option, Delegated Authentication must be disabled. Organizations using both Active Directory (AD) and LDAP can now synchronize their user passwords from AD through to LDAP.
  5. Click Save.
  6. Optional. Map LDAP attributes to attributes in the Attribute Mappings section. The attributes listed in the table are your LDAP attributes. To edit these mappings, click the edit icon. See Work with profiles and attributes.
