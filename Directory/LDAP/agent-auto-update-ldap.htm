Automatically update LDAP agents

Agent auto-update functionality keeps your agents up to date and helps ensure your org has the latest features. You can auto-update single or multiple agents on demand, or you can schedule the updates to occur outside of business hours to reduce downtime and disruption to your users.

With agent auto-updates, you don't need to uninstall and then reinstall agents whenever a new agent is released. Instead, you select the agents you want to update and the installer removes and then installs the latest version of the agents.

To use agent auto-update functionality, you must download and install LDAP version 5.16.0 or later. See Download the latest agent version.

Upon agent installation of agent version 5.16.0 and above on Linux platforms, we now grant the OktaLDAPService user sudo permission to automatically install the newest agent version when using the auto-update feature.
