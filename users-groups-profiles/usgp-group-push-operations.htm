Group Push operations

Use Group Push and Group Link to sync or un-sync groups between and third-party target apps. This is done by either pushing groups to target apps (group push) or by using Group Linking to import groups from target apps and linking them to . The following table lists the supported operations and how they appear in .

Operation Description
Create a mapping for a new group (group push) Group A is a group created in and does not exist in the target app. When Group A is pushed from to the target app, an instance of the group is created in the app.
Create a mapping for an existing group (group linking) Group B is a group created in the target app and then imported to . Group A is created in to link to Group B. Both groups appear on the Groups page. Once linked, Group B is sourced. Group B does not appear on the Groups page.
Unlink and delete a group in the target app (group push) Group A is a group that exists both in and the target app. When Group A is unlinked from the target app, it still remains in , but is deleted in the target app.
Unlink and leave a group in the target app (group linking) Group B is a group created in the target app, then imported to . Group A is created in to link to Group B. When they are unlinked, all memberships and references in Group A are deleted in . Group B appears like any other imported group
