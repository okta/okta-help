Enable the self-service registration widget

Whether you are using the Okta-hosted Sign-in page (URL) or hosting your own Sign-in widget (Dev URL), you need to add the following configuration parameters directly under var config = OktaUtil.getSignInWidgetConfig(); in the Sign-In widget to enable the Sign up link and registration form:

config['features.registration'] = true; config['authScheme'] = 'SESSION';

To enable the Custom Sign In HTML editor, you must have a custom URL domain configured.

If your org is using the Okta-hosted Sign-in page, this procedure shows how to enable registration in the widget. The configuration parameters and specified location within the widget HTML are the same in both hosting models.

  1. In the , go to Settings Customization .
  2. Click the Custom Sign In tab.
  3. In the HTML editor, add the following strings directly under var config = {{{config}}};
config['features.registration'] = true; config['authScheme'] = 'SESSION';

  1. Click Save and Publish.
