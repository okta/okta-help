Override an app username

Change the default app username mapping to meet the specific requirements of your org. You initially define the default username format on the app Sign-on or Provisioning tabs of the app.

If you attempt to change the default username and User sets username and password is selected on the Sign On page, an error occurs.

For Active Directory, the username is referred to as common name (cn).

  2. Select Apps in the Filters list.
  3. Click Mappings for the app, and then select Configure User mappings if a list appears.
  4. Select the [app name] to Okta User tab.
  5. Click Use default username setting for Okta user.
  6. Click Override with mapping.
  7. Select an attribute from the dropdown list, or enter an expression to create the app username.
  8. Optional. Click to set the push frequency for the attribute.
  9. Click Save Mappings and then Apply updates now.
