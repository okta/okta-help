Override application attribute mapping

You can override application attribute mappings when assigning apps to individuals or groups. See Add custom attributes to apps, directories, and identity providers.

This functionality lets you customize or define application entitlements at several levels:

  • Mapping: The default source for an attribute value.

  • Administrator Override: Where an administrator can override the default mapping by editing a user's assignment profile. This can be reset to the default mapping by clicking the Reset link that appears under overridden attributes.

  • Group: Where the attribute value is set in the group app assignment configuration. This mapping can only be overridden by converting from a group assignment to an individual assignment.

Override attribute mapping at the user level

  1. In the , go to Applications Applications .
  2. Select an application.
  3. Click the Assignments tab.
  4. Click the Edit icon for a user.
  5. Edit the necessary attribute mappings using the text boxes.
  6. Click Save.

When changes to one or more attribute mappings have been made but not applied, you may see a Re-apply mapping button at the bottom of the application user's profile. Click this button to update the attribute values in question to those determined by mappings.

Reset application attribute mapping to a default value

  1. In the , go to Applications Applications .
  2. Select an application.
  3. Click the Assignments tab.
  4. Click the Edit icon for a user.
  5. Click Reset for the application attribute mapping value you want to reset.
  6. Click Save.

View default group expressions

  1. In the , go to Applications Applications .
  2. Select an application.
  3. Click the Assignments tab.
  4. Click the Edit icon for a user.

The group attributes are listed in the Public Groups area.
