Learn about app integrations

App integrations are configured connections between and external applications.

These app integrations are either pre-built, in the case of those available in the Okta Integration Network (), or custom-made by admins or developers using the App Integration Wizard (AIW). After configuring an integration, admins can assign it to groups and users in an org. Admins can add the app integrations to the Okta End-User Dashboard for all appropriate users.

Single sign-on (SSO)

Single sign-on enables your users to sign on to multiple systems using a single set of authentication credentials. Users can click an Okta tile and be connected automatically to that external application. See About Single Sign-On.

The four main SSO protocols supported by Okta:

  • OpenID Connect (OIDC). See About OIDC app integrations.
  • Security Authentication Markup Language (SAML). See About SAML app integrations.
  • Secure Web Authentication (SWA). See About SWA app integrations.
  • WS-Federation (WS-Fed). See About WS-Fed app integrations.

Provisioning

Some app integrations also support the provisioning features of Okta Lifecycle Management. The protocol used for the communication between Okta and external applications is the System for Cross-domain Identity Management (SCIM) protocol. See About provisioning app integrations.
