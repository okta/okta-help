Use a noun phrase when providing a description of a product, component, feature, or approach (for example, "Passwords and locking"). Avoid using gerund phrases in titles.

<short description> Provide a short description that provides just enough information so that readers can determine whether or not they need to read the topic itself. Provide the goal that a user can obtain from the topic. Use no more than 50 words.

<note or tip> Include an admonition at the beginning of the topic. If the user must first read the information in the topic to understand the meaning of the tip, include it at the end of the topic instead.

<description> Include a description of the product, feature, approach, or series of tasks. Include as many paragraphs as necessary.

<items and descriptions in an unordered list or jump list> (optional) If you have multiple subjects or subtopics that you want to include within one concept topic, you can include sections to represent an organizational division in the topic.

Include the item in bold, followed by a colon and the description. Use parallel structure and full sentences for the descriptions. Include end punctuation after the description.

<subtopics or subjects with descriptions (sections)> If you have multiple subjects or subtopics that you want to include within one concept topic, you can include sections to represent an organizational division in the topic.
