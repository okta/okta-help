Redirect unassigned users to a custom error page

If end users attempt to access an app integration to which they aren't assigned, they can be automatically redirected to the default Okta URL or a custom URL.

Unassigned users are more likely to try to access app integrations if you use embedded app links in your portal or on other sites outside of Okta.

  2. Select the app integration where you want to add the redirect.
  3. Click the General tab.
  4. Click Edit in the App Embed Link section.
  5. Select one of the following in the Application Access Error Page section:
    • Use the error page setting on the global settings page: Redirects unassigned users to the same default URL specified in the org-level (global) setting. This URL is set through the Settings Customization Custom Error Pages . See Configure a custom error page.
    • Use a custom error page for this application: Redirects unassigned users of this app to a custom Error page URL. This option is specific to the app integration and overrides any default org-level setting for a custom error page.
  6. Click Save.
