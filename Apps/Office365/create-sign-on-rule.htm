Create Office 365 sign on rules

This topic explains how to create sign on rules for the Office 365 app sign on policy.

This topic explains how to create sign on rules for the Office 365 authentication policy.

Before you begin

Review the following docs:

  • Office 365 sign on rules options
  • Office 365 default sign on rules
  • Add an app sign-on policy rule

Start this task

In the Okta Admin Console,

  1. Go to Applications Applications .
  2. Click the app for which you want to create a sign-on policy.
  3. Click the app for which you want to create a policy.
  4. Click the Sign On tab and scroll down to the Sign On Policy section.
  5. Click the Sign On tab and scroll down to the User authentication section.
  6. Click View policy details.

  7. Click Add Rule.

    The App Sign On Rule window pops up.

  8. Give the rule a descriptive name.
  9. Configure the conditions under which the rule will apply and the actions to be taken when the rule applies. See Office 365 sign on rules options.
  11. Repeat steps 4-7 for each new rule.
  12. Repeat steps 5-8 for each new rule.

Prioritize rules

You can determine rule precedence by setting the priority number. For example, a rule with a priority value of 1 has first priority and takes precedence over all other rules. To set the priority of a rule, move it up or down by clicking the blue arrow next to it.

Manage rules

  • Edit a rule: Click the pencil icon.
  • Disable a rule: Click the pencil icon to open the rule. Then select the Disable Rule option under Rule Name.
  • Delete a rule: Click the X icon.
