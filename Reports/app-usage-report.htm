Application Usage report

Prerequisites

  • Ensure that you are signed in to the Okta Admin Console.

  • This report can be run by anyone with one of the following permission levels:

    • Super Administrator

    • Org Administrator

    • Read-Only Administrator

    • Mobile Administrator

    • Report Administrator

Parameters

The Application Usage report can be filtered using any of the following parameters:

  • Start date

  • End date

  • App integration

  • User

  • Group

Procedure

To create the usage report for app integrations during a time period:

  1. From the Admin Console, navigate to Reports Reports .

  2. Click Application Usage.

  3. By default, the report runs using:

    • The previous month, up to and including today's date

    • The first alphabetically sorted app integration in your org

    • All users and groups

  4. Change any of the report parameters in the top selection drop down boxes to filter the report for your specific needs.

    Events are retained by Okta for 90 days, so the earliest available date range is 3 months prior.

  5. Click Run Report to see the result on the report page. You can also download the report as a comma separated value (CSV) file by clicking Download CSV. When the application username is null for the app integration, the report uses a blank character for the entry in the CSV file.

  6. If you select All (for all users and groups) in the app integration drop down box, you can only filter the results by group and the report is only available as a CSV download.

Asynchronous Application Usage

If you activate the Asynchronous Application Usage feature, the procedure to create the usage report is simplified and runs in the background. The process is the same as above; however, instead of clicking Run Report, click Request Report.

Okta generates the report in a background task and sends an email with a link to download the report as a CSV file.

Results

The report page shows the following information:

Field name Field description

Person

The user's name in Okta

Application username

The user's name in the app integration

Logins

Number of logins for that user

Last login

The last time the user signed in to the specified app integration.

The CSV report includes:

Field name Field description

Person

The user's name in Okta

Login

The user's Okta sign in credentials

Application username

The user's name in the app integration

Logins

Number of logins for that user. Included only when a specific app integration is selected

Instance name

The name of the app integration. Included only when All (for all users and groups) is selected

Login Date

The last time the user signed in to the specified app integration.

Login Date_ISO8601

The last time the user signed in to the specified app integration, in ISO 8601 format
