Telephony usage report

The Telephony usage report contains data about your org’s telephony events within a specified period. You can filter the report by voice or SMS events, which helps you identify usage trends over time.

Before you begin

Ensure that:

  • You’re signed in to the Admin Console.

  • You’re signed in as a super admin, org admin, mobile admin, or reports admin.

Parameters

You can filter the Telephony usage report using the following parameters:

  • Event period (last hour, last 24 hours, last 7 days, last 30 days, last 60 days, or last 90 days)

    If your org doesn’t have historical data for the entirety of the selected date range, the report output will only show the dates for which data is available.

  • Telephony type (voice or SMS)

  • Country

Get report

  2. In the Telephony section click Usage report.

  3. Enter an event period to filter the report. Okta retains events for 90 days, so the earliest available date range is 3 months prior.

    Okta recommends applying only one event period filter to the report.

  4. Optional. Specify a telephony type (Voice or SMS) to filter the report.

  5. If you want a detailed comma-separated file (CSV) file of the report, click Export CSV.

Results

The generated report includes aa Telephony requests over time graph that displays the following information:

Field name Field description
Total requests The total number of requests for the selected event period.
Resend rate The percentage of all SMS messages that resulted in a resend request. An increase in the resend rate can indicate a deliverability issue.
SMS requests The number of SMS sent by Okta users.
SMS deliverability rate The percentage of SMS messages that were successfully delivered to their destination.
Voice requests The number of voice requests sent by Okta users.

The report includes a Requests by deliverability stage table that displays the following information:

Field name Field description
Stage The status of each authentication event (user requested, sent to provider, delivered to user, and blocked).
Enrollments The number of new SMS enrollments that were requested, sent, delivered, and blocked.
Sign-ins The number of sign-in messages that were requested, sent, delivered, and blocked.
Password resets The number of password resets that were requested, sent, delivered, and blocked.
Account unlocks The number of account unlock messages that were requested, sent, delivered, and blocked.

Total

The total number of requested, sent, delivered, and blocked messages.

The report also includes a Requests sent for delivery by destination country table that displays the following information:

Field name Field description
Destination country The geographic destination of each delivered message.
Enrollments The number of new SMS enrollments.
Sign-ins The number of user sign-ins.
Password resets The number of password resets (requested, sent, delivered, and blocked).
Account unlocks The number of user accounts that were unlocked.

Total

The total number of enrollment, sign-in, password reset, and account unlock messages.
