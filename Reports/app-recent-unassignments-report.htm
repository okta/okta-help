Recent Unassignments report

Prerequisites

  • Ensure that you are signed in to the Okta Admin Console.

  • This report can be run by anyone with one of the following permission levels:

    • Super Administrator

    • Org Administrator

    • Read-Only Administrator

    • Mobile Administrator

    • Report Administrator

Parameters

The Recent Unassignments report can be filtered using any of the following parameters:

  • App integration

  • Start date

  • End date

Procedure

  1. From the Admin Console, navigate to Reports Reports .

  2. Under the Application Access Audit panel, click Recent Unassignments.

  3. Specify an app integration that you want the report to run against.

  4. Specify a date range to filter the report.

    Events are retained by Okta for 90 days, so the earliest available date range is 3 months prior.

  5. Click Run Report.

  6. If you want a comma separated value (CSV) copy of the report, click Download CSV.

Results

The generated report contains the following fields:

Field name Field description

Name

Display name of the unassigned user

Username

The user's name in their Okta profile

Unassigned On

Timestamp for when the user was unassigned from the app integration

Assignment Status

Displays the current assignment status for the user in the app integration

The CSV report also includes:

Field name Field description

fullName

The user's name in their Okta profile

assigneeOktaUsername

The user's user name in Okta

assigneeAppUsername

The user's user name in the app integration

assigneeOktaId

Unique Okta ID for the user

unassigned

Timestamp when the user was unassigned from the app integration, in ISO 8601 format

appUserStatus

Current assignment status for the user in the app integration
