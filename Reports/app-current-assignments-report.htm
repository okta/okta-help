Current Assignments report

Prerequisites

  • Ensure that you are signed in to the Okta Admin Console.

  • This report can be run by anyone with one of the following permission levels:

    • Super Administrator

    • Org Administrator

    • Read-Only Administrator

    • Mobile Administrator

    • Report Administrator

Parameters

The Current Assignments report can be filtered using any of the following parameters:

  • Start date

  • End date

  • User

  • Application

  • Group

Procedure

  1. From the Admin Console, navigate to Reports Reports .

  2. Under the Application Access Audit panel, click Current Assignments.

  3. By default, the report runs using:

    • Today's date

    • All app integrations in your org

    • All users and groups

  4. Change any of the report parameters in the top selection drop down boxes to filter the report for your specific needs.

    Events are retained by Okta for 90 days, so the earliest available date range is 3 months prior.

  5. Click Request Report. The report will be compiled and a link to the CSV file will be emailed to you. The link in the email is valid for 30 days.

Results

The generated report contains the following fields:

Field name Field description
User The user's Okta name
Login The user name that the user uses to sign in to Okta
Application name The name of the app integration
Application instance The instance name of the app integration
Application login The user name that the user uses for single sign on to the app integration
Assigned on Timestamp of when the user was assigned to the app integration, in ISO 8601 format
Unassigned on Timestamp of when the user was unassigned from the app integration, in ISO 8601 format
Last login Timestamp of when the user last signed in to the app integration, in ISO 8601 format
Last password change Timestamp of when the user last changed their password for the app integration, in ISO 8601 format
