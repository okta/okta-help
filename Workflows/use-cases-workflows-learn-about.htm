Okta Workflows use cases

You can use Workflows features across many use cases, including user identity creation, provisioning and deprovisioning, identity conflicts, and lifecycle management. These are the only use cases that are currently supported by Okta Workflows, and they are subject to system limits.

  • Provision and deprovision
  • Change time- and context-based identity entitlement
  • Resolve identity conflicts
  • Report lifecycle activity
  • Send notifications for lifecycle activity
  • Sync customer identities with CRM and marketing tools
  • Send custom end-user notifications
  • Perform identity proofing with a third-party service
  • Take action on end user identities based on a time-based context

  • Manage data privacy requests

  • Log, reference, and sync user consent data in Okta and third-party tools
