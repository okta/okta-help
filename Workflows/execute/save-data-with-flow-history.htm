Save flow data

If you enable the Save all data that passes through the Flow feature, starts saving the flow data each time the flow executes. This data retention is useful when you need to check prior executions or to debug flow errors.

retains data for each execution for 30 days. However, you can delete all data from the execution history at any time. See Clear all saved data.

Use caution when working with flows that contain sensitive data.

Saving data

Clear saved data

You can delete all execution data collected for your flow. The Execution History page still displays the list of flow executions, showing the success or failure and the date and time of each execution. However, the data that passed through the flow is no longer available.

Helper flow behavior

The following section is review-only state (not published to the live site) until OKTA-584217 is enabled. Helper flow behavior is expected to change so the following examples likely need to be updated.

The Save all data that passes through the Flow option of a parent flow determines the setting for any helper flows called by the parent flow.

The parent flow setting overrides the settings for all helper flows, including any helper flows called by another helper flow.

Examples

Example 1

  • Flow A has the saving data feature turned on.

  • Flow B has the saving data feature turned off.

  • When the flow executes, Flow A calls Flow B.

  • Result: Flow Histories for both A and B are saved because A is the root flow for the execution.

Example 2

  • Flow F has the saving data feature turned off.

  • Flow G has the saving data feature turned on.

  • The flow executes with Flow F calling Flow G.

  • Result: Flow Histories aren't saved for either flow, because F is the root flow for the execution.

Example 3

  • Flow J has the saving data feature turned on.

  • Flow K has the saving data feature turned off.

  • Flow L has the saving data feature turned on.

  • The flow executes with Flow J calling Flow K, which in turn calls Flow L.

  • Result: Flow Histories are saved for all three flows, because J is the root flow for the execution.
