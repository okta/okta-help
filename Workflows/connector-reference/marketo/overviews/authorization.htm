Authorization

When you add a card to a flow for the first time, you’ll be prompted to configure the connection. This will enable you to connect your Marketo account, save your account information, and reuse the connection in new flows that include Marketo.

Tip

You can create multiple connections and manage them from your Connections page.

To create a new connection from an Event or Action card:

  1. Click New Connection.
  2. Enter a Connection Nickname. This is useful if you plan to create multiple connections to share with your team.
  3. Enter your Endpoint. For example, if the endpoint is https://xxx-xxx-xxx.marketo.com/rest, use https://xxx-xxx-xxx.marketo.com.
  4. Enter your Client ID.
  5. Enter your Client Secret.
  6. Click Create.

To use the new connector, you'll need to add your account to the connector. Existing Marketo accounts will not be transferred to Marketo (New). The new Marketo connector uses the same authentication as the old connector, so follow the same steps you would normally to authenticate.

  1. Add a card from the (New) connector to your flow.
  2. Navigate to the Admin panel within . Note: You must be an administrator with API access to find the information needed to authenticate Marketo in Okta Workflows.
  3. Find your endpoint:
    1. Click Web Services.
    2. Scroll down to the REST API section and copy the Endpoint URL. NOTE: If your Endpoint is https://xxx-xxx-xxx.marketo.com/rest, you will enter everything through the .com into Okta Workflows, not the /rest. So your Endpoint for Okta Workflows will be https://xxx-xxx-xxx.marketo.com.

    3. Copy and paste the Endpoint URL into Okta Workflows.

  4. Find your Client ID and Secret:

    1. Navigate to LaunchPoint.
    2. Look for the installed service called Okta Workflows. If you don't have one, follow these steps:
      1. Click New > New Service.
      2. Name this service Okta Workflows.
      3. Select Custom from the Service dropdown.
      4. Add a description that will help you remember this service later.
      5. Select the API only user you wish to give access to this account.
      6. Select Create.
      7. Select View Details next to the Okta Workflows service.
  5. Copy and paste your Client ID and Secret into Okta Workflows
  6. Click Next to save your configuration.
