Authorization

Authorize your account

Adding an connection requires you to first register your app with using the . You need to have a machine-to-machine application for each tenant. See Register Machine-to-Machine Applications.

  1. In your , go to Dashboard Applications Applications and then click Create Application.

  2. Enter a descriptive name for your application.

  3. Select Machine to Machine Applications, and click Create.

  4. For the Authorize Machine to Machine Application dialog, select the Management API from the menu. Because your application only accesses the management API, you don't need to create an API.

  5. Each machine-to-machine app that accesses an API has a set of scopes granted by the authenticated user. Select the Permissions that you want for your application's access token. The minimum required scopes are:

    • create:users

    • read:users

    • read:user_idp_tokens

    • read:connections

    • update:users

    • update:users_app_metadata

    • delete:users

    You can add other scopes later for this application through the .

  6. Click Authorize.

  7. On the Settings tab, copy the Client ID and the Client Secret values from the Basic Information for your application.

  8. Click Settings on the left panel, and on the General tab, copy the Tenant Name and Region information. You need this information to create the connection in .

Return to the Workflows console to create the application connection.

Create a connection

After you have registered and configured your application, you can create the connection from within .

  1. Open the Workflows console, and click the Connections tab.

  2. Click New Connection.

  3. Click the icon.

  4. Enter a Connection Nickname. Unique names are useful if you plan to create multiple connections to share with your team.

  5. Enter the Client ID and Client Secret values that you copied from the application.

  6. For the Region option, select the region where your tenant is located. You can find this information from the when you click your org name in the header bar.

    If your tenant uses a custom domain, select Private Cloud.

  7. Enter your Organization. This is the domain that you chose for your tenant. See Create tenants.

    If you're using a custom domain, enter the full base URL into this field (without the https:// portion of the URL). For example, your_tenant.your_custom_domain.com.

  8. Click Create.

You can now select the configured connection whenever you add an action card to your workflow.
