Authorization

Authorize this connector by creating a connection to your (ASA) account. You can reuse this connection the next time that you build a flow with this connector.

Prerequisites

  • An tenant with enabled.

  • administrator credentials.

  • ASA administrator credentials.

  • Create an ASA team. See Advanced Server Access.

  • A service user account with the Admin role granted. uses this account to create connections for the ASA connector. To create a service user account with Admin permissions, see Service users.

  • The service user account must be added to an ASA group with the Admin role assigned. See Team roles and Add a user to a group.

Procedure

To create a connection from an event or action card:

  1. On the Connections page of the console, click New Connection.

    Optionally, you can create a connection from any action card for the ASA connector.

  2. Enter a value in the Connection Nickname field. This is useful if you plan to create multiple connections to share with your team.

  3. In the Service Account API Key ID field, enter the value from the service user account that you already created.
  4. In the Service Account API Key field, enter the value from the service user account that you already created.
  5. In the ASA Base URL field, enter https://app.scaleft.com.

    6. Use this field for future releases of the ASA connector that support per-team DNS names.

  1. In the ASA Team Name field, enter the value for the team in which you created the service user account.
  2. Click Create.

    3. The Advanced Server Access dialog appears.

  3. Select a team.

    4. The Grant Permissions dialog appears.

  4. Enter a name for the service user created for this application.
  5. Click Approve.

    6. The connection appears in the list of Workflows connections.

  6. Verify that the credentials that you've provided have successfully created a connection to your ASA environment.
