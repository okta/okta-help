Guidance for connector

The connector in Workflows allows you to create custom solutions without the need to customize your ASA deployment.

  • Capabilities of the Advanced Server Access connector

  • Test action cards

Capabilities of the connector

You can perform CRUD operations on the following:

  • Projects

  • Teams

  • ASA Users and Groups

  • Pre-Authorizations

  • Enrollment Tokens

  • Servers

Additionally, for API operations that are not directly supported by the pre-built cards, you can use the Custom API Action card to invoke arbitrary API calls against the API.

Your use of the connector and its action cards assumes a basic understanding of terminology, components, API, and interface. Developer documentation is an essential resource to use when developing flows to understand the various actions and properties of the cards. For the API, see Introduction to the Advanced Server Access API.

Test action cards

Test a flow using action cards from the connector.

  1. Create an ASA Connection object associated with your ASA team.

    2. It is recommended that you use a sandbox, test, or non-production ASA team for this connection.

  2. Use the Create Project action card to create a project that will be used to test the connector itself.

  3. In the ASA dashboard, verify that the project was created successfully.

  4. Optionally, use the Create ASA Group action card to create a local ASA group.

  5. Optionally, use the Add User to Group action card to add an ASA user to the new group.

  6. Use the Add Group to Project action card to ad ann ASA group to a project.

  7. Optionally, configure the group to grant group members access to servers in the project.

  8. In ASA dashboard, navigate to Project > Users and verify that the users who are in the group are listed in the project.

  9. Optionally, use the Create Server Enrollment Token for Project action card to create a server enrollment token for the same project.

  10. Install the ASA agent on a server using the same token.

  11. Verify that users in the group that are assigned to the project can access the server.

Known issues and limitations

The following known limitations to the connector that may impact your flows.

  • The connector does not support all API endpoints.

  • Note the difference in behavior in updates to the connector and connections in Workflows. ASA administrators are responsible for managing the service user account that is used by the connector to authenticate to ASA.
