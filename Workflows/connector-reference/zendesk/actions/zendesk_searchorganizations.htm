Search Organizations

Search for organizations in Zendesk.

Options

Field Definition Type Required
Result Set Choose one of the following options to display your search results:

  • First Matching Org: returns a single record.

  • First 1000 Matching Orgs: returns the first 1000 matching records, in list format.

  • Stream Matching Orgs: pass all matching records from your parent flow to a helper flow; a Streaming input section will be added to the card from which you can select a helper flow for streaming and adding custom extensible fields. No outputs are displayed when this option is selected.

 Dropdown TRUE

When searching against Date & Time fields, the following format is permitted: YYYY-MM-DD.

To learn how to return a large number of records, see Set up the stream matching option with a helper flow.

Steps to use the Stream Matching Orgs option

To use the streaming feature of this card to pass data to a helper flow, perform the following steps:

  1. Select Stream Matching Orgs and click Save.

  2. Click Choose Flow and either choose an existing flow or create a flow.

  3. In the helper flow, create keys for Record and State. Set the type for both output field to Object.

  4. Optionally, if you will not pass any additional data into the helper flow using custom extensible fields in the card's Streaming section, omit the State output field.

    Additionally, if you don't adhere to the naming convention of Record and State for the input fields, a failure will occur when you add any further fields and pass data to the helper flow. For example, input fields to this helper flow with names of x and y (or any other name) will cause an error. However, you can add a single field as an input (for example, x) to the helper flow that doesn't adhere to the naming convention, and all your data will be passed through that field as a key.

Limit Set of user defined properties returned for all matching records.
  • If Limit is set to 0, no records are returned.
  • If Limit is set to greater than 0, records greater than 0 and up the maximum number of records specified are returned.
  • If Limit is empty or not selected, all records are returned.
