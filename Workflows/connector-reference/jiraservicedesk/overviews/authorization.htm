Authorization

Before connecting to Okta Workflows, make sure the Accept Remote API calls option is enabled in the System Admin screen.

For information about configuring API Tokens, see Manage API tokens for your Atlassian account.

Tip

You can create multiple connections and manage them from your Connections page.

To create a new connection from an Event or Action card:

  1. Click New Connection.

  2. Enter an Connection Nickname. Make this nickname unique so you can authorize multiple accounts.

  3. Enter a Subdomain for your instance.

  4. Enter your Username and API Token.

    To create and manage API tokens for your instance, go to the API Tokens tab from the Your profile and settings menu > Account Settings page.

  5. Click Create to save your configuration.

If you revoke or delete your API Token, you must create a new token and update all of your connections to use the new token. See View the Connections page. Jira automatically locks you out of the API if four calls are made with old or incorrect passwords/tokens. If any active flows attempt to use your old token, all calls will fail (including those that use the correct credentials). You'll have to change your password to regain API access.
