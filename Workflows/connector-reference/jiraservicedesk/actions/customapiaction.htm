Custom API Action

Make a custom, authenticated HTTP call to the API.

This action is unlike other cards. See API documentation for more information on using custom HTTP calls.

Options

Input

Field Definition Type Required
Request
Relative URL

URL address on the web server to which you are attempting to interact. Specify the relative URL as /{insert_remaining_URL}. You can specify query parameters in the relative URL using "?", or specify the query parameters as an object key pair in the Query input.

For example, for the API https://{subdomain}.atlassian.net/rest/api/2/issue/{Issue ID}/transitions, the relative URL is /rest/api/2/issue/{Issue ID}/transitions.

For a webhook, for example, if the full address is https://{subdomain}.atlassian.net/rest/webhooks/1.0/webhook, the relative URL is /webhooks/1.0/webhook.

 String TRUE
Query Specify any additional query parameters that should be included in object format (key value pairs).

Example: {"name":"something-urgent"}

Object or string FALSE
Headers Specify any headers required in addition to authorization or content-type (these are already handled by this connector). This will usually be similar to the following:

Example: {"Content-type":"application/json"}

Object FALSE
Body Specify a request body in JSON format. Only available for POST, PUT, and PATCH requests. Object FALSE
