Authorization

Authorize this connector by creating a connection to your account. You can reuse this connection the next time that you build a flow with this connector.

Tip

You can create multiple connections and manage them from your Connections page.

Prerequisites

  • admin account

  • admin credentials

  • API Key containing the following permissions: read, create/update, delete

Okta recommends that non-team API Keys are used with the Opsgenie connector. Unintended behavior may result if team-specific API Keys are used with the Opsgenie connector. If you are using an API Key that belongs to a team integration, teams must be granted access to these schedules.

To configure connector in Workflows:

  1. There are two ways that you can create a new connection:

    • Add a card to your flow and click New Connection.
    • Open the Connections tab in the Workflows platform. Click New Connection and then click .

  2. Enter a Connection Nickname. This is useful if you plan to create multiple connections to share with your team.

  3. Enter an API Key to authorize your connection.

  4. From the Instance drop-down list, select a location.

  5. Click Create.
