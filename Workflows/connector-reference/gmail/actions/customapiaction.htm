Custom API Action

Make a custom, authenticated HTTP call to the Google API.

Options

Input

Field Definition Type Required
Relative URL URL address on the web server to which you are attempting to interact. Specify the relative URL as /{version}/{insert_remaining_URL}. You can specify query parameters in the relative URL using "?", or specify the query parameters as an object key pair in the Query input.

For example, for the API endpoint: https://gmail.googleapis.com/gmail/v1/users/{userId}/messages the relative URL is: /v1/users/{userId}/messages.

See Gmail API.

 String TRUE
Query Specify any additional query parameters that should be included in object format (key value pairs). Object FALSE
Headers Specify any headers required in addition to authorization or content-type (these are already handled by this connector). Object FALSE
Body Specify a request body in JSON format. Only available for POST, PUT, and PATCH requests. Object FALSE

Output

users.getProfile

Return profile information for the provided user id.
See also https://developers.google.com/gmail/api/reference/rest/v1/users/getProfile

Request

Field Value
Request type GET
Relative URL gmail/v1/users/{user id}/profile
Query parameter(s) none
Header fields Not required.
JSON Body Not required.

Response

Field Value
Status code 200 on success
JSON Body A result similar to:

{ "emailAddress": "tom.jerry@atko.com", "messagesTotal": 57167, "threadsTotal": 24459,"historyId": "3594971" }

users.message.get

Return a message for a provided user id and message id.
See also https://developers.google.com/gmail/api/reference/rest/v1/users.messages/get

Request

Field Value
Request type GET
Relative URL gmail/v1/users/{user id}/messages/{id}
Query parameter(s) Optional format: one of minimal, full(default if not specified), raw, metadata.
See the user.message/get API docs for a complete list of query parameters.
Header fields Not required.
JSON Body Not required.

Response

Field Value
Status code 200 on success
JSON Body A result similar to:

 { "id": "17e7cf5bb9ae3f65", "threadId": "17e7cf5bb9ae3f65", "labelIds": [ . . . ], "snippet": "A portion of the message" "payload": { "partId": "", "mimeType": "multipart/alternative", "filename": "", "headers": [ {"name": "Delivered-To", "value": "tom.jerry@atko.com"}, {"name": "Received","value": "by 2002:a05:7110:719b:b0:125:2704:5096 with SMTP id cb27csp754953geb; Fri, 21 Jan 2022 06:06:26 -0800 (PST)"} ] . . . }

See https://developers.google.com/gmail/api/reference/rest/v1/users.messages for more information.

users.message.list

Return the list of user message ids for the provided user id.
See also https://developers.google.com/gmail/api/reference/rest/v1/users.messages/list

Request

Field Value
Request type GET
Relative URL gmail/v1/users/{user id}/messages
Query parameter(s) optional maxrResults=n
Where n is a integer with value 1 or greater representing the number of results to return.
Header fields Not required.
JSON Body Not required.

Response

Field Value
Status code 200 on success
JSON Body A result similar to:

{ "messages": [ { "id": "17f510c519f45ff7", "threadId": "17f510c519f45ff7" }, . . . { "id": "17f5103058f1d7dd", "threadId": "17f5103058f1d7dd" } ], "nextPageToken": "04684018650224186699", "resultSizeEstimate": 8 }

users.settings.forwardingAddresses.create

Create a new forwarding address for the provide useruser id.
See also https://developers.google.com/gmail/api/reference/rest/v1/users.settings.forwardingAddresses/create

Request

Field Value
Request type POST
Note: GET with no JSON body returns all current forwarding addresses.
Relative URL gmail/v1/users/{user id}/settings/forwardingAddresses
Query parameter(s) None supported.
Header fields Not required.
JSON Body { "valid email address": string, "create": enum (VerificationStatus) }

Where valid email address is a valid forwarding email address.

See also: https://developers.google.com/gmail/api/reference/rest/v1/users.settings.forwardingAddresses#ForwardingAddress.

Response

Field Value
Status code 200 on success
JSON Body A result similar to:

{ "valid email address": string, "status": enum (VerificationStatus) }

Where status is one of the know status codes, such as pending or accepted.
