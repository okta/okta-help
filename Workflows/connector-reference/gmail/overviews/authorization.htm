Authorization

When you add a card to a flow for the first time, you'll be prompted to create a connection. Setting up a connection allows you to save your account information and reuse that connection the next time you build a flow with the connector.

Tip

You can create multiple connections and manage them from your Connections page.

To create a new connection from an Event or Action card:

  1. Click New Connection.

  2. Enter a Connection Nickname for your account. The nickname can be the actual name of the account or a generic nickname.

  1. In the OAuth window, enter your Google account and password information.

  2. On the card, authorize access to your Google account by clicking Allow.

You may experience intermittent connection outages when you create a new connection for the connector. Existing connections and flows that include Gmail connector event or action cards are not affected. See Gmail connection returning "This app is blocked" error message.

Guidance for account types and the connector

  • Only users can make proper connections for the connector.

  • If you are using a gmail.com or googlemail.com account, then connections for the connector will fail at runtime, Gmail connector cards will not function as intended, and flows that contain a Gmail card will not run successfully.

  • Project Service Accounts cannot be used to make connections for the connector.

  • users do not need to be assigned the role of Service Account User in their project to make a working connection for the connector and use these cards:

    • Read Email

    • Send Email

    • Send Email with Attachment

  • users must be assigned the role of Service Account User in their project to use these cards:

    • Add Delegate

    • Forward Emails

    • Set Auto Reply

    • Update User Settings

  • When using Google Sign In with the Okta Browser Plugin inside Workflows, you cannot switch accounts without first switching the account that's using the Okta Browser Plugin.
