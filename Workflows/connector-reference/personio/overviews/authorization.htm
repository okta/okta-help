Authorization

When you add a card to a flow for the first time, prompts you to configure the connection. This connection links to your account and saves your account information, so you can reuse the connection for future flows.

For data security, recommends that you create multiple connections that use the principle of least privilege (PoLP) for each of your use cases. For example, create one connector with read-only access and another connector with update privileges.

Before you begin

As outlined in the Subscription Agreement, electronic data submitted by or on behalf of customers through must not contain any health, credit card, financial, or other sensitive data. If you choose to save data within your Flow History, any saved information is retained for 30 days.

To use the connector, you need access to the API. restricts API access to customers on their Enterprise subscription plan. See personio.com to see pricing and features.

Start this task

Set up the API credentials in

  1. In your instance, go to Settings.

  2. On the Integrations card, click API Credentials.

  3. Click Generate new credential or select an existing API credential.

  4. For your API credential:

    • Name: Enter a unique name for your credentials.

    • Integration: Select Other.

    • Personal data: Enable the Read and Write checkbox for Employees data.

    • Readable employee attributes: Select which employee attributes that you want to be able to access. To improve the performance of the List Employees card, limit the number of attributes chosen to only those required.

    • Click Save.

  5. Click Copy client ID and store it temporarily for use when creating the connector.

  6. Click Copy secret and store it temporarily for use when creating the connector.

In the connector, you can access only the employee attributes specified during API credentials creation.

Create the connection

In the console:

  1. Click New Connection.
  2. Enter a Connection Nickname. Make this nickname unique if you want to set up multiple accounts.
  3. Enter your Client ID.
  4. Enter your Client Secret.
  5. Click Create.

Remove a connection

Deleting a connection from your console doesn't delete the corresponding API credential. Delete the API credential directly through your console.
