Known issues

  • For any fields with a Date & Time type, only stores values for the year, month, day, and the time zone. Attempting to provide a full Date & Time value results in a rounding off to the start of the date. For example, if the input is 2023-04-27T22:27:23.163Z, then stores it as 2023-04-27T00:00:00+03:00.

  • Attempting to update a field that uses an integer number results in losing the decimal part of the value. For example, if the number input value is 23.45, then that field stores only 23.

  • When using the List Employees action card, avoid making any updates to employee records, either through the Update Employee action card or direct API calls. Some records can be skipped or returned twice if the source is updated during the card execution.
