Known issues and limitations

API version

The connector in currently supports the API version 2023-07. See API Versioning.

Upgrading to a newer version of the API will result in unexpected behavior when using the action cards in the connector.

If you want to use a specific version, you can add 'API-Version': '<version>' to the headers sent by the Custom API Action card. For example, 'API-Version' : '2023-04'.

The Custom API Action card doesn’t return an error if you use a deprecated or non-existent version.

API rate limits

For paid accounts, the API token has a combined read and write limit of 10 million complexity points per minute. However, some single query operations may have different rate limits. See Rate Limits for details on the complexity points structure.

After the reaching the limit, action cards throw a rate limit exceeded error. To avoid hitting the limit, avoid making multiple concurrent API calls.

Retry attempts

The connector retries the API action three times each minute if it receives one of these HTTP status codes:

  • 429 - Too Many Requests

  • 503 - Service Unavailable

  • 504 - Gateway Timeout

However, the retry logic doesn't apply to these HTTP status codes:

  • 400 - Bad Request

  • 404 - Not Found

  • 500 - Internal Server Error

Authorize your account

When you add a card to a flow for the first time, prompts you to create and configure the connection. This connection links to your account and saves your account information, so you can reuse the connection for future flows. You don't have to authenticate every time you build a flow with this connector. See Authorization.
