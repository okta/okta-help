To learn how to use the Stream Matching Tickets option and set up a helper flow to return many records, see Use search action cards with streaming.

When using the All Matching Tickets option, a limit is placed on the search result.

When using the All Matching Tickets option, the maximum of 10,000 tickets are returned in any search result.

This limit helps ensure proper system performance. However, other custom user properties or user data in the result set may cause a flow to exceed system limitations unexpectedly and cause an out-of-memory error.

To avoid system limitations or possible memory errors, use the Stream Matching Tickets option. With this action card, a much larger set of records can be read, with these expected performance benchmarks:

  • 500,000 tickets can be read in approximately 13 hours.

  • 1 million tickets can be read in about 25 hours.

Steps to use the Stream Matching Tickets option

To use the streaming feature of this card to pass data to a helper flow, perform the following steps:

  1. Select Stream Matching Tickets and click Save.

  2. Click Choose Flow and either choose an existing flow or create a flow.

  3. In the helper flow, create keys for the input fields. Set the type for the output fields to Object.

  4. If you expect to pass any additional data into the helper flow using custom extensible fields in the card's Streaming section, add output fields for them.

Also, if you don't adhere to the naming convention for the input fields, a failure occurs when you add any further fields and pass data to the helper flow. For example, input fields to this helper flow with names of x and y (or any other name) will cause an error. However, you can add a single field as an input (for example, x) to the helper flow that doesn't adhere to the naming convention, and all your data will be passed through that field as a key.

There’s no output field on the card that will display the streamed ticket records.
