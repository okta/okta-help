To learn how to use the Stream Matching Users option and set up a helper flow to return a large number of records, see Use search action cards with streaming.

When using the All Matching Users option, a maximum of 10,000 users are returned in any search result. This limit helps ensure proper system performance. However, additional custom user properties or other user data in the result set may cause a flow to exceed system limitations unexpectedly and cause an out-of-memory error to occur.

To avoid system limitations or possible memory errors, use the Stream Matching Users option. With this action card, a much larger set of records can be read, with these expected performance benchmarks:

  • 500,000 user records can be read in approximately 13 hours.

  • 1 million users can be read in about 25 hours.

Steps to use the Stream Matching Users option

To use the streaming feature of this card to pass data to a child flow, perform the following steps:

  1. Select Stream Matching Users and click Save.

  2. Click Choose Flow and either choose an existing flow or create a flow.

  3. In the child flow, create keys for Users. Set the type for the output field to Object.

  4. If you will pass any additional data into the child flow using custom extensible fields in the card's Streaming section, add output fields for them.

Additionally, if you don't adhere to the naming convention for the input fields, a failure will occur when you add any further fields and pass data to the child flow. For example, input fields to this child flow with names of x and y (or any other name) will cause an error. However, you can add a single field as an input (for example, x) to the child flow that doesn't adhere to the naming convention, and all your data will be passed through that field as a key.

There is no output field on the card that will display the streamed user records.
