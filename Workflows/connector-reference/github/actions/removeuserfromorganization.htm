Remove User from Organization

Removes the specified user from your organization on .

To use this action card in a flow, you must set up SAML support for your app in your instance. You must also be the owner of the organization.

To set up SAML support between and , see Configuring SAML single sign-on and SCIM using Okta.

This removal action doesn't depend on how the user was added to the organization. If you used a SCIM API call to add the user to the organization, then the user's SCIM information is deleted also. See Add User to Organization.

If you use this card to remove a user added through the SCIM API but who hasn't accepted their invitation, then the invitation is canceled. However, the user’s SCIM information remains in your organization. To send the user a new invitation through a SCIM API call, you must first call the Delete a SCIM user from an organization endpoint. This action can be done through the Custom API action card or any other API solution.
