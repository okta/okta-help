Authorization

Before connecting software to Okta Workflows, ask your Jira administrator to enable the Accept Remote API calls option on the System Admin page.

For information about configuring API tokens, click here.

You can create multiple connections and manage them from your Connections page.

To create a connection from an event or action card, complete the following steps:

  1. Click New Connection.

  2. Enter a Connection Nickname. Make this nickname unique so you can authorize multiple accounts.

  3. Enter your Username and API Token.

  4. Enter your Instance URL without the https//: prefix. For example, yourcompany.atlassian.net.

  5. Click Create to save your configuration.

If you revoke or delete your API token, you must create a token and update all of your connections to use the new token. See View the Connections page.
