Guidance for Excel Online connector

Read the following information for guidance and best practices when using the connector in your flows.

  • Authentication

  • Types of accounts

  • Supported scopes

  • Action card or event card-specific limitations

Authentication

Create a connection using an admin or user account. See Authorization.

Reauthorize a connection

If you’ve used your account to create a connection successfully, you should be able to use this account to create as many connections as you want and reauthorize the old connections as long as the admin doesn't change the configuration.

Types of accounts

  • admin account

  • admin credentials

Supported scopes

The following OAuth scopes must be enabled in your environment:

  • Files.ReadWrite
  • offline_access

Action card or event card-specific limitations

Add Row

Due to third-party API limitations, this action card doesn't update workbooks that are stored in shared libraries. Updates are limited to workbooks stored in personal storage.

If a workbook is moved from personal storage to a shared library, the Microsoft API may continue to return the data for the version of the file at the time when it was moved.

If the file is moved back to personal storage, the Microsoft API may return data for both the original file and the updated file after it was moved back to personal storage.

Read All Rows

Due to third-party API limitations, this action card doesn't update workbooks that are stored in shared libraries. Updates are limited to workbooks stored in personal storage.

If a workbook is moved from personal storage to a shared library, the Microsoft API may continue to return the data for the version of the file at the time when it was moved.

If the file is moved back to personal storage, the Microsoft API may return data for both the original file and the updated file after it was moved back to personal storage.

A maximum of 500,000 rows can be read.
