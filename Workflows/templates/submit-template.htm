Submit a flow to the Workflows Templates catalog

You can share flows that address different business needs with other users of the Okta Workflows platform. Once you submit a Workflows template to the Templates repository and it is approved, the template is published to the Okta Workflows console and is discoverable from the Templates catalog.

Create and export a flow

  1. Design and save a flow in Workflows Designer. The flow can contain cards from one or more connectors or functions and tables. When you save the flow, keep all components in the same folder.
  2. To export your flow, click the gear icon of the container folder and click Export.
  3. Rename the file as workflow.flopack.

Create a video tutorial

An entry in the Workflows Templates catalog must be accompanied by a video walk-through. The video serves as a guide to create the flow and set up all of its components. The video should be five to ten minutes in length with proper quality.

The video file should contain the following metadata:

  • Title: Workflows Tutorial: [your template name]
  • Description: Use the same description as that in the workflow.json file below.
  • Meta Description: An abbreviated description.
  • Category: Product or Feature Demo
  • Select both YouTube and Okta web sites.
  • Specify the Okta destination as https://www.okta.com/platform/workflows/

Submit a flow to the Templates repository

To submit a template, you must create three files: the flow itself, a readme.md file, and a workflow.json file to store metadata that's necessary for searches in the Templates catalog. For example, see this submission.

  1. Create a workflow.json file with the following information:
    • Name: Name for your template. This name will be the template's unique identifier. For example, suspend_inactive_users. The Name string is limited to 50 characters.
    • Title: Title of the template. This will appear on the template tile in the catalog. It should be more descriptive than the Name string. The Title string is limited to 72 characters.
    • Description: A fuller description of the business processes that are addressed in this template. Limit the description to a maximum of five sentences.
    • Connectors: A list of the connectors that are used in this template. Connector names should be in lower case. For a list of connectors that are available in the Workflows platform, see Connectors.
    • Resources: Links to additional documentation or videos for this template.
  2. Create a readme.md file. The file contains all the necessary setup information to build and execute your template. The file must be written in markdown and follow the guidelines for submission. For more information, see Workflows Template Submission.
  3. Use the following local directory structure to organize your template assets:

    • workflows/ <my_shared_workflow>/ workflow.json

    • workflows/ <my_shared_workflow>/ readme.md
    • workflows/ <my_shared_workflow>/ workflow.flopack
  4. Make sure you have access to the GitHub repository at https://github.com/okta/workflows-templates/tree/master/workflows.
  5. Upload your three template assets to GitHub in the following directory: https://github.com/okta/workflows-templates/workflows/[your new folder].

Guidelines for file and folder naming

For a template to be properly considered for publication, you must follow guidelines for file and folder naming in your template.

  • Folder names must be in lower case and separated by underscores (regex: ^[a-z0-9_]{2,50}$). For example, create_report_google_sheets.
  • The names for each of the files in your template must be in lower case:
    • workflow.json
    • readme.md
    • workflow.flopack
  • The workflow.json and workflow.flopack files must contain valid JSON structure.
  • The name of the template and its container folder must match.
  • The template must contain valid connector names that are referenced in the Workflows platform connectors.json file.
  • The video and documentation links in the workflow.flopack file must point to the same folder structure.

Considerations for submitting a template

There are several factors that are considered when approving your submission to the Workflows Templates catalog.

  • Does the template serve a need for a business use case?

  • Does the template include all necessary components? Your submission must contain all the assets discussed on this page.

  • Did you provide adequate content to set up and execute the flow? For example, be mindful that a video walk-through and proper documentation will be helpful to other users who want to download and test your template for their business needs.

  • Does the flow provide potential architectural concerns during execution? For example, will the execution of your template cause another user to hit their Workflows rate limits or maximize their tables allowance?
