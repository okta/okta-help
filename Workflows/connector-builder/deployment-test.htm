Test your connector

This area shows all the versions of a connector created by you or your team for a specific project. gives each new version of the connector an incremental version number.

Any icon displayed after the version number indicates the state of the connector:

Private deployments of the connector use this test version of the connector.

If you click the icon, the state changes to Hidden and this version of the connector is hidden.

Private deployments of the connector are hidden from use.

If you click the icon, the state changes to Visible and this version of the connector is available to use.
There are errors or warnings detected in the validation test for this version of the connector. Click the icon to see specific warning messages.

This area only displays test versions that pass the validation process, unlike the previous version of . If a test version fails the validation process, the error dialog shows the issues, but doesn't retain any details about that test version.

If you deploy a new version that breaks existing flows, then keeps the resulting error in the test version table for you to review.

Each row also displays the creation date and the team member who created it.

You can add any information relevant to a specific version in the editable Note column.

The Find a version text field allows you to enter search terms to find a specific test version of your connector.

Create test version

To create a version of your connector for testing purposes, click Create test version. This action makes a new copy of the connector, so you can make changes and tweak settings while still keeping an existing version of the connector privately deployed.

Validate connector

After making any changes, you can test the connector for errors without having to deploy the connector to a local environment. This step confirms that the connector contains the necessary components for a successful deployment, such as at least one action flow and a unique name for each flow.

When you click Validate connector, checks the latest version of the connector. A dialog indicates if the connector is valid or has errors. See Connector Builder deployment validation errors.
