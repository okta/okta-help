Extensible input fields

When an action card is designed to allow extensible fields, a Workflows user can add fields to a card when building a flow. Such fields are not defined by you when building your connector and action cards in Connector Builder. Instead, they are defined by a user on the card in Workflows Designer.

The Extensible Inputs toggle, when turned on, allows a Workflows user to add as many fields to a card as needed to suit their flow. For each new field, the user adds a a string value for the key and an input value.

The Extensible Inputs toggle is turned off by default. Extensibility is available for dynamic input fields only.
