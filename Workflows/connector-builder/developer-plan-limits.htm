About Workflows Developer Plan limits

Access to Connector Builder for users who are not Okta Workflows customers is available through a Developer Plan. The plan is intended to offer Connector Builder features to as many third-party builders as possible, allow functionality to developers to test real use cases, but without compromising the platform.

There are Workflows developer plan limits that may impact the design and success of your flows.

Category Description Limit or Value
Flows Number of parent flows that the user can turn on at any one time 3

Users An unlimited number of users can access an org Yes
Scheduled flows Users can trigger flows using a scheduler No
Connector access Number of connectors that can be added to flows No limit
Connector set Name of the connector set that is available to the user 1
API endpoint flows Users can use API endpoint cards to trigger a flow No
Flow history duration Number of days that Flow History is retained before being deleted 7
Polling monitors Users can run polling monitors at 5-minute intervals YES
Events Users can create flows that respond to events YES
Flow History Users can access Flow History YES

Executions Limit to the number of monthly flow executions in an org 5K per month
Data throughput Limit to the amount of monthly data that can pass through a flow
