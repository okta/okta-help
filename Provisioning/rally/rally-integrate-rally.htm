Integrate Rally Software with

If you experience provisioning errors after integrating Rally Software with , make sure the attribute mappings contain the correct information and confirm the Rally Software user name format is correct. If you enable provisioning after previously assigning users to Rally Software single sign-on (SSO), run a full import to link existing assigned users with Rally Software users.

  1. Add the Rally Software app to if it has not been added previously:
    2. Click Add Application.
    3. In the Search for an application field, enter Rally.
    4. Select Add for Rally Software.
    5. Complete the fields on the General Settings page and click Next.
    6. In the Sign On Methods section of the Sign-On Options pane, select a sign on option. If you select SAML 2.0, click View Setup Instructions and follow the instructions.
    7. Click Done.
  2. If you added the Rally Software app previously, on the click Applications and select Rally Software in the list of applications.
  3. Click the Provisioning tab, click Configure API Integration, and select the Enable API Integration check box.

  4. Complete these fields:

    • Skip Users if Error While Import: Optional. Select this check box if you want to skip user import if an error occurs during import.
    • Admin Username: Enter the Rally Software administrator user name.
    • Admin Password: Enter the Rally Software administrator password.
    • Import Groups: Optional. Clear this check box if you do not want to import Rally Software groups.
  5. Optional. Click Test API Credentials to test the API integration.
  6. Click Save.

  7. Optional. To edit the to Rally Software provisioning settings, select To App in the SETTINGS list, click Edit, edit the settings, and click Save.

  8. Optional. To edit the Rally Software to provisioning settings, select To Okta in the SETTINGS list, click Edit, edit the settings, and click Save.

  9. Optional. Assign users to the Rally Software app. See Assign applications to users.
