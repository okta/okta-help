Provision on-premises applications

The provisioning features of Okta Lifecycle Management automatically manage accounts for your entire organization.

On-premises provisioning combines your on-premises applications and a SCIM server or custom connectors together with the to send user information to and from . makes calls to the applications through your corporate firewall in order to create new user accounts, update attributes, and deactivate users, all based on the user assignment rules configured in .

supports provisioning and deprovisioning for any on-premises application using one of the following methods:

  • An Active Directory (AD) instance that manages user accounts
  • A web services API that can access using a publicly addressable connection
