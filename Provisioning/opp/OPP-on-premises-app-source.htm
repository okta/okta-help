Make an on-premises application the profile source

If you configure your on-premises application as a profile source, it becomes the identity authority for assigned users. User profiles are no longer editable in and changes are synchronized to during provisioning events.

  1. Configure the API timeout. See Configure the API call timeout period.
  3. Enter the name of your on-premises app integration in the Search field.
  4. Click the name of your app integration and click the Provisioning tab.
  5. In the Provisioning section, click Edit.
  6. In the Profile Source section, click Enable.
