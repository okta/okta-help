Configure the API call timeout period

You can specify how long your org waits for an API call to complete before a timeout occurs. The minimum setting is 30 seconds. Your selected timeout value is applied to all requests (GET, PUT, and POST) that are sent to the SCIM server.

  1. Confirm that you have a SCIM connector. See Connect to a SCIM connector.
  3. Enter the name of your on-premises app integration in the Search field.
  4. Click the app integration name and click the Provisioning tab.
  5. In the Connector Configuration section, click Edit.
  6. In the Timeout for API calls list, select a timeout period.
