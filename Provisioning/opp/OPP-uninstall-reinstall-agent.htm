Uninstall and reinstall the

Before you remove an , recommends that you have one or more other agents configured to avoid an interruption of service. When you uninstall and reinstall an , you must decide whether you also want to remove the old token. If you're performing an upgrade, removing the token isn't mandatory. To remove the API token, you must delete the folder, and then deactivate and remove your old agent.

Uninstall on Linux

  1. Sign in to your Linux server as root.
  2. Enter yum remove OktaProvisioningAgent.

Uninstall on Windows

  1. On your Windows desktop, go to Start > Control Panel > Programs > Programs and Features.
  2. Select , and then click Uninstall.

  3. Go to \Program Files\ and delete the folder. Deleting this folder removes the agent configuration data and the API Token from your hard drive. The API token for the server is still valid in so it's important to remove the configuration data.
