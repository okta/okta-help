Upgrade

Upgrade the to get the latest features and functionality. During an upgrade, the service is stopped if it's running and it isn't automatically started after an upgrade. The upgrade echoes the exact command needed to start the service.

Linux

  1. Sign in as root to your server.
  2. Copy OktaProvisioningAgent*.rpm to a scratch directory, and then cd to that directory.
  3. Determine if there's a previous version of the on your server by entering the following command:

    4. rpm -q OktaProvisioningAgent

    If the agent is present on the server, the command returns its name and version.

  4. Use yum to upgrade the existing provisioning agent by entering the following command:

    5. yum localupdate <package name>

    For example, yum localupdate OktaProvisioningAgent*.rpm

  5. To restart your , enter the following command:

    6. service OktaProvisioningAgent start

  6. To confirm that the is running, enter the following command:

    7. service OktaProvisioningAgent status

The process automatically restarts after the upgrade is complete.
