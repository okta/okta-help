Typical workflow for adding provisioning to an existing app integration

To add provisioning to an existing Single Sign-On (SSO) SAML or OIDC app integration, you enable the functionality and configure the provisioning settings.

To test provisioning features or silently provision users using a new app integration, an alternative solution is to create a duplicate instance of the app integration in your org and then grant your users access to this second instance of the provisioning app integration.

This table list the tasks you need to complete to set up a cloud-based provisioning app integration and to configure it to work in your existing org.

Description
