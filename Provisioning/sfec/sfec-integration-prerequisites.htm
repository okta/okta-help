SAP SuccessFactors Employee Central integration prerequisites

These are the prerequisites for integrating SAP SuccessFactors Employee Central with :

  • SuccessFactors Web Services API enabled on the tenant. To enable this functionality, contact SAP SuccessFactors customer support or your implementation partner.

  • SFAPI and Employee Central SOAP API enabled on the SAP SuccessFactors Employee Central Provisioning Console. To access these settings, open the Provisioning Access Console, select your company, and go to Company Settings Web Services .
  • A new or existing role with all permissions selected for Manage Integration Tools. To assign these permissions to a new or existing role, go to the Admin Center, go to Administrator Permissions Manage Integration Tools , select the Select All check box in the right pane, and click Done.
  • A new or existing role with all permissions selected for Employee Central API. To assign these permissions to a new or existing role, go to the Admin Center, go to Administrator Permissions Employee Central API , select the Select All check box in the right pane, and click Done.
  • When adding the SAP SuccessFactors Employee Central app to Okta, the Company ID field must contain your company ID. You can get this value from your sign in URL. For example, https://acme.successfactors.com/login?company=ACME123456789.
