Google Docsコネクターに関するガイダンス

フローでGoogle Docsコネクターを使用する際のガイダンスとベスト プラクティスについては、次の情報をお読みください。

  • サポートされているスコープ

  • アクションカードまたはイベントカード固有の制限

サポートされているスコープ

Google Docs環境で、次のOAuthスコープを有効にする必要があります。

  • https://www.googleapis.com/auth/documents
  • https://www.googleapis.com/auth/drive.metadata.readonly
  • https://www.googleapis.com/auth/drive.file

アクションカードまたはイベントカード固有の制限

Update Userアクションカード

各フィールドの追加情報を確認してください。

フィールド 詳細
Insert Inline Image（インライン画像挿入） このオプションを選択した後で、画像がページの高さ/幅を超過すると、新しいGoogle Docページが追加される場合があります。
テキスト

改行文字を挿入すると、そのインデックスに暗黙的に段落が作成されます。新しい段落の段落スタイルは、リストや箇条書きを含め、現在の挿入インデックスにある段落からコピーされます。

挿入するテキストのテキストスタイルは自動的に決定され、通常は隣接するテキストのスタイルが維持されます。一般に、挿入するテキストのテキストスタイルは、挿入インデックスの直前のテキストと一致します。

一部の制御文字（U+0000～U+0008、U+000C～U+001F）およびUnicode基本多言語面私用域（U+E000～U+F8FF）の文字は、挿入テキストから排除されます。
Segment ID（セグメントID）

セグメントID値の取得には、Read Documentカードが使用されます。

使用法：

  • スラッシュ（/）付きの文字は1文字と見なされます。

  • Segment ID and Index（セグメントIDとインデックス）：指定したセグメント（ヘッダー、フッター、脚注）に、インデックスから順にテキストを挿入します。

  • Segment ID（セグメントID）：ドキュメント本文の最後にテキストを挿入します。

  • Index（インデックス）：ドキュメント本文内の指定されたインデックスにテキストを挿入します。

ドキュメント本文では、startIndexおよびendIndexプロパティのインデックス値を使用してください。これらはそれぞれ、行の先頭の文字と末尾の文字のインデックスを表します。開始および終了インデックスの取得にはRead Documentカードが使用されます。特定のセグメント（ヘッダー、フッター、脚注）では、endIndexは行の長さとして使用されます。開始インデックスは常に0です。
インデックス

ドキュメント本文では、Start Index（開始インデックス）およびEnd Index（終了インデックス）プロパティのインデックス値が使用されます。これらはそれぞれ、行の先頭の文字と末尾の文字のインデックスを表します。開始および終了インデックスの取得にはRead Documentカードが使用されます。特定のセグメント（ヘッダー、フッター、脚注）では、［End Index（終了インデックス）］は行の長さとして使用されます。開始インデックスは常に0です。

使用法：

  • スラッシュ（/）付きの文字は1文字と見なされます。

  • Segment ID and Index（セグメントIDとインデックス）：指定したセグメント（ヘッダー、フッター、脚注）に、インデックスから順にテキストを挿入します。

  • Segment ID（セグメントID）：ドキュメント本文の最後にテキストを挿入します。

  • Index（インデックス）：ドキュメント本文内の指定されたインデックスにテキストを挿入します。
Font Weight（フォントの太さ）

フォントの太さは、表示されるフォント深さの1要素に過ぎません。表示される太さは、継承を考慮した上で、太さと、テキストスタイルの太字の解決値の組み合わせによって決まります。

  • テキストが太字でフォントの太さが400未満の場合、太さは400で表示されます。
  • テキストが太字でフォントの太さが400以上700未満の場合、太さは700で表示されます。
  • フォントの太さが700以上の場合、表示される太さはフォントの太さと等しくなります。
  • テキストが太字ではない場合、表示される太さはフォントの太さと等しくなります。

フォントの太さは、［Font Family（フォントファミリー）］の値と一緒に渡す必要があります。一方の値のみが渡されると、エラーがスローされます。

Image Height（画像高さ） 高さが指定されていない場合、画像のデフォルトサイズは解像度に基づいて計算されます。一方の寸法のみが指定されている場合、もう一方の寸法は画像のアスペクト比を維持するように計算されます。幅と高さの両方が指定されている場合、画像はアスペクト比を維持しながら指定の寸法に収まるように拡大縮小されます。
Image Width（画像幅） 幅が指定されていない場合、画像のデフォルトサイズは解像度に基づいて計算されます。一方の寸法のみが指定されている場合、もう一方の寸法は画像のアスペクト比を維持するように計算されます。幅と高さの両方が指定されている場合、画像はアスペクト比を維持しながら指定の寸法に収まるように拡大縮小されます。

Read Documentアクションカード

各フィールドの追加情報を確認してください。

フィールド 詳細
Suggestions View Mode（提案表示モード）

  • Default（デフォルト）：返されるドキュメントに適用されるSuggestions View Mode（提案表示モード）は、ユーザーの現在のアクセスレベルによって異なります。ユーザーが表示アクセス権のみを持つ場合、PREVIEW_WITHOUT_SUGGESTIONSが適用されます。それ以外の場合は、SUGGESTIONS_INLINEが適用されます。これがデフォルトの提案表示モードです。

  • Inline（インライン）：返されるドキュメントにはインラインの提案があります。提案される変更は、ドキュメント内の基本コンテンツとは区別されます。提案される変更を表示する権限がユーザーにない場合、このモードでドキュメントの取得をリクエストすると、403エラーが返されます。

  • Accepted（受け入れ）：返されるドキュメントは、提案されたすべての変更を受け入れたプレビューです。提案される変更を表示する権限がユーザーにない場合、このモードでドキュメントの取得をリクエストすると、403エラーが返されます。

  • Rejected（拒否）：返されるドキュメントは、ドキュメント内に提案が含まれる場合に、提案されたすべての変更を拒否したプレビューです。
