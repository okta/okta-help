フローバックアップ用のテンプレートの設定

フローとフォルダーのバックアップでは、フローをGitHubやGoogle Driveなど外部のプラットフォームにオンデマンドまたは自動的に保存およびバックアップできます。

このタスクのセットでは、フローをエクスポートする方法を示します。Export Flow関数カードをExport Folder関数カードと取り換えて、個別のフローの代わりにフォルダー内のフローのセットをエクスポートすることもできます。

開始する前に

次のアクセス権があることを確認してください。

  • 組織でOkta Workflowsが有効になっており、Oktaテナントへのアクセス権がある

  • GitHubテナントへのアクセス権がある

  • GitHubリポジトリーへのアクセス権があり、そのリポジトリーでプル要求をコミットまたはオープンする権限がある

フローバックアップ用のテンプレートのダウンロード

新しいテンプレートは、Workflowsインスタンスの［Templates（テンプレート）］ページにあります。

フローのバックアップ用テンプレートのサンプルは次の方法で見つけられます。

  1. Workflowsの［Templates（テンプレート）］ページで、［Search（検索）］フィールドに「versioning for flows and folders（フローとフォルダーのバージョニング）」と入力します。

  2. 検索結果の［Versioning for Flows and Folders with GitHub（フローとフォルダーのGitHubによるバージョニング］テンプレートをクリックします。

  3. テンプレートのページで［Add template（テンプレートを追加）］をクリックし、環境にダウンロードします。

  4. これらの手順を繰り返し、環境でフローをバックアップするためのテンプレートを複数ダウンロードすることもできます。

テンプレートのセットアップ

このタスクには、［Versioning for Flows and Folders with GitHub（フローとフォルダーのGitHubによるバージョニング）］テンプレートで提供される2つのフローのうち1つを設定するため必要なステップが含まれています。このタスクでは、［Export Flow and Make Commit to GitHub（フローをエクスポートしてGitHubにコミットする）］というフローを設定します。

  1. ［Versioning for Flows and Folders with GitHub（フローとフォルダのGitHubによるバージョニング）］テンプレートをダウンロードした上で、［Export Flow and Make Commit to GitHub（フローをエクスポートしてGitHubにコミットする）］というフローを開きます。

  2. Export Flow関数カードで、環境からエクスポートするフローを選択します。そのフローに含まれるすべての要素が、JSONファイルとしてエクスポートされます。

  3. エクスポートするファイルの名前を選択します。デフォルトはexportedFlow.flowです。ファイル名は変更できますが、ファイルの拡張子は必ず.flowにしてください。

  4. コメント履歴に追加するコメントを編集し、コミットした理由がリポジトリーの他のユーザーに伝わるようにします。

  5. Update File Contentカードの場合、GitHubコネクタ用の接続を作成します。

    1. 接続を確立し、OAuthプロンプトに従って、Workflowsが自分の代わりにこのGitHubリポジトリーにアクセスすることを認可します。

    2. テストに使用する組織、リポジトリー、ブランチを選択します。まだ行っていなければ、テスト用のテストレポジトリとブランチを作成することをお勧めします。レポジトリで［Settings（設定）］ > ［Manage Access（アクセスの管理）］メニューを開くために適切な権限があることを確認します。

    3. 次のフィールドに値が含まれていることを確認します。

      • ［Message（メッセージ）］（コミットのコメント）

      • ［File Content（ファイルコンテンツ）］（エクスポートされるJSONファイル）

      • ［Path（パス）］（フォルダのパスとファイル名）：フォルダやサブフォルダ内にファイルを作成または更新するには、ファイル名に加えてパスを、たとえばMainFolder/SubFolder1/exportedFlow.flowのように定義する必要があります。この文字列はPath入力の値として使用されます。

    4. そのファイルが存在しない場合、フローはエラーを出力してから、新しいファイルの作成を試みます。この問題を解決するには、次のようにします。

      • Try/If Errorブロック内の［If Error］をクリックし、上述と同じリポジトリーへの接続を確立します。

      • ［Message（メッセージ）］、［File Content（ファイルコンテンツ）］、［Path（パス）］に正しい値が入力されていることを確認します。

フローをエクスポートしてGitHubでPRを開くという名前のフローも参照してください。このフローでは、GitHubのリポジトリーブランチに直接コミットする代わりに、新しいブランチを作成してプルリクエストをオープンできます。

テンプレートのフローのテスト

  1. ［Test Flow（フローのテスト）］をクリックします。

  2. GitHubで、作成または更新したファイルを選択します。

  3. ファイルの［History（履歴）］を選択し、コミットの履歴が表示されることを確認します。最後のコミットがOkta Workflowsによって行われ、以前のコミット時に送信したコメントが含まれていれば、正しい結果です。
