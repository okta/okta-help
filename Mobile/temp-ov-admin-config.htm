User experience

The settings that you enabled in step 4 determine the user’s sign-in experience.

  • Push Notification
  • Push Notification with Number Challenge
  • Touch ID and Face ID prompt on iOS devices
  • FIPS-mode encryption

Push Notification

If you selected Enable Push Notification, users are prompted to configure it for their account the next time they sign in to . The app guides them through the setup.

If you also select Prompt users to upgrade, users who use older versions that don’t support Push authentication receive a Please update your profile message. Users can upgrade immediately or postpone the upgrade by selecting Remind me later. If they choose to be reminded, the prompt is shown again the next time they sign in. See (Documentation for end users).

Push Notification with Number Challenge

Number Challenge works with Android, iOS, and Apple Watch devices that are enrolled in .

If you enabled Push Notification with Number Challenge, users receive a number matching request: The displays a number. Users are prompted to select this number out of three random numbers in an notification on their devices. Users gain access to their app accounts only if they tap the correct number. See the end-user documentation: Sign in with an push notification (iOS) or Sign in with an push notification (Android).

Touch ID and Face ID prompt on iOS devices

When you enable Touch ID or Face ID verification, users are prompted to configure Touch ID or Face ID for their device when they enroll or authenticate. The device guides users through this configuration.

Users who authenticate with on devices that don’t support biometrics are not affected if you enable Touch ID or Face ID.

If users are already enrolled in with Push Notification and you enable Touch ID or Face ID, minimal setup is required. The next time they authenticate with Push Notification, users must also scan their fingerprint or facial image to gain access to their app account.

  • If the user's fingerprint or facial image is already set up in the device Settings, the Touch ID or Face ID option is activated in the app as soon as the users clicks Send Push in the .
  • If the user's fingerprint or facial image isn’t set up in the device Settings yet, the user receives a Touch ID Required or Face ID Required message when they click Send Push in the .

See Authenticate with on Android devices or Authenticate with on iOS devices.

FIPS-mode encryption

To meet FIPS compliance, users must ensure that is installed on a supported operating system version: 

  • iOS 7 or higher
  • Android 6 or higher

Additional solutions

Use hardware key storage for Android devices

To enhance security on Android devices, enable Use Hardware Key Storage on Android through Settings.

This allows you to implement the Federal Identity, Credential and Access Management (FICAM) security architecture, which applies access control and hardware protection to keys stored on Android devices.

Users must have a secure PIN set up in order to ensure FICAM compliance.

Use S with Push Notification and RADIUS agents

To use with Push Notification and you also use the RADIUS agent, you must upgrade the RADIUS agents to version 2.1.5 or later. To find the current agent version, see RADIUS Server Agent Version History for instructions.
