Event hooks

Event hooks are outbound calls from that trigger process flows within your own software systems. They're sent when specific events occur in your org, and they deliver information about the event. Unlike , event hooks are asynchronous and don't offer a way to execute the process flow. After sending the call, the process flow continues without waiting for a response from the called service.

To set up an event hook, you need to implement a web service with an internet-accessible endpoint. It's your responsibility to arrange hosting of your code on a system that is external to . defines the REST API contract for the requests that it sends to your custom code, and for the responses your custom code can send back.
