Fixes (-8 pt)

Bug Vuln. Product. OIE Product. Only Doc after all Prod cell deploy

OKTA-401626: The attempt limit for self-service account recovery was different for valid and invalid users.

Bug Vuln. OIE Product.

OKTA-493073: The error message Your org’s available authenticators do not satisfy the selected constraints. Enable the appropriate authenticators to continue for authentication policy errors was no longer applicable in certain use cases.

Bug. OIE Product.

OKTA-493531: Users were unable to login using passcodes when Permit Automatic Push for Okta Verify Enrolled Users was enabled.

Bug. Product. OIE Product.

OKTA-801372: The People’s page used an incorrect field name as the sorting key, we changed the sorting key from apiLastUpdate to lastUpdated.

Bug. Product. OIE Product.

OKTA-840894: Users attempting to cancel signing in with a PIV/CAC card were not being redirected back to the custom domain.

Bug. Product. OIE Product.

OKTA-844814: When clicked the Show More button in API > Trusted Origins tab, an Invalid search criteria error was displayed.

Bug. OIE Product.

OKTA-852341: After completing an MFA challenge and signing out, the username on the Sign In page was populated with the full Okta username and not the username prefix that was used for sign in.

Bug. Product. OIE Product.

OKTA-854006: Clicking Save and Add another to add new attributes on the Profile Editor page sometimes didn’t function as expected.

Bug. Product. OIE Product.

OKTA-855768: Improved New Device Behavior Evaluation incorrectly identified a previously used device as new when the admin accessed the Okta Admin Dashboard.

Bug. OIE Product.

OKTA-860752: In the Admin Console, the Japanese version of the MFA Enrollment by User report contained some English.

Bug. Product. OIE Product.

OKTA-866469: The Coupa integration URL displayed under the application Sign On tab was incorrect.

Bug. Product. OIE Product.

OKTA-867511: Users weren’t assigned to applications through group assignments following an incremental import from AD into Okta.

Bug. Product. OIE Product.

OKTA-867991: Logging in to the End-User Dashboard through a third-party identity provider (IdP) displayed an incorrect error message if the password had expired.

Bug. Product. OIE Product.

OKTA-868319: In the End-User Dashboard, the link to access the Okta Browser Plugin installation guide redirected users to a broken page.

Bug. OIE Product

OKTA-872333: After an org upgraded to Identity Engine, some apps with the default app sign-on policy weren’t assigned to the Classic Migrated policy.

Bug. Product. OIE Product.

OKTA-872600: Custom email domain configurations sometimes didn’t appear on the Domains page in the Admin Console.

Bug. API bug. Product. API.

OKTA-873320: The max_age and login_hint parameters in the authorize request were sometimes ignored when a client used the private_key_jwt client authentication method.

Bug. Product

OKTA-873738: Some field widths rendered improperly.
