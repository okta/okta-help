Fixes

General Fixes

  • Some orgs were unable to configure their global session policies to display the password-first Sign-In Widget. OKTA-508888
  • Staged and provisioned user accounts received different error messages when they clicked Forgot password? on the Sign-In Widget. This occurred in orgs with User Enumeration Prevention turned on. OKTA-509453
  • Okta Provisioning agent version 2.0.10 didn't use the correct Java version. OKTA-522077
  • Routing rules incorrectly redirected some users to an IdP before they could enter their username. OKTA-527215
  • Linked objects didn’t show up in logs after they were created or deleted. OKTA-532256
  • Some YubiKeys didn't work for authentication even though they were successfully enrolled. OKTA-532720
  • AD-sourced users could continue to use ADSSO or IWA to sign in to Okta after being moved to an out-of-scope OU. OKTA-534260
  • Admins with a custom role couldn’t edit the users in a group if the group was assigned to an app with profile sourcing enabled. OKTA-534595
  • When a DELETE request to the /api/v1/authorizationServers/<authServerID>/clients/<clientID>/tokens endpoint was called for large scale operations, an HTTP 500 error was returned. OKTA-536037
  • The Remind me later button on the factor enrollment page didn’t redirect to the End-User Dashboard. OKTA-537535
  • Some admins weren't able to delete network zones after they upgraded to Identity Engine. OKTA-538402
  • Changing the Username on the Assignment page for the Box app failed with an HTTP 500 error. OKTA-540825
  • Errors during federation sometimes didn't display the cause of the error. OKTA-541442
  • The authn_request_id information was missing from the user.authentication.auth_via_mfa System Log event for Okta Verify Push verifications. OKTA-542472
  • The Norwegian translation of the end-user settings and preferences menu was incorrect. OKTA-544783
  • Admin roles that were constrained to a group with group rules couldn't be assigned to a user or group. OKTA-546310
  • The Welcome page, SMS reminder prompt, and security image prompt weren’t displayed for users accessing Okta using AD SSO in incognito mode. OKTA-547525
  • After upgrading to Identity Engine, orgs with custom domains couldn’t use getRequestContext in the Sign-in page code editor. OKTA-549174
  • The Box integration provisioning menu didn’t display the correct settings. OKTA-549537
  • When the Admin Global Search UI Enhancement Early Access feature was enabled, admins couldn’t select groups on the App Sign On Rule screen. OKTA-549770
  • Using an Agentless DSSO test endpoint without any routing rules configured to use ADSSO resulted in a 404 error. OKTA-549886
  • Some orgs didn’t correctly recognize a sign-in attempt using a smart card. OKTA-550773
  • Provisioning new users from Okta to Office 365 failed. OKTA-550789
  • The Forgot Password windows on the End User Settings page displayed Calling now… and Sending code… messages before users entered their phone number. OKTA-551022
  • The Email Authenticator challenge lifetime was sometimes set to five minutes regardless of its value in the authenticator settings. OKTA-551130
  • The Done button wasn't displayed after YubiKey was successfully deleted. OKTA-552440
  • Users were sometimes signed out of Okta right after signing in if the tokens returned were too large. OKTA-552637
  • Customized sign-in pages for orgs using a custom domain didn’t render properly. OKTA-552810
  • When the full-featured code editor was enabled, updates to email customizations, custom error pages, and the sign-in page didn't trigger System Log events. OKTA-553284
  • Internet Explorer 11 users were blocked from signing in to orgs that used custom domains. OKTA-557858
