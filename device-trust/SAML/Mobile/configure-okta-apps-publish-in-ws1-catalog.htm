Publish Okta apps to the catalog

Prerequisites

Step 2: Configure Okta application source in [%=Mobile.VIDM%]

End users can continue to access apps from either the Okta dashboard or the Workspace ONE dashboard. Both experiences are fully supported. This section describes how to configure the catalog to publish applications federated through Okta without the need to first import them into . This allows admins to manage federated applications and user entitlements completely from the Okta Admin console.

If you decide to integrate Okta applications into the catalog:

  1. Configure Okta as an application source in (you should have done this already in STEP 2: Configure Okta application source in [%=Mobile.VIDM%]).
  2. Enter your Okta tenant details in the console as described in the procedure below. You do not need to add individual applications to the catalog.

When end users log into the , the Okta apps to which they are entitled appear automatically in the catalog, along with their other apps.

uses the Okta tenant information you configure to connect to the Okta tenant and retrieve apps and user entitlements whenever a user logs into . When a user clicks an Okta app in , uses the application source configuration to launch the app.

You manage apps and user entitlements in the Okta Admin console, not in the console. When you add or delete apps or entitlements in the Okta Admin console, the changes are replicated in end users' catalogs directly. Okta apps do not appear in the administration console.

This integration supports the following types of Okta apps:

  • SAML 2.0
  • WS-Federation
  • Bookmark
  • OpenID Connect
Note: SWA apps are not supported.
