Configuring Okta Mobility Management

Okta Mobility Management (OMM) enables you to manage your users ' mobile devices, applications, and data. Your users enroll in the service and can then download and use managed apps from the Apps Store. Managed apps are typically work-related, such as Box or Expensify. As an administrator, you can remove managed apps and associated data from users' devices at any time. You can configure policies, such as data sharing controls, on any of your managed apps.

For other OMM topics, see:

Before You Begin

  • To install Okta Mobility Management and register your devices, you must have Okta Mobile installed on each target device. Okta Mobile is only required for the registration procedure. After registration is complete, any security policies that you configure remain active even if Okta Mobile is deleted on the device.

    Refer to Using Okta Mobile for more information.

  • Required OS versions:
    • For Apple devices: iOS 7.1 or higher.
    • For Android devices: Android OS 4.0 or higher.
  • To install Okta Mobility Management on an Apple iOS device:
    • Administrators must have an Apple ID so they can have Apple sign the Okta Push Notification Service Certificate. You can create an Apple ID at http://appleid.apple.com.
  • You must have groups configured in order to configure your mobile policies. You can create groups in Okta or import them from your directory. See Adding and Using Groups, below, for more information.

Performing Initial Setup

Your initial setup steps differs for each operating system.

Google Android Configuration

There is no additional configuration necessary for Okta Mobility Management for your Google Android devices.

Apple iOS Configuration

To configure Okta Mobility Management for your Apple iOS devices, you must first configure the Apple Push Notification Service (APNS) certificate. This process includes downloading a Certificate Signing Request (CSR) from Okta, uploading the CSR to Apple for digital signature, and finally uploading the signed certificate to the Okta org; as follows:

  1. From your Administrator dashboard, either:

    • From the Security tab, select the Policies page, then select the Mobile tab.

    • From the Devices tab, select Mobile Policies.

  2. Click the Apple Certificate Setup button.

    One of the following icons will appear on the Apple Certificate Setup button:

    • Yellow exclamation mark: This indicates a push certificate has not yet been configured.

    • Green check mark: This indicates a valid push certificate has already been configured.

    • Red exclamation mark: This indicates your current push certificate has either expired, or is close to expiring.

  3. Follow the instructions in the Apple Certificate Setup dialog:

  4. Download Certificate Signing Request:

    Click Download to obtain your Certificate Signing Request (okta-apns-CSR.dat) from Okta.

  5. Obtain Apple Push Notification Service Certificate

    Navigate to the Apple Push Certificates management portal using the link provided (https://identity.apple.com/pushcert)

    Note that you will need an Apple ID to log into this portal.

    If this is the first time you are setting up a certificate, you will be asked to accept Apple 's terms and conditions. If you have previously configured one or more certificates, you will see them listed in the portal.

    • Click Create a Certificate.
    • Click Choose File and navigate to the CSR file you previously downloaded for Apple to sign (okta-apns-CSR.dat), then click Upload. Once the request has been successfully uploaded, a confirmation screen is displayed.

    • Click Download on the confirmation screen to download the APNS MDM certificate.

      Note that APNS certificates have an expiration date. Log in to the Apple Push Certificates Portal to Renew or Revoke certificates in the Certificates for Third-Party Servers section.

  6. Upload Apple Push Certificate

    Return to the Apple Certificate dialog box, click Browse to locate the APNS downloaded in the previous step, then click Upload to complete the CSR signature process.

Once you have configured the Apple Push Notification Service (APNS) certificate, your users must then download and install the Okta Mobile app from the Apple App Store. They should search for Okta Mobility Management and proceed through the download and installation process.

Renewing Your Apple Push Notification (APNS) Certificates

It's not possible to overwrite an existing certificate in Okta â€“ don't worry about accidentally renewing the wrong certificate. However, you can avoid the hassle of reloading the same certificate by carefully following these instructions.

Requirements

Your Apple ID.

Managing APNS Certificates

Navigate to the Apple Push Certificates Portal, here: https://identity.apple.com/pushcert/ (you need your Apple ID to log into this portal).

If you have previously configured one or more certificates, you will see them listed in the portal, as shown below. You can Renew, Download, and Revoke APNS certificates from this portal.

Important: We recommend you do not use the Revoke option. If you revoke a certificate, all your end users will subsequently need to re-enroll in Okta Mobility Management.

You can click the information (i) icon to view details (highlighted in yellow below) about each certificate. Use this information to compare certificates in this portal to the one in Okta.


Renewing APNS Certificates

APNS certificates expire after one year. If you need to renew your certificate you need to first download a new Certificate Signing Request (CSR) from Okta, as follows:

  1. From your Admin dashboard, select Policies > Security.

  2. Select the Mobile tab on the Policies page.

  3. Click the APNS OMM vendor certificate button.

    Note that a green check box on the APNS OMM vendor certificate button indicates that a push certificate has already been configured.

    The APNS MDM vendor certificate dialog appears:

    Note that step 2 on this screen displays information (highlighted in yellow, above) about your current APNS certificate, expired or not. Use this information to identify the certificate in the portal that you want to renew.

  4. Click Download to obtain your Certificate Signing Request (okta-apns-CSR.dat) from Okta.

  5. Navigate to the Apple Push Certificates Portal, here: https://identity.apple.com/pushcert/ (your Apple ID is required to log into this portal).

  6. Locate the certificate that has expired/is expiring, and click Renew.

    Note that you can find information about each certificate by click the information (i) icon. Use this information to compare your certificates.

  7. Click Choose File, then navigate to the CSR file you previously downloaded for Apple to sign (okta-apns-CSR.dat), then click Upload.

    Once the request has been successfully uploaded a confirmation screen is displayed:

  8. Click Download on the confirmation screen.

  9. Return to the APNS OMM Vendor Certificate dialog box in Okta, in the Upload Apple Push Certificate section, click Browse to locate the renewed APNS that was just downloaded, then click Upload to complete the process.

     

Adding Users

If you already have imported your users, proceed to the following sections. If not, there are many ways to add users to your org. You can import them as described in Importing People or by individually adding them as described in Adding People in Using the Okta People Page. You can import users from your existing directories as well. Refer to the Directories and On-Premises Infrastructure documentation category for information on importing users from Active Directory, LDAP, and other directories.

Adding and Using Groups

Your Okta Mobility Management security policies are configured and enforced at the group level. Policies cannot be assigned to individual users. You can add groups in Okta or use groups that you have imported from directories or apps. Refer to the Groups section in Using the Okta People Page for more information about adding groups in Okta. For a complete overview of using groups in Okta including detailed descriptions of importing groups from directories, refer to Importing and Using Groups in Okta.

End User Setup

After you configure Okta Mobility Management and set up your security policies, your users can sign into Okta Mobile to enroll.

See Okta Mobility Management - End User Setup for enrollment instructions for your end users.

Now you have OMM configured, you need to configure one or more Mobile Policies, see Configuring Mobile Policies for details.
