Configure Okta MFA for PingFederate

You can extend Adaptive MFA to interoperate with PingFederate. This guide details how to configure PingFederate to interoperate with the Okta IDP MFA Adapter.

There are four steps to this installation.

Preconfiguration

Before starting, verify that your setup includes the following:

  • Windows Server 2012 or 2016 machine
  • PingFederate 9.0.1, configured as an identity provider
  • Java Runtime Environment (JRE) 8 update 161 or higher

Note: Okta has thoroughly tested the recommended configuration shown above. Under certain circumstances, the following other configurations can be configured for Okta MFA: PingFederate 9.0.3 with JRE 8, PingFederate 8.0 with JRE 8, and PingFederate 8.0 with JRE 7, update 131 or higher.

Install the Okta MFA Adapter
  1. Copy the file OktaPingFederateAdapter-1.0.0.jar into the folder <pf_install_directory>\pingfederate-9.0.1\pingfederate\server\default\deploy, where folder <pf_install_directory> is the PingFederate installation directory on your system.
  2. Copy the file okta.form.login.template.html into the folder <pf_install_directory>\pingfederate-9.0.1\pingfederate\server\default\conf\template.
  3. Restart the PingFederate service.
Create an App for the OIDC Flow
  1. From the Okta Dashboard, select Applications > Add Application > Create New App to start the wizard
  2. Select Web for the platform, select OpenID Connect for the Sign on method, and then click Create, as shown below.

  3. On the next screen, enter the application name (PingFederate OIDC App is suggested) and the login redirect URIs, as shown below. The redirect URI should be the protocol, followed by the host and port of your PingFederate installation. The default port on which PingFederate server listens is 9031. When done, click Save.

  4. The General tab opens Click the Edit button at the top right. In the Allowed Grant Types section, select Implicit (Hybrid), as show below. Allow ID Token with implicit grant type is automatically checked. When done, click Save.

  5. Scroll down to the Client Credentials section, as shown below. Note the Client ID and Client Secret. You will need them later.

  6. On the Assignments tab, assign the app to at least one user for testing. Assign other users and groups, as desired. For more information on adding and assigning apps, see The Applications Page.
  7. On the Sign On tab, scroll down to the Sign On Policy section, and add a Sign On rule for MFA. An example is shown below.

  8. Navigate to Settings > Customization and scroll down to the iFrame Embedding section. Click Edit and then select Allow iFrame Embedding, as shown below.

Create the Okta MFA Adapter Instance
  1. Go to admin.pingone.com and sign in with an account that has Administrator privileges.
  2. Navigate to Setup > Identity Repository > Settings > Admin Settings Link, as shown below. Click Edit.

  3. In the Identity Provider section, click Adapters under Application Integration, as shown below,

  4. Click Create New Instance, as shown below.

  5. Enter an Instance Name and Instance ID of your choice. From the drop down list for Type, select Okta IDP Adapter, as shown below.

  6. Enter appropriate values in the IdP Adapter configuration screen from Okta. These values are in your PingFederate MFA app in your Okta org. To find these values, see Step 5 in the Create an App for the OIDC Flow section above.

  7. Follow the prompts in the screens that follow and leave all the values at their defaults. When done, click Save, in the screen shown below.

Create the Composite Adapter

  1. Repeat steps 1-4 above in the Create the Okta MFA Adapter Instance section above. In the screen that opens, choose Composite Adapter from the Type drop down, as shown below.

  2. Add the PingOne HTML Form and Okta adapter instances to the list of Chained adapters within the composite adapter, as shown below. Configure the INPUT USER ID MAPPING fields as follows

    • Target Adapter: Okta
    • User ID Selection: Username

    This configuration causes PingFederate to pass the username field from the previous adapter into the Okta adapter.

  3. Select the Extended Contract tab, and click Add to add the following items, as shown below.

    • username
    • memberOf
    • sn
    • givenName

  4. Follow the prompts to save these settings.

Configure the SP Connection to Use the Composite Adapter
  1. Click PingOne in the Identity Provider screen, as shown below.

  2. Select the Assertion Creation tab, and then click Configure Assertion Creation, as shown below.

  3. Select the Authentication Source Mapping tab, and then click Map New Adapter Instance, as shown below.

  4. Choose the composite adapter you created in the previous section from the Adapter Instance drop down list.

  5. Select the Attribute Contract Fulfillment, and select Attribute Contracts that match your system definitions, as shown below. The values should match the following names that you defined previously.

    • username
    • memberOf
    • sn
    • givenName

  6. Optionally, delete the previously configured adapters for this SP connection. If you do not delete them, screens similar to the one shown below appear during log on, as the user must select from any configured adapters.

  7. Done! PingFederate is now configured with Okta MFA.

Test the Adapter
  1. Create a user in Okta with the same username prefix as one of the users in your AD setup, and assign them to the MFA for PingFederate app.
  2. Ensure that your organization has at least one MFA factor defined create one and that it is required in the app-specific sign-on policy for the MFA for PingFederate app.

  3. Enable Cross-Origin Resource Sharing (CORS) in your Okta org with your domain.

    Note: For more information about CORS, see CORS Overview.

    • Login to your Okta org.
    • Navigate to Security > API.
    • Select the Trusted Origins tab, then click CORS.

    • Select Add Origin.
    • Enter the Name and Origin URL information for your system, as shown below. When done, check the box for CORS, and click Save.

  4. Sign into the PingOne dock URL using the same User ID as in step 1 in this section. The enrollment screen appears, as shown below.

  5. Click Configure factor, as shown above, to enroll your device and sign in.

  6. Close your browser, and sign in again. The verification screen appears, as shown below.
