Importing People

As an administrator, you decide how Okta should be displayed and/or integrated with your org. through provisioning (i.e., from an application) or by using a CSV file.

Importing People Using Provisioning

You also have control over how imported users are confirmed and, through mapping rules, how application fields are mapped to an Okta username.

Provisioning must be enabled to use the following features. To enable provisioning, do the following:

  1. From the Administrator Dashboard select Applications and click the application you want to configure.

  2. Click the Provisioning tab and then click the Edit button.

  3. Scroll down to User Import.

Scheduling User Imports

When you import users from an application, you are creating new Okta users. Using the Schedule Import drop-down menu, you can specify how often an import should occur. This can span from every hour to every 2 days. Use the default selection, never, if you prefer to perform manual imports.

To perform an immediate import, do the following:

  1. From the Administrator Dashboard select Applications and click the application you want to configure.
  2. Click the Import tab.
  3. Click the Import Now button.
  4. After the import is complete, select the desired users and then click Confirm Assignments to queue them for activation. Confirmed users are sent an activation email.

Confirming Imported Users

When one or several users are imported into Okta, they must be confirmed before they are considered successfully imported. This can be a manual process performed under the Import tab, or completed automatically for each import.

Note: This feature is currently only available for Active Directory and Workday integrations.

From the Automatic confirmation section,

  • Select Never if you prefer to perform manual confirmation on the Import Results page under the Import tab.
  • Select New users if you want all new users to be automatically confirmed.
  • Click Immediately activate new user accounts on confirmation if you want the additional step of automatically activating confirmed users.

Import Safeguard

If an unusual number of app unassignments occurs during an import, Okta triggers an alert to warn against the unintended deprovisioning of a large number of apps from users within an org. The Import Safeguard feature stops the import and suspends subsequent imports.

Note: For the Import Safeguard feature to be triggered, an org must contain a minimum 100 app assignments.

When triggered, the following message appears on the Okta Dashboard.

If the app unassignment was expected

If these app unassignements were intentional, you can simply resume the import by clicking the Resume All Imports button. You can also view the event in the system log and, if you feel the current setting caused the unnecessary alert, adjust the threshold of allowed app unassignments. For details, see Increase the App Threshold below.

If the app unassignment was not expected

If the number of unassignments was not intentional, clicking the Cancel the Affected Import and Resume Other Imports button allows you to arrest imports from the offending app and resume other imports. Okta recommends viewing the event in the system logs and, if the issue remains unexplained, call Okta customer support.

Increase the App Threshold

If you click the set app assignment removal limit link, the following modal appears. It allows you to define a percentage of acceptable app unassignments before Import Safeguard is automatically triggered.

Note: This rate can also be changed from the Provisioning tab during the app provisioning process.

Importing People From a CSV File

For apps that don't support provisioning, you can import people directly from a CSV file.

  1. From the Administrator Dashboard, select Applications and click the application you want to configure.

  2. Click the Import tab.

  3. Click the Import from CSV button.

  4. Click on the CSV template link to download the template and enter your user's information.

    Note: You can add up to 5,000 users to your CSV file.

  5. Browse to your saved file and click on the Upload CSV button.

    Import begins automatically and a summary of your upload results appears.

  6. Click the Import users button.

  7. Confirm your imports and click on the people tab to begin assigning them.

For more details about importing people from a .csv file, see Importing Users From a CSV File.
