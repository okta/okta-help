Devices

The Devices page displays all the devices in your orgAn abbreviation of organization, but can also be thought of as a company. A company that uses Okta as their SSO portal is generally referred to as an org. As an administrator, you decide how Okta should be displayed and/or integrated with your org. that have at one time been enrolled in Okta Mobility Management (OMMAn acronym for Okta Mobility Management. OMM enables you to manage your users' mobile devices, applications, and data. Your users enroll in the service and can then download and use managed apps from the Apps Store. Managed apps are typically work-related, such as Box or Expensify. As an administrator, you can remove managed apps and associated data from users' devices at any time. You can configure policies, such as data sharing controls, on any of your managed apps. See Configuring Okta Mobility Management for more information.), with numerous options for display and depth of information about each device. The top-level Dashboard provides a wide overview of the device population, while the lower-level Dashboard displays device-specific data and a reporting option with .csv file output.

View Device Data

From the Devices > Overview page, you can view and search the status of the device population and deprovision or wipe (factory reset) them. You can also monitor all devices to see if they remain unencrypted, have been jailbroken or rooted. Clicking the buttons under the Actions column allows you to deprovision OMM functions or wipe the device directly from this overview page.

Deprovisioning

Deprovisioning removes all managed apps and configurations, and unenrolls the device from OMM. Deprovisioned devices are clearly listed under the Status column on the lower-level Dashboard page of the Devices page. When an end user is deactivated from Okta, their OMM-enrolled devices are automatically deprovisioned, and all company-managed apps and data are removed from the device in OMM. The device appears in the device report as "Deprovisioned". If the user is reactivated later, they must re-enroll their device.

To deprovision a device, go to Devices > Overview > device and click the Deprovision button under the Actions column.

End user deactivation

We recommend that, rather than having end users remove OMM from their own devices, you first deprovision the device. If an end-user removes themselves from OMM, the Okta servers may never know of the removal (due to a poor network or a device that is off-line). This creates a state where a device is enrolled, but deprovisioned. This behavior can exist for both rooted and non-rooted devices.

Note: Deprovisioning does not require end-user confirmation. After you deprovision users, their managed apps and configuration are removed. They see the effects of the deprovisioning the next time they sign in.

Jailbreak/Root Detection

Okta provides Jailbreak (iOS) and Root (Android) detection for OMM devices.

  • Jailbreak Detection: If an iOS device has been jailbroken, it means your end user has installed unsupported software on their device to either run custom software without Apple's permission, or to pirate copies of software or in-appAn abbreviation of application. Essentially, it is a web-based site used to perform any number of specific tasks, and requires authentication from end users by signing in. purchases.
  • Root Detection: If an Android device has been rooted, while it doesn't mean the end user has necessarily done anything inappropriate with their device, it does mean they have access to alter the firmware and/or OS that Google and the manufacturer/carrier shipped with the device.

Admins can quickly see if a device has been jailbroken or rooted directly from the Dashboard of the Devices page. Device Attributes Page Actions

There are several options that can only be accessed from the Device Attributes page of a device. To access a device's Device Attribute page, go to Admin Dashboard > Devices > Overview. From the lower-level Dashboard, select or search for the device name/user.

Clearing or Resetting Passcodes

When passwords are lost or need to be reset for any reason:

• For Android devices, the admin resets the passcode.

• For iOS devices, the admin clears the passcode and the end user must set up a new one themselves within one hour.

Note: This option has a minimum requirement of Okta Mobile 1.2.8 for Android or Okta Mobile4.2 for iOS.

To reset or clear a device's passcode remotely: 

  1. Select the Device Attributes page of the device whose password you want to clear or reset.
  2. Click the Device Actions button on the upper-right side of the page.
  3. Select Clear Passcode (for iOS) or Reset Passcode (for Android) .

iOS

For iOS devices, click Clear Passcode to confirm. From that point on, the end user has one hour to set a new passcode.

Android

For Android devices, enter the new passcode you want to assign to this device, then click Reset Passcode.

Remote Locking

Admins can remotely lock a device. This option is useful for when devices appear to be in an uncertain state, but wiping (factory reset) is not warranted.

  1. Select the Device Attributes page of the device you wish to remotely lock.
  2. Click the Device Actions button on the upper-right side of the page.
  3. Select Remote Lock.
  4. Optionally send a message and contact number to the device. For example, "Please return this phone to Tom B" and a contact number.
  5. Click the Remote Lock button.
Reporting

For granular data about each device, you can run a report by clicking the Download CSV button. The resulting .csv file provides a very comprehensive output of each device, including device status, serial number, platform, device capacity (MB), and available device capacity (to name a few).
