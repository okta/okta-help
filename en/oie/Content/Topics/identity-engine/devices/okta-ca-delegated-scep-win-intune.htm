Configure Okta as a Certificate Authority for Windows using Intune

Configuring a Certificate Authority (CA) allows you to issue client certificates to your targeted Windows devices. This topic describes how to create a SCEP profile in Microsoft Intune and generate a SCEP URL in Okta.

 

To configure a static SCEP challenge type for Windows using Workspace ONE, see Configure Okta as a Certificate Authority for Windows using Workspace ONE

 

Before you begin

  • Okta tested this configuration using AD CS and NDES integrated with Microsoft Intune
  • Deploy certificates for digital signature and but not for other purposes, such as encryption

Make sure you have access to the following:

  • Okta Admin Console
  • Microsoft Endpoint Configuration Manager Admin Center (MECM, formerly SCCM)
  • Microsoft Azure

Start this procedure

  • Task 1: In Okta, download the x509 certificate
  • Task 2: In MECM, create a Trusted Certificate profile
  • Task 3: In Microsoft Azure, register AAD app credentials for Okta
  • Task 4: In Okta, configure management attestation and generate a SCEP URL
  • Task 5: In MECM, create a SCEP profile
  • Task 6: On a Windows computer, verify the certificate installation

Task 1: In Okta, download the x509 certificate

  1. In the Admin Console, go to Security > Device Management.
  2. Click the Certificate Authority tab.
  3. Click the Download icon for Okta CA in the Action column to download the x509 certificate. You will upload the certificate to Microsoft Endpoint Configuration Manager (MECM) in Task 2.

Task 2: In MECM, create a Trusted Certificate profile

  1. In Microsoft Endpoint Configuration Manager, go to Devices.
  2. Click Configuration profiles.
  3. Click + Create profile.
  4. In Create a profile, select:
    1. Platform: Windows 10 and later
    2. Profile: Trusted certificate
    3. Click Create.
  5. In the Trusted certificate wizard:
    1. Enter a name and (optionally) a description.
    2. Click Next.
    3. Select the x509 certificate that you downloaded from Okta in Task 1.
    4. In Destination store, select Computer certificate store - Intermediate.
    5. Click Next.
    6. Assign the Trusted certificate profile to one or more user groups. The user group(s) must be the same to which you assign the SCEP profile in Task 5

      7. Make sure the user group(s) specified in both profiles are the same.

    7. Click Next.
    8. Set Applicability Rules.
    9. Click Next.
    10. Review the configuration, and then click Create.

Task 3: In Microsoft Azure, register AAD app credentials for Okta

  1. In Microsoft Azure, go to App registrations.
  2. Click New registration.
  3. Enter a user-facing display name for Okta Verify.
  4. In Supported account types, select Accounts in this organizational directory only (Okta only - Single tenant).
  5. Click Register.
  6. In the app page under Essentials, copy and make a note of the following values. You will paste these values in the Okta Admin Console in Task 4.
    • Application (client) ID
    • Directory (tenant) ID
    • Object ID
  7. Add a client secret:
    1. In the left pane, click Certificates & secrets.
    2. Under Client secrets, click + New client secret.
    3. In Add a client secret:
      • Add a Description (optional).
      • Specify an expiration period.
    4. Click Add.The secret appears under Client secrets.
    5. In the Client secrets section, copy and make a note of the Value.
  8. Set API permissions:
    1. In the left pane, click API permissions.
    2. Click + Add a permission.
    3. In Request API permissions scroll down and click the Intune tile.
    4. Under What type of permissions does your application require? click Application permissions.
    5. In the Select permissions search field, search for scep, and then select scep_challenge_provider in the search results.
    6. Click Add permissions.
    7. In the Configured permissions section, click Grant admin consent for Okta and then click Yes in the message that appears.
    8. Click scep_challenge_provider permissions.
    9. Close the left pane.

Task 4: In Okta, configure management attestation and generate a SCEP URL

  1. In the Admin Console, go to Security >Device Management.
  2. Click the Device Management tab.
  3. Click Add Platform.
  4. In Select platform, select Desktop (Windows and macOS only).
  5. Click Next.
  6. In Configure management attestation:
    1. Certificate authority: Select Use Okta as certificate authority
    2. SCEP URL challenge type: Select Delegated SCEP URL (Microsoft Intune only).
    3. In the following fields, enter the values that you copied from Microsoft Azure in Task 3:
      • AAD client ID
      • AAD tenant
      • AAD secret
    4. Generate, copy, and make a note of the Okta SCEP URL. You will paste the URL in Microsoft Endpoint Configuration Manager in Task 5.

      Important: Make sure to make a note of the SCEP URL as this is the only time it will appear in Okta.

Task 5: In MECM, create a SCEP profile

  1. In Microsoft Endpoint Configuration Manager, go to Devices.
  2. Click Configuration profiles.
  3. Click + Create profile.
  4. In Create a profile:
    1. Platform: Windows 10 and later
    2. Profile: SCEP certificate
    3. Click Create.
  5. In the SCEP certificate wizard enter a name and (optionally) a description.
  6. Click Next and configure settings:
    1. Certificate type: User
    2. Subject name format (recommended; other formats will also work): CN={{UserPrincipalName}} ManagementAttestation {{AAD_Device_ID}}
    3. Key storage provider: Enroll to Trusted Platform Module (TPM) KSP if present, otherwise Software KSP
    4. Key usage: Digital signature
    5. Key length: 2048
    6. Hash algorithm: Select SHA-2
  7. Click + Root Certificate.
  8. In the Root Certificate pane, select the trusted certificate that you created earlier in Task 2, and then click OK.
  9. Under Extended key usage, set Predefined values to Client Authentication.
  10. Copy the SCEP URL you generated in Task 4 and paste it in the SCEP Server URLs field.
  11. Click Next.
  12. Assign the SCEP certificate to the same user group(s) to which you assigned the Trusted certificate profile in Task 2.

    13. Make sure the user group(s) specified in both profiles are the same.

  13. Click Next.
  14. Set Applicability Rules.
  15. Click Next.
  16. Review the configuration, and then click Create.

Task 6: On a Windows computer, verify the certificate installation

Verify client certificate installation

  1. On the Windows computer, click Start and type cert and then click Manage user certificates.
  2. Look in Personal > Certificates.

Verify Certificate Authority

  1. On the Windows computer, click Start and type cert and then click Manage user certificates.
  2. Look in Intermediate Certificate Authority > Certificates.
  3. In Issued To, find and double click on Organization Intermediate Authority.
  4. See Issuer: Organization Root Authority.

If you don't find the certificate, check logs as described in Verify successful SCEP certificate installation and flow.

Verify successful SCEP certificate installation and flow

  1. On the Windows computer, click Start and type Event and then click Event Viewer.
  2. Look in Applications and Service Logs > Microsoft > Windows > DeviceManagement-Enterprise > Admin.
  3. In the General tab, find:
    • SCEP: Certificate installed successfully.
    • SCEP: Certificate request generated successfully
