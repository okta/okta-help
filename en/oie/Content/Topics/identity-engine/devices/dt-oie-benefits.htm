Device Trust on Okta Identity Engine benefits

Device Trust on Okta Identity Engine offers many benefits over Device Trust on Okta Classic.

 

 Device Trust in Okta Classic Device Trust on Okta Identity Engine
All platforms Enforce Device Trust on SAML and WS-Fed apps
  • Enforce Device Trust on SAML, WS-Fed, and OIDC apps
  • A unified approach built around the Okta Identity Engine-powered Okta Verify app for assessing the device posture of desktop and mobile devices by collecting trust signals.
  • Option to integrate with Okta Verify to deliver passwordless login experiences to any user on any device
  • Support for integrating with endpoint security tools
  • Simplified deployment with no dependencies on any specific on-prem directory (such as Active Directory, LDAP) or endpoint management tool, providing a vendor-agnostic approach across all platforms
Windows

Client-based approach that requires:

  • Active Directory
  • Okta IWA agent
  • Okta Certificate Authority (CA)
  • An msi or exe file for certificate deployment which is not used for other Okta functions

Limited support for:

  • Microsoft Intune-managed or other lightweight device management-based Windows management tools
  • Azure Active Directory-only managed devices

No dependency on:

  • Active Directory
  • Okta IWA agent

Strong support for:

  • Microsoft Intune and other lightweight device management-based Windows management tools
  • Azure Active Directory-only managed devices

Choice of Certificate Authority (CA) options:

  • 3rd party CAs (upload your Root CA certificate)
  • Use Okta as a CA
macOS

Client-based approach requires:

  • Integration with Jamf Pro
  • Requires Okta Certificate Authority (CA)
  • Integrate with any macOS management tool 

Choice of Certificate Authority (CA) options:

  • 3rd party CAs (upload your Root CA certificate)
  • Use Okta as a CA
Android and iOS
  • Client-based approach requires Okta Mobile which may not be widely deployed to users in some organizations.
  • Okta Device Trust isn't supported on Android devices running Android Q and later.
 Client-based approach built around Okta Verify which can also serve as a second factor for MFA and therefore more likely to be widely deployed to your user base.
