Windows Okta Verify installation options

You can silently install Okta Verify on Windows devices, as an alternative to end users manually installing it.

Before you begin

End users might see the following notice and prompt:

  • End user license agreement (EULA) and Crash Reporting notice

If you install the app silently on end user Windows devices (whether through your MDM or Microsoft Endpoint Configuration Manager (MECM)), end users are not shown the EULA nor the Crash Reporting screen. Users who want to see those notices need to uninstall Okta Verify and then reinstall it by double-clicking the Okta Verify Setup file. The EULA and the Crash Reporting screens are shown during reinstallation.

  • User Account Control (UAC) prompt
    • MECM – If you use MECM to silently install Okta Verify on end-user Windows devices, end users are shown the UAC consent and credential prompts.
    • MDM – If you configure your MDM to silently install Okta Verify on end-user Windows devices, end users are not shown UAC prompts.

Choose your installation option

  • Install Okta Verify using your MDM solution
  • Install Okta Verify using MECM

Install Okta Verify using your MDM solution

Perform the steps provided in the Microsoft Intune document Add a Windows line-of-business app to Microsoft Intune. Make sure to select the Device Context option in Intune (not User Context).

Install Okta Verify using MECM

Installation occurs on a system level; user-based installation is not supported.

  1. Copy the file Okta Verify Setup file to your MECM (SCCM) server.
  2. Launch Configuration Manager Console.
  3. Go to \Software Library\Overview\Application Management\Applications.
  4. Launch the Create Application Wizard by right-clicking Applications and then selecting Create new application.
  5. Select Automatically detect information about this application from installation files.
  6. In Type, select Windows Installer (*.exe or *.msi).
  7. In Location, browse to the Okta Verify file.
  8. Click Next.
  9. Follow the onscreen prompts.
  10. In the Specify information about this application screen, do the following:
    1. Enter a name.
    2. Specify the installation program:

      3. Option 1: Install without specifying installer flags

      EXE

      OktaVerifySetup-1.x.x.x-yyyyyyy.exe /q2

      MSI

      msiexec /I OktaVerifySetup-1.x.x.x-yyyyyyy.msi EXEOPTIONS=”/q2”

      Option 2: Install and specify installer flags

      Available values for the log level are None, Critical, Error, Warning, Info, Debug.

      EXE

      OktaVerifySetup-1.x.x.x-yyyyyyy.exe -q2 [<OPTION1>=<Value1>] [<OPTION2>=<Value2>] [...]

      MSI

      msiexec /I OktaVerifySetup-1.x.x.x-yyyyyyy.msi /qn EXEOPTIONS="-q2 [<OPTION1>=<Value1>] [<OPTION2>=<Value2>] [...]"

EnrollmentOptions
  • SilentEnrollmentDisabled: Default. Users are not prompted to add an Okta Verify account unless they click Sign in with Okta Verify.

  • Enabled: Users are always prompted to add an Okta Verify account.

  • Disabled: Users are never prompted to enroll in Okta Verify unless they open the app and click Add an account.

SilentEnrollmentDisabled

2.0.1

These options allow customers to control when users are prompted to enroll in Okta Verify. This is useful if you want to reduce the number of user prompts and/or control the rollout of Okta Verify and Okta FastPass in your org.
  1. Click Next.
  2. Confirm settings and click Next.
  3. At the success screen, click Close.
  4. In the center pane of the Applications list, right-click the app and select Deploy.
  5. Follow the prompts in the Deploy Wizard.
