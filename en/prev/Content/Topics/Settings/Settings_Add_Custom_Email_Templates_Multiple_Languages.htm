Add Custom Email Templates for Multiple Languages

Settings > Email & SMS

Custom email templates allow you to send end usersIn Okta literature, we generally refer to "end users" as the people who have their own Okta home page (My Applications), using chiclets to authenticate into all of their apps. End users do not have any administrative control. When we refer to "users" we are generally referring to the individual(s) who have administrative control. custom Okta-generated email messages in multiple languages.

Procedure
  1. Go to Settings > Email & SMS.
  2. Click the Email tab.
  3. In the left pane, select the email template for which you want add a new translation.
  4. Under Custom Language, click Add Translation.

    Tip: You can copy the html tags from the Default template

    1. Click the pencil icon to the right of the Default template. Screenshot

    2. Copy the body of the message and then click Cancel to close the Default email. Screenshot

    3. Paste the copied html message into a text editor and compose your translation.

  5. Click Add Translation.

  6. Select a language in the Language drop-down menu.

  7. Enter a Subject.

  8. Paste your custom translation into the message body. Screenshot

  9. Click Add Translation to save your changes.

    When you add a translation, or save a change to the default template, the email template functionality enters Custom mode, indicated by the word Custom in the Default template label. Screenshot

  10. (Optional) If you want to delete all custom translations and revert to the standard Okta template, click Reset to Default.
Examples

Suppose you added two custom email templates, one in French, another in Japanese. You would then have templates in three languages:

  • Default (Custom)
  • French
  • Japanese

Which users will receive the French translation?

  • (First priority) UsersIn Okta literature, we generally refer to "users" as the people who serve as Okta administrators. When we refer to "end users" we are generally referring to the people who the administrators serve. That is, those who use Okta chiclets to access their apps, but have no administrative control. who have set their Display Language to French

  • (Second priority) Users whose locale property is fr_FR

Which users will receive the Japanese translation?

  • (First priority) Users who have set their Display Language to Japanese

  • (Second priority) Users whose locale property is ja_JP

Which users will receive the Default (Custom) template?

Everyone else will receive the Default (Custom) template.

If a user's Display Language is French and their locale property is Japanese (ja_JP), which translation of the email will they receive?

They'll receive the French translation because their Display Language setting has precedence (first priority).

Troubleshooting

Users received emails in the language of the Default (Custom) template instead of an added translation, as intended.

Users receive email in the language of the Default (Custom) template when:

  • Their Display Language is set to a language that does not appear in the Custom Language list. For example, the user has set their Display Language to German, but the adminAn abbreviation of administrator. This is the individual(s) who have access to the Okta Administrator Dashboard. They control the provisioning and deprovisioning of end users, the assigning of apps, the resetting of passwords, and the overall end user experience. Only administrators have the Administration button on the upper right side of the My Applications page. did not provide a German translation for that template. Screenshot

  • There is no locale property configured in the user's profile, or the user's locale property is set to the same language as the Default (Custom) template.

User receive emails in a different language than expected.

Check the user's locale property and ask them to check their Display Language setting. Remember, the user's Display Language setting is first priority, so it takes precedence over all other language settings.

Users no longer receive emails in the standard Okta-provided Default language that was selected before a custom translation was added.

Standard Okta-provided translations are not available when either of the following are true:

  • The standard email template was modified.
  • One or more translations were added.

Either of the above conditions causes the email template functionality to enter Custom mode, which is indicated by the word Custom in the Default template label. Screenshot
