Manage Dashboard Tabs for End Users

Settings > Customization

You can create dashboard tabs and apply them to end usersIn Okta literature, we generally refer to "end users" as the people who have their own Okta home page (My Applications), using chiclets to authenticate into all of their apps. End users do not have any administrative control. When we refer to "users" we are generally referring to the individual(s) who have administrative control. as follows:

  • Create dashboard tabs for new end usersIn Okta literature, we generally refer to "users" as the people who serve as Okta administrators. When we refer to "end users" we are generally referring to the people who the administrators serve. That is, those who use Okta chiclets to access their apps, but have no administrative control. that have not yet been activated and choose the apps that you want to appear on them. This allows you to manage the first day experience for new end users.
  • Create dashboard tabs and apply them to all activated end users in your orgAn abbreviation of organization, but can also be thought of as a company. A company that uses Okta as their SSO portal is generally referred to as an org. As an administrator, you decide how Okta should be displayed and/or integrated with your org.. For important details, see below.

Key features
  • Create up to three additional tabs
  • Drag and drop to move chicletsThe "buttons" that appear on an end user's Home page and represent each application they wish to access through Okta. Clicking the chiclet allows the end user to instantly sign in and authenticate themselves into their chosen app. within the current tab or to another tab
  • Apply the dashboard to new end users that have not been activated, or to all activated end users
  • Preview individual end users' dashboard
  • (Applies only to new end users who have not been activated) Provide different dashboards for different sets of new, unactivated end users. For example, you can configure a dashboard for one set of new end users, activate those end users; then configure a different dashboard for another set of new end users, and then activate them. Each set of new end users has a different dashboard. Note: To ensure that new end users see apps on the specific tab where you placed them using the managed dashboard feature, you must assign these apps to end users and apply the custom dashboard before you activate the end users. Apps you assign to already-activated end users appear on the initial tab. Screenshot

Procedure
  1. Go to Settings > Customization.
  2. Click End-user dashboard.

    After end users are activated, they will only see the apps that were assigned to them before they were activated.

  3. You can do any of the following:
    • Create a tab. Click the + sign, enter a name, and then click Save.You can create up to 3 additional tabs.
    • Rename a tab. Hover over a tab and click the pencil icon.
    • Move apps. Drag and drop apps within the current tab or to another tab.
    • Delete a tab. First remove all the apps out of the tab, and then click Delete.
    • Preview a users' homepage. After your dashboard updates are complete, you can use the Preview field at the bottom of the page to search for end users and preview their dashboards. Visibility into end users' dashboards can help you answer questions and resolve issues. Only company-provided apps are visible to admins in this preview; you cannot see self-service apps or end users' personal tabs.
    • Apply dashboard to all users or new users. For details, see below.
  4. To apply your changes, click Apply dashboard to all users or Apply dashboard to new users.

    If you Apply dashboard to all users, note the following:

    • All of your end users' personal apps are consolidated into a single a tab titled Personal Apps.
    • Applying dashboard changes to all end users overrides the current dashboard configuration for all activated end users in your org, and all their appAn abbreviation of application. Essentially, it is a web-based site used to perform any number of specific tasks, and requires authentication from end users by signing in. chiclets are moved to new locations. These changes cannot be undone.
    • It takes some time for your dashboard updates to complete. You cannot edit the dashboard during this time.
    • A message displays before your dashboard updates are applied asking if you are sure that you want to continue.
    • Okta automatically sends an email to the adminAn abbreviation of administrator. This is the individual(s) who have access to the Okta Administrator Dashboard. They control the provisioning and deprovisioning of end users, the assigning of apps, the resetting of passwords, and the overall end user experience. Only administrators have the Administration button on the upper right side of the My Applications page. who applied changes to all end users. The email acknowledges that the new dashboard was applied. If any apps are experiencing errors, a unique ID is provided in the email that you can give to Okta Support for remediation.
    • A banner appears on end users' home page to notify them when their admin has customized their dashboard. Screenshot

  5. If you are applying the dashboard to new end users, make sure to activate them.
Caveats
  • Applying dashboard changes to all end users overrides the current dashboard configuration for all activated end users in your org, and all their app chiclets are moved to new locations. These changes cannot be undone.
  • All apps on end users' personal tabs are consolidated into a single tab titled Personal Apps.
  • You cannot assign or delete apps from the End-user dashboard feature. For these operations, see The Applications Page.
