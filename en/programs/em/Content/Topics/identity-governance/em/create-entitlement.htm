Create entitlements

Entitlement creation is a two-part process. First, create custom attributes that your downstream applications can use from the Profile Editor page. Next, promote them to Governance Engine to use the attributes as entitlements in Okta. Okta doesn’t allow you to create entitlements directly.

Prerequisites

  • Sign in as a super admin and ensure that you're assigned to the Okta Entitlement Management application.

  • Ensure that you’ve enabled Governance Engine for the app you want to create entitlements for. See Enable Governance Engine.

  • Refer to Add custom attributes to apps, directories, and identity providers.

  • Keep the following considerations in mind to avoid errors:

    • Entitlement Management supports the Personal attribute type, but not the Group attribute type.

    • Entitlement Management doesn't support dragging and dropping groups to establish group priority for custom attributes you've promoted as entitlements. It considers the priority displayed for the group on the application's Assignments tab as the group's priority order. To prioritize groups, add the groups in the order of priority.

    • For non-promoted attributes, group priority still applies. However, priority order is static and can't be modified.

    • Don’t mark an attribute as required. Marking it as required can result in incorrect or failed entitlement assignments.

    • Previously promoted entitlements aren’t updated when you edit attribute values from the Profile Editor page.

    • Adhere to the following character limits to avoid errors in promoting entitlements:

      • Entitlement name: 255 characters

      • Entitlement value: 1024 characters

      • Entitlement description: 1000 characters

Create custom attributes for entitlements

  1. In the Admin Console, go to DirectoryProfile Editor.

  2. In the Actions column, click Profile.

  3. Click + Add Attribute. Populate the fields on the Add Attribute dialog as required. Keep the following considerations in mind:

    1. Set the Data type to String or String array.

    2. Enter a display name and a variable name. The variable name must be a value that your downstream application can map to and consume.

    3. Select Define enumerated list of values checkbox.

    4. Enter a display name and a variable name for the Attribute members field. The variable name must be a value that your downstream application can map to and consume.

    5. Don’t mark this attribute as a required attribute. The attribute can’t be promoted as an entitlement if you select Yes.

    6. Select the Attribute type as Personal. Entitlement management doesn’t support the Group attribute type.

  4. Click Save or Save and add another.

Promote attributes as entitlements

  1. In the Admin Console, go to ApplicationsApplications.

  2. Select an application.

  3. Go to the Governance tab, and then click Continue.

  4. On the Entitlements tab, click Add entitlements to use existing custom attributes as entitlements.

  5. Select custom attributes that you want to use as entitlements.

  6. Click Add entitlements. It can take a few minutes for the entitlements to appear. You may have to refresh the page.

After you promote attributes, changes that you make to the custom attribute aren’t reflected in the entitlement. You can’t edit the entitlement directly. To make changes, you must delete the entitlement, modify the attribute, and promote the attributes as an entitlement again.

Next, create an entitlement policy using Okta Workflows to assign entitlements to groups. See Create an entitlement policy. You can also create entitlement bundles and configure Request Types for your users to request bundles. See Create an entitlement bundle.

You may need to provision entitlements to users in the downstream application using Workflows after you create entitlements and before you assign entitlements to users.
