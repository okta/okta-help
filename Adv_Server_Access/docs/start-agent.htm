Start the server agent

If the server agent is stopped, you must manually start the service so it can reestablish a connection with .

Operating System Process
Systems that use systemd Run sudo systemctl start sftd
Systems that use upstart Run  sudo service sftd start
Windows From the command line, run  start-service sftd

Start the agent on systems that use systemd to manage services

On systems that use the systemd system and service manager, you can start the agent by entering:

sudo systemctl start sftd

Start the agent on systems that use upstart to manage services

On systems that use upstart to manage services, you can start the agent by entering:

sudo service sftd start

Start the agent on Windows

You can start the agent on Windows by using Windows PowerShell or from the Services app.

To start the agent by using a Windows PowerShell:

  1. Open Windows PowerShell.
  2. Enter the following command:start-service sftd

To start the agent by using the Services app:

  1. Open the Services application.
  2. Right-click ScaleFT Server Tools and select Start.
