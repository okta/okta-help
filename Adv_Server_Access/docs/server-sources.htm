Server sources

The Source and Source Details fields provide additional information to help teams identify discovered and enrolled servers.

Source Details Description
AD Discovered Lists the associated AD connection. Servers discovered within an Active Directory domain. See Active Directory server discovery.

These servers are not managed by the server agent.
Cloud Discovered Lists the associated cloud account ID. Servers discovered by the AWS server discovery feature. See AWS server discovery.

These servers are not managed by the server agent.
Cloud Managed Lists the associated cloud provider and, if available, the cloud account ID.

Servers that indicate that they belong to a cloud provider such as AWS or GCP. These servers have installed the server agent and are enrolled either via an enrollment token or with a linked cloud account. See Deploy cloud servers.
Managed - Servers that have installed the server agent and enrolled via an enrollment token.
Unknown - Servers with multiple sources, undefined origins or bad states. For example, if a Cloud discovered server has also installed the server agent it would be included here. This is uncommon.
Unmanaged - Servers added manually through the dashboard or API.

These servers are not managed by the server agent.
