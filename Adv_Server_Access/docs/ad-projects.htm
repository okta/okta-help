Configure projects for Active Directory support

Before you can discover or connect to your Active Directory (AD) servers, you must configure AD support for one or more projects.

Normally, creates and manages local user accounts for every server. AD-Joined allows you to configure delegated authentication for your projects and eliminates the need for to create and manage local accounts and allows users to access servers with their existing AD credentials.

When you configure your project for AD support, you must also forward all traffic through a gateway. Okta recommends configuring your network and gateway to restrict direct access to AD servers; doing so allows you to use for MFA and auditing purposes, and ensures the security of your network. This will likely involve additional configuration of your network and gateway.

During Early Access, AD support is allowed for projects that only contain Windows servers without the server agent installed. You can't add AD support for projects that already contain Linux servers, FreeBSD servers, or Windows servers that have installed the server agent. After configuring a project to support AD, you can’t reverse the change. Okta recommends that you always create new projects for use with AD-managed servers.

Existing projects

You can't disable AD support after it's enabled for a project.

  1. Open the dashboard.
  2. Go to Projects.
  3. Identify a project and click Edit.
  4. Select an existing gateway connected to the AD domain.
  5. Select Enable Active Directory Support.
  6. Select Enable traffic forwarding.
  7. Click Update.

enables AD support for the project.
