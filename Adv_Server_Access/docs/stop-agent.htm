Stop the server agent

The server agent on enrolled servers checks in with the system on a regular basis. After four days of no contact from a server, hides the server in the dashboard. You can halt communications between a server and by stopping the server agent service on the device. If the agent is restarted before the server's authentication token expires, it will reappear in the list of enrolled servers.

Operating System Process
Systems that use systemd Run sudo systemctl stop sftd
Systems that use upstart Run  sudo service sftd stop
Windows From the command line, run  stop-service sftd

Stop the agent on systems that use systemd to manage services

On systems that use the systemd system and service manager, you can stop the agent by entering:

sudo systemctl stop sftd

Stop the agent on systems that use upstart to manage services

On systems that use upstart to manage services, you can stop the agent by entering:

sudo service sftd stop

Stop the agent on Windows

You can stop the agent on Windows by using Windows PowerShell or from the Services app.

To stop the agent by using a Windows PowerShell:

  1. Open Windows PowerShell.
  2. Enter the following command:

    stop-service sftd

To stop the agent by using the Services app:

  1. Open the Services application.
  2. Right-click ScaleFT Server Tools and select Stop.
