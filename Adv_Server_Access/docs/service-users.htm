Service users

Service users enable you to grant permissions to your automation to access specific operations in , including:

  • Enabling trusted services to be granted access to your infrastructure
  • Integrating with the API to automatically add users to groups
  • Retrieving audit events from the API to store in your ELK setup
  • Building other custom integrations with the API

You can add service users to groups and grant them the same permissions as regular users. The main difference between a user and service user is how they authenticate.

While users are pushed from , which handles their authentication, each service user is given a pair of credentials that are used to generate a short-lived authentication token to be used with the API.

Authentication

In order to authenticate as your service user to the API, you will need to create an API key. The API key is a pair of strings known as the ID and secret. You will need both to authenticate, which generates an authentication token that is sent with each request you make.

Expire a service user API key

Immediately upon expiring an API key, any requests that use a token generated using the key are prevented from succeeding.

To expire the API key of a service user:

  1. From the dashboard, click Users.
  2. Select the Service Users tab.
  3. Click the name of the service user whose key you want to expire. The service users details page appears.
  4. Click the gear next to ID of the expiring key. Click Expire Now. The Expire API Key window appears.
  5. Click Expire to expire the API key ID, or click to cancel the operation.
