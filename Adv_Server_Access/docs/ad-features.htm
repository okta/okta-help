Features and usage

Features

Feature Description
Delegated authentication for RDP connections Normally, creates and manages local accounts for members of a team. After configuration, users can launch RDP sessions with an existing AD account. See Configure Active Directory account mapping.
Centralized AD connections centrally stores all authentication and routing information related to a specific AD domain on the Connections page. Teams can add multiple connections to support multi-domain environments. See Create an Active Directory connection.
Passwordless authentication Teams can add signed certificates to allow users to launch RDP sessions without needing to enter their AD password. An AD domain controller can sign the certificates or teams can use self-signed certificates for testing purposes. See Passwordless certificates.
User discovery

User sync jobs enable teams to discover AD users and their corresponding Security Identities (SIDs). See [%=3rd Party Integrations.AD%] user discovery.
Server discovery Server sync jobs let teams automatically sync (add new servers, update details for existing servers, and remove unavailable servers). See Server discovery.
Unmanaged server connections

Teams can connect to discovered servers without installing the server agent. See AD-Joined server connections.
Auditing records event data and allows teams to track when servers are added, removed, or accessed by users.
Integrated PolicySync AD-Joined integrates support for PolicySync, a separate feature. PolicySync adds fine-grained access control for projects. AD-Joined can use PolicySync to locate and import metadata from AD for use as server labels used via selectors. See [%=okta-feature-names.PolicySync%]: Attribute-Based Access Control.

Supported usage scenarios

Scenario Account type Server agent required? Connection type Sign in Experience
User connects directly to an managed Windows server Local account Yes Direct Password required
User connects through an gateway to an managed Windows server Local account Yes Gateway Password required
User connects through an gateway to an unmanaged Windows server with an AD account. Users must manually enter their AD password. AD account No Gateway SSO
User connects through an gateway to an unmanaged Windows server with an AD account. Users don't need to enter a password. AD account No Gateway SSO

Limitations

  • Limited server discovery to 10,000 computers for all connections
  • Limited server discovery to 100 assignment rules per job and ten jobs per connection
  • Limited the maximum number of server labels to 10 AD attributes
  • Projects can only contain Windows servers without the server agent installed
    These projects can't contain Linux servers, FreeBSD servers, or Windows servers managed by the server agent.
  • gateways configured for RDP connections must run Ubuntu 20.04 (x86 only)
  • Previously created gateways must update to v1.55.1 or later and create a resource list entry for Ubuntu 20.04
    For details, see Install the [%=okta-feature-names.ScaleFT%] gateway on Ubuntu or Debian
  • Limited gateways to a maximum of 20 simultaneous RDP connections by default
    Teams can adjust this setting.
  • Removed support for RDP connections with the RoyalTSX client
