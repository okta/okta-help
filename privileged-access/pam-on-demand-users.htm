On-demand users

creates an account when a user attempts to access a server. While the account is active, on-demand users have regular user permissions (non-administrative permissions) when using their account. After a session ends, the server agent removes the account.

On Windows servers, the associated user account and home directory are removed when the account expires. This also includes any data stored within the home directory.

You can configure on-demand users for a project at any time by configuring the On Demand User TTL (Time to Live) setting. See Projects.

If you enable on-demand users, you must make the server accessible through port 4421 of the previous network hop. For connections through a bastion or gateway, the server must be accessible from port 4421 of the bastion or gateway. For direct connections, the server must be accessible from port 4421 on the client. For more information, see Configure the [%=okta-feature-names.Okta Privileged Access%] server agent.
