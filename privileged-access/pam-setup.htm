Set up

To set up , you must add the OIN application to your Okta org, add an admin user, enable provisioning, and assign users and groups.

Prerequisites

You must have an admin role in your org.

Task 1: Add app to your org

You can only install one instance of to your Okta org. This set up can be performed only once for every Okta org. the same org.

  1. From the Okta , go to Applications > Applications.
  2. Click Browse App Catalog.
  3. Search for in the search field and select it from the list.
  4. Click Add Integration.
  5. Enter a team name.
  6. Click Done.

Once app is integrated to your Okta org, the single sing-on (SSO) is automatically enabled. To view the SSO configuration, click the Sign On tab on your app.

Task 2: Assign an admin to

  1. From the Applications page, click .
  2. Go to the Assignments tab and click Assign > Assign to People.
  3. Identify your account in the list and click Assign.
  4. Click Save and Go Back, then click Done.

Task 3: Verify SSO and user configuration

After you successfully install the app, it will appear on your dashboard under My Apps. The single sign-on is automatically configured. You can now sign in to the app.

  1. From your dashboard, click the app.
  2. In the , you can view assigned Users and Groups. At this stage, one user is registered, and admins are not configured. We will add more users and groups in the next steps.

Task 4: Enable provisioning

  1. Go to the Provisioning tab and click  Configure API Integration.
  2. Select Enable API Integration.
  3. Click Authenticate with .
  4. On the dialog that appears, select you team.

  5. Enter a name for a service account and click Approve. Any name such as svc-scim can be used. The service account will be automatically created in for use by the Okta SCIM integration.

  6. Click Save.
  7. Click Edit on Provisioning to App page.
  8. Select Create Users, Update User Attributes, and Deactivate Users provisioning options.
  9. Click Save.

Task 5: Assign users and groups

You can now assign users and groups to the app in your Okta org. You can create new groups that reflect the admin structure in . For example, create a PAM Admin group, Security Admin Group, and Resource admin group, assign these groups to the app, and push the groups to your Okta Privilege Access team.

  1. Go to the Assignments tab.

  2. Users added to the will display.

    To assign users or groups to an app, see Assign an app integration to a user and Assign an app integration to a group.

Task 6: Push groups to

You can use Group Push to push existing Okta groups and their memberships to Okta Privileged Access. See Configure group sync.

After the Group Push operation is completed, user groups pushed from the Okta org to Okta Privileged Access will be visible under Okta Privileged Access. Check Directory Users and Directory Groups.

This confirms the integration is working and you can now start configuring .
