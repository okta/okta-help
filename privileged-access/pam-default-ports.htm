port requirements

In order to provide access to server resources, teams must allow traffic through several different network ports.

client

Port Description
22 Used for outgoing SSH connections to servers.
443 Used for outgoing connections to and the platform.
4421 Used for outgoing RDP connections to servers.
7234 Used for outgoing connections to gateways.

server agent

Teams can modify the default ports through the server agent configuration file. See Configure the [%=okta-feature-names.Okta Privileged Access%] server agent.

Port Description
22 Used for incoming SSH connections.
443 Used for outgoing connections to and the platform.
3389 Used locally on Windows servers for RDP loopbacks. Does not need to be publicly available.
4421 Used for incoming connections to help provision On Demand users. See Projects.

On Windows servers, this port is also used to proxy RDP sessions to port 3389.

gateway

Teams can modify the default ports through the gateway configuration file. See Configure an [%=okta-feature-names.ScaleFT%] gateway.

Port Description
443 Used for outgoing connections to and the platform.

Also used for outgoing connections to AWS or GCP if session capture stores logs in a cloud bucket. See Session capture.
7234 Used for incoming connections from the client.

3389

Used for outgoing connections when using AD-Joined. It is not required when using agent.

53

Used for resolving hostnames through DNS.

389

Used in AS-Joined to query devices from the domain.

Proxy Information

Organizations that use a web proxy or perform deep packet inspection to restrict network traffic may encounter issues with . To ensure can operate correctly, teams should add exceptions for the following characteristics:

Characteristic Value
domain

Teams can allow access to the entire domain. This is the simplest option and ensures that all traffic to is allowed through a proxy.

  • pam.oktapreview.com (beta)
subdomains

Teams can allow access to specific subdomains.

  • app.scaleft.com
  • dist.scaleft.com
  • pkg.scaleft.com
User Agent strings

Teams can allow access based on specific user strings. Teams will need to modify the values below based on a specific version of .

  • client: scaleft.go/<version> (sft;)
  • sever agent: scaleft.go/<version> (sftd;)
  • gateway: sft-gatewayd/<version>
Minimum TLS version
  • TLS 1.2 or later
SSL inspection (MITM) leverages Certificate Pinning to allow communication between the platform, clients, and servers. To work around the restrictions of SSL inspection, teams should consider allowing traffic to the domain (pam.oktapreview.com)
