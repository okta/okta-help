Import user attributes using custom mappings

Teams can port existing user attributes into from a pre-existing deployment. This is commonly used if the team wants to retain any user and group metadata as they migrate infrastructure to .

To import a configuration using custom mappings, complete the following steps:

  1. Create custom attributes for your user profile, adding one custom attribute for each supported user attribute. You can pick the attribute name, but set the data type for every attribute to string, except for Unix UID and Unix GID, which should be set to integer.
  2. Populate the user profile attributes that you created. You can do this in many ways, including importing a comma-separated values (CSV) list of users, or setting up application-based mappings to your user profile from the Provisioning tab of another application in the Integration Network ().
  3. Set up application-based mappings from your user profile to . Make these changes from the Provisioning tab of the application, ensuring that the mappings are from to . Assign mappings for the custom attributes that you created and save your changes.

Any users that have the custom attribute set in Universal Directory will now automatically have that custom attribute provisioned when they are assigned to .
