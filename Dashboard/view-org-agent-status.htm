View your org agents' status

Use the Status section to view the real-time performance updates and the agents that your org uses. There are four codes for agents:

  • Operational: All agents and app instances are green.
  • Degradation: An agent is down but the app instance is still connected through a backup agent.
  • Disruption: All agents for an app instance are down and the app isn't connected.
  • Failed to load: The agent status retrieval failed. Click Refresh page.

A red indicator appears next to app instances or agents with an aggregate status of Degradation or Disruption. Click Learn more to open the new Agents page.

You can click Agents in the Status section to view:

  • Agent tabs for RADIUS, IWA, and RUM.
  • App instances and agent tabs for Active Directory (AD), LDAP, and on-premises multifactor authentication.
Each tab displays groupings of associated agents. The groupings list individual agents’ status, description, connection, and version in sortable columns:
  • Status: Operational, disruption, degradation, or inactive. The aggregate of these individual agent codes appears in the Agent Status section of the Admin Dashboard.
  • Description: The agent ID
  • Connection: Live, down since (with the applicable time stamp), or N/A
  • Version: The version number of the agent

For RADIUS agents only

You can hide agents regardless of their status and filter the list of RADIUS agents by hidden and visible agents.

Hiding an agent removes it from the default list of visible agents and its status isn't included in the count in the Agent Status section of the Admin Dashboard. To hide a RADIUS agent, select the Mark as hidden agent option from the dropdown box associated with the agent. Similarly, select Mark as visible agent to make it visible.

Hiding a RADIUS agent doesn’t deactivate it. To deactivate an agent, you must manually deactivate it from the device that it’s installed on.
