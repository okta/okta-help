アドバンストサーバーアクセスのクライアント

リリース：1.62.3

デプロイ日：2022年8月24日

アドバンストサーバーアクセスのクライアントがUbuntu 22.04をサポートするようになりました。

サポートされるオプションのリストについては、「サポートされるオペレーティングシステム」をご覧ください。

修正

macOSクライアントGUIからのアドバンストサーバーアクセスのチーム登録でエラーが返されました。

通知

2022年12月より、アドバンストサーバーアクセスのクライアントは以下のオペレーティングシステムのサポートを終了します。

  • Debian 9
  • Fedora 33
  • Fedora 34
  • macOS 10.12
  • macOS 10.13
  • macOS 10.14
  • Windows 8

これらのオペレーティング システムを使用しているチームは、この期限までにアップグレードを完了してください。「サポートされるオペレーティングシステム」をご覧ください。

リリース：1.60.0

デプロイ日：2022年5月4日

2022年6月初旬より、アドバンストサーバーアクセスは以下のオペレーティングシステムのサポートを終了します。

  • Ubuntu 12.04
  • Red Hat Enterprise Linux 6

前述のオペレーティングシステムを利用している場合、2022年6月までにサポートされるオペレーティングシステムにアップグレードすることをお勧めします。「サポートされるオペレーティングシステム」をご覧ください。

機能拡張

  • sft sshコマンドにSFT_ALLOW_INSECURE_USERNAMES=1環境変数を含めることで、標準外の文字が含まれるユーザー名を使ったssh接続を一時的に許可できます。「アドバンストサーバーアクセスのクライアントを使用する」をご覧ください。
  • クライアントがDiffie Hellman Group Exchange SHA256 Kexアルゴリズムをサポートするようになりました。
  • AD-Joinedにいくつかの変更が加えられました。「AD-Joined」をご覧ください。
    • 複数のADアカウントを持つユーザーは、検出されたADデバイスとの接続に使用するアカウントを選択できます。
    • ユーザーは、パスワード入力なしでADデバイスに接続できます。「パスワードなし証明書」をご覧ください。
    • ユーザーは、macOSとWindowsの両方のGUIからADデバイスに接続できます。
  • ユーザーは、client.timeout_secondsオプションを構成することで、サーバーからの応答待機の最大時間を定義できます。「クライアントを構成する」をご覧ください。
  • 未登録のクライアントが、より詳細なエラーメッセージを返すようになりました。

修正

  • SSH接続を使ったユーザー名の指定ではエラーが返されました。
  • macOSクライアントでは、アドバンストサーバーアクセスのドキュメントへのリンクが誤っていました。
  • アドバンストサーバーアクセスのAPIレートまたは同時並行処理の制限をすでに超過している場合、クライアントは特定の操作で接続を再試行しませんでした。
  • ライフサイクルフックのSFT_Hook_Events環境変数をスクリプトで利用できませんでした。
