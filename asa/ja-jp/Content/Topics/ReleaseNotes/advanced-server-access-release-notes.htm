アドバンストサーバーアクセスのクライアント

リリース：1.56.1

デプロイ日：2022年1月10日

修正

  • 同じチームで複数のASAユーザーアカウントの単一クライアントインスタンスを使用するとエラーが発生するという問題を修正しました。

過去のリリース

リリース：1.55.1

デプロイ日：2021年11月10日

機能強化

  • クライアントは、ssh.allow_rsa_sha1_keysオプションを付けてレガシーRSA/SHA1ホストキーアルゴリズムを使用できるようになりました。

リリース：1.54.1

デプロイ日：2021年8月4日

アドバンストサーバーアクセスの専用ヘルプサイト：アドバンストサーバーアクセスを公開しました。

この改善により、help.okta.comからアドバンストサーバーアクセス向けの個別のオンラインヘルプに直接アクセスできるようになりました。

新しいサイトの利点：

  • コンパクトにまとめられた製品中心のコンテンツ

  • 合理的なナビゲーション

  • より効率的なコンテンツのアップデートと顧客フィードバックに対する俊敏な応答性

リリース：1.54.0

デプロイ日：2021年6月30日

2021年9月より、アドバンストサーバーアクセスは以下のオペレーティングシステムのサポートを終了しました。

  • CentOS 6

  • FreeBSD 10

記載されているオペレーティングシステムをご利用の場合、Oktaは2021年9月までにサポートされているオペレーティングシステムにアップグレードすることを推奨します。「サポートされているオペレーティングシステム」を参照。

修正

  • RoyalTSXサーバーリストの形式が誤っていた問題を修正しました。

  • アドバンストサーバーアクセスのクライアントインストーラーがApple M1搭載Mac上で、誤ってRosettaのインストールを求めるという問題を修正しました。


リリース：1.53.2

デプロイ日：2021年6月1日

機能強化

  • sft list-serversコマンドでPolicySyncが有効になっているチームでサーバーラベルの出力をサポートできるようになりました。


リリース：1.52.2

デプロイ日：2021年4月29日

機能強化

  • アドバンストサーバーアクセスのクライアントがネイティブでM1プラットフォームをサポートするようになりました。


リリース： 1.51.3

デプロイ日： 2021年3月23日

修正

  • クライアントはサーバーバナーの印刷時にLogLevelオプションを無視するという問題を修正しました。

リリース： 1.51.1

デプロイ日： 2021年2月16日

機能および機能強化

  • sft config editを使用し、アドバンストサーバーアクセスのクライアント構成ファイルをシステムのデフォルトのエディターを使用して編集します。
  • クライアントでは、失敗前にSSH認証リクエストの再試行頻度を最大5分間に制限するようになりました。

リリース： 1.50.1

デプロイ日： 2020年11月16日

機能および機能強化

  • ホストキーのキャッシュがクリアされる頻度を減らし、並列処理されるクライアントのパフォーマンスを改善しました。

早期アクセス機能

  • 共有プライマリユーザーグループ識別子(GUD)がサポートされるようになりました。

修正

  • クライアントの信頼の転送を有効にすると、インタラクティブでないセッション中のエージェント転送が破損する問題を修正しました。

リリース： 1.45.4

デプロイ日： 2020年7月10日

修正

  • クライアントトラスト転送の使用時にパニックが生じてクライアントがクラッシュするという問題を修正しました。(OKTA-312023)

リリース： 1.45.3

デプロイ日： 2021年6月16日

修正

  • どのチームにも登録されていないクライアント上での使用時に、'sft resolve -q'コマンドの出力がquietではなかった問題を修正しました。
  • 並行して多数の 'sft' コマンドを実行中に、内部 'known_hosts' ファイルを閉じることができないため、その後の接続に失敗するという問題を修正しました。(OKTA-292731)
  • 'ProxyCommand'オプションの使用時にSecureCRT sshクライアントがサーバーに接続できないという問題を修正しました。(OKTA-259170)

リリース： 1.44.2

デプロイ日： 2020年3月24日

機能および機能強化

  • macOS クライアントがRDPでRoyal TSX をサポートするようになりました。
  • Windows RDP クライアントのタイトルバーに接続先のホスト名が表示されるようになりました。
  • ssh_configのBoolean SSH 構成で yes と no だけでなく、true と false もサポートするようになりました。

修正

  • macOS クライアントのsft list-servers-rjsonにあったパスの問題が修正されました。
  • 複数のログインがコマンドラインから同時に試行されると、最初のログインが完了するまでシステムが待機しないという問題を修正しました。

アドバンストサーバーアクセスのサーバーエージェント

リリース：1.56.1

デプロイ日：2022年1月10日

修正

  • 一部のsshd構成で一致ステートメントに関連して生じていた問題を修正しました。

過去のリリース

リリース：1.55.1

デプロイ日：2021年11月10日

このリリースは予定されたリリースサイクルの一部であり、ユーザーに関連する変更点はありません。

リリース：1.54.0

デプロイ日：2021年6月30日

2021年9月より、アドバンストサーバーアクセスは以下のオペレーティングシステムのサポートを終了しました。

  • CentOS 6

  • FreeBSD 10

記載されているオペレーティングシステムをご利用の場合、Oktaは2021年9月までにサポートされているオペレーティングシステムにアップグレードすることを推奨します。「サポートされているオペレーティングシステム」を参照。

修正

  • sshd構成ファイルのホストキーの検索の際にホワイトスペースとコメントが無視されていた問題を修正しました。


リリース：1.53.2

デプロイ日：2021年6月1日

機能強化

  • アドバンストサーバーアクセスのサーバーエージェントが、構成オプションを通じたカスタムブローカーアクセスポートとカスタムリッスンホストおよびリッスンポートの使用をサポートするようになりました： BrokerAccessPort、BrokerListenPort、BrokerListenHost。「アドバンストサーバーアクセスのサーバーエージェントを設定して使用する」を参照。

修正

  • Windowsでログが適切に保存されない問題を修正しました。


リリース：1.53.2

デプロイ日：2021年4月29日

機能強化

  • アドバンストサーバーアクセスのサーバーエージェントが ARM64アーキテクチャーでも利用できるようになりました。

早期アクセス機能

  • PolicySync属性ベースのアクセスコントロール (ABAC) がサポートされるようになりました。これにより、個別のサーバーに割り当てられたラベルでサーバーアクセスを決定できます。


リリース： 1.51.3

デプロイ日： 2021年3月23日

機能強化

  • サーバーエージェントがsshd_configファイルの変更内容を検証するようになりました。形式が正しくない構成ファイルが接続を中断するのを防ぎます。
  • サーバーエージェントは、sshd_configファイルに変更を加える前にそのバックアップを作成するようになりました。これにより、構成ファイルが無効になったような場合に、動作している構成が確実に復元できます。
  • 膨大な数のプロセッサーを搭載するマシンのデフォルトのスレッド数が削減されました。これにより、個々のプロセッサーのパフォーマンスが向上します。

修正

  • オンデマンドでユーザーがRDPを使用してWindowsサーバーへの接続を試みたとき、最初の試行が失敗することがあった問題を修正しました。

リリース： 1.51.1

デプロイ日： 2021年2月16日

機能および機能強化

クライアントツール：

  • sft config editを使用し、アドバンストサーバーアクセスのクライアント構成ファイルをシステムのデフォルトのエディターで編集します。
  • クライアントでは、失敗前にSSH認証リクエストの再試行頻度を最大5分間に制限するようになりました。

ゲートウェイ：

  • ゲートウェイにはサンプルの構成ファイルが追加されました。
  • ゲートウェイはエージェント転送をサポートするようになりました。

修正

サーバーツール：

  • Red Hat Enterpriseを実行しているサーバーがアップグレードされたとき、時折アドバンストサーバーアクセスのサーバー項目が重複する問題を修正しました。
  • 特定のプロジェクトでクライアントの信頼の転送を管理者が有効または無効にしたときに、時折無効なsshd構成になる問題を修正しました。(早期アクセス)

リリース： 1.50.4

デプロイ日： 2020年12月17日

修正

  • 同じユーザーに対してUNIXユーザーID (UID)の変更とユーザーの削除操作の適用を同時に試みると、時折エージェントのクラッシュが生じる問題を修正しました。

リリース： 1.50.3

デプロイ日： 2020年11月17日

修正

  • 存在しないローカルUNIXグループのメンバーだったユーザーにより、時折エージェントがクラッシュする問題を修正しました。
  • エージェントを新たにWindowsにインストールした際に、リモートデスクトッププロトコル (RDP)ブローカーが開始できない問題を修正しました。

リリース： 1.45.3

デプロイ日： 2020年6月16日

修正

  • ユーザーがディアクティベートまたは削除されたときに、リモートデスクトッププロトコル(RDP)セッションが終了できない問題を修正しました。(OKTA-294736)

リリース： 1.44.6

デプロイ日： 2020年4月20日

修正

  • 一部のユーザーグループのメンバーシップが正しく追加されないといったLinuxエージェントの問題が修正されました。

リリース： 1.44.4

デプロイ日： 2020年4月15日

機能および機能強化

  • 強化されたユーザーとグループの属性が利用可能になりました。詳細は「アドバンストサーバーアクセスのユーザーとグループの属性」参照してください。

リリース： 1.44.2

デプロイ日： 2020年3月24日

機能および機能強化

  • UNIXクライアントがシェル、homeディレクトリ、アカウントのコメントを含む顧客ユーザー属性を更新できるようになりました。
  • SSHログのRSA 証明書ID 部分が解析されました。

修正

  • sudoエンタイトルメントがUNIXで名前変更されたときに生じていた問題を修正しました。
  • TLS 証明書の読み込み時のエラー処理が向上しました。
  • ファイルを変更する際に、sshd_configのファイルの許可が時折変更されていた問題を修正しました。

リリース： 1.41.0

デプロイ日： 2019年11月13日

機能および機能強化

  • イベントフックのロギングが減少しました。

修正

  • サーバーを別のプロジェクトに移行する際に、時折生じていたユーザー同期の問題を修正しました。


アドバンストサーバーアクセスのゲートウェイ

リリース：1.56.1

デプロイ日：2022年1月10日

修正

  • 要塞がAccessAddressとしてロードバランサーまたはFQDNを使用できなかった問題を修正しました。

過去のリリース

リリース：1.55.1

デプロイ日：2021年11月10日

早期アクセス機能

  • AD-JoinedがActive Directory (AD)サーバーを自動検出できるようになりました。
  • AD-JoinedがAD資格情報を使ってリモートデスクトッププロトコル (RDP)プロキシを許可できるようになりました。

詳細は、AD-Joinedを参照してください。

機能強化

  • sft-gatewayd.serviceを使用した一部Linuxシステム機能へのアクセスが制限されるようになりました。
  • ゲートウェイがRefuseConnectionsオプションを介したSecure Shell (SSH)接続とRDP接続を拒否できるようになりました。

修正

  • 診断用サポートバンドルに機密データが含まれていた問題を修正しました。

リリース：1.54.0

デプロイ日：2021年6月30日

2021年9月より、アドバンストサーバーアクセスは以下のオペレーティングシステムのサポートを終了しました。

  • CentOS 6

  • FreeBSD 10

機能強化

  • SSHセッションログファイル項目に、重複する接続IDが含まれなくなりました。


リリース：1.53.2

デプロイ日：2021年4月29日

機能強化

  • ゲートウェイがアクティブな接続数を報告するようになりました。これにより管理者ゲートウェイの現在の接続負荷を確認できます。

  • SSHセッションログのログ整合性検証アルゴリズムが改善されました。これによりSSHセッションログの確実性が確保されました。

  • ログストレージの一時ディレクトリが有効なデバイスでない場合にゲートウェイが起動しなくなりました。これにより、ランタイム時ではなく起動時に無効なログディレクトリを検出できます。

  • ゲートウェイログストレージの一時ディレクトリがSessionLogTempStorageDirectory 構成オプションで指定できるようになりました。これにより、一時ディレクトリを構成して、一時ディレクトリとストレージのディレクトリを別のボリューム上に構成できるようになりました。

修正

  • SSHセッションメタデータが誤って記録されていた問題を修正しました。


リリース： 1.51.3

デプロイ日： 2021年3月23日

機能強化

  • 外付けストレージの使用時に、使用中のローカルディスクストレージまたはゼロをゲートウェイが報告するようになりました。この変更により、管理者はゲートウェイファイルがローカルまたは外部に保存されているか判断し、使用中の容量を確認できます。

リリース： 1.51.1

デプロイ日： 2021年2月16日
