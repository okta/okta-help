Verwendung des Advanced Server Access-Clients

Jeder Advanced Server Access -Client-Befehl nutzt die folgende Syntax:

sft [globale Optionen] Befehl [Befehlsoptionen] [Argumente ...]

Globale Optionen

Sie können bei jedem Client-Befehl die folgenden Optionen verwenden:

  • -h, --help Zeigt die Hilfe an.
  • -v, --version Zeigt die Version an.

Client-Befehle

Befehl Beschreibung Optionen
sft config Ruft die Client-Konfigurationsoptionen ab und legt sie fest. Siehe Client-Konfiguration.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • --append: Fügt den angegebenen Wert zu einem Array hinzu.
sft dash Öffnet das Dashboard Ihres Teams in Ihrem Browser.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
sft device-info Zeigt die Geräteinformationen Ihres Clients als JSON an. -
sft enroll Fügt den neuen Client zu Ihrem Client-Bestand auf der Advanced Server Access-Plattform hinzu.
  • --default: Legt ein neues Standardteam fest. (Standard: true)
  • --url: Legt die URL für den Zugriff auf eine Advanced Server Access-Instanz fest.
  • --team: Verwendet das angegebene Team.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • --force: Registriert den Client auch dann, wenn bereits ein Duplikat vorhanden ist. (Standard: false)

sft help Zeigt eine Liste mit Befehlen oder die Hilfe für einen Befehl an. -

  • -l, --selector: Filtert die Ergebnisse nach dem angegebenen Selektor (Bezeichnungsabfrage). Siehe Selektoren.
  • --output, -o: Formatiert die Ergebnisse im angegebenen Format. Zu den verfügbaren Optionen gehören: default (Standard), json (JSON) oder describe (Beschreibung).
  • --columns: Zeigt die angegebenen Spaltennamen in der Ausgabe an. Spaltennamen sollten in Kleinbuchstaben geschrieben und in einer durch Trennzeichen getrennten Liste gesammelt werden.
sft list-projects Listet die für das Team verfügbaren Projekte auf.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • -l, --selector: Filtert die Ergebnisse nach dem angegebenen Selektor (Bezeichnungsabfrage). Siehe Selektoren.
  • --output, -o: Formatiert die Ergebnisse im angegebenen Format. Zu den verfügbaren Optionen gehören: default (Standard), json (JSON) oder describe (Beschreibung).
  • --columns: Zeigt die angegebenen Spaltennamen in der Ausgabe an. Spaltennamen sollten in Kleinbuchstaben geschrieben und in einer durch Trennzeichen getrennten Liste gesammelt werden.
sft list-servers Listet die im aktuellen Team verfügbaren Server auf.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • -l, --selector: Filtert die Ergebnisse nach dem angegebenen Selektor (Bezeichnungsabfrage). Siehe Selektoren.
  • -p, --project: Filtert die Ergebnisse nach dem angegebenen Projektnamen.
  • --output, -o: Formatiert die Ergebnisse im angegebenen Format. Zu den verfügbaren Optionen gehören: default (Standard), json (JSON) oder describe (Beschreibung).
  • --columns: Zeigt die angegebenen Spaltennamen in der Ausgabe an. Spaltennamen sollten in Kleinbuchstaben geschrieben und in einer durch Trennzeichen getrennten Liste gesammelt werden.
sft list-servers-rjson Listet die im aktuellen Team verfügbaren Server im RJSON-Format auf. Sie können diese Ausgabe in einer Datei speichern und in Royal TSX zum Erstellen eines dynamischen Ordners verwenden. Siehe Verwendung von Royal TSX mit Advanced Server Access.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • -l, --selector: Filtert die Ergebnisse nach dem angegebenen Selektor (Bezeichnungsabfrage). Siehe Selektoren.
  • -p, --project: Filtert die Ergebnisse nach dem angegebenen Projektnamen.
  • -f, --filename: Legt den Pfad fest, unter dem die Ausgabe-Datei gespeichert wird.
sft login Erstellt nach der Abmeldung beim aktuellen Team Ihres Clients eine neue Sitzung und authentifiziert sich mit dem Identitätsanbieter Ihres Teams.

Eine aktive, autorisierte Client-Sitzung ermöglicht es dem Advanced Server Access-Client, die Anmeldedaten nach Bedarf im Hintergrund anzufordern.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
sft logout Führt die Abmeldung von der aktuellen Sitzung durch.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
sft proxycommand Wird mit OpenSSH ProxyCommand verwendet, um die transparente Nutzung von sft mit ssh, scp, rsync, ftp etc. zu ermöglichen.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • --via, --bastion: Stellt über die angegebene SSH-Bastion eine Verbindung zum Ziel her.
  • --config: Veraltet. Verwenden Sie stattdessen sft ssh-config.
sft rdp Stellt über RDP eine Verbindung zu einem Ziel her, das als Argument übergeben wird.
  • --via, --bastion: Stellt über die angegebene SSH-Bastion eine Verbindung zum Ziel her.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • --share: Gibt den absoluten Pfad zu einem Verzeichnis an, das mit dem Ziel geteilt werden soll.
sft resolve Löst einen einzelnen Server auf, der mit dem angegebenen Hostnamen oder den Instanzdetails übereinstimmt.
  • --q, --quiet: Sorgt dafür, dass nur schwerwiegende Warnungen in stderr ausgegeben werden.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
sft session-logs verify Verifiziert die Integrität eines angegebenen Sitzungsprotokolls anhand des Gateway-Signaturschlüssels, der bei Advanced Server Access registriert ist. Protokolldateien ohne gültige Signaturen sind möglicherweise fehlerhaft oder wurden durch einen Angreifer beschädigt.
  • --stdin: Gibt die Sitzungsdaten von stdin anstatt aus einer Sitzungsprotokolldatei zurück.
sft session-logs export Exportiert die Sitzungsprotokolle in einem bestimmten Format. Standardmäßig werden Protokolle im JSON-Format exportiert. Während des Exportvorgangs werden die Sitzungsprotokolle auch verifiziert. Protokolldateien ohne gültige Signaturen sind möglicherweise fehlerhaft oder wurden durch einen Angreifer beschädigt.
  • --stdin: Gibt die Sitzungsdaten von stdin anstatt aus einer Sitzungsprotokolldatei zurück.
  • --insecure: Hält den Client davon ab, die Integrität der Sitzungsprotokolldatei zu verifizieren.
  • --format: Exportiert Protokolle in einem bestimmten Format. Zu den verfügbaren Optionen gehören: json (JSON) oder asciinema (Asciinema).
  • --output: Speichert die exportierten Protokoll in der angegebenen Datei anstatt in stdout.
sft ssh Stellt über SSH eine Verbindung zu einem Ziel her, das als Argument übergeben wird.

Im Allgemeinen verwendet Advanced Server Access mit der Integration von OpenSSH ProxyCommand SSH. Der Befehl sft ssh dient zur SSH-Unterstützung in Umgebungen oder Kontexten, in denen OpenSSH nicht verfügbar ist, oder in Fällen, in denen Sie Advanced Server Access-spezifische Optionen wie --via explizit übergeben möchten.
  • --via, --bastion: Stellt über den angegebenen SSH-Bastion-Host eine Verbindung zum Ziel her.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • -L, --local-port-forward: Leitet den angegebenen lokalen Port an eine Remote-Adresse weiter.
  • -R, --remote-port-forward: Leitet den angegebenen Remote-Listener an eine lokale Adresse weiter.
  • --command: Führt den angegebenen Befehl über SSH aus.
sft ssh-config Gibt einen OpenSSH-Konfigurationsblock aus, den Sie in Ihrer ~/.ssh/config-Datei verwenden können, wodurch Ihre lokale SSH-Binärdatei die Advanced Server Access-Authentifizierung nutzen kann. Diese SSH-Konfiguration wird nur verwendet, wenn Ihr Client eine derzeit aktive und autorisierte Sitzung hat.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --account: Verwendet das angegebene Konto.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • --via, --bastion: Stellt über die angegebene SSH-Bastion eine Verbindung zum Ziel her.
sft support collect Sammelt lokale Diagnosedaten für den Okta-Support. -
sft support submit Versendet lokale Diagnosedaten für den Okta-Support. -
sft unenroll Entfernt den derzeit aktiven Client aus Ihrem Client-Bestand in der Advanced Server Access-Plattform.
  • -y, --yes: Verwendet das angegebene Team.
  • --team: Verwendet das angegebene Team.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.
  • --all: Hebt die Registrierung aller lokalen Clients auf.
sft use Legt ein registriertes Team als aktuellen Standard für die aktuelle Sitzung fest.
  • --instance: Verwendet die angegebene Instanz der Advanced Server Access-Plattform.
  • --config-file: Verwendet die angegebene Konfigurationsdatei.

Selektoren

  • -l, --selector Der Selektor (Bezeichnungsabfrage), nach dem gefiltert wird.

Befehle, die einen Selektor als optionales Argument akzeptieren, können ihre Ergebnisse basierend auf einer beliebigen Selektorabfrage filtern.

Die Selektor-Syntax basiert auf Kubernetes-Bezeichnungsabfragen. Siehe Bezeichnungen und Selektoren.

Beispiel:

sft list-servers -l os_type=windows,project_name=Demo

In diesem Beispiel wird mit einem Selektor die Liste der Server, auf die Sie zugreifen können, gefiltert. Der Befehl gibt eine Liste der Windows-Server zurück, die beim Demo-Projekt registriert sind.

Client-Konfiguration

Mit dem Befehl sft config können Sie die Konfigurationsoptionen anzeigen oder festlegen.

Wenn der Advanced Server Access-Client installiert ist, gibt es keine Konfigurationsdatei. Die Konfigurationsdatei wird erstellt, wenn Sie die erste Konfigurationsoption festlegen.

Bis Sie explizit einen Konfigurationswert angeben, werden die Standardeinstellungen verwendet. Die Standardeinstellungen des Advanced Server Access-Client bieten die höchste Sicherheit und Benutzerfreundlichkeit für die gängigsten Situationen. Abgesehen von persönlichen Einstellungen wie der Einstellung für rdp.screensize müssen Sie möglicherweise gar keine Client-Konfigurationen festlegen.

Advanced Server Access-Client-Konfigurationen sind in mehrere Bereiche gruppiert. Diese Bereiche sind derzeit rdp, ssh, ssh_agent, service_auth und update.

Anzeigen Ihrer Konfiguration

  • sft config Zeigt Ihre aktuellen Konfigurationen an.
  • sft config [section.key] Zeigt den aktuellen Wert einer bestimmten Konfiguration an, die durch section.key angegeben wird.

Festlegen eines Konfigurationswerts

Mit der folgenden Befehlssyntax können Sie einen Konfigurationswert festlegen: sft config [section.key] [value].

RDP-Konfigurationsoptionen

Schlüssel

Beschreibung

Beispiele
rdp.screensize Legen Sie hier einen Zeichenfolgewert wie 1024x768 fest, der Ihre gewünschte Größe für das RDP-Fenster angibt.

sft config rdp.screensize 800x600

sft config rdp.screensize 1024x768
rdp.fullscreen Legen Sie hier true fest, damit sich RDP-Sitzung im Vollbildmodus öffnen. Wenn Sie true festlegen, wird der Wert von rdp.screensize ignoriert.

sft config rdp.fullscreen true

sft config rdp.fullscreen false
rdp.client (Nur bei macOS) Geben Sie hier den gewünschten RDP-Client an: entweder royaltsx für Royal TSX oder macfreerdp für MacFreeRDP. Wenn Sie diese Option nicht konfigurieren, versucht Advanced Server Access, Royal TSX zu verwenden, und verwendet MacFreeRDP, wenn Royal TSX nicht verfügbar ist.

sft config rdp.client royaltsx

sft config rdp.client macfreerdp

SSH-Konfigurationsoptionen

Schlüssel

Beschreibung

Beispiele
ssh.save_privatekey_passwords Wenn Sie hier true festlegen, speichert der Advanced Server Access-Client alle Passphrasen, die Benutzer eingeben, im lokalen kryptographischen Speicher der Arbeitsstation.

sft config ssh.save_privatekey_passwords true

sft config ssh.save_privatekey_passwords false
ssh.port_forward_method Legen Sie hier netcat fest, damit Advanced Server Access netcat (nc) remote als Portweiterleitung ausführt, anstatt die standardmäßige native SSH-Portweiterleitung zu verwenden.

sft config ssh.port_forward_method netcat

sft config ssh.port_forward_method native

ssh.insecure_forward_agent

Legen Sie hier host fest, um den ForwardAgent für die Ausführung von SSH-Befehlen festzulegen. Hinweis: Von Advanced Server Access ausgegebene Anmeldedaten werden nicht zum SSH-Agenten hinzugefügt. Diese Option dient also zur Nutzung mit Hosts, die so konfiguriert sind, dass sie extern verwaltete Anmeldedaten akzeptieren, z. B. einen öffentlichen SSH-Schlüssel, der nicht von Advanced Server Access verwaltet wird.

Wenn Sie diese Option nicht konfigurieren oder als Wert „none“ festlegen, leitet Advanced Server Access den SSH-Agenten nicht weiter.

sft config ssh.insecure_forward_agent host

sft config ssh.insecure_forward_agent none

Konfigurationsoptionen für den SSH-Agenten

Schlüssel

Beschreibung

Beispiele
ssh_agent.enable Wenn Sie hier true festlegen, verwendet der Advanced Server Access-Client bei der Authentifizierung einen SSH-Agenten.

sft config ssh_agent.enable true

sft config ssh_agent.enable false
ssh_agent.keys Legen Sie hier ein JSON-Array mit mindestens einem Pfad zu privaten SSH-Schlüsseln fest, die in den SSH-Agenten geladen werden sollen. Sie können mit dem Flag --append Werte an diese Liste anhängen.

Tipp: Wenn Sie ein JSON-Literal in der Windows PowerShell schreiben, maskieren Sie innere Anführungszeichen. Beispiele:

sft config ssh_agent.keys '[\"C:\\Users\\alice\\.ssh\\id_rsa\"]'

sft config ssh_agent.keys '["/Users/alice/.ssh/id_rsa"]'

sft config ssh_agent.keys --append /Users/alice.ssh/id_rsa

sft config ssh_agent.keys '[]'

Netzwerkkonfigurationsoptionen

Schlüssel

Beschreibung

Beispiele
network.forward_proxy Wenn Sie diese Option konfigurieren, verwendet der Advanced Server Access-Client die angegebene HTTP- bzw. HTTPS-URL als HTTP-Tunnel.

sft config network.forward_proxy https://Ihr-Proxy.beispiel.de:3141

network.tls_use_bundled_cas

Wenn Sie hier true festlegen, verwendet der Advanced Server Access-Client eine gebündelte CA-Zertifikatsliste für die TLS-Validierung. Wenn Sie false festlegen, verwendet der Client die CA-Liste des Betriebssystems.

Hinweis: Diese Option ist standardmäßig auf „true“ festgelegt. Wir empfehlen dringend, diesen Wert nicht in „false“ zu ändern, da es bei bestimmten Betriebssystemen Probleme mit der CA-Liste gibt und die Leistung daher bedeutend schlechter ausfallen kann als bei der Verwendung gebündelter CA-Zertifikatslisten.

sft config network.tls_use_bundled_cas true

sft config network.tls_use_bundled_cas false

Sonstige Konfigurationsoptionen

Schlüssel

Beschreibung

Beispiele
service_auth.enable Wenn Sie hier true festlegen, unterstützt der Advanced Server Access-Client die Authentifizierung für Dienstbenutzer. Siehe Dienstbenutzer.

sft config service_auth.enable true

sft config service_auth.enable false
update.release_channel Der Advanced Server Access-Client verwendet standardmäßig den Updatekanal stable. Sie können aber auch festlegen, dass der Client den Updatekanal test verwendet und häufiger Aktualisierungen erhält.

sft config update.release_channel test

sft config update.release_channel stable
client.timeout_seconds

Legt fest, wie lange der Client auf eine Antwort des Servers wartet, bevor er die Anforderung erneut sendet.

  • Wenn Sie diese Option nicht konfigurieren oder auf „0“ festlegen, verwendet der Client den Standardwert von 300 Sekunden.
  • Wenn Sie diese Option auf einen negativen Wert festlegen, wird die Zeitüberschreitung deaktiviert.

sft config client.timeout_seconds 60

sft config client.timeout_seconds -1
