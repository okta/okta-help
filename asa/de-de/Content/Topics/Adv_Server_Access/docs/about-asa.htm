Komponenten von Advanced Server Access

Eine Bereitstellung von Advanced Server Access umfasst eine Kombination der folgenden Komponenten:

Komponenten Beschreibung
Teams Ein Team ist ein Container der obersten Ebene, der alle Ressourcen für eine bestimmte Bereitstellung enthält. Jedes Team besitzt einen eindeutigen Namen und einen verknüpften Identitätsanbieter (IdP).

Alle anderen Konfigurationsobjekte in Advanced Server Access gehören zum Bereich eines bestimmten Teams.
Gruppen Eine Gruppe ist eine Zusammenstellung von Benutzern, der bestimmte Berechtigungen zugewiesen sind. Für jede Bereitstellung werden zwei Standardgruppen erstellt: „Alle“ und „Besitzer“.

Sie können einer Gruppe eine Teamrolle oder mehrere Teamrollen zuweisen. Jedes Gruppenmitglied übernimmt die zugewiesenen Rollen.
Projekte Ein Projekt ist ein Autorisierungsbereich, ähnlich einer Domäne bei Active Directory. Jedes Projekt verknüpft eine Sammlung von Ressourcen (darunter Benutzer und Server) mit einem Satz an Konfigurationen, die die rollenbasierte Zugriffssteuerung und Zugriffsrichtlinien umfassen.
Dynamische Anmeldedaten Die Anmeldedaten von Advanced Server Access sind kurzlebige Objekte, die den Zugriff auf Projektressourcen ermöglichen. Teams können sich Projekte als programmierbare Zertifizierungsstellen vorstellen, die diese kurzlebigen Zertifikate ausstellen. Auf Basisebene enthalten Zertifikate Folgendes:
  • das Advanced Server Access-Projekt, für das das Zertifikat ausgestellt wurde
  • den Benutzernamen, der auf dem Server des Advanced Server Access-Benutzers verwendet werden soll, für den das Zertifikat ausgestellt wurde
  • die Zeit, zu der das Zertifikat abläuft
Benutzer Ein Benutzer ist eine Person, die zu einem Team gehört und sich mit dem Identitätsanbieter dieses Teams authentifiziert. Advanced Server Access definiert die Benutzerberechtigungen basierend auf Gruppenmitgliedschaften.

Benutzer autorisieren Clients, die zum Client-Bestand hinzugefügt werden, damit sie Anmeldedaten erhalten.
Dienstbenutzer Dienstbenutzer sind spezielle Konten, die nicht mit einer realen Person verknüpft sind. Teams können einen Dienstbenutzer verwenden, um Aktionen mit der Advanced Server Access-API zu automatisieren oder den Zugriff auf bestimmte Vorgänge in der Advanced Server Access-Plattform zu gewähren. Siehe Dienstbenutzer.
Clients Der Advanced Server Access-Client ist ein Befehlszeilentool, das auf einer Arbeitsstation installiert ist. Wenn Benutzer den Client installiert und bei einem Advanced Server Access-Projekt registriert haben, bietet er Zugriff auf die Serverressourcen, die beim selben Projekt registriert sind. Siehe Advanced Server Access-Clients.
Server Der Advanced Server Access-Server-Agent steuert den Zugriff über Secure Shell (SSH) und Remotedesktopprotokoll (RDP) auf Remote-Server, die bei einem Advanced Server Access-Projekt registriert sind.

Ein Server ist nur bei einem einzigen Projekt registriert. Teams können Server mit einem verknüpften Cloud-Konto automatisch oder mit einem Registrierungstoken manuell bei Projekten registrieren. Siehe Advanced Server Access-Server-Agenten.
Server-Benutzerkonten Der Advanced Server Access-Server-Agent verwaltet Benutzerkonten auf Windows- und Linux-Servern.

Wenn Benutzer in Okta deaktiviert werden, entfernt der Server-Agent alle zugehörigen Benutzerkonten auf dem Server, um unerwünschten Zugriff zu verhindern.
Berechtigungen Berechtigungen gestatten Benutzern, die keine Administratoren sind, die Verwendung von bestimmten sudo-Befehlen, ohne dass ihnen Administratorrechte gewährt werden müssen.

Teams können ein System mit abgestuften Berechtigungen erstellen, basierend auf der Gruppenzugehörigkeit der Benutzer.
Attribute Attribute legen verschiedene Merkmale von Benutzern und Gruppen fest. Dazu können Unix- und Windows-Benutzernamen, UIDs und GIDs gehören.

Teams können Attribute mit vorab festgelegten Parametern erzeugen. Administratoren haben die vollständige Kontrolle über die Benutzer- und Gruppenattribute in ihrem Team. Sie können die Attribute bearbeiten, um Attributkonflikte bei bestehenden Bereitstellungen zu vermeiden oder aufzulösen.
