Configure optional settings

Cisco ASA IKEv2 VPN supports two optional settings: Client IP Reporting and Groups response.

Topics

  • Configure Client IP Reporting
  • Configure Groups Response

Configure Client IP Reporting

To configure Okta to be able to parse, report on and eventually enforce policy based off of the source client IP Address you need to configure the Cisco ASA VPN (RADIUS) App in Okta as follows:

Enter the following settings in Advanced RADIUS Settings found on the Sign On tab for the RADIUS app in your Okta Admin Console, as shown below.

  • Client IP: Check Report client IP.
  • RADIUS End User IP Attributes: 31 Calling-Station-Id

Assigning Profiles using GROUPS RESPONSE settings

The app can work together with Cisco ASA ASDM to assign profiles to user on login.
Using this configuration users assigned a group in their Okta tenant with matching group name in the group policies section of Cisco ASA will be assigned the matching Cisco group policy.

After configuring group membership within the GROUP RESPONSE section of the app as shown above. Within Cisco ASA ASDM create a matching group.

  1. Navigate to Remote Access VPN >Network (Client) Access >Group Policies.
  2. Create a new group policy with the matching group name as the Okta group.

  3. Verify - Using the prior setting, noting the value in the banner field:
    1. Open a commend prompt
    2. Execute the ipconfig command.
    3. Examine the result. The assigned IP address should match that assigned to the given group.
      In this example, 192.168.104.1 was defined in the associated policy.
