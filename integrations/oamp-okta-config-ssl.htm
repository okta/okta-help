Enable SSL on OAM servers

[Optional] During this task we will use the WebLogic Server console to enable SSL (HTTPS) on OAM servers.

To enable SSL options for OAM servers:

  1. Login to the WebLogic Server Administration Console.
  2. In the Domain Structure pane expand DomainName Environment Servers .
  3. Select a server.
  4. In Change Center, click Lock & Edit.

    If the WebLogic Server Console preference Automatically Acquire Lock and Activate changes is not set then you will not need to click Lock & Edit.

  5. Select Configuration SSL
  6. Expand the Advanced section.
  7. From Hostname Verification select None.
  8. Check Use JSSE SSL.
  9. Click Save.
  10. In Change Center click Activate Changes.
  11. Restart the server instance.

Locating Logs

Oracle WebLogic Server maintains logs by domain and individual server. The file, server-diagnostic.log, contains useful debugging information. The server diagnostic log file can be found in:

Server Location
Administration Server ${DOMAIN_HOME}/servers/AdminServer/logs
Managed Server ${DOMAIN_HOME}/servers/servername>/logs

Where servername is one of the OAM server names.

Restarting WebLogic Server instances from the command line

WebLogic Server instances can be started and stopped from the command line.

To manage WebLogic Server instances at the command line:

  1. On the Administration server machine, open a command prompt.
  2. Change directory to ${DOMAIN_HOME}/bin.
  3. Excute a stop or start command as described below:
  4. Server & Command Command
    Administration Server - Shut Down
    • Linux: nohup ./stopWebLogic.sh &
    • Windows: stopWebLogic.cmd
    Administration Server - Start
    • Linux: nohup ./startWebLogic.sh &
    • Windows: startWeblogic.cmd
    Managed Server - Shutdown
    • Linux: nohup ./stopManagedWebLogic.sh serverName &
    • Windows: startManagedWebLogic.cmd serverName
    Managed Server Start
    • Linux: nohup ./startManagedWebLogic.sh serverName &
    • Windows: startManagedWebLogic.cmd serverName

Restaring WebLogic Server instance from the Admin Console

When configured to use Node Manager, WebLogic Server instances can be started and stopped from the Administration Console. To manage WebLogic Server instances using the WebLogic Administration Console:

  1. Login to the WebLogic Server Administration Console.
  2. In the Domain Structure pane expand DomainName Environment Servers .
  3. Select the Control tab.
  4. Select one or more rows representing the WebLogic Server instances to manage.
  5. To start servers click Start.
    To shutdown servers click Shutdown Force shutdown now

Node manager will then perform the requested operation in the background.
