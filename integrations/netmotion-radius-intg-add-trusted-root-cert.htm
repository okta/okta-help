Netmotion Mobility - Add trusted root certificate

The NetMotion Mobility gateway requires a trusted root certificate. You must add the root certificate authority used to issue the server certificate Trusted Root CA certificate store of your client device.

These steps must be performed on the computer running the NetMotion Mobility VPN client.

Before you begin

  • Ensure that you have sufficient access rights to add a certificate as a trusted CA using the Microsoft Management Console.

To add a certificate to the Trusted root CA in Windows 10:

  1. On the computer running the NetMotion Mobility VPN client, open the Microsoft Management Console.
  2. From the file menu select File Add/Remove Snap-in.
  3. In the Add/Remove Snap-in dialog, in the Available snap-ins section, select Certificates and click Add.

  4. In the Certificates snap-in dialog, select Computer account and click Next.
  5. In the Select Computer dialog, select Local computer and click Finish.
  6. Click OK.
  7. Navigate to Console Root Certificates (Local Computer) Personal .Right click Personal and select All Tasks Import... The Certificate Import wizard starts.

  8. Click Next.
  9. Enter the fully qualified path or click Browse to navigate to the directory that contains the certificate.
  10. Select the certificate and click Next.
  11. Click Finish to import the certificate.
  12. From Certificates (Local Computer) Personal Certificates , select the certificate that you added and drag it to Certificates (Local Computer) Trusted Root Certification Authority Certificates .

The new certificate has been successfully added as a trusted root certificate.
