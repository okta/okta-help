SNMPモニタリングを管理する

SNMP（Simple Network Management Protocol）は、ネットワーク管理者がさまざまな情報のクエリをデバイスに対し実行することを可能にします。Access Gatewayでは、SNMPポーリングを実行することにより、アプライアンスから直接情報を収集できるようにします。SolarwindsやNagiosなどのサードパーティ製ネットワークモニタリングツールは、特定のパラメーターをモニタリングするためにSNMPを使用できます。

Access Gatewayは、Simple Network Management Protocol (SNMP) v2をサポートします。

以下の情報をアプライアンスからポーリングを実行できます：

  • システム

    • 説明 (.1.3.6.1.2.1.1.1.0)

    • SysUpTime (.1.3.6.1.2.1.1.3.0)

    • 連絡先 (.1.3.6.1.2.1.1.4.0)

    • 名前 (.1.3.6.1.2.1.1.5.0)

    • ロケーション (.1.3.6.1.2.1.1.6.0)

  • hrSystem稼働時間 (.1.3.6.1.2.1.25.1.1.0)

標準MIB

Access Gatewayアプライアンスは、次のMIBのサブセット内にあるOIDをサポートします。

  • UCD-SNMP-MIB ：.1.3.6.1.4.1.2021

  • IF-MIB (ifMIBObjects) ：.1.3.6.1.2.1.31.1

  • HOST-RESOURCES-MIB ：.1.3.6.1.2.1.25

  • SNMPv2-MIB ：.1.3.6.1.2.1

これらのMIBは独自のものではなく、ほとんどのネットワークモニタリングシステムで利用できます。

独自MIB

Access Gatewayアプライアンスには独自のMIBは含まれていません。

構成

デフォルトでは、Okta SNMPモニタリングパッケージはインストールされていません。

バージョン2020.04.04以降では、SNMPがプリインストールされています。

SNMPモニタリングを有効にする方法：

  1. SSHを使用してAccess Gateway 管理者コンソールに接続します。

  2. 5を入力し、[System（システム）]サブメニューに進みます。

  3. 2を入力し、[Install（インストール）]サブメニューに進みます。

  4. okta-monitoring-snmpパッケージを入力します。

  5. プロンプトが表示されたら、パッケージをインストールする場合はy、インストールを中止する場合はNを入力します。

SNMPテストツール

さまざまなオペレーティングシステムでSNMP機能をテストするためのツールが複数あります。以下の情報は、一般的なオペレーティングシステムの概要を示しています。

Microsoft Windows

Windowsオペレーティングシステムでは、net-snmpと呼ばれるオープンソースツールを使用できます。以下の手順に従いnet-snmpをインストールし、Access GatewayアプライアンスへのSNMPポーリングをテストします。

  1. net-snmp をダウンロードして、マシンにインストールします。

  2. サンプルテストコマンドからsnmpwalkコマンドをコピーし、コマンドプロンプトに貼り付けて実行します。

Linux（Debianベース）

Linuxベースのオペレーティングシステムでは、同じ結果を得るためにSNMPパッケージをインストールできます。Debianシステムでは、以下の手順に従います。

  1. snmpdをインストールします。apt-getを持つシステムの場合：

    sudo apt-get install snmpd

  2. サンプルテストコマンドからsnmpwalkコマンドをコピーし、ターミナルに貼り付けて実行します。

サンプルテストコマンドと出力

snmpwalkを使用して、アプライアンスごとにSNMPをテストします。

Access Gatewayコミュニティ文字列

SNMPアクセスには、通常「コミュニティ文字列」と呼ばれるものが必要です。-cスイッチを使用してコミュニティ文字列を指定します。実際のコミュニティ文字列値は記録されておらず、このページではAccessGatewayCommunityStringに置き換えられます。

SNMPコミュニティ文字列は、Okta Access Gatewayサポートからのリクエストに応じて提供されます。
