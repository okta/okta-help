Access Gatewayをデプロイする

デプロイメントは、Access Gateway仮想アプリケーションをダウンロードしてインストールするプロセスです。Access Gatewayは通常、次の図に示すようなアーキテクチャを使用してデプロイされます。ただし、高可用性の使用は必須ではありません。Access Gatewayインスタンスは、仮想環境でホストする必要があります。

前提条件

以下を確認します。

  • Access Gatewayデプロイメントのすべての前提条件を確認します。
  • Access GatewayはDNSを使用して解決可能であり、DNSでホスト名が割り当てられている必要があります。
  • 静的IPアドレスを持っていることを確認します。Access Gatewayは、静的IPアドレスにDHCPを使用することをサポートしていません。
  • DNSサーバーとDNSホスト名（FQDN）があることを確認します。Access Gatewayは、エンドユーザーが利用するDNSソリューションを使用して解決できる必要があります。エンドユーザーがインターネットから発信している場合、Access Gatewayソリューションはパブリックに解決できる必要があります。内部ユーザーが内部IPアドレスに接続し、外部ユーザーがパブリックIPアドレスに接続するように、分割DNSを活用することをお勧めします。

Access Gatewayと統合されたすべてのアプリケーションのFQDNは、DNSで解決できる必要があります。

VMWare vSphereへのデプロイ

Access Gatewayは、サポートされるデプロイメント環境のリストに記載されている任意の環境にデプロイできます。次のセクションでは、Access GatewayをVMware vSphereにデプロイする方法を説明します。

最新のOVAイメージをダウンロードする

最新のOkta Access GatewayOVAイメージをダウンロードする方法：

  1. Okta orgに管理者としてサインインします。

  2. Okta Access Gatewayイメージを[Settings（設定）] > [Downloads（ダウンロード）]ページからダウンロードします。

  3. 指示に従って、OVAファイルを適切な場所に保存します。

VMWare ESXi/vSphereにインポートする方法：

VMWare ESXi、vSphere、およびvSphere Clientバージョン6.5以降は、SHA-256OVFファイルをサポートしています。

SHA256からSHA1に変換する方法：

$ ovftool --shaAlgorithm=SHA1 /path/to/the/original/ova_file.ova /path/to/the/new/ova/file-SHA1.ova

  1. ESXi/vSphereサーバーからVMware vSphere Clientをダウンロードしてインストールします。

  2. VMware vSphereクライアントを開きます。

  3. サーバー名またはIPアドレスと資格情報をそれぞれのフィールドに入力し、[Login（ログイン）]をクリックします。

  4. vSphere Clientウィンドウで、[File（ファイル）][Deploy OVF Template（OVFテンプレートのデプロイ）]を選択します。

  5. [Deploy OVF Template（OVFテンプレートのデプロイ）]ウィンドウで、[Browse（ブラウズ）]をクリックします。

  6. Okta Access Gateway OVAファイルを選択し、[Open（開く）]をクリックします。

  7. [Next（次へ）]をクリックします。

  8. 提供されたテンプレートの詳細をレビューし、[Next（次へ）]をクリックします。

  9. Access Gatewayライセンス契約への同意を求めるメッセージが表示されたら、[Accept（同意する）]をクリックし、[Next（次へ）]をクリックします。

  10. Access Gatewayテンプレートの名前を入力し、[Next（次へ）]をクリックします。

  11. ストレージの場所を選択し、[Next（次へ）]をクリックします。

  12. 要件に基づく適切なディスクフォーマットオプションを選択し、[Next（次へ）]をクリックします。

  13. [Finish（終了）]をクリックします。vSphere Clientがデプロイメントプロセスを開始します。

  14. デプロイメントが完了したら、[Close（閉じる）]をクリックします。

  15. vSphere Clientウィンドウで[Inventory（インベントリ）]をクリックします。

  16. 現在サーバーにデプロイされているVMを表示するには、[Virtual Machines（仮想マシン）]タブを選択します。

  17. Access Gateway VMを選択し、ツールバーの[Power On（電源オン）]（再生アイコン）をクリックします。

  18. VMを右クリックし、[Open Console（コンソールを開く）]をクリックしてVMにサインインします。

サポートされるデプロイメント環境

  • VMware Workstation Player：

    VMWare Workstation Playerを使用してデプロイするには、「VMware Workstationデプロイメント」を参照してください。

  • VMware vSphere/ESXi：

    VMWare ESXi/vSphereを使用してデプロイするには、「VMware vSphere/ESXiデプロイメント」を参照してください。

  • Oracle VirtualBox：

    Oracle VirtualBoxはテストおよび開発での使用のみがサポートされます。Oktaでは、Oracle VirtualBoxの本番環境でのデプロイメントをサポートしていません。

    Oracle VirtualBoxを使用してデプロイするには、「Oracle VirtualBox（OVB）デプロイタスク」を参照してください。

  • Amazon Web Services：

    Amazon Web Servicesを使用してデプロイするには、「Amazon Web Servicesデプロイメントタスク」を参照してください。

  • Oracle Cloud Infrastructure：

    Oracle Cloud Infrastructureを使用してデプロイするには、「Oracle Cloud Infrastructureデプロイメントタスク」を参照してください。

  • Microsoft Azure：

    Microsoft Azureを使用してデプロイするには、「Microsoft Azureのデプロイメントタスク」を参照してください。
