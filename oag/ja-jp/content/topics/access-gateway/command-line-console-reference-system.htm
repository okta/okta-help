システムメニュー

システムメニューでは、ホスト名の変更、パッケージのインストールや削除、アップデートの実行、Access Gatewayインスタンスのリブート、シャットダウン、 リセットを実行できます。

5を選択して［System（システム）］サブメニューを開きます。

システムコマンド

  1. ホスト名を変更：Access Gatewayインスタンスのホスト名を変更します。
  2. 高可用性を構成する：Access Gateway高可用性を設定して管理します。
  3. パッケージをインストール：名前付きパッケージをインストールします。
  4. パッケージを削除：名前付きパッケージを削除します。
  5. アップデート：システムをアップデートします。
  6. リブート：システムをリブートします。
  7. シャットダウン：システムをシャットダウンします。
  8. リセット：システムのオリジナルのデフォルト設定にリセットします。

ホスト名の変更

  1. 1を押してホスト名を変更します。
    現在のホスト名が表示されます。
  2. 新しいホスト名を入力し、改行キーを押します。
  3. yを押して確定します。
  4. Enterを押してシステムメニューに戻ります。

高可用性を構成する

［High Availability Configuration（高可用性の構成）］メニューには、Access Gatewayノードの現在のロールが表示されます。

  • ［Single（単一）］: ノードはまだワーカー/管理者として構成されていません。
  • ［Admin（管理者）］:ノードは高可用性の管理者として構成されています。
  • ［Worker（ワーカー）］：ノードは高可用性のワーカーとして構成されています。

  1. 1：ノードキーをリセットします。
  2. 2：新規ワーカーノードとの接続を管理者ノード側で準備します。
  3. 3：ワーカーノードになるようにノードを準備します。
  4. 4：既知のすべてのワーカーノードがリスト表示されます。これは、管理者ノードのみに適用されます。
  5. 5：既知のワーカーノードを削除します。管理者ノードのみに適用されます。
  6. 6：高可用性ステータスを確認できます。
  7. 7：クラスターマネージャーを使ったローリングアップグレードを実行します。

高可用性設定の詳細については、「高可用性を設定して管理する」をご覧ください。

ローリングアップグレードの実行については、「管理者の再指定を実行する」をご覧ください。

クラスターマネージャ

クラスターマネージャメニューが表示されます。

コピー 
Access Gatewayクラスターマネージャー（ワーカー）
1 - ノード指定を承認
2 - ノードを管理ノードとして認可

X- 終了
選択：

ワーカーノードで、[1-Authorize Node Nomination（ノードの再指定を認可）]を選択し、「オペレーションはワーカーノードでサポートされていません」というエラーが発生した場合、任意のキーを押して続行します。

別の承認プロセスの進行中にノード指定承認プロセスの実行を試みると、「Admin Nomination in progress for node - <nominatedNode Hostname>（ノードの管理者指定が進行中）」というエラーが発生します。

  1. ノードの指定を承認する
    1. ［1 - Authorize Node Nomination（ノードの指定を承認する）］を選択します。

      管理ノードで[2-Authorize Node as Admin node（管理ノードとしてノードを認可する）]を選択すると「オペレーションは管理者ノードでサポートされていません。任意のキーを押して続行します」というエラーになります。

      別の承認が進行中の場合に管理ノードとしてノードの認可プロセスを試みると、「管理者指定が進行中」というエラーが発生します。

    2. 現在の管理者ノードが次のメッセージを表示します。

      コピー
      注意：管理者ノードのセットアップ準備ができており、ワーカーノードに表示された権限付与
      トークンを保有していることを確認してください。

      指定された管理者ノードに表示された権限付与トークンを入力し：

      完了したら、Enterキーを押します。

      再指定プロセスが開始します。

  2. ノードを管理者ノードとして認可します。

    1. [2-Authorize Node as Admin node（ノードを管理者ノードとして認可する）]を選択します。

    2. 次にワーカーノードは確認ダイアログボックスを提示します。続行する場合はYを入力し、中断する場合はNを入力します。

      クラスターマネージャーパッケージがインストールされていないまたは管理者ノードでサポートされていない場合は、以下のエラーメッセージが表示されます。

      コピー
      管理者ノードでクラスターマネージャーパッケージが見つかりませんでした - OAGバージョン - 2020.7.0

      続行する前に管理者ノードをアップグレードしてください。

    3. ワーカーノードは、管理ノードに提供する必要のある認可コードを生成および表示します。

      次の認可コードをコピーします：

      コピー

      指定された管理者ノードのセットアップ開始に必要な権限付与トークン：ワーカー-c:8ba1c123-715d-4b70-ab5d-0e41493bef73の場合
      、トークンをコピーして、プロンプトが表示されたら管理ノードにペーストします。
      Enterを押して続行します。

パッケージをインストールする

  1. 2を押して新しいパッケージをインストールします。
  2. パッケージの名前を入力し、改行キーを押します。
  3. yを押して確定します。
  4. Enterを押して［System（システム）］メニューに戻ります。

パッケージを削除する

  1. 3を押して既存のパッケージを削除します。
  2. パッケージの名前を入力し、改行キーを押します。
  3. yを押して確定します。
  4. Enterを押して［System（システム）］メニューに戻ります。

リブート

  1. 5を押してリブートします。

  2. yを押して続行するか、nを押してキャンセルします。

  3. システムがリブートされます。

シャットダウン

  1. 6を押してシャットダウンします。

  2. yを押して続行するか、nを押してキャンセルします。

  3. システムがシャットダウンされます。

リセット

  1. 7を押してリセットします。

  2. yを押して続行するか、nを押してキャンセルします。

  3. システムはすべての設定をデフォルト値に戻します。

リセット後、ゲートウェイアプライアンスがシャットダウンするので再起動する必要があります。
