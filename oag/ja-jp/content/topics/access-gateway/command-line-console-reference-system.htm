システムメニュー

システムメニューでは、ホスト名の変更、パッケージのインストールや削除、アップデートの実行、Access Gatewayインスタンスのリブート、シャットダウン、 リセットを実行できます。

5を選択し、［System（システム）］サブメニューを開きます。

システムコマンド

  1. ホスト名を変更：Access Gatewayインスタンスのホスト名を変更します。
  2. 高可用性の構成：Access Gateway高可用性を設定して管理します。
  3. パッケージをインストール：名前付きパッケージをインストールします。
  4. パッケージを削除：名前付きパッケージを削除します。
  5. アップデート：システムをアップデートします。
  6. リブート：システムをリブートします。
  7. シャットダウン：システムをシャットダウンします。
  8. リセット：システムのオリジナルのデフォルト設定にリセットします。

ホスト名の変更

  1. 1を押してホスト名を変更します。
    現在のホスト名が表示されます。
  2. 新しいホスト名を入力し、改行キーを押します。
  3. yを押して確定します。
  4. Enterを押してシステムメニューに戻ります。

高可用性の構成

高可用性メニューにはAccess Gatewayノードの現在のロールが表示されます。次のロールがあります。

  • 未設定：ノードはワーカーまたは管理者のいずれにも設定されていません。
  • 管理者：ノードは高可用性の管理者として設定されています。
  • ワーカー：ノードは高可用性のワーカーとして設定されています。

  1. 1を押してノードキーをリセットします。
  2. 2を押して管理者ノードが新規ワーカーノードに接続するよう準備します。
  3. 3を押してノードがワーカーノードになるよう準備します。
  4. 4を押して既知のすべてのワーカーノードをリスト表示します。管理者ノードのみに適用されます。
  5. 5を押して既知のワーカーノードを削除します。管理者ノードのみに適用されます。
  6. 6を押して高可用性ステータスを確認します。
  7. 7を押してクラスターマネージャを使ったローリングアップグレードを実行します。

高可用性設定の詳細については、「高可用性を設定して管理する」をご覧ください。

ローリングアップグレードの実行については、「管理者の再指定を実行する」をご覧ください。

パッケージをインストールする

  1. 2を押して新しいパッケージをインストールします。
  2. パッケージの名前を入力し、改行キーを押します。
  3. yを押して確定します。
  4. 改行キーを押してシステムメニューに戻ります。

パッケージを削除する

  1. 3を押して既存のパッケージを削除します。
  2. パッケージの名前を入力し、改行キーを押します。
  3. yを押して確定します。
  4. 改行キーを押してシステムメニューに戻ります。

アップデート

Access Gatewayのアップグレードに関する詳しい説明は「Access Gatewayをアップグレードする」をご覧ください。

Access Gateway 2021.8.0以前で利用可能。

  1. 4を押してアップデートプロセスを開始します。
  2. アップデートの準備ができたら、yを押して続行するかnを押してキャンセルします。
  3. yを押してパッケージのダウンロードを確定します。ダウンロードが始まり、アップデートが開始されます。
  4. Enterを押してシステムメニューに戻ります。

Access Gatewayの選択的アップグレードは、Access Gatewayのバージョンから後のバージョンへのアップグレードと、最新バージョンへのアップグレードを両方サポートします。

Access Gatewayのアップグレードに関する詳しい説明は「選択的アップグレードのワークフロー」をご覧ください。

Access Gatewayバージョン2021.9.3より利用可能

Access Gatewayを最新バージョンにアップグレードする

アップグレードするノードで、以下の手順を実行します。

  1. ターゲットインスタンスのAccess Gateway管理者コンソールに接続します。

    >ssh oag-mgmt@[node.tld]

  2. [5-Sysetem（システム）]を選択します。
  3. [4- Upgrade（アップグレード）]を選択します。
    アップグレードメニューが表示され、以下のようになります。
    Upgrade Access Gateway
    1 - Upgrade to the latest Access Gateway version. . . 2 - View list of Access Gateway versions to upgrade. . . x - Exit
  4. 1 - Upgrade to the latest Access Gateway version. . .（最新のAccess Gatewayバージョンにアップグレードする）を入力します。
  5. アップグレード確認ダイアログが表示され、以下のようになります。
    1を入力すると確定され、アップグレードが開始されます。
    終了し、アップグレードをキャンセルするにはX を入力します。Okta Access Gateway 202X.Y.Z-abcdefgにアップグレードする。 注: 操作を確認すると、アップグレードプロセスが開始されます。システム構成は、更新前に自動的にバックアップされます。この作業を行う前に、システムのスナップショットを撮るすることを強くお勧めします。 1 - 確認 X - 終了 選択肢:
  6. 1を入力して続行します。

    Xを入力するとアップグレードが中止され、前のメニューに戻ります。

  7. アップグレードが開始され、以下のように結果が表示されます。 Update in progress... okta-datastores-202X.Y.Z-abcdefg.x86 64 . . . Installed: okta-monitoring-rest-202X.Y.Z-abcdefg.x86_64 Complete! Dependencies resolved. . . . Complete! Upgrade to Okta Access Gateway 202X.Y.Z-abcdefg successful. いずれかのキーを押して、結果の確認を行います。
  8. yumビューページが表示されます。
    yを入力し、yumアップグレードログを表示します。
    xを入力すると、終了し、メインのアップグレードメニューに戻ります。

選択したAccess Gatewayバージョンにアップグレードする

アップグレードするノードで、以下の手順を実行します。

  1. ターゲットインスタンスのAccess Gateway管理者コンソールに接続します。

    >ssh oag-mgmt@[node.tld]

  2. [5-Sysetem（システム）]を選択します。
  3. [4- Upgrade（アップグレード）]を選択します。
    アップグレードメニューが表示され、以下のようになります。
    Upgrade Access Gateway
    1 - Upgrade to the latest Access Gateway version. . . 2 - View list of Access Gateway versions to upgrade. . . x - Exit
  4. ［2 - View list of Access Gateway versions. . .（Access Gatewayバージョンのリストを表示）］を選択します。次のような使用可能なバージョンのリストが表示されます。

    Available versions Okta Access Gateway 2021.A.x-abcdefg Okta Access Gateway 2021.B.x-abcdefg Okta Access Gateway 2021.C.x-abcdefg Okta Access Gateway 2021.D.x-abcdefg Okta Access Gateway 2021.E.x-abcdefg Up/Down - scroll list Return - select x - exit

  5. 矢印キーを使って目的のバージョンを選択し、改行キーを入力するとアップグレードプロセスが開始されます。

リブート

  1. 5を押してリブートします。

  2. yを押して続行するか、nを押してキャンセルします。

  3. システムがリブートされます。

シャットダウン

  1. 6を押してシャットダウンします。

  2. yを押して続行するか、nを押してキャンセルします。

  3. システムがシャットダウンされます。

リセット

  1. 7を押してリセットします。

  2. yを押して続行するか、nを押してキャンセルします。

  3. システムはすべての設定をデフォルト値に戻します。

リセット後、ゲートウェイアプライアンスがシャットダウンするので再起動する必要があります。
