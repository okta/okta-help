ログのフォーマットと例

Access Gatewayは、管理者のアクション、ユーザーアクセス、承認などの状態を含むすべてのイベントおよびアクションをログします。

  • 一般的なログの形式
  • 管理者 UIコンソール
  • ゲートウェイ
  • プロセスモニター
  • アクセスログ

一般的なログの形式

Access Gatewayは監査イベントを以下の形式でログに記録します：

TIMESTAMP HOSTNAME APPLICATION PROCID COMPONENT SUB-COMPONENT LOG_LEVEL EVENT [STRUCTURED_DATA] MESSAGE

ログステートメントのフィールド

フィールド

説明

タイムスタンプ 現在のシステムの日付と時間
ホスト名 マシンのホスト名
アプリケーション Access Gateway
PROC_ID プロレス層
COMPONENT（コンポーネント） プロセスのコンポーネント
SUB-COMPONENT（サブコンポーネント） プロセスのサブコンポーネント
LOG_LEVEL ログレベル
EVENT（イベント） イベントの種類
STRUCTURED_DATA

分析/トラブルシューティングに重要な発生したイベントに関するデータ
メッセージ

フィールド 説明

名前

ドメイン

IDPドメイン

タイプ

IDPタイプ

RESULT（結果）

成功/失敗

REASON（理由）

VALID / INVALID_NETWORK_CONN (FAIL)、INVALID_TOKEN (FAIL)

  1. 有効化なIDP

    1. ログレベル：INFO

    2. 結果: PASS（成功）

    3. 理由: VALID（有効）

    4. メッセージ: Success confirming IDP status with: <IDP Domain>（IDPドメインを用いたIDPステータスの確認に成功しました）

    5. ログサンプル：

      Oct 9 04:00:00 Access Gateway WEB_CONSOLE IDP LOCAL INFO SYSTEM_IDP_STATUS [NAME="<IDP Name> IDP" DOMAIN="<IDP URL>" TYPE="<Identity Provider type>" RESULT="PASS" REASON="VALID"] Success confirming IDP status with: <IDP URL>

  2. IDPはネットワーク到達不可能

    1. ログレベル：ALERT

    2. 結果: FAIL（失敗）

    3. 理由: INVALID_NETWORK_CONN

    4. メッセージ: Failure confirming connectivity with IDP: <IDP Domain>.（IDPドメインを用いたIDPステータスの確認に失敗しました）ネットワーク構成を確認してください。

    5. ログサンプル：

      Oct 9 04:02:00 Access Gateway WEB_CONSOLE IDP LOCAL INFO SYSTEM_IDP_STATUS [NAME="<IDP Name> IDP" DOMAIN="<IDP URL>" TYPE="<Identity Provider type>" RESULT="FAIL" REASON="INVALID_NETWORK_CONN"] Failure confirming connectivity with IDP: <IDP URL>>. ネットワーク構成を確認してください。

  3. IDPセキュリティ鍵が有効でなくなった

    1. ログレベル：ALERT

    2. 結果: FAIL（失敗）

    3. 理由: INVALID_TOKEN

    4. メッセージ：Failure validating security token with IDP（ IDPを使用したセキュリティトークンの検証に失敗）: <IDP Domain>（IDPドメイン）。トークンが存在し、有効になっていることを検証してください。

    5. ログサンプル：

      Oct 9 04:02:23 Access Gateway WEB_CONSOLE IDP LOCAL INFO SYSTEM_IDP_STATUS [NAME="<IDP Name> IDP" DOMAIN="<IDP URL>" TYPE="<Identity Provider type>" RESULT="FAIL" REASON="INVALID_NETWORK_CONN"] Failure validating security token with IDP: <IDP Domain>. トークンが存在し、有効になっていることを検証してください。
