サービス

[Services（サービス）]メニューを使用すると、Access Gatewayアプライアンスでサービスを開始、停止、再起動したり、実行中のサービスのステータスを確認したり、SSL証明書を再生成したりできます。表示できるサービスは、NGINX、Access Gateway 管理者、NTP、およびセッションキャッシュです。

メインメニューで2を入力すると、Access Gatewayのサービスサブメニューに進みます。

サービスコマンド

  • NGINX：NGINXエージェント管理サブメニューを開きます。
  • Access Gateway管理者：Access Gateway Admin管理の管理サブメニューを開きます。
  • NTP：NTP管理サブメニューを開きます。
  • セッションキャッシュ：セッションキャッシュサブメニューを開きます。
  • DNSリゾルバ：DNSリゾルバサブメニューを開きます。

NGINX

NGINXメニューを使用して、NGINXサービスの開始、停止、再起動、サービスのステータス確認、およびSSL証明書の再生成を行うことができます。

  1. 開始：サービスを開始します。
  2. 停止：サービスを停止します。
  3. 再起動：サービスを再起動します。
  4. ステータスを確認：このオプションを使用して、サービスが実行中か停止中かなど、サービスのステータスを表示できます。
  5. SSL証明書を再生成：SSL証明書を再生成します。
  6. SSL証明書をアップデート：SSL証明書をアップデートします。
  7. TLS 1.1を無効化：TLS 1.1のサポートを無効にします。
  8. TLS 1.1を有効化：TLS 1.1のサポートを有効にします。
  9. 暗号を管理:暗号を管理します。

開始、停止、再起動

  1. NGINXメニューで1、2、または3を押します。NGINXサービスが要求された操作を実行します。

  2. Enterを押すとNGINXメニューに戻ります。

ステータスを確認

  1. NGINXメニューで4を押すと、NGINXサービスのステータスが表示されます。

  2. Enterを押すとNGINXメニューに戻ります。

SSL証明書を再生成

  1. NGINXメニューで5を押します。

  2. Access Gatewayアプライアンスの設定に使用するドメイン名を入力し、Enterを押します。

  3. 改行キーを押してNGINXメニューに戻ります。

SSL証明書を更新

  1. NGINXメニューで6を押すと、既知の証明書がリスト表示されます。Enterを押して続行します。

  2. 更新する証明書のドメイン名を入力します。置換する証明書ファイルの内容が手元にあり、エディターにペーストする準備ができている必要があります。

  3. 画面の指示に従って証明書の内容を挿入して変更し、更新された証明書を保存します。

  4. Enterを押すとNGINXメニューに戻ります。

TLS 1.1を無効化／有効化

  1. 7または8を押してNGINXメニューでTLS 1.1を無効／有効にします。

  2. 改行キーを押してNGINXメニューに戻ります。

暗号を管理

  1. NGINXメニューで9を押します。現在指定されている暗号セットが表示されます。

  2. 新しい暗号セットを入力します。暗号のセットは「:」で区切られます。暗号セットを除外するときは、プレフィックス「!」を使用します。

  3. Enterを押すとNGINXメニューに戻ります。

Access Gateway Admin（Access Gateway管理者）

Access Gateway Admin（Access Gateway管理）メニューでは、Access Gateway Adminサービスの開始、停止、再起動、およびサービスのステータス確認を実行できます。

サブメニューには次のコマンドが含まれます。

  1. 開始：Access Gateway Admin（Access Gateway管理者）を開始します。
  2. 停止：Access Gateway Admin（Access Gateway管理者）を停止します。
  3. 再起動：Access Gateway Admin（Access Gateway管理者）を再起動します。
  4. ステータスを確認：Access Gateway Admin（Access Gateway管理者）ステータスを表示します。

NTP

NTPサブメニューには次のコマンドが含まれます。

  1. NTPの設定：Access Gatewayが時間の同期に使用するNTPサーバーを1つ以上設定します。

  2. 開始：NTPサービスを開始します。

  3. 停止：NTPサービスを停止します。

  4. 再起動：NTPサービスを再起動します。

  5. ステータスを確認：NTPサービスのステータスを表示します。

  6. システム時刻を設定：システム時刻を設定します。

セッションキャッシュ

セッションキャッシュサブメニューには次のコマンドが含まれます。

  1. 開始：セッションキャッシュサービスを開始します。

  2. 停止：セッションキャッシュサービスを停止します。

  3. 再起動：セッションキャッシュサービスを再起動します。

  4. セッションキャッシュのステータス：キャッシュサービスのステータスを表示します。

  5. 実行中の情報を表示：キャッシュサービスのステータスを表示します。

  6. メモリ使用量を変更[...]：セッションキャッシュに割り当てられるメモリーを変更します。

    1. 6の[メモリー使用量を変更[...]]を選択します
    2. 次のいずれかを入力します。
      • 値：MB単位の具体的な値。たとえば、1500はメモリ割り当てを1500MBに設定します。
      • -値：総メモリから差し引く値。たとえば、-500はメモリ割り当てを500MBに設定します。
      • 値%：メモリの割合。たとえば、30%はメモリ割り当てを総メモリの30%に設定します。
      • -値%：総メモリから差し引く割合。たとえば、-10%はメモリ割り当てを総メモリから10%差し引いた値に設定します。

DNSリゾルバ

DNSリゾルバサブメニューには次のコマンドが含まれます。

  1. 開始：DNSリゾルバを開始します。

  2. 停止：DNSリゾルバを停止します。

  3. 再起動：DNSリゾルバを再起動します。

  4. ステータスを確認：DNSリゾルバサービスのステータスを表示します。

  5. リセット：DNSリゾルバをリセットします。
