選択的アップグレードワークフロー

選択的アップグレードは、Okta Access Gatewayのノードを現在のバージョンから新しいバージョンにアップグレードします。アップグレードは、現在リリースされているバージョンにすることができますが、現在ノードにインストールされているバージョンより新しいバージョンにすることもできます。

Access Gatewayバージョン2021.9.3より利用可能

開始する前に

以下を確認します。

  • Access Gatewayコマンドラインコンソールへのアクセス権を持っている。

  • 各Access Gatewayノードは、yum.oag.okta.comにアクセスできます。Access Gateway 管理者コンソールを使用すると、Access Gateway yumレポジトリへの接続性をテストできます。

    接続性をテストする方法：

    1. 安全なシェルを使って、Access Gateway 管理者コンソール：ssh oag-mgmt@adminに接続します。
    2. [1-Network（ネットワーク）]を選択します。
    3. [7-Connectivity Test（接続テスト）]を選択します。
    4. ホスト：yum.oag.okta.comを入力する
    5. ポート：443を入力する

    yum.oag.okta.com接続確認メッセージをレビューします。エラーが発生した場合、ホストyum.oag.okta.comが、Access Gatewayがデプロイされているネットワークからポート443を使って接続していることを確認します。

選択的アップグレードを実行する

以下の選択的操作がサポートされています。

  1. 最新のAccess Gatewayバージョンにアップグレードする。
  2. 選択したAccess Gatewayバージョンにアップグレードする。

最新のAccess Gatewayバージョンにアップグレードする

アップグレードするノードで、以下の手順を実行します。

  1. ターゲットインスタンスのAccess Gateway 管理者コンソールに接続します。

    >ssh oag-mgmt@[node.tld]

  2. [5-Sysetem（システム）]を選択します。
  3. [4- Upgrade（アップグレード）]を選択します。
    アップグレードメニューが表示され、以下のようになります。
    Upgrade Access Gateway
    1 - Upgrade to the latest Access Gateway version. . . 2 - View list of Access Gateway versions to upgrade. . . x - Exit
  4. [1 - Upgrade to the latest Access Gateway version ...（最新のAccess Gatewayバージョンにアップグレードする）]を選択します。
  5. アップグレード確認ダイアログが表示され、以下のようになります。
    1を入力して確定をし、アップグレードを開始します。
    終了し、アップグレードをキャンセルするにはX を入力します。Okta Access Gateway 202X.Y.Z-abcdefgにアップグレードする。 注: 操作を確認すると、アップグレードプロセスが開始されます。システム構成は、更新前に自動的にバックアップされます。この作業を行う前に、システムのスナップショットを撮るすることを強くお勧めします。 1 - 確認 X - 終了 選択肢:
  6. 1を入力して続行します。

    Xを入力するとアップグレードが中止され、前のメニューに戻ります。

  7. アップグレードが開始され、以下のように結果が表示されます。 Update in progress... okta-datastores-202X.Y.Z-abcdefg.x86 64 . . . Installed: okta-monitoring-rest-202X.Y.Z-abcdefg.x86_64 Complete! Dependencies resolved. . . . Complete! Upgrade to Okta Access Gateway 202X.Y.Z-abcdefg successful. いずれかのキーを押して、結果の確認を行います。
  8. yumビューページが表示されます。
    yを入力し、yumアップグレードログを表示します。
    xを入力すると、終了し、メインのアップグレードメニューに戻ります。

選択したAccess Gatewayバージョンにアップグレードする

アップグレードするノードで、以下の手順を実行します。

  1. ターゲットインスタンスのAccess Gateway管理者コンソールに接続します。

    >ssh oag-mgmt@[node.tld]

  2. [5-Sysetem（システム）]を選択します。
  3. [4- Upgrade（アップグレード）]を選択します。
    アップグレードメニューが表示され、以下のようになります。
    Upgrade Access Gateway
    1 - Upgrade to the latest Access Gateway version. . . 2 - View list of Access Gateway versions to upgrade. . . x - Exit
  4. [2 - View list of Access Gateway versions. . .（Access Gatewayバージョンのリストを表示）]を入力します

    次のような使用可能なバージョンのリストが表示されます。

    Available versions Okta Access Gateway 2021.A.x-abcdefg Okta Access Gateway 2021.B.x-abcdefg Okta Access Gateway 2021.C.x-abcdefg Okta Access Gateway 2021.D.x-abcdefg Okta Access Gateway 2021.E.x-abcdefg Up/Down - scroll list Return - select x - exit
  5. [up（上）]/[down（下）]または矢印を使用して該当するものを選択し、Enterを押してアップグレードプロセスを開始します。
  6. アップグレードの確認ダイアログで、1を入力し、アップグレードを開始します。Upgrade to Okta Access Gateway 202X.Y.Z-abcdefg NOTE: Confirming the operation will start the upgrade process. The system configuration will be automatically backed up before the update. It is strongly recommended that to take a system snapshot before proceeding. 1 - Confirm X - Exit Choice:
  7. アップグレードが完了したら、任意のキーを使用して続行します。Update in progress... okta-datastores-202X.Y.Z-abcdefg.x86 64 . . . Complete! Upgrade to Okta Access Gateway 202X.Y.Z-abcdefg successful.
  8. yumの表示ページには、2つのオプションがあります。
    • yを入力し、yumアップグレードログを表示します。
    • xを入力すると終了し、メインのアップグレードメニューに戻ります。
