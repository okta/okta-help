Oracle E-Business Suite for Rapid SSOを構成し、DBCファイルを作成する

E-Business Suite（EBS）環境の構成セクションは、Access Gateway Rapid SSO統合を使用する統合にのみ適用されます。Oracle EBS classic（Oracle AccessGateおよびOracle Internet Directory/Universal Directory）を使用して統合する場合は、このセクションをスキップしてください。

  • 必要なユーザーを作成し、ロールを割り当てる
  • EBSシングルサインオン用の有効化
  • Okta Access GatewayをOracle E-Business Suiteに登録する
  • テスト

必要なユーザーを作成し、ロールを割り当てる

  1. Oracle E-Business Suiteコンソールを参照し、sysadminとしてサインインします。例：http://ebs-internal.example.com:8000/OA_HTML/AppsLogin
  2. Oracle E-Business Suiteホームページから、［Navigator（ナビゲーター）］パネルを下方にスクロールし、［User Management（ユーザー管理）］を展開します。
  3. [Users（ユーザー）]をクリックします。[User Maintenance（ユーザーメンテナンス）] ページが表示されます。
  4. [Register（登録）]ドロップダウンリストから[User Account（ユーザーアカウント）]を選択し、[Go（続行）]をクリックします。[Create User（ユーザーの作成）]アカウントページが開きます。
  5. OAGSSOUSERユーザーを作成するため以下の詳細を入力し、[Submit（送信）]をクリックします。
    フィールド値
    User Name（ユーザー名）OAGSSOUSER
    Password（パスワード）適切なパスワード
    Description（説明）SSOのためのOAGユーザーアカウント
    Password Expiration（パスワード有効期限）なし
  6. [Assign Roles（ロールの割り当て）]をクリックします。[Update User（ユーザーの更新）]ページが表示されます。
  7. [Assign Roles（ロールの割り当て）]をクリックします。
  8. 検索フィールドにAPPS_SCHEMA_CONNECTを入力し、[Go（続行）]をクリックします。このロールは、EBSデータベースと通信する権限をユーザーに付与します。
  9. 適切な理由（例えば、OAG SSOに必要）、現在の日付からの有効な開始日を入力し、[Apply（適用）]をクリックします。
  10. 後ほど使用するため、ブラウザウィンドウを開いたままにします。

シングルサインオン用のEBSの有効化

  1. Windowsまたは該当するOS固有のメニューから、Javaコントロールパネルを開き、[Security（セキュリティ）]タブを選択します。
  2. 例外リストにOracle E-Business Suiteホストを追加します。
  3. Oracle E-Business Suiteコンソールブラウザに戻ります。
  4. [Navigation（ナビゲーション）]ペインで[System Administrator（システム管理者）]にスクロールし、 [System Administrator（システム管理者）] > [Profile（プロファイル）] > [System（システム）]を展開します。Javaアプレットが起動します。
  5. セキュリティ警告が表示された場合、［I accept the risk（リスクを受け入れる）］と［Run（実行）］を順にクリックします。この警告は、2回表示される場合があります。
  6. ［System Administrator（システム管理者）］>［Profile（プロファイル）］>［System（システム）］を選択し、アプリケーション認証エージェントのプロファイルを探します。
  7. サイトAccess Gatewayがユーザーを認証するために使用するEBSルート（例えば、https://ebssso.example.com）にを更新します。このフィールドには、Access Gatewayで保護されたアプリケーションの完全修飾ドメイン名が含まれている必要があります。[Save（保存）]をクリックします。
  8. ウィンドウメニューで[Find System Profile Values（システムプロファイル値を探す）]を選択します。また、代わりに[System Administrator（システム管理者）] > [Profile（プロファイル）] > [System（システム）]を選択することもできます。
  9. アプリケーションSSOのタイプを検索します。
  10. アプリケーションSSOのタイプをSSWAからSSWA/w SSOに変更します。[Save（保存）]をクリックします。
  11. ウィンドウメニューで[Find System Profile Values（システムプロファイル値を探す）]を選択します。また、代わりに[System Administrator（システム管理者）] > [Profile（プロファイル）] > [System（システム）]を選択することもできます。
  12. アプリケーションSSOのログインタイプを検索します。
  13. アプリケーションSSOのログインタイプが[BOTH（両方）]に設定されていることを確認します。[Save（保存）]をクリックします。
  14. ウィンドウメニューで[Find System Profile Values（システムプロファイル値を探す）]を選択します。また、代わりに[System Administrator（システム管理者）] > [Profile（プロファイル）] > [System（システム）]を選択することもできます。
  15. %Session Cookie%を探します。
  16. Oracleアプリケーションセッションクッキードメインを[DOMAIN（ドメイン）]に変更します。[Save（保存）]をクリックします。
  17. Javaアプレットを終了します。
  18. コンソールを終了します。
  19. Oracle E Business Suiteを再起動します。これには、15～30分程度かかる場合があります。

Access Gatewayは、E-Business Suite APIを使用して、E-Business Suiteデータベースと通信します。E-Business Suite管理者に、この通信がどのポートを使用しているかを確認し、Access GatewayとバックエンドのE-Business Suiteデータベースの間でこのポートが開いていることを確認します。通常、ポート1521を使用しますが、他のポートを使用している場合もあります。

Okta Access GatewayをOracle E-Business Suiteに登録する

  1. Oracle E-Business Suiteサーバーで、ebs.war Webアプリケーションを探し、展開します。アプリは、Oracle Identity Cloud サービスコンソールからダウンロードできます。

  2. ebs.warファイル内のWEB-INF/libフォルダーにあるfndext.<Major>.<minor>.<revision>.jarファイルを解凍します。インストールされている製品によっては、findext.jarがすでに解凍されている場合があります。

    $find . -name 'fundext*.jar. /u01/install/APPS/fs1/EBSapps/comn/java/classes/oracle/apps/fnd/jar/fndext.jar
  3. コマンドプロンプトを開き、zipを解凍した場所にディレクトリを変更します。
  4. 次のようなコマンドを使用して、fndext-M.m.rev.jarファイルをE-Business Suiteホストにコピーします：

    scp fndext-2.0.8.jar oracle@<EBS_IP>:/home/oracle

  5. 次のようなコマンドを使ってEBSホストへのセキュアシェルセッションを開きます。

    ssh oracle@<EBS_IP>

  6. 次のようなコマンドを使ってE-Business Suiteの環境を構成します：

    . /u01/install/APPS/EBSapps.env run

  7. 次のようなコマンドを使ってEBSSDKディレクトリを準備します：

    cd $HOME; mkdir EBSSDK; mv fndext-2.0.8.jar; cd EBSSDK

  8. 次のようなコマンドを使ってデスクトップアプリセキュリティを構成します。<your ebs domain name>は、自分のEBSドメインのアドレスに置き換えてください：

    java oracle.apps.fnd.security.AdminDesktop apps/apps CREATE NODE_NAME=<your ebs domain name> DBC=/u01/install/APPS/fs1/inst/apps/EBSDB_apps/appl/fnd/12.0.0/secure/EBSDB.dbc

  9. 次のようなコマンドを使ってDBCファイルが作成されたことを確認します：

    cat EBSDB.dbc

    このコマンドを実行すると、次のような出力が表示されます。

    #Desktop DB Settings #Tue Nov 19 20:15:56 EST 2019 FNDNAM=APPS APPL_SERVER_ID=97BDEC3E76E2113EE05304FD140A235E23699099323052021210255403175980 APPS_JDBC_URL=jdbc\:oracle\:thin\:@(DESCRIPTION\=( ADDRESS_LIST\=(LOAD_BALANCE\=YES) (FAILOVER\=YES) (ADDRESS\= (PROTOCOL\=tcp) (HOST\=apps.example.com) (PORT\=1521))) (CONNECT_DATA\=(SERVICE_NAME\=EBSDB))) GWYUID=APPLSYSPUB/PUB

Access GatewayでRapid EBSアプリを作成する場合は、このファイルの内容が必要になります。

テスト

  1. http://ebs.example.com:8000/OA_HTML/AppsLocalLogin.jspにアクセスしてローカル認証を確認します。
  2. 次のようなコマンドを使用して、EBSデータベースを実行しているホストへのシェルを保護します：

    ssh oracle@<EBS_IP>

  3. listener.oraが含まれるディレクトリに移動します。
  4. 次のコマンドを実行してlistener.oraの内容を確認します：

    tcp.validnode_checking = NO tcp.invited_nodes = ( ebs.example.com )

  5. リスナーを再起動します。
