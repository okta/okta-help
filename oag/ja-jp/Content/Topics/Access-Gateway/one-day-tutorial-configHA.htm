高可用性の構成

高可用性アーキテクチャは、今日必要とされているフォールトトレランスを提供します。

Access Gatewayの高可用性は以下のもので構成されます：

  • Access Gatewayの単一管理インスタンス。ワーカーノードに対する構成変更を維持および伝播には、管理インスタンスまたは 管理ノードが使用されます。さらに、管理ノードを通常のAccess Gatewayインスタンスとして使用することができます。
  • リクエストをサービスする管理ノードにバインドされた1つ以上のワーカーインスタンス。
  • Access Gatewayの高可用性クラスターにリクエストをルーティングする顧客によって提供されたロードバランサー。

Access Gatewayの高可用性インスタンスライフサイクルの概要：

  • Access Gatewayのインスタンスはプロビジョニングされています。このインスタンスは管理ノードと呼ばれています。このノードは、保護済みのアプリケーションを定義することが含まれるよう通常通りに構成されます。高可用性を構成する前にアプリケーションまたはIDPサポートを構成する必要はありません。
  • Access Gatewayの2回目以降のインスタンスがプロビジョニングされます。これらのインスタンスはワーカーノードと呼ばれています。これらのノードはアプリケーションと構成はされませんが、管理ノードからすべての構成を取得できます。
  • ワーカーノードの場合は、すべての構成に管理ノードを使用するよう特別に構成されています。これらの構成後、ワーカーノードはAccess Gateway 管理 UI コンソールを公開しません。コマンドラインインターフェースを使用した場合にのみアクセスできます。

この図では、管理ノードはワーカーノードとしても動作するように示されています。管理者がリクエストも同様にサポートすることを意図していない場合は、ロードバランサーの構成から省くことができます。

管理ユーザーインターフェイス（UI）は、管理ノードと、まだワーカーノードとして構成されていないノードでのみ使用できます。インスタンスがワーカーとして構成されると、管理UI使ってアクセスすることはできなくなります。Access Gatewayクラスターの管理はすべて、管理ノードとその管理UIで行う必要があります。

概要

高可用性の構成には、次のプロセス全体が含まれます。

  1. 管理ノードを構成する。
    このステップでは、通常管理ノードが構成されます。
  2. ワーカーノードを構成する。
    このステップでは、アプリケーションなしでワーカーノードが構成されます。
  3. 管理ノードを準備します。
    このステップでは、1つまたは複数のワーカーノードが追加にされる場合に、管理ノードのアラートまたは準備に、コマンドラインインターフェースが管理ノードで使用されます。
  4. ワーカーノードを準備します。
    このステップでは、コマンドラインインターフェイスは、Access Gatewayクラスターの一部となるワーカーを準備するためにワーカーノードで使用されます。
  5. クラスターへのワーカーの統合。
    前のステップの後で、ワーカーは自動的にクラスターに統合されます。この段階では、ワーカー管理UIは無効になっていますが、ワーカーがキーを交換すると、管理ノードが構成を提供します。

操作

高可用性の構成方法

  1. Access Gatewayノードと関連するキーをリセット - 管理インスタンスおよびワーカーインスタンス両方のキーをリセットします。
  2. Access Gatewayクラスターへのワーカーノードの追加 - 1つ以上のワーカーノードをクラスターに追加します。
  3. クラスター構成の確認 - 最後の構成を確認します。

Access Gatewayノードに関連付けられるキーをリセット

Access Gatewayノードは、さまざまなキーを使って相互に通信します。Access Gateway高可用性クラスターの一部としてインスタンスを使用する場合は、キーを再生成する必要があります。
キーのリセットが必要なのは、インスタンスごとに一度のみです。

  1. Access Gateway管理コンソール.
    ssh oag-mgmt@[admin or worker]に接続
  2. [5 - Sysetem（システム）]を選択します。
  3. [8 - High Availability Configuration（高可用性の構成）]を選択します。
  4. [1 - Reset Key（キーをリセット）]ノードを選択します。
  5. [y]を入力して高可用性同期プロセスで使用するキーをリセットするか、[N]を入力してリセットプロセスを中断します。
  6. [X]を入力して終了するか、他のメニュー項目を入力して続行します。

ワーカーノードをAccess Gatewayクラスターに追加

ワーカーノードを追加する場合、管理とワーカーノードの両方が以下の状態でなくてはなりません：

  • 以前にプロビジョニングされている。
  • SSHを使って接続できる。
  • キーがリセットされている。「Access Gatewayノードと関連付けられるキーをリセットする」をご参照ください

ワーカーを準備するときには、管理者ではなくワーカーノードに接続されていることを確認してください。クラスター管理者側でワーカーオペレーションの準備を行うと、Access Gateway 管理 UI コンソールが動作しなくなります。Access Gatewayバージョン2021.11.2以降では、このオペレーションは禁止されています。管理者として以前使用したノードは、ワーカーとして再利用する際にリセットする必要があります。「Access Gatewayコマンドラインのリセット」をご参照ください。

  1. 管理ノードでは：

    1. インスタンスAccess Gateway 管理コンソール.ssh oag-mgmt@[admin.tld]に接続します
    2. [5 - Sysetem（システム）]を選択します。
    3. [8 - High Availability Configuration（高可用性の構成）]を選択します。高可用性構成メニューが表示されます：Access Gateway高可用性の設定（ロール） 1 1 - Reset Key（リセットのキー） 2 - Prepare Admin（管理者を準備） 3 - Prepare Worker（ワーカーを準備） 4 - List Nodes（ノードを一覧にする） 5 - Remove Node（ノードを削除する） 6 - Check Status（ステータスをチェックする） X - Exit（終了） Choice（選択）：

      高可用性メニューにはAccess Gatewayノードの現在のロールが表示されます。

      ロールは以下のものである可能性があります：

      • 単一 - ノードはまだワーカーまたは管理者として構成されていない。
      • 管理者 - ノードは高可用性の管理者として構成されています。
      • ワーカー - ノードは高可用性のワーカーとして構成されている。

    4. [2 - Prepare Admin（管理者を準備）]を選択する。

      最初に高可用性の管理者ノードを構成するときは、[1 - Reset Keys（キーのリセット）]オプションを選択および実行し、インスタンスのSSHキーをリセットします。

      キーのリセットが必要なのは、インスタンスごとに一度のみです。「コマンド ライン管理コンソールレファレンス」をご参照ください。

      Access Gatewayのレプリケーションでは、コマンドラインコンソールからホスト名の設定を使用します。

      管理ノードとワーカーノードのホスト名の更新には、コマンド ライン コンソールの[System（システム）（5）]>[Change Hostname（ホスト名の変更）（1）] の順に使用するようにしてください。

    5. 管理ノードは、ワーカーノードに提供される必要のある権限付与トークンを生成および表示します（例：oag.okta.com:8ba1c123-715d-4b70-ab5d-0e41493bef73）。提供された権限付与トークンをコピー： ワーカーノードからのセットアップを初期化するのに必要な権限付与トークンが以下で与えられます。この行の下に次のテキストをコピーし、メッセージが表示されたらワーカーノードに貼り付けます。 oag.okta.com:8ba1c123-715d-4b70-ab5d-0e41493bef73 これまで利用できたワーカーノード： . . . 次に管理ノードはワーカーノードを待機します。

    この時点で、管理ノードはワーカーノードからの接続を待機しています。ワーカーノードが追加されるまでウィンドウは開いたままにします。Xの入力が早すぎると、管理ノードが、プロセスが完了したものと想定し、追加のワーカーノードのリスト化を停止します。[X]の入力は、すべてのワーカーノードが追加された後に行います

    1. ワーカーノードとして添付されているインスタンスのコマンドプロンプトに戻ります。
  2. 各ワーカーノードでは、
    1. インスタンスAccess Gateway管理コンソール.ssh oag-mgmt@[worker.tld]に接続します
    2. [5 - Sysetem（システム）]を選択します。
    3. [8 - High Availability Configuration（高可用性の構成）]を選択します。
    4. [3 - Prepare Worker（ワーカーを準備）]を選択します。

      最初に高可用性の管理者ノードを構成するときは、[1 - Reset Keys（キーのリセット）]オプションを選択および実行し、インスタンスのSSHキーをリセットします。キーのリセットが必要なのは、インスタンスごとに一度のみです。「コマンドライン管理コンソールレファレンス」をご参照ください。

    5. ワーカーは以下を表示します：ホストワーカーのHAの準備状態を確認しています. . . 注意：管理ノードのセットアップの準備ができており、管理ノードに表示された権限付与トークンを保有していることをご確認ください。管理ノードに表示された権限付与トークン： <admin...com>:927da506-7efb-4520-bd32-dd03b86f2a9bを入力します入力後、ワーカーノードはを管理ノードに接続し、権限付与と次のような確認情報を交換します：接続を許可する管理ノード　　　　　　　　　　　　　　　　　　　　　　　　　<admin...com>をリクエストしています　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　 管理ノードにノード<worker...com>が正常に追加されました 現在の構成を同期中 Enterを押して続行します....
    6. メッセージが表示されたら、いずれかのキーを押して続行します。
    7. [X]を入力して終了するか、他のメニュー項目を入力して続行します。

    ワーカーインスタンスが構成され、使用の準備が完了します。

管理ノードでは：

  1. 管理インスタンスAccess Gateway管理コンソールに戻ります。以下のように、新規ワーカーノードを追加した結果を表示します：　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　ワーカーノードからの初期設定に必要な権限付与トークンが以下に付与されます。この行の下のテキストをコピーし、メッセージが表示されたらワーカーノードにペーストします。 <admin...com>:927da506-7efb-4520-bd32-dd03b86f2a9b これまでに利用可能なワーカーノード: <worker1...com> <worker2...com> <worker3...com>
  2. [X]を入力して終了するか、他のメニュー項目を入力して続行します。

クラスター構成をチェック

Access Gateway高可用性クラスターのステータスをレビューまたはチェックする：

  1. Access Gateway 管理コンソール.ssh oag-mgmt@[admin or worker]に接続
  2. [5 - Sysetem（システム）]を選択します。
  3. [8 - High Availability Configuration（高可用性の構成）]を選択します。
  4. [6 - Check Status（ステータスチェック）]を選択します。

    ステータスチェックを行うと、構成が変更された後あるいはNGINXエンジンが再起動したときのみクラスターの最新ステータスが表示されます。

    新しく追加されたノードが表示されない場合は、基板となるエンジンステータスが変更されるAccess Gateway 管理 UI コンソール機能のいずれかを実行します。または、NGINXエンジンを再起動します。「 Access Gateway 管理コンソール サービスセクションのNGINXサブメニュー 」をご参照ください。

  5. クラスターインスタンスのリストが、以下のような関連ステータスと共に表示されます：HA Synch Status Up/Down/Page Up/Page Down/Home/End - Scroll x-exit worker1.yourdoman.tld: Pass worker2.yourdoman.tld: Pass . . . workern.yourdoman.tld: Fail ここでは、
    • Pass（成功）： アクセス可能で機能しているワーカーノード。
    • Fail（失敗）：機能していないワーカー。詳細については、「ノードログ」をご参照ください。
  6. [x]を入力して終了します。
