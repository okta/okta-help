ログフォワーダーを構成する

Access Gatewayはログフォワーダー（Graylogなど）をサポートします。

Access Gatewayバージョン2020.04.4より利用可能

  • ログフォワーダーレシーバーを定義する
  • ログフォワーダーを追加する
  • ログフォワーダーをテストする
  • Access Gatewayログ

開始する前に

次のものが必要です：

  • syslogイベントを受信するように構成されたsyslogサーバー。
  • リモートロギングコンシューマーの接続情報。例：
    パラメーター例

    完全修飾IPアドレスまたはDNS解決可能なロガーの名前

    192.168.1.1

    my.graylog.server

    ロガープロトコル

    TCP/UDP

    ロガーリッスンポート

    5514

    Access Gatewayとログサーバー間の通信に使用されるポートは、オープンである必要があります。

    Access Gatewayは、ログサーバーの接続を検証します。

ログフォワーダーレシーバーを定義する

次の例は、ログフォワーダーレシーバーとしてGraylogを構成する方法を説明しています。フォワードされたログを受信するシステムのタイプを構成するには、そのシステムのドキュメントを参照してください。

Graylogでログフォワーダーを作成するには：

  1. 管理者としてGraylogコンソールにサインインします。
  2. [System（システム）] > [Inputs（入力）]を選択します。
  3. [Select Input（入力を選択]ドロップダウンリストでSyslog UDPを検索します。
  4. [Launch new input（新しい入力を起動）]をクリックします。
  5. [Global（グローバル）]チェックボックスを選択します。

  6. [Title（タイトル）]フィールドにフォワーダーの名前を入力します。

  7. [Port（ポート）]フィールドに、Access Gatewayとログサーバー間の通信に使用するポート番号を入力します。syslog入力レシーバーの構成時に、オペレーティングシステムの制限を避けるために、ポート2048以上を使用してください。ポートは、Access Gateway管理者インスタンスからアクセス可能でなければなりません。

  8. [Save（保存）]をクリックします。
  9. Access Gateway 管理者 UI コンソールに戻ります。

ログフォワーダーを追加する

リモートロガーを追加するには：

  1. Access Gatewayインスタンスに進みます。
  2. [Backups and Logs and Backups（バックアップおよびログとバックアップ）]を選択します。
  3. [Log Forwarder（ログフォワーダー）]を選択します。
  4. [(+)] > [Syslog remote（リモートSyslog）をクリックします。
  5. 以下を入力します。

    フィールド

    値

    Name（名前）

    フォワーダーの名前。

    Feed（フィード）

    以下のいずれか：

    • AUDIT（監査）
    • ACCESS（アクセス）
    • MONITOR（モニタリング）

    各フィードの詳細については、「フィードの例」を参照してください。

    Protocol（プロトコル）

    UDPまたはTCPを選択します。このプロトコルはログリスナーと一致する必要があります。

    Host（ホスト）

    リモートSyslogリスナーのDNS解決可能なホスト名またはIPアドレス。

    Port（ポート）

    リモートsyslogリスナーのポート。

  6. [Validate Forwarder（フォワーダーの検証）]をクリックします。Access Gatewayは、リモートロガー接続情報を検証しようとします。必要に応じて、入力エラーを修正します。検証が成功すると、[Validate Forwarder（フォワーダーの検証）]ボタンが[Forwarder Validated（フォワーダー検証済み）]に変更されます。
  7. [Okay（OK）]をクリックします。
  8. ログフォワーダーの定義が、ログフォワーダーのリストに表示されます。初期ステータスはTesting（テスト中）で、テストに合格するとValid（有効）に変更されます。

ログフォワーダーをテストする

ログ転送をテストするには、次のものが必要です。

  • 構成済みのログレシーバー。「ログフォワーダーレシーバーを定義する」で概説されるステップに従ってください
  • Access Gatewayノードでのログフォワーダーの定義。Access Gateway 管理者 UI コンソールへのサインインに基づきイベントを生成するため、テストを最も簡単に行うことができるのはアクセスロガーです。
  • 1つ以上のイベントを生成できる。

ログフォワーダーをテストする方法：

  1. ログサーバーでシステムロガーを構成します。
  2. Access Gatewayでログフォワーダー（アクセスロガーが望ましい）を構成します。
  3. システムロガーが開始され、イベントの受信準備が完了していることを確認します。
  4. サインアウトし、Access Gateway Admin Consoleに再度サインインします。
  5. ログサーバーを調べます。次のように、複数のイベントが生成されます：

フィードの例

タイプ 説明
AUDIT（監査） 監査ログイベントには、ユーザー認証を表すログエントリが含まれます。

監査イベントの詳細と例については、「Access Gateway監査ログ」を参照してください。

サンプルイベント：

2020-06-24T10:05:56.000-05:00 example.myaccessgateway.com ACCESS_GATEWAY WEB_CONSOLE - - INFO SYSTEM_STARTUP [] Startup complete, system ready.2020-06-24T04:00:01.000-05:00 example.myaccessgateway.com ACCESS_GATEWAY WEB_CONSOLE IDP LOCAL INFO SYSTEM_IDP_STATUS [NAME="MyIDP" DOMAIN="someorg.oktapreview.com" TYPE="IDP_OKTA" RESULT="PASS" REASON="VALID"] Success confirming IDP status with: someorg.oktapreview.com.
ACCESS（アクセス） アクセスログイベントには、ユーザー承認およびアプリケーションのアクセスを表すログエントリが含まれます。例えば、特定のユーザーが、所定のIPアドレスから特定のアプリケーションにアクセスしたなどです。アクセスイベントの詳細と例については、「Access Gatewayアクセスログ」を参照してください。

サンプルイベント：

2020-06-24T09:41:08.000-05:00 example.myaccessgateway.com auth header.myexample.com 10.0.0.110 - - "GET /assets/images/image.png HTTP/2.0" 200 1229 "https://gw-admin.example.com/" "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_6) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/80.0.3987.149 Safari/537.36" "-" 0.029 0.028 .
MONITOR（モニタリング） モニタリングログイベントには、アプリケーション構成（追加、削除、変更）、証明書構成、および認証モジュール構成を表すログエントリが含まれます。「Access Gatewayモニタリングログ」を参照してください

サンプルイベント：

2020-06-25T07:00:02.119-05:00 example.myaccessgateway.com OAG_MONITOR MONITOR DISK_USAGE INFO DISK_USAGE [FILESYSTEM="/dev/mapper/centos-root" MOUNT="/" USAGE="12%"] Mount / is 12% full
