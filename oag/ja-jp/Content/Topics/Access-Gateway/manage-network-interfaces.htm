ネットワークインターフェイスを管理する

Access Gatewayは、複数のネットワークインターフェイスを使用できます。定義されているネットワークインターフェイスの数と使用方法は、主に基本設定に基づいています。

ネットワークインターフェイスの管理には、以下のタスクが含まれます。

  • ネットワークインターフェイスの作成：仮想環境による2番目以降のネットワークインターフェイスを追加する方法については、「ネットワークインターフェイス」を参照してください。
  • インターフェイスにIPアドレスを割り当てる：インターフェイスにIPアドレスを割り当てます。
  • インターフェイスルーティングを管理する：特定のインターフェイスに適切なルーティング情報を定義します。
  • DNSを管理する：Access GatewayノードまたはクラスターのDNSサーバーを追加します。
  • NICボンディング：パフォーマンスを向上させるために、インターフェイスを結合させます。

すべてのネットワーク操作は、Access Gateway 管理コンソールを使用して実行されます。

コマンドライン管理コンソールにアクセスする方法：

  1. SSHを使用してAccess Gateway 管理コンソールに接続します。
  2. 管理コンソールの資格情報を使用してサインインします。

    • ユーザー名： oag-mgmt

    • パスワード： デフォルト資格情報

  3. [1 - Manage network interface（1 - ネットワークインターフェイスの管理）]を選択し、[Network（ネットワーク）]メニューに進みます。
  4. 管理したいインターフェイスを番号で選択します。

インターフェイスIPアドレスを割り当てる

特定のネットワークインターフェイスに関連付けられているIPアドレスを管理する方法：

  • Access Gateway 管理コンソールに接続し、前述のようにネットワークインターフェイスに移動します。
  1. 管理するインターフェイスの行番号を入力するか、xを入力して終了します。Available networking interfaces. Select interface to manage: 1: eth0 2: eth1 3: eth2 . . . x: Exit 選択したインターフェイスに関する情報が表示され、続いて管理インターフェイスのサブメニューが表示されます。たとえば、eth2を表示するために3を入力すると、次の例のように表示されます。 Managing interface eth2 (MAC 09:. . . :00) Status: UP DHCP IP Address: 192.168.1.230/24 Routes: default via 192.168.1.1 proto dhcp metric 103 dev eth2 192.168.1.0/24 proto kernel scope link src 192.168.1.230 metric 103 dev eth2 a: Assign static IP address d: Set up DHCP r: Manage Routes x: Exit
  2. 静的IPアドレスを割り当てるにはaを入力します。

    インターフェイスにすでに静的IPアドレスがあるか、DHCPで割り当てられたアドレスがある場合、このオプションは使用できません。

    1. IPアドレスを入力し、Enterを押します。

      IP Address: 192.168.1.230

    2. アドレスのネットマスクを4桁の16進数で入力し、Enterを押します。

      Netmask: 255.255.255.0

    3. 静的IPアドレスのゲートウェイアドレスを入力し、Enterを押します。

      Gateway: 192.168.1.1

    4. 次の確認メッセージが表示されます。 The following network configuration change will be made to your running configuration: Configuring static IP address 192.168.1.230/255.255.255.0 on device eth3 with gateway 192.168.1.1 a: abort change c: continue with change
    5. 変更を継続する場合はcを、中止する場合はaを入力します。
  3. インターフェイスのDHCPを設定するにはdを入力します。Setup DHCP for device ethX Are you sure you want to setup DHCP (y/n):
  4. yを入力します。 The following network configuration change will be made to your running configuration: Configuring DHCP for device ethX a: abort change c: continue with change

インターフェイスルーティングを管理する

指定されたネットワークインターフェイスのルートを管理する方法：

  • Access Gateway 管理コンソールに接続し、前述のようにネットワークインターフェイスに移動します。

ルートの管理

rを入力して、管理ルートサブメニューを開き、ルート追加プロセスを開始します。

Managing routes for interface eth3 Routes: default via 192.168.1.1 proto static metric 103 dev eth1 192.168.1.0/24 proto kernel scope link src 192.168.1.230 metric 103 dev eth3 a: Add route d: Delete route x: Exit

  1. aを入力してルートを追加します。

    1. このルートをデフォルトルートにするには、yを入力し、それ以外の場合はnを入力します。 Enterを押します。

      Default? (y/n)

    2. ルートのゲートウェイを入力し、Enterを押します。

      Gateway: 192.168.1.1

    3. ルートの宛先を入力し、Enterを押します。

      Destination (CIDR): 192.168.195.78

    4. 次の確認メッセージが表示されます。The following network configuration change will be made to your running configuration: adding route to 192.168.1.1 on dev eth3 a: abort chance c: continue with change
  2. ルートを削除するにはdを入力します。削除を選択できるルートのリストが表示されます。
    1. 削除するルートの番号を入力します。
    2. yを押して確定します。ルートがインターフェイスから割り当て解除され、[Add/Delete IP addresses（IPアドレスの追加/削除）]サブメニューに戻ります。
  3. 変更をコミットします。
    1. メインのネットワーキングメニューに戻ります。
    2. 変更をコミットするにはcを入力します。

DNSを管理する

[Manage DNS Network（DNSネットワークの管理）]サブメニューでは、DNS設定を管理することができます。

  1. ネットワークメニューから8を選択します。現在のDNSネームサーバー設定が表示されます。
    Managing DNS Setting Current Name Servers: Primary: 192.168.8.8 Secondary: Tertiary: Search Domains: okta.com someotherdomain.com 1: Manage DNS servers 2: Manage search domains x: Exit

  2. xを入力すると終了して親メニューに戻ります。

NICボンディング

NICボンディングは、2つのイーサネットポートをボンディングされた仮想ポートに組み合わせるプロセスです。通常、NICボンディングは、単一のポート上に単一のネットワーク接続を飽和させるのに十分なトラフィックがある場合に使用されます。

  1. 5を入力してNICボンディングをセットアップします。

  2. IPアドレスを入力し、Enterを選択します。

  3. ネットマスクの値を入力し、Enterを選択します。

  4. デフォルトゲートウェイの値を入力し、Enterを選択します。

  5. プライマリDNSサーバーの値を入力し、Enterを選択します。

  6. セカンダリDNSサーバーの値を入力し、Enterを選択します。

  7. 検索ドメインアドレスを入力し、Enterを選択します。

  8. cを押して変更をコミットします。

  9. ネットワーク設定をレビューしてyを入力し、変更を確定するためEnterを選択します。または、nを入力してEnterを選択すると、変更が破棄され、ネットワーキングメニューに戻ります。

  10. 変更が完了したら、いずれかのキーを入力してネットワーキングメニューに戻ります。
