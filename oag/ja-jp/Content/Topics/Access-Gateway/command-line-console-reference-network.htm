ネットワーク

[Network（ネットワーク）]メニューにはネットワークのステータスを確認したりネットワーク設定を変更するオプションが含まれています。

メインメニューから1を選択し、Access Gateway[Network Setup（ネットワークセットアップ）]サブメニューに入ります。

ネットワークコマンド

  • [Manage Interface（インターフェイスの管理）]：Access Gatewayアプライアンスのルーティングなど、ネットワークインターフェイスを管理するサブメニューに進みます。
  • [Test network configuration（ネットワーク構成のテスト）]：複数のWebサイトへの接続を試み、NGINX設定およびステータスを確認するサブメニューに進みます。
  • /etc/hostsファイルの編集：Access Gatewayアプライアンスの/etc/hostsファイルを編集できます。
  • [Set up NIC bonding（NICボンディングのセットアップ）]：スタティックネットワーク構成と同様のNICボンディング設定を行うサブメニューに進みます。
  • [Proxy Settings（プロキシ設定）]：Access Gatewayアプライアンスのプロキシ接続をセットアップしたり、アプライアンスからプロキシの構成を削除するサブメニューに進みます。
  • Ping：接続先ホストがAccess Gatewayアプライアンスから到達可能か判断します。
  • 接続テスト：Access Gatewayアプライアンスや他のシステムとの間の接続を検証します。このツールを使用してバックエンドアプリケーションまたはサーバーがAccess Gatewayに到達できるか検証します。
  • DNS設定の管理 - プライマリおよびセカンダリDNSサーバーなどのDNSを設定します。
  • [Manage Trusted Domains（信頼できるドメインの管理）] - 信頼できるドメインの有効化、無効化、および表示を行います。
  • [Show a running configuration（実行中の設定を表示）]：現在のAccess Gatewayアプライアンスの構成を表示します。
  • [Network（ネットワーク）]：構成変更プランを表示します。
  • [Commit changes to system（変更をシステムで確定）]：保存されていない変更をAccess Gatewayアプライアンスで確定します。
  • [Restart Networking（ネットワーキングの再起動）]：Access Gateway アプライアンスでネットワークサービスを再起動します。
  • [Exit（終了）]：管理コンソールを終了します。

[Manage Interfaces（インターフェイスを管理する）]

[Manage Interfaces（インターフェイスを管理する）]サブメニューでは、以下のことが行えます。

  • インターフェイスのDHCP管理のセットアップ
  • インターフェイスからIPアドレスを割り当てまたは削除（Access Gatewayバージョン2020.07.1より利用可能）
  • インターフェイスからルートを割り当てまたは削除。

[Manage Interfaces（インターフェイスの管理）]メニューを使用して行われたネットワーキング関連のすべての変更が収集され、[Commit changes to system（変更をシステムで確定）]メニュー要素を使用して完了した時に保持される必要があります。変更は直ちに行われますが、[Commit changes to system（変更をシステムで確定）]を使用して確定された時に保存されます。

  1. 管理したいインターフェイスの行番号を入力するか、xを入力して終了します： Available networking interfaces. Select interface to manage: 1: eth0 2: eth1 3: eth2 . . . x: Exit 選択したインターフェイスのインターフェイス情報が、インターフェイス管理サブメニューとともに表示されます。例として、3を入力すると： Managing interface eth2 (MAC 09:. . . :00) Status: UP DHCP IP Address: 192.168.1.230/24 Routes: default via 192.168.1.1 proto dhcp metric 103 dev eth2 192.168.1.0/24 proto kernel scope link src 192.168.1.230 metric 103 dev eth2 a: Assign static IP address d: Set up DHCP r: Manage Routes x: Exit

    2. インターフェイスのIPアドレスを管理するにはaを入力します。
    インターフェイスのDHCPを設定するにはdを入力します。
    インターフェイスのルーティング情報を管理するにはrを入力します。

ルートの管理
rを入力してルート追加プロセスを開始します
例：

ルート管理サブメニューでは次のような画面が表示されます：
Managing routes for interface eth3 Routes: default via 192.168.1.1 proto static metric 103 dev eth1 192.168.1.0/24 proto kernel scope link src 192.168.1.230 metric 103 dev eth3 a: Add route d: Delete route x: Exit

  1. xを入力して終了し、インターフェイス管理メニューに戻ります。
  2. rを入力した新規ルートを追加します
    Adding route for interface ethX（インターフェイスイーサネットXのルートを追加）が表示されます。
    1. このルートがデフォルトルートかどうかを指定し、Enterを選択します。
      Default? (y/n) 新規ルートをデフォルトルートにするにはyを入力し、それ以外の場合はnを入力します。
    2. ルートのゲートウェイを入力してEnterを選択します。
      Gateway: 192.168.1.1
    3. ルートの宛先を入力し、Enterを選択します。
      Destination (CIDR): 192.168.195.78
    4. 確認が表示されます。
      例：
      The following network configuration change will be made to your running configuration: adding route to 192.168.1.1 on dev eth3 a: abort chance c: continue with change
  3. ルートを削除するにはdを入力します。
    ルートリストピッカーが次のような画面を表示します： 
    Pick an route to delete
1: 192.168.128.0/16
2: 192.168.4.0/16
x: Exit
    1. 削除するルートの行番号を入力します。
    2. 確定するにはyを入力し、キャンセルする場合はそれ以外のキーを入力します。
      ルートはインターフェイスから割り当てが解除され、[Add/Delete IP addresses（IPアドレスの追加/削除）]サブメニューに戻ります。
  4. 変更を確認
    1. メインのネットワーキングメニューに戻ります。
    2. 変更をコミットするにはcを入力します。

ネットワーク構成のテスト

  1. 3を押してネットワーク設定のテストを開始します。

  2. 次に、システムがwww.okta.comへの接続を試み、ステータスを表示します。改行キーを押して続行します。

  3. 最後に、システムがNGINX設定を確認してステータスを表示します。Enterを押して[Networking（ネットワーキング）]メニューに戻ります。

/etc/hostsファイルの編集

/etc/hostsは、静的に割り当てられたホスト名を設定するために使用されます。管理者が特定のホスト名のDNSを上書きする必要がある場合、またはDNSに存在しないホストのアドレスを提供する必要がある場合、そのホストを/etc/hostsファイルに追加できます。通常、これはOkta Access Gatewayが正しいホストに接続できるようにするために実施されます。

  1. 4を押して/etc/hostsファイルを編集します。これはホストの現在のエントリを行番号ごとに表示します。

  2. aを押して新しいエントリを追加します。

  3. 以下の標準フォーマットに従ってエントリを追加します：

  4. エントリを削除するには、dを押して削除したいエントリに対応する行番号を入力します。

  5. cを押してホストファイルへの変更を確定します。

  6. xを押してネットワーキングのメインメニューに戻ります。

NICボンディングのセットアップ

NICボンディングは、2つのイーサネットポートをボンディングされた仮想ポートに組み合わせるプロセスです。通常、NICボンディングは、単一のネットワーク接続を飽和させるために単一のポート上に十分なトラフィックがある場合に使用されます。

  1. 5を入力してNICボンディングをセットアップします。

  2. IPアドレスを入力し、Enterを選択します。

  3. ネットマスクの値を入力し、Enterを選択します。

  4. デフォルトゲートウェイの値を入力し、Enterを選択します。

  5. プライマリDNSサーバーの値を入力し、Enterを選択します。

  6. セカンダリDNSサーバーの値を入力し、Enterを選択します。

  7. 検索ドメインアドレスを入力し、Enterを選択します。

  8. cを押して変更をコミットします。

  9. ネットワーク設定をレビューしてyを入力し、変更を確定するためEnterを選択します。nを入力してEnterを選択すると、変更を破棄し、ネットワーキングメニューに戻ります。

  10. 変更の完了後、いずれかのキーを入力してネットワーキングメニューに戻ります。

[Proxy Settings（プロキシ設定）]

プロキシの設定

  1. 7を押してインターネットへのプロキシ接続を設定または無効にします。
  2. 1を押してプロキシを設定します。
  3. プロキシのホスト名を入力し、改行キーを押します。
  4. プロキシポートを入力し、改行キーを押します。
  5. 必要に応じてプロキシのユーザー名を入力し、改行キーを押します。
  6. 必要に応じてプロキシのパスワードを入力し、改行キーを押します。
  7. プロキシをバイパスする必要があるホスト名を入力します（カンマ区切り）。
    入力したプロキシ設定が表示されます。
  8. yを押してプロキシ設定を確定するか、Nを押して処理を中断します。
  9. 改行キーを押してプロキシメニューに戻り、改行キーを再度押してネットワーキングのメインメニューに戻ります。

プロキシの設定解除

  1. 7を押してプロキシメニューに進みます。

  2. 2を押してプロキシ設定を削除します。

  3. yを押して確定します。

  4. 改行キーを押してプロキシメニューに戻り、もう一度改行キーを押してネットワーキングのメインメニューに戻ります。

Ping

Pingオプションを使用して、接続先ホストがAccess Gatewayアプライアンスから到達可能か確認できます。

  1. 8を押してpingメニューに進みます。

  2. pingしたい接続先ホストのホスト名かIPアドレスを入力し、改行キーを押します。

  3. システムにpingの結果が表示されます。

  4. 改行キーを押してネットワーキングのメインメニューに戻ります。

[Connectivity Test（接続テスト）]

[Connectivity Test（接続テスト）]を使用して、Access Gatewayアプライアンスと他のシステムとの間の接続を検証できます。また、このツールを使用してバックエンドアプリケーションまたはサーバーがAccess Gatewayから到達できるかも検証できます。

  1. ネットワークメニューから9を選択します。

  2. テストしたいマシンのホスト名かIPアドレスを入力し、改行キーを押します。

  3. テストを必要とするポート番号を入力し、改行キーを押します。

  4. システムに接続結果が表示されます。

  5. 改行キーを押して[Networking（ネットワーキング）]メニューに戻ります。

DNS設定の管理

[Manage DNS Setting（DNS設定の管理）]サブメニューを使用して、ドメイン名サーバーの追加、削除および変更、ならびにドメインの検索が実施できます。

  1. ネットワークメニューから8を選択します。次のような現在のDNSネームサーバー設定が表示されます：
    Managing DNS Setting Current Name Servers: Primary: 192.168.8.8 Secondary: Tertiary: Search Domains: okta.com someotherdomain.com 1: Manage DNS servers 2: Manage search domains x: Exit

  2. xを入力すると終了して親メニューに戻ります。

DNSサーバーの管理

  1. 1を入力して[Manage DNSservers（DNSサーバーの管理）]サブメニューに進みます。
    現在のDNSサーバーが表示されます。例：
    Managing DNS Servers Primary: 192.168.8.8 Secondary: Tertiary: . . .
  2. 1を入力して第1DNSサーバーを設定します。
  3. 2を入力して第2DNSサーバーを設定します。
  4. 3を入力して第3DNSサーバーを設定します。

    DNSサーバーが何も設定されていない場合、第2または第3DNSサーバーの設定をするとそれが第1DNSサーバーに設定されます。

    第1DNSサーバーのIPアドレスは空白にできませんが、第2、第3DNSサーバーは省略できます。

  5. 選択したDNSサーバーのIPアドレスを入力してEnterをクリックします。
    アップデートされたDNSサーバーの一覧が表示されます。
  6. xを入力すると終了して親メニュに戻ります。

[Manage search domains（検索ドメインの管理）]

Access Gatewayは最大6つの検索ドメインをサポートします。

  1. 2を入力して[Manage search domains（検索ドメインの管理）]サブメニューに進みます。現在の検索ドメインが以下のように表示されます：
    Current search domains: okta.com someotherdomain.com . . .
  2. aを入力して新しい検索ドメインを追加します。
    1. 新しい検索ドメインを入力してEnterを押します。
    2. 新しい検索ドメインが追加され、現在のリストが以下のように表示されます：
      Current search domains: okta.com someotherdomain.com . . .
  3. rを押して既存の検索ドメインを削除します。既存の検索ドメインの一覧が以下のように表示されます：
    Remove which search domain: 1: okta.com 2: someotherdomain.com . . .
    1. 削除される検索ドメインに関連付けられている番号を入力します。
    2. 検索ドメインが削除され、現在のリストが表示されます。例えば、someotherdomain.comの削除後の出力は次のようになります：
      Current search domains: okta.com . . .

信頼できるドメインを管理する

信頼できるドメインの管理メニューは、信頼できるドメインの有効化、無効化、および表示を行うために使用されます。
Access Gatewayバージョン2020.08.3より利用可能

  1. ネットワークメニューから9を選択します。信頼できるドメインの管理サブメニューが以下のように表示されます：
    Manage Trusted Domains (status: Enabled/Disabled) 1 - Enable/Disable trusted domain 2 - View trust domains x - Exit

  2. xを入力すると終了して親メニューに戻ります。

信頼できるドメインを有効または表示する

  1. 1を入力して信頼できるドメイン有効化／無効化設定を切り替えます。
  2. 信頼できるドメインの管理サブメニューが再表示され、信頼できるドメインの現在のステータスが示されます。

信頼できるドメインを表示

  1. 2を押して信頼できるドメインを表示します。以下のように、現在信頼できるドメインの一式が表示されます：
    UP/DOWN/HOME/END - scroll list x - exit trusted-one.domain.com trusted-two.domain.com . . . trusted-n.domain.com . . .

    表示オプションは存在しますが、無効化されている時に信頼できるドメインを表示を試みると警告が表示されます。

  2. xを入力して表示を終了します。
    信頼できるドメインの管理サブメニューが再表示され、信頼できるドメインの現在のステータスが示されます。

実行中の構成を表示

実行中の設定を表示するオプションを使用して、インターフェイス、DNS、ルーティング、ホストファイルの内容など、現在の設定の詳細を表示することができます。

  1. ネットワークメニューからsを選択します。

  2. 改行キーを押して実行中の設定詳細を表示します。

  3. 改行キーを押して[Networking（ネットワーキング）]メニューに戻ります。

DNS検索

所定のホスト名に対してDNS検索を実行

  1. ネットワークメニューからdを選択します。

  2. ホスト名またはIPアドレスを入力します。

  3. ホスト名またはIPアドレスを入力します。

  4. DNSサーバーのホスト名またはIPアドレスを入力します。

  5. 改行キーを押して検索を実行します。

  6. 改行キーを押して[Networking（ネットワーキング）]メニューに戻ります。

変更をシステムで確定

  1. ネットワークメニューからcを選択します。
    注：現在の変更一式が表示されます。

  2. yを選択して確定するか、nを選択して変更を無視します。

  3. Enterを押して[Networking（ネットワーキング）]メニューに戻ります。

ネットワーキングの再起動

  1. ネットワークメニューからrを選択します。

    2. ネットワーキングが再起動します。

  2. Enterを押して[Networking（ネットワーキング）]メニューに戻ります。
