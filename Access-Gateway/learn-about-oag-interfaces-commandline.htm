The provides options and settings specific to the system as a whole. Using the Command Line console, you can manage networking, specify NGINX settings, configure high availability, and more.

First sign in

  1. Start VM and open a terminal window. Sign in using credentials:

    1. Username: oag-mgmt
    2. Password: <default-password>

The first time you access , you must change the default password.

After you sign in, the system provides you with various menu options, which are described in the following sections.

Network menu

The Network menu contains options for checking the status of the network and modifying the network settings.

Service menu

Kerberos menu

The Kerberos menu allows you to list or destroy Kerberos tickets.

When listing a ticket, the software provides the Kerberos principal and Kerberos ticket held in the credential cache or keytab file.

You can destroy a user’s active Kerberos authorization by overwriting and deleting the credential cache that contains them.

Monitoring menu

The Monitor menu allows you to view the logs, enable and disable debug mode in logging and display memory usage statistics. See About [%=okta-feature-names.Access Gateway%] Logging.

System Menu

The System menu allows you to change the hostname of the instance, configure high availability, add or remove packages, and reboot/shut down the instance.

NGINX

The NGINX menu allows you to start, stop, and restart the NGINX service, check the status of the service, regenerate the associated SSL certificate and more.

Change Console Password

The Change Access Gateway Console Password menu allows you to change the password for the oag-mgmt user.

Change Web Console Password

The Change Access Gateway Web Console Password menu allows you to change the password for the administrator on the .

Support Connection

The Support Connection menu option allows you to activate and deactivate support connections.
