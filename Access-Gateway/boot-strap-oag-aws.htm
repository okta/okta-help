Bootstrap with Amazon Web Services

Okta Access Gateway Bootstrap with Amazon Web Services details how to deploy and configure for development use quickly and easily using an AWS  environment.
Overall the goal is to help you perform the specific tasks required to get your application up and running as quickly as possible. See Manage [%=okta-feature-names.Access Gateway%] deployment for all currently supported virtual environments and the underlying tasks required to deploy to each.

Audience

bootstrap with AWS is for anyone who needs to perform basic deployment and integration tasks with . Administrators following this guide will have an development environment, running on AWS, when complete.  

No experience with is required, but you'll need a basic knowledge of networks and firewalls.
Specifically this guide covers these tasks:

  1. Deploy the gateway - How to deploy the virtual appliance into and Amazon Web Services virtual environment.
  2. Configure the gateway - Initialize the instance, including basic networking settings.
  3. Integrate the with Okta as IdP .
  4. Create SAML access application - configure an application to simplify login and access.
