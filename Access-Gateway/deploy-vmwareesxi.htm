Deploy the OVA file to VMware vSphere and ESXi

This topic describes how to deploy the OVA installation file to VMware vSpere and ESXi environments.

Deploy the OVA to VMware vSphere and ESXI

  1. Download and install the VMware vSphere Client from the ESXi/vSphere server.
  2. Open the VMware vSphere Client.
  3. Enter the server name or IP address and credentials in their respective fields and click Login.
  4. In the vSphere Client window, select FileDeploy OVF Template .
  5. In the Deploy OVF Template window, click Browse.
  6. Select the Okta-Access Gateway OVA file, and then click Open.
  7. Click Next.
  8. Review the template details provided, and then click Next.
  9. Click Accept to accept the licence agreement, and then click Next.
  10. Enter a name for the template, and then click Next.
  11. Select a storage location, and then click Next.
  12. Select the appropriate disk format option based on your requirements, and then click Next.
  13. Click Finish. The VMware vSphere Client begins the deployment process.
  14. Click Close when the deployment is complete.
  15. In the vSphere Client window, click Inventory.
  16. Select the Virtual Machines tab to display the virtual machines that are currently deployed to the server.
  17. Select the virtual machine and click Power On (symbolized by a play icon) in the toolbar.
  18. Right-click the virtual machine and select Open Console to sign in to it.

Troubleshooting compatibility errors

On some versions of VMware EXSi, the appliance doesn't start due to a compatibility error. Perform this procedure to correct the error.

  1. Open VMware vSphere Client.
  2. Enter the server name or IP address and credentials in the respective fields, and then click Login.
  3. Go to the Okta virtual machine.
  4. Right-click the virtual machine name, and then select Compatibility Upgrade VM compatibility Yes .
  5. In the Compatible with dialog box, select the default option. Typically, this is ESXi 6.5 and higher.
