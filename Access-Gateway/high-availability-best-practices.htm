Best practices for high availability

Before you configure high availability for , follow these best practices:

  • Rest keys for all cluster members, including admin and worker instances. Go to System High Availability Reset Keys .
  • Confirm that the host name of each new instance is set. The default host name is oag.okta.com. To change the host name or verify it, open the , and go to System Hostname .
  • When you initially configure an instance, reset it to an initial state. Go to System Reset .

See Command Line Management console reference for more information on command line operations.

While configuring an high availability cluster, you can add your Okta tenant as an IDP at any time. Additionally, you can also add applications any time.
