Logging levels

supports the following logging levels in order of severity. In general, higher level log events also include lower level log events. For example, INFO log events also include WARN, and ERROR log events.

Level Description
DEBUG Fine-grained events that are most useful for debuging itself.
INFO Informational messages that highlight the progress of the application at coarse-grained level.
Default.

WARN

Potentially harmful events. However, these events usually don't result in a loss of function. For example, expiring certificates.

ERROR Error events that allow to continue running but typically require attention.

FATAL

Severe errors that might result in shutting down.
