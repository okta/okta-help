Add an application

The purpose of this task is to step through the process of adding a second administrator app, assignable and accessible from your Okta org dashboard.
Using the administrator application Okta org administrators can assign the app to any user, allowing those users to access the and administer .

Create group and assign users

You must either assign users or define groups representing the users who can access the .

To define an group within your Okta tenant:

  1. Sign in to your Okta tenant as an administrator.
  2. In the , navigate to Directory Groups .
  3. Click Add Group.
  4. Enter a name for the group such as Access Gateway Admins.
  5. Add an option description, such as Members of this group can administer .
  6. Click Add Group.

To add users to the newly added groups:

  1. Click the name of the newly added group.
  2. Click Manage People.
  3. Using the Search by people field, find those users who should be able to administer .
  4. From the Not Members list click Add All.
  5. Click Save when complete.

Create the application in

  1. Select the Access Gateway AdminUI option from the left column menu, and click Create.

    The New Protected Application wizard starts and displays the Setting tab.

  2. In the Essentials pane enter:
    Field Value
    Label The name of the application,
    For example: Access Gateway Admin UI
    Public Domain The external facing URL of the gateway. Typically similar to gw-admin.[domain.tld].
    For example: gw-admin.myaccessgateway.com
    Post Login URL

    Leave unchanged.
    Group Enter the name of group created in the prior step.

  3. Click Next. The Attributes page appears.

    The attribute set specified by the add application wizard is required by . Do not modify this list.

  4. Click Next. The Policy page will appears.
  5. Click Done.

Test

  1. Sign in to your tenant as a user in the Administer group.
  2. From the applications dashboard, find the app that you added and select it. The opens as the same user.
