Supported exporters

supports monitoring metrics for various exporters and models.

supports the following models:

  • Pull: Servers access the endpoint /v1/metrics that's running on a known port (8889) on using HTTP GET operations. Servers pull metrics from on demand.
  • Push: An administrator defines a fully qualified domain and port combination used by a server, security, protocol, and other information. periodically pushes metrics to the endpoint in one of the supported formats.

Supported exporters include:

Exporter

Model

Description
Opencensus Push exports metrics periodically in Opencensus format. A push model is used to send metrics to a fully qualified top-level domain/port.
Open Telemetry(OTLP) gRPC Push exports metrics periodically in the Open Telemetry format. A push model uses gRPC to send metrics to a fully qualified top-level domain/port.
Open Telemetry(OTLP) HTTP Push exports metrics periodically in the Open Telemetry format. A push model uses HTTP POST requests to send metrics to a fully qualified top-level domain/port.
Prometheus Pull exports metrics on demand in the Open Metrics format. Metrics are sent using ReST over port 8889 using a known endpoint.
Prometheus Pull exports metrics on demand in the Open Metrics format. Metrics are obtained through a push model with a fully qualified top-level domain/port, or through a pull model over a port 8889 that uses the /metrics endpoint.
