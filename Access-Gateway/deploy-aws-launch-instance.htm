Launch Okta and obtain an Elastic IP address

To complete the import process, you must do the following steps:

  1. Launch an instance
  2. Obtain an Elastic IP Address

Launch an instance

  1. Go to Amazon Web Services (AWS) and sign in with an appropriately privileged account.

  2. Click Services.

  3. Click Compute. Click EC2.

  4. From the navigation pane, go to IMAGES, and click AMIs.

  5. Select the Okta AMI and click Actions Launch .

    1. In the Choose an Instance Type select an appropriate type, then click Next: Configure Instance Details. For development purposes, t2.micro is sufficient. See Capacity planning and sizing for instance sizing information.

    2. Skip to security configuration by clicking 6. Configure Security.

    3. Create a security group that contains three rules for SSH, HTTP, and HTTPS with the following values:

      Type Protocol Port
      SSH TCP 22
      HTTP TCP 80
      HTTPS TCP 443

    4. When complete, the new rules should resemble:

    5. Click Review and Launch.

    6. Click Launch.

    7. Select Proceed without a Keypair.

    8. Select the acknowledgment and click Launch. It can take several minutes for your instance to launch.

    9. Click View Instances. Wait for the instance to start.

Obtain an Elastic IP address

  1. From the navigation pane, go to NETWORK & Security.

  2. Click Elastic IPs.

  3. Click Allocate new address.

  4. .Select Amazon pool and click Allocate. A new address is allocated.

  5. Click Close.

  6. Select the row associated with the new Elastic IP address and select Actions Associate .

  7. Select the instance that you started from the Instance dropdown list and click Associate.

  8. Verify that the IP address was associated with the instance by clicking Services EC2 .

  9. Go to Instances.

  10. Click Instances. The newly acquired elastic IP address is now associated with the instance.
