Selective upgrade

Selective upgrade, also referred to as upgrading to a specific version, is the process of taking an node and upgrading that node from its current version to any of the follow on versions up to, and including the most recently released version.

Selective upgrade supports the following scenarios:

  • Current version (assumed to be most recent) - Upgrade not possible, no newer versions exist.
  • Current version - Upgrade to version possible.
  • Current version - Upgrade to version , or are possible.
  • Current version - Upgrade to version , , or are possible.
  • And so on.

Selective upgrade is performed using the standard upgrade process and must be performed on each cluster node individually. Typically such updates are done using a rolling upgrade process where nodes are taken out of rotation, updated and returned back to the cluster.
