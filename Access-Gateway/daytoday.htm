Manage day to day

After initial deployment and integrating applications, administrators must perform normal day to day functions, such as monitoring, configuring logging, and similar tasks.
The following list includes common day to day administrative tasks:

Task Description
Administer the support VPN The Support VPN is a mechanism used by select Okta support individuals to access an instance at the operating system level.

See also About [%=okta-feature-names.Access Gateway%] support VPN.

Backup and restore

Create backups of your configuration. Use a backup to restore your configuration.
Configure [%=okta-feature-names.Access Gateway%] proxy server Configure to use a proxy. instances must be rebooted after proxy configuration.
Configure log forwarders Configure log forwarders to work with . See Administer logging.
Configure High Availability supports high availability by creating a cluster of nodes or instances.
Download logs Download and expand system logs.

Less common system tasks

Task Description
Administer Auth Modules How to add authorization modules (auth modules) to establish a secondary source of truth for authentication. Helpful in the case where can't reach your tenant.

The use of auth modules is very rare. Consider consulting with your Okta support representative before you implement auth modules.
Administer local Identity Providers Add a local identity provider. If no org IdP can be reached, the local IdP can be used to provide application access.
Administer SNMP monitoring How to enable Simple Network Management Protocol (SNMP) support and what functionality is available.
