Initialize the

Once an instance is running, you initialize it by assigning it a cookie domain. This domain represents the general domain that the instance is protecting.

  1. Open https://admin/ in your browser and sign in using the default credentials. See Default [%=okta-feature-names.Access Gateway%] credentials.
  2. prompts you to change the default password. See Initial sign in to [%=okta-feature-names.Access Gateway UI%].
  3. Configure the DNS entries for the instance:

    Use lower case letters for the domain names.

  4. Click Done.
  5. Return to the browser and sign in again using the new admin domain.
  6. The Admin Topology page appears.
