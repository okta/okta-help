Okta Browser Plugin

Okta Browser Pluginを使うと、通常は認証情報の手動入力が必要なさまざまなアプリケーションに自動的にOktaサインインを開始できます。

Okta End-User Dashboardからアプリが開始されると、アプリのURLを指す新しいブラウザータブが開きます。プラグインは暗号化された接続を使用してOktaから認証情報やその他の必要なデータを取得し、その情報をページに適用します。認証が完了したら、プラグインには認証情報は保存されません。

セキュリティを強化するため、プラグインは信頼できる検証済みのサイトでのみ機能します。Okta Browser Pluginをインストールしていないが、Okta Browser Pluginを必要とするアプリケーションがエンドユーザーダッシュボードにある場合、プラグインをインストールするようにダッシュボードにプロンプトが表示されます。

プラグインは頻繁に更新されます。必要に応じて最新バージョンをインストールするよう求められます。「Oktaのブラウザー用プラグインのバージョン履歴」を参照してください。

Okta Browser Pluginでできること

アプリに自動的にサインインする

Oktaで特定のアプリのサインインページに直接アクセスすると、プラグインによって自動的に認証情報が入力され、それ以上の操作を行わなくてもサインインできます。

Oktaサインインを自動的に開始する

Oktaにサインインせずに、Okta対応のアプリケーションに直接アクセスすると、ポップアップバナーにサインインボタンが表示されます。サインインしようとすると、Oktaウィンドウが起動し、Okta End-User Dashboardを開かなくてもアプリにサインインできます。

サインインページで認証情報を自動入力する

自動サインインが有効になっていない場合、一部のOkta対応アプリのサインインページに直接アクセスすると、ポップアップバナーに認証情報を自動入力するオプションが表示されます。

パスワードの更新ページでパスワードを自動挿入する

一部のOkta対応アプリのパスワード更新ページを表示している場合、ポップアップバナーで現在のパスワードが自動挿入されます。

パスワードを更新する

一部のOkta対応アプリでパスワードを変更すると、新しいパスワードでOktaを更新するオプションがプラグインで提供されます。

複数のOktaアカウントを切り替える

Oktaにサインインすると、後続のOktaアカウントに初めてアクセスするときに、そのアカウントを信頼または拒否することが求められます。 時間と共に、プラグインを介してOktaアカウントのリストが構築されていくことによって、さまざまなアプリに容易にアクセスできるようになります。

「Oktaのブラウザ用プラグインでアカウントを切り替える」をご覧ください。

ウェブブラウザによってサインイン認証情報が保存されないようにする

Chromeなどのウェブブラウザが、Oktaのサインイン認証情報や、Okta End-User Dashboardからアクセスするサードパーティアプリのサインイン認証情報を保存しないようにすることができます。

「ブラウザーがパスワードの保存を求めないようにする場合」を参照してください。

新しいサービスへのサインアップ時にパスワードを作成する

注：この機能はまだ早期アクセスであり、管理者がこの機能を有効にしている場合にのみ利用できます。

アカウントを作成する際に、プラグインを使用してアプリのパスワードを作成できます。Okta End-User Dashboardにアプリを追加したら、アプリ設定パネルを使用してアプリのサインインURLを変更できます。

「アプリ設定ページの表示」を参照してください。

注意

Firefoxブラウザでオプション［Never remember history（履歴を一切記憶させない）］を選択しないでください。Okta Browser Pluginが動作しなくなります。
