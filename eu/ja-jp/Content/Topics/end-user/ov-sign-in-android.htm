AndroidデバイスのOkta Verifyで認証する

デバイス上のOkta Verifyアプリを使用して、Oktaで保護された複数のアプリのアカウントに安全にサインインできるよう本人確認を行います。

開始する前に

  • Okta Verifyの最新バージョンがデバイスにセットアップされていることを確認します。詳細は、「AndroidデバイスにOkta Verifyをセットアップする」をご覧ください。
  • バッググラウンドでOkta Verifyの実行が制限されていないか確認します。[Settings (設定)] > [Battery (バッテリー)] > Adaptive Battery (アダプティブバッテリー) > [Restricted apps (制限されたアプリ)]を開き、Okta Verifyが制限されたアプリとしてリストされていないことを確認します。Okta Verifyがこのページにリストされている場合、チェックボックスをオフにしてバッググラウンドでアプリが実行できるようにします。

デバイスがサインイン方法として構成されている場合、Okta Mobileからアプリにアクセスすることはできません。

手順

  • Okta Verifyで認証する
  • Android Wearで認証する

Okta Verifyで認証する

  • 認証コード： Okta Verifyで生成されたコードを使用します。
  • プッシュ通知： プッシュ通知で[Yes (はい)]をタップします。
  • デバイス： デバイスを使って本人確認を行います（組織でサポートされている場合のみ）。
    利用可能なサインインオプションを確認するには、Okta Verifyを開き、アカウントをクリックします。[Account details (アカウントの詳細)]ページで、サポートされているサインイン方法にチェックマークが付いています。

デバイスがサインイン方法として構成されており、デバイスの生体認証を変更またはリセットする場合、[Use Okta Verify on this device (このデバイスでOkta Verifyを使用する)]オプションが認証のリストに表示されますが、動作していません。この問題を解決するには、生体認証を無効にしてからもう1度有効にします。ヘルプが必要な場合、IT 部門にお問い合わせください。

IT 部門の構成およびデバイスのセットアップに従い、次の異なる方法でOkta Verifyを認証できます。

認証コードを使用してサインインする

  1. コンピューターのウェブブラウザーで組織のOktaアカウントにサインインするか、Oktaで保護されたアプリにアクセスします。
    [Okta Verify Authentication (Okta Verify認証)] チャレンジ画面が表示されます。
  2. [Or enter code (またはコードを入力)]リンクをクリックします。

    IT部門で実装されている機能に基づいて、Oktaで保護されたアプリにアクセスする際に、Okta Verifyからコードを入力してサインインするよう求められる場合があります。

  3. モバイルデバイスでOkta Verifyを開きます。
  4. 画面に表示される認証コードを覚えておいてください。以下の例では、Oktaのコードは123345です。
    6桁コードは30秒ごとに更新されます。進行状況バーに新しいコードが生成されるまでの残り時間が表示されます。
    Okta Verifyに複数のアカウントがある場合は、アクセスする組織に関連付けられているアカウントを使用します。
  5. ウェブブラウザーにコードを入力します。[Verify (確認する)]をクリックして、続行します。認証コードを使用した後、同じコードを使って再度サインインできません。

    入力したコードが正しければ、アプリのアカウントにサインインできます。

プッシュ通知を使用してサインインする

  1. コンピューターのウェブブラウザーで組織のOktaアカウントにサインインするか、Oktaで保護されたアプリにアクセスします。
    [Okta Verify Authentication (Okta Verify認証)] チャレンジ画面が表示されます。
  2. デバイスにプッシュ通知を送信して本人確認を行います。[Send Push (プッシュ通知を送信する)]をクリックします。

    注意: このオプションは、IT 部門が有効にしていなければ利用できません。
    これはオプションです。[Send push automatically (プッシュ通知を自動送信する)]チェックボックスをオンにし、今後の通知がデバイス上に自動でポップアップされるようにします。
    IT部門で実装されている機能に基づいて、Oktaで保護されたアプリにアクセスする際に、プッシュ通知を取得して本人確認を行うよう求められる場合があります。[Select (選択)]をクリックします。

  3. Okta Verifyからのプッシュ通知があるかどうかモバイルデバイスを確認します。5分以内に認証リクエストを承認または拒否しない場合、通知は期限切れとなります。
    モバイルデバイスで新しい通知を受信するには、もう一度Oktaにサインインし、[Send Push (プッシュ通知を送信する)]ボタンをクリックします。
  4. Oktaへのサインインを試みたことを確認するオプションをタップします。
    ご利用のウェブブラウザーがアプリのアカウントのOktaアプリに移動します。

デバイスを使用してサインインする

Okta Verifyの最新バージョンを使用しており、IT部門がこの機能を有効にした場合、デバイスで認証を処理できます。

Okta Verifyの未認証の使用を防止するには、指紋検出テクノロジーがデバイスで有効であり、Okta Verifyでセットアップされている場合、この機能を使用できます。メッセージが表示されると、指紋をスキャンします。

Android Wearで認証する

ウォッチ画面のプッシュチャレンジを使ってOkta Verifyを表示したり、承認または拒否したりできます。

リクエストを承認したら、自動的にOktaアカウントにサインインされます。
