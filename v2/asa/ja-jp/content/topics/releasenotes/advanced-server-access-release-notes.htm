過去のリリース

リリース：1.68.2

デプロイ日：2023年3月02日

修正

  • OktaAdvanced Server Accessのクライアントバージョン1.13.1～1.65.0は、サードパーティライブラリのWebブラウザー が原因でコマンドインジェクションに対して脆弱です。詳細については、CVE-2023-0093のOktaセキュリティアドバイザリーを参照してください。

リリース：1.68.1

デプロイ日：2023年2月22日

機能強化

  • Linux、FreeBSD：1.67.4は、LinuxおよびFreeBSDユーザーがapt/yum 設定を更新せずに利用できる最終リリースです。更新すると、1.68.1を利用できるようになります。これは、前リリースと同じですが、新しいGPGキーで署名されています。LinuxとFreeBSDの今後のリリースはすべて、新しい場所に置かれます。

  • Windows + macOS：WindowsおよびmacOSユーザーは、1.67.4のインストールを求められます。これをインストールすると、新しい署名キーとリポジトリ場所を使用する1.68.1への更新が再度求められます。

リリース：1.67.4

デプロイ日：2023年2月22日

機能強化

  • sftでのエラー処理を改善しました。
  • state.jsonファイル内の<team-user-url>エントリの重複は許可されなくなりました。
  • 登録のエラー処理を改善しました。
  • ログイン中にキーリングモードを切り替えると、state.jsonファイルが暗号化された状態に自動的に移行します。
  • macOS向けのScaleFTクライアントアプリを構築するために、ScaleFTをOkta証明書に変更しました。
  • 新しいリリースパッケージングシステム、新しいGPGキー、および新しいDULLキーのサポートを導入しました。
  • macOSソフトウェアアップデートフレームワークを更新しました。

修正

  • macOSのsftオートコンプリートコマンドのバグを修正しました。

  • Ubuntu 12.04
  • Red Hat Enterprise Linux 6

前述のオペレーティングシステムを利用している場合、2022年6月までにサポートされるオペレーティングシステムにアップグレードすることをお勧めします。「サポートされるオペレーティングシステム」を参照してください。

機能強化

  • sft sshコマンドにSFT_ALLOW_INSECURE_USERNAMES=1環境変数を含めることで、標準外の文字が含まれるユーザー名を使ったssh接続を一時的に許可できます。「Advanced Server Accessのクライアントを使用する」をご覧ください。
  • クライアントがDiffie Hellman Group Exchange SHA256 Kexアルゴリズムをサポートするようになりました。
  • AD-Joinedにいくつかの変更が加えられました。「AD-Joined」をご覧ください。
    • 複数のADアカウントを持つユーザーは、検出されたADデバイスとの接続に使用するアカウントを選択できます。
    • ユーザーは、パスワード入力なしでADデバイスに接続できます。「パスワードなし証明書」をご覧ください。
    • ユーザーは、macOSとWindowsの両方のGUIからADデバイスに接続できます。
  • ユーザーは、client.timeout_secondsオプションを構成することで、サーバーからの応答待機の最大時間を定義できます。「クライアントを構成する」をご覧ください。
  • 未登録のクライアントが、より詳細なエラーメッセージを返すようになりました。

修正

  • SSH接続を使ったユーザー名の指定ではエラーが返されました。
  • macOSクライアントでは、Advanced Server Accessのドキュメントへのリンクが誤っていました。
  • Advanced Server AccessのAPIレートまたは同時並行処理の制限をすでに超過している場合、クライアントは特定の操作で接続を再試行しませんでした。
  • ライフサイクルフックのSFT_Hook_Events環境変数をスクリプトで利用できませんでした。
