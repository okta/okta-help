What’s the difference between the terms register, reset, restore, set up, and enroll?

When working with , those terms all mean essentially the same thing: establishing or re-establishing a connection between your account and the specific app installed on your device.

Can I transfer my existing enrollment from one device to another?

For security reasons, you might be able to transfer from one device to another only if you first remove the existing enrollment tied to your original device. Then, enroll your new device in as described in Set up your current [%=okta-feature-names.Okta Verify%] account on a new iOS device. If you can't access your , contact your company's help desk.

Codes and Keys

What is a QR code and why do I need to use it for ?

A QR code is a universal barcode that can be scanned and read by your device's camera. uses QR codes to enroll your device with your account. Usually, you only do this once, but if you have to reset your account, you scan a new QR code to enroll again successfully. For how to scan the QR code, see Set up [%=okta-feature-names.Okta Verify%] with a QR code on your iOS device.

If you can't scan the QR code, see Set up [%=okta-feature-names.Okta Verify%] on your iOS device by signing in to your account or Set up [%=okta-feature-names.Okta Verify%] on your iOS device by using an activation link or secret key.

What is the secret key and do I need it to sign in to my account?

When you enroll your device with for the first time, you have different options, depending on your company’s configuration. If you can’t set up your account with a QR code, generates a secret key that you can enter in to set up your account. The secret key is not the 6-digit code that you enter in the sign-in window when you access your -protected apps. To learn more about using the secret key, see Set up [%=okta-feature-names.Okta Verify%] on your iOS device by using an activation link or secret key.

When I open , I see a six-digit code. What is it for? 

When you access apps protected by , generates a code that you can enter in the sign-in window to confirm your identity. You need this code only if you select the code option in the authentication window.

You don’t have to use the code to sign in with Okta Verify. You can also authenticate with push notifications if your organization configured this option.

The six-digit code is renewed every 30 seconds. After you close , the code is no longer generated.

The performance of your phone is not affected.

What is the blue loading bar above the code in Okta Verify?

This is not a loading bar. It’s a progress bar. The six-digit code is renewed every 30 seconds. The progress bar indicates how much time is left until a new code is generated.
