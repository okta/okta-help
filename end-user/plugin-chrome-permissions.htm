: Plugin permissions for extensions

The requires several permissions to work properly with -based browser extensions.

Permission Required
tabs

This permission opens a new tab in the following scenarios.

  • When the user selects an app on the pop-up window. The plugin opens a new tab to the application URL and signs in the user
  • When the user wants to sign in to their through the Admin link on the pop-up window
  • When the user wants to switch between their accounts using the account chooser feature

  • When the user wants to allow disabling browser password prompts from the pop-up window settings
cookies When the plugin inherits the session ID and device token cookies from the . The plugin uses these to make API calls for SWA applications. This setting permits the server to verify the user and ensure that POST requests are coming from a valid plugin user.

privacy To prevent browser extensions from prompting to save application passwords during single sign-on (SSO) operations. This is an optional permission that end users can opt into if they choose, as the Okta extension manages these particular passwords.
storage

To access the chrome.management setting. stores and accesses third-party app metadata in this location. This includes app login links, app logo links, and other info that identifies the app. caches the data in the extension's local storage to minimize server-side API calls for that metadata information.

unlimitedStorage To provide an unlimited quota for storing client-side third-party app data. This rarely exceeds 5 MB of local storage.
