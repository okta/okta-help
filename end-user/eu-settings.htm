End-user settings

You can edit personal information, specify the display language, change password settings, set up additional security methods, and see a list of recently used apps on the Settings page.

To view the Settings page from the End-User Dashboard, click your name in the top right corner and select Settings.

  • Personal Information: Change your personal information for their Okta account without having to contact an administrator.
  • Change Password: Change your password after verifying their identity through credentials, as determined by your password policy.
  • Display Language: Select your preferred language for all end-user pages.
  • Security Methods: Manage your security methods they use for signing in.
  • Recently Used Apps: Enable a section at the top of their dashboard that displays a list of recently used apps.

The Security Image and Forgotten Password Question authenticators settings features are not available in the End-User Dashboard.

You can only set up security methods that are assigned to them in the MFA Enrollment policy.

Before you begin

Before you can manage your settings, you must verify your credentials using the Edit Profile button. You must then verify your password and a second security method, if one is enrolled.

After you have verified your settings, you can manage their settings on the Settings page.

Secure account changes

You must reauthenticate after a period of five to 15 minutes, as configured by your admin. By clicking Edit Profile, you can initiate credential verification.

When you remove or add a security method in the Security Methods section, you must first verify your credentials with a password and a second security method, if one is enrolled.

The End User Dashboard > Settings menu has different options in .

  • Profile re-authentication is not supported in orgs using .
  • Forgotten authenticator settings: Authenticators used for recovery are the same as authenticators used for sign in. Users manage all authenticator enrollments from the Extra Verification section.
  • Password reveal: This setting is now always enabled. There is no toggle to turn it on and off.
  • Self-service password reset: End users can now perform self-service password resets through security question, email, voice call, and SMS, as determined by your password policy.
  • Keep me signed in: This feature is not available for . End users who do not have a valid session open are directed to the sign-in widget to sign in again.
