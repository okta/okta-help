Configure email notifications for an admin role

Super admins can set default email notifications for all admins in their org. Most admins have their own set of available Email notifications. The exceptions are report, group membership, and third-party admins, who may receive global Okta communications but aren't eligible for system notifications.

After you configure these settings, admins will only receive emails for groups, apps, or users that they have permission to view. For example, an app admin who is subscribed to app-related email notifications only receives notifications about the apps they're constrained to.

Before you begin

  • Ensure that you're signed in as a super admin.

Start this task

  2. Click the Roles tab.

  3. Locate the role that you want to configure email notifications for and select Edit Edit role notifications .

  4. Select the email notifications that the role should have.

  5. Click Save.

Notes:

  • Individual admins can set their own email notification preferences. See Admin email notifications.

  • When you edit an admin type's email notifications, any custom notification preferences that individual admins configure are restored to their type defaults (this occurs regardless of the admin type that you edited).

  • Rate limit warning and violation: All admins receive notification banners in the Admin Console and emails about their org-wide rate limit usage. Notifications flag warnings and violations. See Rate limits overview to better understand rate limits and Set up rate limit notifications for instructions on enabling rate limit notifications.

  • Admins with more than one role receive all of the email notification types selected for each of their assigned roles.

  • Super admins for all orgs are only notified in the Admin Console and by email about their org-wide rate limit usage and not for their client and operations-based rate limit usage. See Rate limits overview.
