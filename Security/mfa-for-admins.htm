Multifactor authentication for admins

Multifactor authentication (MFA) reduces the risk of admin account compromise if credentials are obtained maliciously by a third party. Super admins can enable mandatory multifactor authentication for all administrators signing in to Administration.

HealthInsight task recommendation

Enable MFA for Admins to reduce the risk of admin account compromise if credentials are obtained maliciously by a third party.

  4. Select Require Multifactor for Administrators signing in to Administration.
  5. Click Save.

Benefits and impacts

After this feature is enabled:

  • Admins will be prompted for multifactor authentication during sign-in or when they access Administration. If an org's sign-on policy already requires multifactor authentication, the admin won't be prompted a second time.
  • Admins who have not yet enrolled into an MFA factor will be prompted to enroll for the first time.
  • At least one factor should be turned on for your organization to enable this setting. If the org doesn't have any MFA factors enabled, with one-time passwords (OTP) will be enabled as the default factor. If factors have already been configured, then no changes will be made.
