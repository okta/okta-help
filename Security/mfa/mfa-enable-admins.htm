Enable MFA for the

Super admins can enable mandatory multifactor authentication (MFA) for all admins who access the .

  • After this feature is enabled, the MFA policy for the is enabled by default. The next time an admin signs in, they're prompted to set up MFA for access to the . Admins who haven't enrolled in MFA are prompted to enroll for the first time.
  • At least one factor must be turned on for your org to enable this setting. If the org doesn't have any MFA factors enabled, with one time passwords (OTP) are enabled as the default factor. If factors have already been configured, then no changes are made.
  • At least one authenticator must be enabled for your org to enable this setting. If the org doesn't have any authenticators enabled, with one time passwords (OTP) are enabled as the default authenticator. If authenticators have already been configured, then no changes are made.
  • You can also make additional changes to your MFA policy. See Configure an app sign-on policy
  • You can also make additional changes to your MFA policy. See Add an [%=okta-feature-names.app sign-on policy%] rule
  • Never disable MFA for admins. This decreases the overall security posture of your org and increases the risk that admin accounts might be compromised.
