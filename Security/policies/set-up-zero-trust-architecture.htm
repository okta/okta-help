Set up a Zero Trust architecture

A Zero Trust architecture is one in which all users must be authenticated and authorized before they can receive access to Okta or an app.

First, you create groups of users according to the type of access they need to do their jobs. Next, you create network zones so you can establish authentication requirements according to where users sign in from. For example, you might want to have fewer restrictions when users sign in from your in-house network, and more restrictions when they sign in from outside that network. You can also create Network Zones that let you block sign-ins from specific countries or regions.

After you've created groups and network zones, you create and configure policies that set the requirements for signing in to Okta, determine when users must enroll in MFA, and determine which users can access which apps.
