Access requests administrators

For orgs with Identity Governance enabled, access requests administrators perform tasks related to managing an instance. When a super admin assigns this role to a user, the user is automatically assigned to the Request Access app. This role allows users to perform a variety of tasks within : configuring app integrations, creating teams, exporting data, and more. The access requests admin role has a fixed set of permissions:

  • View all users
  • View all groups
  • Manage user access to applications with the application
  • Act as an administrator for the platform

After this role is removed from a user account, the system waits two minutes before finalizing the removal. This helps reduce the chances of mistaken role changes.
