Configure VPN notifications

You can use zones in VPN notifications to specify when to display an alert to end users that a VPN connection is required to connect to an application. These notifications are customizable and are disabled by default for all applications. If you configure a VPN notification message, you can select the network zones where this information is displayed.

The VPN notification doesn't appear if the end user has selected the Auto-launch option in the General settings of the app integration.

To configure VPN notification:

  2. Select the application that requires a VPN connection from the list of applications.

  3. Click General.

  4. In VPN Notification, click Edit.

  5. Select the appropriate location for displaying the VPN requirement message:

    • Disabled: Disable VPN notification for the selected application if it doesn't require a VPN connection.

    • Inside Any Zones: Show VPN connection information only when a browser's client IP address matches any network zone you've configured.

    • Outside Any Zones: Show VPN connection information only when a browser's client IP address doesn't match any network zone you have configured.

    • Anywhere: Show VPN connection information regardless of the browser's client IP address.

  6. Enter the VPN notification message that you want to display.

  7. Optional. Add a URL with detailed VPN instructions.
