aggregates data about sign-in activity across the customer base to analyze and detect potentially malicious IP addresses and to prevent credential-based attacks such as:

  • password spraying
  • credential stuffing
  • brute-force cryptographic attacks

Because ThreatInsight collects information about the origin of sign-in activity directed at organizations and endpoints, it provides a security baseline for all customers. You can choose to log events for auditing or to log events and block traffic that ThreatInsight has identified as suspicious. If you choose to log and block traffic, automatically denies access to sign-in requests that come from potentially malicious IP addresses that ThreatInsight has detected.
