Add a location behavior

A location behavior is used to define policies based on changes in the end user's geographical location when they sign in. For example, admins can configure to prompt an end user for a second authenticator when a user attempts to sign in from a new country, but permit access when there's a change of city.

Start this task

  2. Click Add Behavior.
  3. From the dropdown, select Location.
  4. In the Behavior Name field, enter a name for the behavior.
  5. In Location granularity, select the method and level of detail used to evaluate the behavior:
    • Latitude - Longitude
    • City
    • State or Region
    • Country

    If you select Latitude - Longitude, proceed to step 6. Otherwise, proceed to step 7.

  6. In Radius from location, enter the distance in kilometers from the location. The value must be five kilometers or greater.
  7. In Evaluate against past, enter the number of recent authenticators used to evaluate this behavior. The value must be one or greater.
  8. Click Save.
