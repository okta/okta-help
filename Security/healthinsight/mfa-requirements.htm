MFA requirements

This security task ensures that multifactor authentication (MFA) requirements aren't in conflict with Behavior Detection. It also ensures that MFA policy rules aren't bypassed.

These settings appear on the Sign-On Policy Add Rule or Edit Rule page. This combination creates a mismatch between the policy's condition and its action:

Option

Setting
Behavior is New Device
Users will be prompted for MFA

You've selected one of these options:

  • When signing in with a new device cookie
  • After MFA lifetime expires for the device cookie

When users select this security task, recommendations to correct the configuration appear.
