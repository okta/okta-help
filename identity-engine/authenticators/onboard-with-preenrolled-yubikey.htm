Onboard users with pre-enrolled

This configuration ensures that new users sign in with a phishing-resistant hardware security key from the day they join your organization. This phishing-resistant onboarding configuration is created using the authenticator and policies. The setup ensures that the users can only use when signing in to or accessing your company resources. Part of this configuration is automated using a pre-designed template.

When a new user is added to the specified group in , receives a shipment request for them through . After the request is fulfilled and the credential is activated in , a PIN is generated and emailed to the user. Later, when you activate the user in , they receive a welcome email with sign-in information. Users can then use this information and the PIN on their first day to sign in securely using .
