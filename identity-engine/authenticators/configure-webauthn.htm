Configure the authenticator

The authenticator lets you use a biometric method to authenticate. This authenticator supports two authentication methods:

  • Security keys, such as or Google Titan.
  • Platform authentication that's integrated into a device and uses biometric data, such as Windows Hello or Apple .

This authenticator also lets you manage which authenticators are allowed in your org for new enrollments, authentication enrollment policies, and user verification.

follows the FIDO2 Web Authentication (WebAuthn) standard. After you enable this authenticator, end users can select it when they sign in to or use it for additional authentication.

To set up and manage YubiKeys to use the one-time password (OTP) mode, see Configure the [%=3rd Party Integrations.YubiKey company name%] OTP authenticator.

Passkey Management

Passkeys are an implementation of the FIDO2 standard in which the FIDO credential may exist on multiple devices. Passkeys enable credentials to be backed up and synchronized across devices. This preserves the strong key-based/non-phishable authentication model of /FIDO while trading off some enterprise security features, such as device-bound keys and attestations, that are available with some authenticators. Users no longer need to carry their security key or phone to pass multifactor authentication challenges. Instead, they can use any device they have already enrolled to authenticate themselves because their credential isn't confined to a single device.

In managed-device environments, users may be able to enroll unmanaged devices with a passkey credential and use these devices to gain access to corporate systems. allows admins to block the use of passkeys for new enrollments for their entire org. When this feature is turned on, users aren't able to enroll new, unmanaged devices using pre-registered passkeys.

Block the use of passkeys

  2. Click the toggle switch for the Block Passkeys for Authenticators option.

When you block the use of passkeys in your org, users running macOS Monterrey can't enroll in using the Safari browser.

In addition, when you block the use of passkeys, iPhone users running iOS 16 on their devices can't use the authentication. If you need to block the use of passkeys, recommends that you enable or security keys that support NFC or USB-C. Enrollments of devices running iOS 16 are supported after you block the use of passkeys for non-passkey uses.
