Authentication scenarios

To demonstrate how the sign-on policies interact, consider a Global Session Policy that uses Password / IDP / any factor allowed by app sign on rules. This setting offers the most flexibility when configuring app-level policies for each app in your org (for example, passwordless authentication for one app and secure MFA for another). Combining this setting with the following settings results in different authentication experiences for end users.

factor settings

Prompts for authentication

Password only

End user signs in with a password or is federated and is not prompted for password again until the first of these events occurs:

  • the session expires (Okta sign-on policy)
  • the Re-authentication frequency point is reached ()
Password + possession factor (per device setting)

End user signs in and is not prompted for password again until the session defined in the expires. End user is not prompted for possession factor again unless they clear cookies on their device.

Password + possession factor (per session

End user signs in and is not prompted for password or factor again until the session defined in the expires.

Password + possession factor (every time)

End user signs in and is not prompted for password or factor again until they return to the app authentication page.

Password + possession factor (per Re-authenticate after setting)

End user signs in and is not prompted for password again until the session defined in the expires. End user is not prompted for possession factor again unless they clear cookies on their device or the factor lifetime setting expires and they return to the app authentication page.

Possession factor only (per Re-authenticate after setting)

End user signs in with any enrolled possession factor. End user is not prompted for the possession factor again unless they clear cookies on their device or the factor lifetime setting expires and they return to the app authentication page.
