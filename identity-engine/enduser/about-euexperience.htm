End-user experience

End users will encounter a new experience with that streamlines registration, authenticator setup and configuration of settings.

  • New users register and activate their profiles in the sign-in widget instead of the welcome wizard, allowing you to fully customize and brand the onboarding experience with custom Okta-hosted sign-in pages.

  • During sign-in, end users authenticate through the primary and secondary security methods among those you’ve made available in your policies.

  • Profile Enrollment policies allow for progressive profiling and self registration.

End-user experience UI changes


Safe mode

During maintenance or deployments, Okta servers may briefly go into safe mode. Safe mode can affect the end user’s MFA enrollment in the following ways:

  • If Okta servers are in safe mode when an end user attempts to access an app, the user must meet the ’s authenticator requirements.

  • If Okta servers go into safe mode while an end user is in the process of enrolling, an “Authenticator setup is not available” message appears. If the end user has already met the ’s authenticator requirements, they may skip enrollment and go directly to the app. If the app sign-on policy’s requirements have not been met, the user is advised to either return to the settings menu or sign out and try again later.
