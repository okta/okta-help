You can deploy to macOS devices using one of the following methods:

  • Allow end users to download it directly from the App Store or a server location that you set up. This approach makes sense if you don't have a device management solution. If you want to post to a private server location, download the app from the ( Settings Downloads ).
  • Use your device management solution to deploy to devices, using one of the following methods:
    • Deploy [%=okta-feature-names.Okta Verify%] directly to devices from the Apple Business Manager (recommended approach)
    • Download [%=okta-feature-names.Okta Verify%] from Okta, and then deploy it to devices

Deploy directly to devices from the Apple Business Manager (recommended approach)

You can configure your device management tool to grab from Apple Business Manager, and then deploy the app directly to targeted end-user devices.

After the app is installed, if your organization uses devices with macOS Big Sur (11) or later, Okta recommends that you configure your device management tool to automatically force app updates.

If you need assistance configuring to automatically force app updates for , email Jamf at success@jamf.com and reference PI-009232.

Download from Okta, and then deploy it to devices

If your device management tool does not support Okta Verify deployment from the Apple Business Manager, you can download directly from Okta.

This deployment method does not automatically force app updates.

  1. In the , go to Settings Downloads .
  2. For for macOS, click Download Latest.

    A PKG file with the latest version of for macOS is downloaded.

  3. Configure your device management tool to deploy the app to targeted end-user devices.

This deployment method does not support auto-update. As required, repeat this process to provide end users with the latest version of .

If you are using a custom PKG file to install , make sure you include a post-install script to relaunch when the installation is complete. Without this post-install script, if a user has open, will not update until it is manually closed and re-opened.
