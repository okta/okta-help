Device registration

Device registration binds a device to the app instance on the device. Each registered device is a unique object in the Universal Directory and is visible on the Devices page.

Device registration happens automatically when users set up an account in . Admins can't create devices from the .

When end users add an account in , the device is automatically registered in the Okta . For account setup details, see the Okta end-user documentation.

Registration workflow

  1. Users create an account in .
  2. A unique key is created and stored on the device. The key is stored in a hardware-backed keystore (for example, the Trusted Platform Module, or Secure Enclave) or in a software-backed keystore.
  3. creates a device record in the Universal Directory. The device is bound to the app instance. The device is registered in and appears on the Devices page in the (Directory Devices).
  4. When users access an -protected app from the device, probes the device:
    • Checks if is installed on the device
    •  Checks if the device is registered (an account exists)
    •  Checks if the device is managed (the device is managed by a device management solution, the device is configured for device management in Security Device Integrations, and the user successfully authenticated with on that device).
    • Checks if secure hardware is present (TPM, Secure Enclave)
    • Checks if the proof of possession key is hardware-protected
