Configure a global session policy for

You can create a global session policy that supports , or edit an existing policy.

This configuration removes the global password requirement from the global session policy and transfers responsibility for defining and enforcing authentication criteria to each of your . Before you complete this configuration, create strong authentication policies for all of your apps so that users can't access them with a single enrolled factor.

Create a global session policy for

  1. Create a global session policy.

  2. Add a rule to your new policy.

  3. In the Rule Name field, enter a name for the rule. For example, Okta FastPass rule for managed devices.

  4. Optional. In the Exclude Users field, enter the names of users (that exist within the group) to exclude from the rule.

  5. Specify the conditions for the rule. At step 6, configure the following conditions: 
    • Establish the user session with: Select Any factor used to meet the Authentication Policy requirements. This allows users to have passwordless authentication.
    • ANDMultifactor authentication (MFA) is: Select the Not required option. This allows users to sign in without using biometrics.
  6. When you've finished configuring the rule conditions, click Create rule.

Edit an existing global session policy for

Complete this procedure if you want to configure an existing global session policy for .

  1. Follow the steps for Add a global session policy rule. At step 6, configure the following conditions:
    • Establish the user session with: Select Any factor used to meet the Authentication Policy requirements. This allows users to have passwordless authentication.
    • AND Multifactor authentication (MFA) is: Select the Not required option. This allows users to sign in without using biometrics.
  2. If required, change the rule priority.
