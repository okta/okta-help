Beta release: Endpoint security integrations - Tanium support

You can integrate with your organization’s endpoint detection and response (EDR) solution. When users try to access a resource protected by , probes their device for context and trust signals. If an EDR client is running on the device, Okta Verify also captures signals collected by the client. Okta evaluates all signals against the that you configured in the .

The high-level endpoint security integration flow involves the following steps:

  1. communicates through plugins with the EDR client running on the same device: 
    • macOS: See Manage endpoint security integration plugins for macOS.
    • Windows: See Manage endpoint security integration plugins for Windows.
  2. The plugin is invoked when a user signs in with to access a resource with an that requires EDR signals.
  3. captures the signals collected by the plugin and provides these to the server.
  4. caches EDR signals for up to eight hours or until the session times out, whichever occurs first. The cache is updated whenever new signals are processed.
  5. The server evaluates the signals against the and allows or denies access to the resource.
  6. If access is denied, users receive a message: You do not have permission to perform the requested action. In the System Log, a message confirms that the authentication policy evaluation resulted in denied access. You can evaluate the context of the System Log message together with information from the EDR dashboard to determine why access was denied.
