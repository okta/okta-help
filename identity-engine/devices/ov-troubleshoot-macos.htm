Troubleshoot Okta Verify on macOS

If you encounter issues with Okta Verify on macOS devices, try the following general troubleshooting steps:

  1. Update Okta Verify to the latest version.

  2. Close Okta Verify and re-open it.

  3. Restart the macOS device and then re-open Okta Verify.

  4. Report the issue using the AppCenter for macOS.

    If you aren't able to access the AppCenter, you can collect logs on the device ~/Library/Containers/com.okta.mobile/Data/Documents/logFile.txt.

Managed devices

If the macOS device is a managed device, try the following troubleshooting steps:

  1. Open the login keychain to verify that a Simple Certificate Enrollment Protocol (SCEP) certificate is installed on the device.

  2. Check whether Okta is the certificate authority that issued the certificate installed on the device.

  3. Check whether the certificate was issued for all users at the device level or for an individual user of the device.

  4. Check whether the private key for the certificate has Access Control set to Allow all applications to access this item.

  5. Check the status of the device in your mobile device management software.

  6. Check the configuration of the SCEP profile in your mobile device management software to make sure allow all apps access is selected.

  7. Verify that your device management software is set to push managed app configurations to the following preference domains:

    • Preference domain 1: com.okta.mobile

    • Preference domain 2: com.okta.mobile.auth-service-extension

If you receive error code 25300 or Failed to create Identity reference for given certificate, this indicates that the SCEP certificate was not installed correctly. Delete the certificate and reinstall it.

Endpoint detection and response integration

If your endpoint detection and response integration is not working as expected, try the following troubleshooting steps:

  1. Verify that the software you are integrating with is installed and configured correctly.

  2. Verify that the software you are integrating with supports managed app configuration.

  3. Verify the Okta plugin for the integration was successfully deployed.

  4. Check that the device is not registered using the same certificate in another Okta org.

  5. Verify that your device management software is set to push managed app configurations to the following preference domains:

    • Preference domain 1: com.okta.mobile

    • Preference domain 2: com.okta.mobile.auth-service-extension
