Add an endpoint security integration

When you add an endpoint security integration, the server requests device integration signals from this specific endpoint security client. The client collects the signals from the plugins on the local device, and then sends a response back to the server. The server uses these signals make app access decisions ( are evaluated based on EL signals).

currently supports integrations with CrowdStrike, Microsoft Windows Security Center, and Chrome Device Trust. Support for additional endpoint security integration vendors is planned for the future.

  2. Click the Endpoint security tab.

  3. Click Add endpoint integration. The list displays available endpoint integration options.

  4. Select one of the following:

    • Windows Security Center

    • CrowdStrike

    • Chrome Device Trust

    You can only add one of each endpoint integration. If all endpoint integrations are already added, none are available to add.

  5. If you selected CrowdStrike in step 4, select one or more platforms, and then click Add.

  6. If you selected Chrome  in step 4, follow these steps to integrate the connector:

    1. Sign in to your Google Admin cConsole.

    2. Open Devices Chrome Connectors, and then click New provider configuration. Scroll down to Okta in the provider list and click Set up.

    3. Enter a Configuration name, and then add the URL Pattern and Service account information from the integration that you created in the Okta to the provider configuration. Click Add configuration.

    4. Apply the provider configuration to your Organizational unit. To ensure that the configuration is applied to the appropriate org unit, check that the managed browser configuration is mapped to the same org unit.
