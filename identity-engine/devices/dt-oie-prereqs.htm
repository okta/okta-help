Device management

To control app access based on device management state, targeted devices must be managed by a device management solution. Note: Devices don't need to be managed for end users to add an account in and for the device to appear in Directory Devices .

Mobile devices

Mobile device management requires third-party Mobile Device Management (MDM) software that supports managed app configuration to configure the key-value pair. For best results, integrate with MDM software that can set to install silently and automatically on end-user devices when users enroll in your MDM software.

Okta tested with the following MDM software but other platform-supported solutions should work:

  • Android: VMware , Microsoft Intune
  • iOS: VMware

Desktop devices

Desktop device management requires a certificate authority (CA) that can issue client certificates to targeted devices, allowing to verify that devices are managed. You can use as CA or provide your own.

Using Okta as a CA

  • macOS: Use any device management solution that supports the deployment of an Apple SCEP MDM payload. The procedure in this document assumes you are managing your devices with Jamf Pro and configuring a Static SCEP profile.

    • See Configure Okta as a certificate authority for macOS using Jamf Pro.

  • Windows: Okta tested two configurations:
    • Delegated (Dynamic) SCEP method using AD CS and NDES integrated with
    • Static SCEP method using

    • Procedures for both methods are provided in this document.

    See Configure Okta as a certificate authority for Windows using Intune and Configure Okta as a certificate authority for Windows using [%=Mobile.WS1%].

Provide your own CA

If you are not using Okta as a CA, you need to provide your own public key infrastructure (PKI) or CA infrastructure integrated with your MDM software. Okta tested this feature with Microsoft Intune (for Windows) and Jamf Pro (for macOS). To prove that devices are managed by your organization, you must configure your CA infrastructure or MDM software to deploy certificates, signed by your organization's certificate authority, to your registered Windows and macOS devices. Deploy certificates for digital signature but not for other purposes, such as encryption.

See Device registration, and Provide your own certificate authority for managed devices.
