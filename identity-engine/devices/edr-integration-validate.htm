Validate your endpoint security integration

It's important to validate your endpoint security integrations before widely deploying them. These steps describe how to validate your integration for SP- and IdP-initiated authentications.

  1. Sign in to using as the authentication method (for example, passwordless authentication where you're prompted to Use this device or Sign in using on this device).
  2. Validate SP-initiated authentication:
    1. Open a browser.
    2. Try to access the relevant policy-protected app directly.
    3. Validate Allow and Deny by doing one or both of the following:
      • Change the condition of the device (for example, by turning the firewall off and on).
      • Change the EDR signals that you specified in the .
    4. Open another tab.
    5. Try to access the relevant policy-protected app directly.
    6. Validating both Allow and Deny flows in the new tab.
  3. Validate IdP-initiated authentication:
    1. On the , click the relevant policy-protected app.
    2. Validate Allow and Deny by doing one or both of the following:
    3. Change the condition of the device (for example, by turning the firewall off and on).
    4. Change the EDR signals that you specified in the .
  4. Change the setting User must authenticate with to Password + Another factor and repeat the SP- and IdP-initiated authentication flows as described in the preceding steps.
