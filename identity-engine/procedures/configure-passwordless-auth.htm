Configure passwordless authentication

Deprecated because we've a new doc (Set up passwordless sign-in experience) replacing this.

You can configure your and to create a passwordless sign-in experience for your end users.

  1. Create a [%=okta-feature-names.Okta sign-on policy%].
  2. Enable email as an authenticator and clear the Do not use for authentication (MFA/SSO) checkbox.
  3. Create an MFA enrollment policy and set Email as required for your user group.
  4. Create a rule for your according to Add a [%=okta-feature-names.Okta sign-on policy%] rule. In step 6, select the Password / IDP / any factor allowed by app sign on rules condition. This allows you to defer the authenticator requirement to the app sign-on policy.
  5. Add an [%=okta-feature-names.app sign-on policy%] rule. In step 8 of that task, select one of the following:

    • One required factor AND user must provide possession factor. In this scenario, end users click an email magic link or enter phone codes. End users are never prompted for a password.

    • One required factor AND any enrolled authentication factor. In this scenario, end users can enter a password but any other enrolled factor also works.
