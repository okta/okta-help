Set up a default app redirect

If Okta is unable to determine which application your end users are attempting to access when they sign in, they are directed to the End-User Dashboard.

However, you can set up a default app redirection so that this doesn't happen to end users who sign in to Okta from outside of a specific application. This setting also applies to end users who perform a self-service password reset or MFA reset.

Start this task

  2. Select a brand.
  3. On the Pages tab in the Sign-in page section, click Configure.
  4. In the Sign-in page section, click Settings.
  5. In the Default Application for Sign-In Widget section, click Edit.
  6. In the Application menu, begin typing the name of the app you want to serve as the default redirect. If you have multiple SAML and OIDC apps, they appear in the dropdown list. The AppLink changes according to the application you choose.
  7. Click Save to draft.

The End-User Dashboard remains available at https://{orgname}.okta.com/app/UserHome.
