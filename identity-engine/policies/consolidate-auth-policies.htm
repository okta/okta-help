Merge duplicate policies

The merge feature finds authentication policies with the same rules, moves their apps to a single policy, and then deletes the duplicates. You can merge all authentication policies in your org except for the ones for specific apps, like RADIUS, O365, and the Okta Admin Console.

The default policy is a unique case. You can merge duplicates into the default policy, but you can't merge it into another policy.

  2. Select a policy to start the merge. Be sure this is the policy you want to keep. All apps from the duplicates move here.
  3. Click the Actions menu, and then select Merge duplicate policies.

The merge can take a few minutes, and the Authentication Policies page is read-only during this time. When it's complete, you receive an email with merge results.
