App sign-on policies

After you upgrade to , learn about the changes to app sign-on policies.

Change summary

The app sign-on policy allows or denies access to an app. Every app has its own app sign-on policy.
App sign-on policies are now called authentication policies, and they're shareable across apps.
Admin experience To configure authentication policies, go to Security .

You can view all in one location. You can create a policy yourself, instead of only adding rules to an app's default policy, and you can also share one policy among many apps. This allows you to create and maintain policies at scale and evaluate how each policy impacts application access.

are still viewable on an app's Sign On tab in Applications Applications . However, you can no longer modify the policy from this location.

User experience Changes to the user experience depend on how you configure new conditions in the policy.

Evaluation of authentication policies is different for OIDC apps. In , an OIDC app sign-on policy is evaluated immediately when a user selects it. In , you can configure OIDC apps with the Redirect to app to initiate login (OIDC compliant) setting. Users who select these apps go first to the initiate login URI. Then, when the app issues an authorize request, the authentication policy is evaluated. MFA prompts appear when the users return to Okta.
