Secondary email as an optional user account field

Learn about the changes to enabling a secondary email address in Optional User Account Fields.

Change summary If you've enabled Secondary Email in Customizations Other Optional User Account Fields, it's treated as a Required profile enrollment attribute.
Admin experience If you've enabled secondary email, end users who don't have one should add it before you upgrade your org. You can create a group rule to find users who are missing the secondary email address.
User experience If secondary email is enabled but the user hasn't provided it, they're prompted for it the first time they sign in after the upgrade.
