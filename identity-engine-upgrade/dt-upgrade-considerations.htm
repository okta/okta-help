Device Trust for mobile devices

Learn about the changes to Device Trust for mobile devices.

Change summary In , you must use to secure your mobile devices.
Admin experience There’s no automated upgrade for mobile devices. To secure your mobile devices, enable in after the upgrade.

Before you upgrade:

  • Turn off Mobile [%=okta-feature-names.Device Trust%]

  • Disable [%=okta-variables.ProductName%] Mobility Management

After you upgrade:

  • Management attestation for mobile devices

  • Managed app configurations
User experience
  • Users can enroll multiple devices in (in , they can enroll only one device).
  • If the authentication policy has the Hardware protected constraint enabled, enrollments with push notifications don't work.
  • If users already have an account, it continues to work. A Set up Okta FastPass button appears in the app's Account Details page.

  • In , users can't add more than one account per org. If a user has two accounts in a org, both continue to work after upgrading. A Set up Okta FastPass button appears in both accounts. If the user clicks the button in the second account, an error appears.
