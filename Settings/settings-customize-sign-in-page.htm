Customize your sign-in page

You can customize the labels and URLs on your sign-in page. You can customize the labels, URLs, and visibility settings on your sign-in page. If your org uses a custom sign-in page and you're familiar with HTML, you can make more advanced customizations with the code editor.

Before you begin

The browser must be at least 750 pixels in height to properly display the sign-in page.

  5. Enter new values for any of the headings, labels, and links that you want to customize. Leave the fields blank if you want to keep the default values.

  7. To add a custom link that appears on the authenticator page of the , enter values in the Custom link text and Custom link URL fields.

    To customize the link on the Authenticator verification page, enter new values in the Factor Page and Custom Link fields. Note: This is an Early Access feature. To enable it, contact Okta Support.

  9. To customize whether the user's identifier appears on authentication and enrollment pages, open the Settings tab and then change the Sign-In Widget User Identifier toggle.
  10. Click Save to draft.

    When a draft is saved but not yet published, yellow dots appear next to the tab label, top control buttons, and the section headings that contain changes.

  11. To see a preview of your changes, click Preview.
  12. Click Publish.

Default links appear in the end users' display language or the browser language, but Okta doesn't localize custom links. If your custom link isn't in the end users' display or browser language, the sign-in page will have text in more than one language.
