Configure a custom Okta-hosted Sign-In page

When end users attempt to sign in to your applications, they are redirected to an Okta-hosted sign-in page. You can customize this sign-in page to provide a seamless user experience that fits your brand. The Custom Sign In tab contains a Sign-In Widget and an embedded HTML editor to help you get started.

If you're using a custom domain and want to use a custom sign-in page or custom error page, you must customize the default CSP for the domain.

About admin roles for these tasks

Only super admins and org admins can perform these tasks.

Task

Description
Customize the Okta URL domain The custom sign-in page is available to end users only when they sign in to through your custom URL domain.
Style the Okta-hosted Sign-In Widget You can add any HTML, CSS, or JavaScript to the sign-in page and also customize the sign-in page per application and with multiple brands.

The latest versions of Safari and Firefox block third-party cookies by default. Your applications may be impacted if they rely on a cookie from an okta.com domain to maintain a session on a non-Okta domain. See FAQ: How Blocking Third Party Cookies Can Potentially Impact Your Okta Environment.
