Configure optional user account fields

Optional user account fields include a secondary email address and a security image. When you enable the secondary email option, end users can enter a secondary address for password reset and account creation notices. When you enable the security image option, users are prompted to select an image the first time that they sign in to Okta. Then, their selected image appears whenever they enter a valid username in the Okta sign-in page.

  1. In the Admin Console, go to Customizations Other .
  2. Scroll to the Optional User Account Fields section, and then click Edit.
  3. Set the Secondary Email option to Enabled.
  4. Set the Security Image option to Enabled.
  5. Click Save.

The browser must create a cookie before it can display a security image. Therefore, the image doesn't appear during a user's initial sign-in attempt, their first attempt after clearing cookies, or their first sign-in attempt from a new device or browser.

When you enable the secondary email option, end users can enter a secondary address for password reset and account creation notices.

  1. In the Admin Console, go to Customizations Other .
  2. Scroll to the Optional User Account Fields section, and then click Edit.
  3. Set the Secondary Email option to Enabled.
  4. Click Save.
