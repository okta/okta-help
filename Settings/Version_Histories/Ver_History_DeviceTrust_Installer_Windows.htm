for Windows Desktop Registration Task Version History

This page lists the current and past versions of the Registration Task for for Windows Desktop. This page is updated when a new version of the Task is released (Generally Availability (GA) or Early Access (EA)). You can download the Generally Available version of the Registration Task from the Downloads page of the ( Settings Downloads ).

Although the Generally Available version of the registration task is available from the through the Settings Downloads page, Early Access versions are not. To obtain an Early Access version, you must configure a link as described in "Procedures, section 2.2" of Enforce [%=okta-feature-names.Okta Device Trust%] for managed Windows computers.

For details about this device trust solution, see Enforce Okta Device Trust for managed Windows computers.

Release

1.4.1

This Generally Available release fixed the following issues:

  • If there was a space in the sAMAccountName, an error appeared when installing the Device Registration task and the installation completed but didn't function.
  • An "unknown publisher" warning appeared after double-clicking the Device Registration MSI file.

Affected customers should uninstall the registration task and install 1.4.1 or later. See Enforce [%=okta-feature-names.Okta Device Trust%] for managed Windows computers.

EA: 2020.0.2.0

1.4.0

This Early Access version provides the following:

  • Support for Trusted Platform Module (TPM 1.2 or 2.0) on Windows 10 devices with TPM. Admin action is required. For installation instructions and other details, see Enhance Windows Device Trust security with Trusted Platform Module (TPM).
  • Fixed an issue where installing the Device Registration Task caused Chrome browser settings to be removed.
  • Fixed an issue where, in some circumstances, installing the Device Registration Task failed to set the automatic certificate selection setting in Chrome. This setting is designed to prevent the browser from prompting end users to select the certificate during the device trust flow.

2019.09.0

1.3.1

This Early Access version provides the following:

  • Improved handling of private keys to ensure successful certificate renewal.
  • To fix an issue in earlier versions where a failed certificate renewal could leave computers in a bad state, this version allows admins to trigger certificate renewal on a per-computer basis. See Enforce Okta Device Trust for managed Windows computers.

2019.03.3

1.3.0

This Generally Available version provides the following:

  • Improved support for organizations that route internet traffic through a proxy server.
  • Fixes an issue in which some system log events reported the Windows operating system version inaccurately on Windows desktops running Windows 8.1 or later.

2019.1.0

1.2.2

This Early Access version provides the following:

  • Organizations that have complex proxy configuration rules defined in a proxy auto-config (PAC) file can now specify the PAC location instead of the server address. See Enforce [%=okta-feature-names.Okta Device Trust%] for managed Windows computers.
  • Prevents the Device Trust certificate installation prompt from appearing to end users who use 32-bit versions of Internet Explorer.
