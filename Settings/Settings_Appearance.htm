Appearance

From the Appearance page, you can manage the look and feel of your end user Home page. Customize the end user onboarding experience, the sign-in page, and the end user Home page.

Start this procedure

In the , go to Settings Appearance to do one of the following tasks:

  • Enable the Onboarding screen
  • Set the sign-in background image
  • Customize your Home page

Enable the Onboarding screen

The Onboarding screen has some helpful pointers on navigating through the console for end users. The screen is only visible when end users sign in to the org for the first time.

  1. In the Display Options section, click Edit.
  2. From the drop-down box associated with Onboarding screen, select Enable.
  3. Click Save.

Set the sign-in background image

  1. In the Sign-In Configuration section, click Edit.
  2. Click Browse to locate the image you want to appear as the background on Sign-in pages.
  3. Click Upload Image. To revert to the default image, click Use Default.
  4. Click Save.

Customize your Home page

Task Procedure

Add a logo
  1. Prepare a logo file. The file must be a .jpg, .png, or .gif.. The file size should be less than 100kB and dimensions should be less than 3000px x 500px.
  2. In the Organization Logo section, click Upload Logo. Then, browse to find the logo file you previously created.
  3. Click Upload Logo.

Associate a URL with the logo
  1. In the Display Options section, click Edit.
  2. In the Logo URL field, enter a URL that you want to associate with the logo, usually your company's website URL.
  3. Click Save.

Select a color scheme for the Home page
  1. In the Application Theme section, select a thumbnail to apply a different color scheme to your end user's My Applications home page. End users see the new theme when they sign in or refresh the page.
  2. Click Change Theme.

Enable the footer on the Okta Home page

The footer contains a shortcut for requesting an app and the email address of your IT admin. It also shows the end users when they last signed in to their account.

  1. In the Display Options section, click Edit.
  2. From the drop-down box associated with Okta Home footer, select Enable.
  3. Click Save.



The New Okta Sign-in Experience

Okta's new sign-in experience introduces a new look and new functionality. It is powered by the Okta Sign-In Widget, a full-featured, embeddable JavaScript toolkit for authenticating users, performing multi-factor enrollment and verification, recovering forgotten passwords, and unlocking accounts.

Requirements
  • For the new sign-in page to display correctly, your browser must be at least 750px in height.
  • To enjoy the new sign-in experience in Windows Internet Explorer, you must use version IE8 or later.
 Enable the New Sign-in Experience
  1. In New Sign-In Configuration, click Edit and then click Enable.
  2. In Sign-In Background Image, click Browse to locate the background image you want to appear in sign-pages, and then click Upload Image. Or, if you want to restore the default image, click Use Default.
  3. Click Save.
 Adding Links to Helpful Resources

Click Need help signing in? to expand the sign-on widget and display links to additional help resources.

Managing Duo Devices

The new sign-in experience uses the Duo Web SDK instead of direct API integration. This change allows admins to leverage the latest updates from Duo, such as the ability to manage Duo devices directly from the login page. And if the Duo self-service portal is enabled for your org, your users can also add, update, and remove authentication methods by clicking the Manage Devices button. For more information, see Configuring Duo Security.

If multifactor authentication (MFA) is configured for your org, please note the following changes:

 Sign-In Page Optimization for Embedded Browser Flows

The new sign-in experience removes obstructive banners and optimizes text size to ensure that users no longer need to scroll when logging in on screens smaller than 700 px wide, such as embedded browser flows.
