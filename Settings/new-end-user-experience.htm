Control access to the

Use this guide to control access to the for your org.

New organizations get access to the dashboard by default, so existing orgs can use this guide to remove access for users that don’t use the dashboard.

About admin roles for this task

  • You must have a Super Admin permissions to change access to the .

  • If you choose to deny access to the , affected users keep the existing access level until the next time they sign in.

Disable the Okta End-User Dashboard for specific groups

You can disable access to the for groups and specific user types. For example, you can create a sign-on policy to prevent dashboard access for groups that only access apps through your company's intranet or groups who only access a custom-built app. Use the following instructions as an example for how to prevent access to the .

  2. Select the Okta Dashboard app.
  3. In the Sign On Policy tab, click Add Rule.

  4. Under Who does this rule apply to?, click the following groups and users and select the specific groups where this rule applies. 

    If you add a sign-on policy to deny access to the , ensure that admins are either not included in the affected group or have a higher priority rule so they retain access to their dashboard.

  5. In the Access section at the bottom of the page, use the dropdown menu to select Denied.
