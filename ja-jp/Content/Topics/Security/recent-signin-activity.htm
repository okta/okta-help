最近のアクティビティ

Okta End-User Dashboardの［Recent Activity（最近のアクティビティ）］ページには、サインインとセキュリティイベントのアクティビティが表示されます。

  • ［Sign-ins（サインイン）］には、アカウントの可視性を高める情報として、エンドユーザーによるOktaへの最近のサインインアクティビティがまとめて表示されます。
  • ［Security Events（セキュリティイベント）］には、エンドユーザーのOktaアカウントに関する最近のセキュリティイベントがまとめて表示されます。

最近のアクティビティの有効化

  1. Okta Admin Consoleで、［Security（セキュリティ）］>［General（一般）］の順に進みます。

  2. ［Organizational Security（組織のセキュリティ）］セクションに移動します。

  3. ［Edit（編集）］をクリックします。

  4. ［Users can see Recent Activity（ユーザーが最近のアクティビティを表示可能）］フィールドに関連付けられたドロップダウンから、［Enabled（有効）］を選択します。

  5. ［Save（保存）］をクリックします。

最近のアクティビティへのアクセス

Okta End-User Dashboardでは、次のいずれかの方法を使用して最近のアクティビティにアクセスできます。

  • 自分の名前をクリックし、［Recent Activity（最近のアクティビティ）］を選択します。
  • ナビゲーションバーの［last sign in: 1 hour ago（直近のサインイン：1時間前）］をクリックします。

サインインとセキュリティイベントの詳細

  • サインイン機能は、System Logのuser.authentication.verifyというイベントに基づいています。このイベントは、エンドユーザーがOktaにサインインするときに発生します。
  • セキュリティイベントには、メインとセカンダリのメールの更新、パスワードの変更、要素の登録とリセットに関する最近のイベントが記録されます。

サインイン

［Sign-ins（サインイン）］には、エンドユーザーによるOktaへの最近のサインインがまとめて表示されます。

有効にすると、ユーザーは次のタスクを実行できます。

  • 過去30日間のサインインの試行またはアクティビティを表示する（最大100件）。
  • Oktaへのサインインに使用したデバイスごとに情報をまとめる。
  • デバイスのリンクをクリックして最近のサインインの詳細を表示する。
  • 最近のサインインアクティビティのおおよそのロケーションを表示する。
  • 疑わしいアクティビティをOkta管理者に直接報告する。


Internet Explorerではサインインテーブルの配置の問題が発生する可能性があります。「ブラウザー」を参照してください。

サインインの情報

この機能を有効にした後に構成が必要な追加の設定はありません。エンドユーザーは、アカウントに関する次の情報にアクセスできます。

デバイス

エンドユーザーがOktaにサインインするために使用したデバイス。このリンクをクリックすると、各サインインアクティビティの詳細情報が表示されます。

  • IPアドレス
  • ロケーション
  • 時刻
  • ブラウザー
  • レポート：ユーザーは［Report（レポート）］をクリックして、心当たりがないアクティビティをorgの管理者に直接報告できます。
前回のサインイン このデバイスを使用して最後にサインインした時刻。
前回のサインイン場所 ユーザーがサインインを行った、わかる範囲でのおおよその場所。
合計サインイン回数

このデバイスからの合計サインイン数。
管理者に報告 ユーザーが疑わしいアクティビティを管理者に直接報告するためのオプション。

セキュリティイベント

［Security Events（セキュリティイベント）］には、エンドユーザーのOktaアカウントに関する最近のセキュリティイベントがまとめて表示されます。［Security Events（セキュリティイベント）］では、メインと予備のメールアカウントの更新、アカウントパスワードの変更、MFA要素のリセットと登録に関する情報が提供されます。

有効にすると、ユーザーは次のタスクを実行できます。

  • 各セキュリティイベントの概要を含む最近の3つのセキュリティイベントカードを表示する。
  • セキュリティイベントカードをクリックしてダイアログを開き、イベントの詳細を確認する。［Report（レポート）］をクリックして、疑わしいアクティビティをOkta管理者に直接報告する。
  • ［See all security events（すべてのセキュリティイベントを表示）］をクリックして、さらに多くのセキュリティイベントを表示する。
  • 利用可能な情報がないことを通知するメッセージを参照する。


セキュリティイベントの情報

この機能を有効にした後に構成が必要な追加の設定はありません。エンドユーザーは、次のセキュリティイベント情報にアクセスできます。

セキュリティイベント

ユーザーがセキュリティイベントをクリックすると、各セキュリティイベントの詳細情報が表示されます。

  • イベント
  • IPアドレス
  • デバイス
  • ブラウザー
  • ロケーション
  • 時刻
  • レポート

元のイベントが十分なログ情報を取得できない場合、デバイス、IPアドレスおよびロケーションのイベントが［Unknown（不明）］として表示される場合があります。
