OAuthおよびREST ShareFile統合の設定

要件

  1. ShareFileで管理者アカウントを作成します。このアカウントを使用してShareFile REST API V3統合で使用するOAuthコンシューマーキーとコンシューマーシークレットを作成します。
  2. APIキーの生成：

    • https://api.sharefile.com/apikeysに移動して新しいOAuth資格情報を作成します。必要に応じてログインします。

    • ［Create New（新規作成）］をクリックします。

    • Application Name（アプリケーション名）：統合に名前を付けます。

    • Redirect URI（リダイレクトURI）：「https://system-admin.okta.com/admin/app/generic/oauth20redirect」と入力します。

    • ［Generate Api Key（APIキーを生成する）］をクリックします。APIキーがテーブルとして表示されます。クライアントIDとクライアントシークレットを取得します。これらは後のプロビジョニング設定で使用します。

  3. OktaでShareFile REST API V3のアーリーアクセス機能を有効にする：
    • Okta Admin Consoleで、［Settings（設定）］>［Features（機能）］に移動します。
    • ［Early Access（アーリーアクセス）］セクションで［Enable newest ShareFile REST API V3（最新のShareFile REST API V3を有効化）］を見つけて有効にします。

構成の手順

注：

  • さきほど構成した基本資格情報（管理者のユーザー名とパスワード）は［Provisioning（プロビジョニング）］→［Integration（統合）］セクションにOAuth資格情報を入力するまで、プロビジョニング操作に使用します。［Enable newest ShareFile REST API V3（最新のShareFile REST API V3を有効化）］機能を有効にすると、OAuth設定だけが表示され、以前の認証モード設定は非表示になります。

  • 基本資格情報（管理者ユーザー名およびパスワード）が未設定で、かつOAuth資格情報を設定していない場合（次のステップ）、プロビジョニング関連の操作で「無効なAPI資格情報」エラーが発生します。

  • ［Enable newest ShareFile REST API V3（最新のShareFile REST API V3を有効化）］機能（［Settings（設定）］→［Features（機能）］）を無効にすることで、基本資格情報（管理者のユーザー名とパスワード）ワークフローに戻すことができます。

  • 既に基本管理者資格情報を入力している場合、もう一度入力する必要はありません。そうでない場合、［Provisioning（プロビジョニング）］→［Integration（統合）］で入力できます。

  • ［Push Profile Update（プロフィールの更新をプッシュ）］は［First Name（名）］フィールドと［Last Name（姓）］フィールドでのみサポートされています。

  1. OktaでShareFile統合の［Provisioning（プロビジョニング）］タブを選択し、［Settings（設定）］で［Integration］を選択します。

  2. 以下を入力します。

    • OAuth Consumer Key（OAuthコンシューマーキー）：さきほど生成したクライアントIDを入力します（要件を参照）。

    • OAuth Consumer Secret（OAuthコンシューマーシークレット）：さきほど生成したクライアントシークレットを入力します（要件を参照）。

  3. ［Authenticate with ShareFile（ShareFileで認証）］をクリックします。新しいhttps://secure.sharefile.comウィンドウが開きます。

  4. アカウントのサブドメインを入力して［Continue（続行）］をクリックしたら、管理者のユーザー名とパスワードを入力します。

  5. ［Save（保存）］をクリックしてOAuthの設定を保存します。

これで、ShareFile統合が認証されました。
