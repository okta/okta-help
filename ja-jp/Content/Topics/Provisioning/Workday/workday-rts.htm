Workday Real Time Sync | Okta

概要

Workday Real Time Sync (RTS) では、OktaはWorkdayからユーザーの作成、更新、および退職イベントをリアルタイムに受信できます。Workdayでのユーザー変更は、直ちにOktaとその下流のアプリケーションに反映されます。

RTSはWorkdayからOktaへのアップデートをリアルタイムにトリガするために使用されます。従業員の突然の退職など、タイムリーな変更が重要な場合に使用されるべきです。Oktaにこのプロセスを開始するようトリガを送るために、Workdayで業務プロセスを構成する必要があります。RTSインポートには、ベース属性、将来（または将来でないが）有効な日付のカスタム属性が含まれています。

RTSを、1～2日のインターバルで実行するようスケジュールされたインポートと組み合わせて使用することを強く勧めます。その理由は、Workdayで一部の頻度の低いアクションはRTSをトリガせず、これらのアクションを調整するためにスケジュールされたインポートが必要であるためです。

前提条件

Workdayがプロファイル マスターとしてセットアップされている。手順については「Workday向けのプロビジョニングの構成」を参照してください。

機能

以下のWorkdayアクションはReal Time Syncでサポートされません。

  • 予備Eメール属性のアップデートはRTSをトリガしません。
  • 管理者ユーザー名属性のアップデートはRTSをトリガしません。
  • RTSは雇用開始日前のユーザーをOktaに追加しません。

周知事項

RTS非アクティブ化：

  • RTSはユーザーをリアルタイムでクエリする時と同じ非アクティブ化ルールに従います。非アクティブ化に関する情報はここで見つかります。

  • 即時非アクティブ化の理由が設定されている場合、ユーザーは同期時にすぐに非アクティブ化できます。詳細については、 を参照してください。

  • RTSと組み合わせてグループ名を変更することはお勧めしません。詳しくは を参照してください。

手順

Workdayのセットアップ

セットアップの構成は以下のとおりです。

A.    統合システムの作成
  1. Workdayに管理者としてログインします。
  2. 画面の左上隅にあるテキスト入力検索ボックスにCreate Integration Systemと入力します。
  3. 以下を入力します。
    • システム名：統合システムに必要な名前
    • テンプレート：テンプレートを使用して新規を選択し、リストからOkta-Workerを選択します
    • ［OK］をクリックします。

  4. ［Enable All Services（すべてのサービスを有効化）］を選択して［Enabled（有効化）］列の下のすべてのチェックボックスが選択されていることを確認します。

  5. 以下のエラーが表示される場合があります。次のセクションで設定を完成させるため、ここでは無視してください。

  6. 値を確定した後、統合システムのページが開きます。

B.    統合システムに統合属性を追加
  1. Integration System（統合システム）の横にある［Actions（アクション）］ボタンをクリックし、［Integration System（統合システム）］>［Configure Integration Attributes（統合属性の設定）］の順に進みます。

  2. Okta APIエンドポイントとOkta APIトークンでプラス符号（＋）をクリックし、それぞれに新規エントリを追加します。

OktaAPIエンドポイント

Use the bulleted elements in the following URL: https://<ENVIRONMENT>/api/v1/app/<Identity Provider ID>/activities.

  • Environment：例：acme.okta.com、mycompany.okta.com
  • Identity Provider ID: From the View Setup Instructions link under the Sign On tab for the Workday app. Use the value generated for Issuer in the Setup Instructions.

Okta APIトークンの取得

  1. From the Okta Dashboard page, navigate to Security > API.
  2. ［Create Token（トークンを作成）］をクリックしてトークンを作成し、関連する名前を入力します。
  3. トークンをコピーしてそれを上述したフォームで使用します。
C.    統合システムへサブスクリプションの追加
  1. Click the ellipsis (three dots) button next to the Integration System, then navigate to Integration System > Edit Subscriptions:

  2. Subscribe to specific Transaction Types（特定のトランザクションタイプに登録）の下で、必要なイベントのタイプごとに項目を選択します。（トランザクション タイプの仕様については  表3を参照してください）。

  3. ［External Endpoints（外部エンドポイント）］の下にあるマイナス符号（―）をクリックして 外部エンドポイントの設定を削除します。

  4. ［Add Launch Integration（起動する統合を追加）］をクリックし、下の 表1に示される値を追加します。

  5. 下のアラートが表示される場合があります。次のセクションで修正するため、ここでは無視できます。

  3. Select Hire for <tenant>. For example: Hire for Acme Inc.   Do not select the default business process.
  4. ［Edit Definition（定義の編集）］に進みます。

  5. 採用プロセスが完了した後に起動する新規ステップを追加する必要があります。［Complete（完了）］列が［Yes（はい）］となっている行で［Order（順序）］列の文字を見つけます。 （この例ではaに設定されています）。

  8. タイプで［Integration（統合）］を選択し、［OK］をクリックして保存します。これは、ビジネスプロセスのランディングページを返します。
