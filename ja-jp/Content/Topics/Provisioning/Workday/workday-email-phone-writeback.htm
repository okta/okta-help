Workdayの電子メールおよび電話の書き戻し

概要

メールと電話番号の属性をWorkdayに書き戻す機能を追加しました。

この新しいライトバック機能により、Okta、Active Directory、またはその他のプロファイルソースが電話番号とEメールのWorkday属性を更新できるようになります。現時点では、他のWorkday属性はサポートされていないことに注意してください。

電子メールと電話番号のWorkdayの書き戻しには、次の機能フラグが必要です。有効にする場合は、Oktaサポートにお問い合わせください。

  • ATTRIBUTE_LEVEL_MASTERING

前提条件

Workdayのプロビジョニングの説明に従って、Workdayのプロビジョニングを設定します。

手順

Workdayの設定

ライトバック機能を正常に機能させるには、以下にリストされている統合システムユーザー/システムグループに追加の必要な権限を付与してから、以下に説明するようにそれらの権限をアクティブ化する必要があります。

  1. ドメインセキュリティポリシーのアクセス許可。

    • 次のドメインについて、「Workday でのインテグレーター システムユーザーの作成」手順のa～e（手順6の一部）に従います。

      • 「労働者データ：連絡先情報の維持」–取得して配置

  2. ビジネスプロセスセキュリティポリシーのアクセス許可。

    • bp：contact changeを検索し、 ビジネス プロセス ポリシーを編集します。

    • で作成した統合システムユーザーのセキュリティグループを ［Initiating Action（開始するアクション）：Maintain Contact Information（連絡先情報の更新）（Webサービス）］に割り当てます。

  3. [WorkdayでのIntegratorシステムユーザーの作成] の手順7に従って、WorkdayでのIntegratorシステムユーザーの作成の要件に記載されている上記のセキュリティポリシーをアクティブにします。

Oktaセットアップ

  1. Oktaで、Workdayアプリを開き、 ［Provisioning（プロビジョニング）］ タブに移動します。

  2. 左側のパネルの SETTINGSの下にある To App ]を選択し、次に Workday Attribute Mappingsの下で、 Go to ProfileEditorを選択します。

  3. ここで、現在ライトバックでサポートされているWorkday属性を追加する必要があります。これは、Workdayから属性を検出し、関連するOkta属性にマッピングすることによって行われます。［Add Attribute（属性を追加）］をクリックします。

  4. ［Refresh Attribute（属性を更新）］をクリックします。

  5. 必要なWorkday属性を追加し、 ［保存］をクリックします。

    各属性には、以下で説明する3つの異なるデータ型があることに注意してください。この表では、例として属性 Work Email を使用しています。

    属性名タイプ必須/オプションコメントサンプル値
    EmailAddressData Work EmailAddress文字列必須Oktaユーザー属性にマップOktaCustomWorkEmail
    EmailAddressData Work Primaryブール値必須プッシュされた仕事用メールをプライマリとして設定する必要がある場合は、値をtrueに設定します。デフォルトはfalseです。true
    EmailAddressData Work Publicブール値必須プッシュされた仕事用メールを公開に設定する必要がある場合は、値をtrueに設定します。デフォルトはfalseです。False

    Workdayの連絡先情報には Primary（プライマリ） と Public（パブリック）の2つのパラメータがあります。

    Primary：このパラメーターは、連絡先情報のタイプがPrimaryであるかAdditionalであるかを定義します。

    Public：このパラメーターは、連絡先情報の可視性がパブリックかプライベートかを定義します。

    現在サポートされているWorkday属性のリストについては、この記事の最後にある 付録A を参照してください。

  6. 新しく追加された属性を関連するOkta属性にマップします。［Map Attributes（属性をマッピング）］をクリックします。

  7. 必ず ［Okta to Workday（OktaからWorkdayへ）］を選択し、以下に示すように属性をマップしてから、 ［Save Mappings（マッピングを保存）］をクリックしてください。

    8. 式言語関数の例

    注：参照については、 https://developer.okta.com/reference/okta_expression_language/#string-functions を参照してください。

    電話番号を解析するには：

    の場合user.mobilePhone = +1（415）123-4567 x 12345

    仕事用携帯電話AreaCode： String.substringBefore(String.substringAfter(user.mobilePhone, "(" ), ")" ) -> 415

    仕事用携帯電話InternationalPhoneCode：String.removeSpaces(String.substringBefore(String.substringAfter(user.mobilePhone, "+" ), "(" )) -> 1

    仕事用携帯電話電話番号：String.removeSpaces(String.substringBefore(String.substringAfter(user.mobilePhone, ")" ), "x" )) --> 123-4567

    仕事用携帯電話内線電話：Work Mobile Phone PhoneExtension: String.removeSpaces(String.substringAfter(user.mobilePhone, "x")) -> 12345

  8. 属性レベルのマスタリング

    典型的なWorkdayライトバックのユースケースでは、Workdayは全体的なプロファイルマスターとして設定されます。ただし、メールや電話番号をWorkdayに書き戻すには、Okta、Active Directory、またはその他のアプリを使用して、これらの特定の属性のプロファイルマスターとして属性レベルのマスタリングを構成する必要があります。次の例を参照してください。

    • ［Directory（ディレクトリ）］ >［Profile Editor（プロファイルエディター）］の順に進みます。

    • 左側のフィルターパネルで Okta を選択し、 プロファイルをクリックします。

    • ［Edit（編集）］を選択します。

    • 選択したプロファイルマスターを選択し、［プロファイルの保存］をクリックします。

    • 属性に移動し、 カスタム>フィルターを選択します。

    • ユーザー権限 と マスター優先度を選択します。

      以下のスクリーンショットでは、カスタム ユーザー権限属性 読み取り/書き込み 権限を付与し、 マスター優先度 を OKTAから継承するように設定しています。これは、この属性がOKTAによってマスタリングされることを意味します。

    • ［Save attribute（属性を保存）］ をクリックして、セットアップを完了します。

既知の問題

  • Workday v15の場合：Workday UIとは異なり、Workday APIには電話番号または電子メールの形式検証/ルールがありません（例：電話番号の形式は3桁+オプションのダッシュ（またはドット）+ 4桁である必要があります）。

    Workday v29 は、電話番号と電子メールのフォーマット検証/ルールを実行します 。

  • OktaがWorkdayプライマリ属性を設定すると、それをNULLに戻すことはできなくなります。これは、NULL値を許可しないWorkday検証ルールが原因です。
