エンドユーザー向けのダッシュボードタブの管理

カスタマイズ設定では、エンドユーザー向けにダッシュボードのタブを追加・削除したり、表示させたいアプリを選択することができます。


概要

  • ダッシュボードのタブは最大4つまで作成できます。
  • タブ間やタブ内でアプリをドラッグ＆ドロップすることができます。
  • 個々のエンドユーザーのダッシュボードをプレビューすると、組織で管理されているアプリがの割り当てと、そのアプリが配置されているタブが表示されます。
  • 構成されたダッシュボードを 新規の非アクティブなユーザーに適用するか、またはアクティブ化されたすべてのユーザーに適用するかを選択します。

警告

  • この機能を構成するには、管理者がスーパー管理者、組織管理者、モバイル管理者のいずれかの 権限を持っている必要があります。
  • この機能を使って、アプリの割り当てや削除をすることはできません。これらの操作については、［Application (アプリケーション)］ページを参照してください。


新規の非アクティブなエンドユーザーへのダッシュボードの適用

この手順を実行した後にアクティベートしたすべての新しいエンドユーザーには、Oktaへの初回サインイン時に、作成したタブと アプリ が表示されます。エンドユーザーには、割り当てられた組織の管理対象アプリのみが表示されます。これにより、エンドユーザーの最初の日の操作体験を管理することができます。いったんサインインすると、エンドユーザーはタブ内やタブ間でアプリを移動したり、自身のタブを作成して個人用アプリを追加したり、タブを削除したりすることができます。ヒント 新規でアクティブ化されていない異なるエンドユーザーに、異なるダッシュボードを適用することができます。例えば、新しいエンドユーザー1組に対してダッシュボードを構成し、それらのエンドユーザーをアクティベートすることができます。続いて、別の新しいエンドユーザーの1組のために別のダッシュボードを構成し、それらをアクティベートすることができます。アクティベートしてOktaにサインインした2組の新しいエンドユーザーには、異なるダッシュボードが表示されます。

  1. ［Settings (設定)］ > ［Customization (カスタマイズ)］に進みます。
  2. ［End-user dashboard (エンドユーザーダッシュボード)］をクリックします。
  3. ダッシュボードを構成します。以下のいずれかを行うことができます。
    • タブを作成する。+ 記号をクリックし、名前を入力してから［Save (保存)］をクリックします。タブは最大3つまで作成できます。
    • タブ名を変更する。タブの上にカーソルを置き、鉛筆アイコンをクリックします。
    • タブにアプリを追加する。アプリをタブにドラッグ＆ドロップしたり、現在のタブ内でアプリを移動させたりすることができます。
    • タブを削除する。まず、タブからすべてのアプリを削除してから、［Delete (削除)］をクリックします。
    • ユーザーのホームページをプレビューする。詳細は、こちらをご覧ください。
  4. ［Apply dashboard to new users (ダッシュボードを新規ユーザーに適用)］をクリックします。
  5. ユーザーアカウントのアクティベート。

アクティベートされたすべてのエンドユーザーへのダッシュボードの適用

  1. ［Settings (設定)］ > ［Customization (カスタマイズ)］に進みます。
  2. ［End-user dashboard (エンドユーザーダッシュボード)］をクリックします。
  3. ダッシュボードを構成します。以下のいずれかを行うことができます。
    • タブを作成する。+ 記号をクリックし、名前を入力してから［Save (保存)］をクリックします。追加のタブは最大3つまで作成できます。
    • タブ名を変更する。タブの上にカーソルを置き、鉛筆アイコンをクリックします。
    • タブにアプリを追加する。アプリを現在のタブ内、または別のタブにドラッグ＆ドロップすることができます。
    • タブを削除する。まず、タブからすべてのアプリを削除してから、［Delete (削除)］をクリックします。
    • エンドユーザーのホームページをプレビューする (下記参照)。
  4. ［Apply dashboard to all users (ダッシュボードをすべてのユーザーに適用)］をクリックします。

重要

  • ダッシュボードをすべてのエンドユーザーに適用すると、組織内でアクティベートされたすべてのエンドユーザーの現在のダッシュボード構成が上書きされ、アプリはすべて新しい場所に移動します。エンドユーザーの個人用アプリは、［My Apps (マイアプリ)］という新しいタブに統合されます。これらの変更は元に戻せません。
  • このオプションはエンドユーザーのダッシュボードの外観や構成を大きく変える可能性があるため、Oktaではエンドユーザーに表示される変更について、事前に通知することをお勧めします。
  • ダッシュボードの更新が完了するまでには、多少時間がかかります。 処理中はボタンが無効になり、ダッシュボードの編集ができなくなります。
  • 新しいダッシュボードが適用されたことを確認するために、Oktaではすべてのエンドユーザーにダッシュボードを適用した管理者に、メールを自動送信します。エラーが発生しているアプリがあれば、Oktaサポートに修正を依頼できるメールに、一意のIDが記載されます。

  • 管理者がダッシュボードをカスタマイズすると、エンドユーザーのホームページにバナーが表示されます。

エンドユーザーのダッシュボードをプレビューします。

ページ下部の［Preview (プレビュー)］をクリックすると、特定のユーザーのダッシュボードをプレビューすることができます。エンドユーザーのダッシュボードを表示することで、質問に答えたり、問題を解決したりすることができます。 このオプションは、ダッシュボードの適用 ...機能とは別に、いつでも使用することができます。

注

組織マネージドアプリのみが管理者に表示されます。管理者は、セルフサービスアプリやエンドユーザーの個人タブを表示することはできません。
