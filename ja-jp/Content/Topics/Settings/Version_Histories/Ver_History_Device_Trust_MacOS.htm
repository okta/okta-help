Device Trust for macOS Registration Taskバージョン履歴

このページでは、Okta Device Trust for macOSデバイスのRegistration Taskの現在 および過去のバージョンを記録しています。このページは、タスクの新しいバージョンがリリースされるたびに更新されます。登録タスクの一般提供バージョンは、Okta管理コンソールの［Downloads (ダウンロード)］ページ（［Settings (設定)］> ［Downloads (ダウンロード)］）からダウンロードできます。

このDevice Trust （デバイスの信頼）ソリューションの詳細については、Jamf Proの管理対象macOSデバイスへのOkta Device Trustの適用を参照してください。

現在のGAバージョン 1.3.1
現在のEAバージョン – –

バージョン

説明

リリース

1.3.2

このリリースには、内部コードのリファクタリングが含まれています。

GA 2021.07.0

1.3.1

このリリースには、以下が含まれています。

  • 以前のバージョンでは、Registration TaskはPython 2.7をベースにしていました。 1.3.1以降は、Registration TaskはPython 3をベースにしています。これは、macOS 10.15.xx（Catalina）以上に対応しています。

  • デバイスの登録が失敗するという既知の問題に対処しています。

GA 2021.06.0

1.2.1

このリリースには、以下が含まれています。

  • これまでのmacOSでは、Device Trust（デバイスの信頼）の登録時に、何らかの理由（破損している、または欠落しているなど）でキーチェーンが使用できない場合、エンドユーザーにデフォルトのキーチェーンをリセットするよう求めていました。 バージョン1.2.1では、デフォルトのキーチェーンが利用できない場合、登録プロセスを停止し、その結果、ユーザーにキーチェーンのリセットを求めるプロンプトが表示されないようになっています。
  • 以前のバージョンでは、Registration Taskのアップデート処理において、何らかの理由でアップデート処理自体が失敗した場合に、キー素材（クライアント証明書、Oktaキーチェーン、キーチェーンパスワード）や更新タスクが意図せず削除され、現在登録されているデバイスが登録解除されることがありました。 バージョン1.2.1からは、アップデート処理自体が失敗しても、これらの項目が削除されなくなりました。
  • 管理者は、以下のクエリを使用して、デバイスにインストールされているRegistration Taskのバージョンを判別できるようになりました。

    • python ~/Library/Okta/okta_device_trust.py version

  • 内部修正とパフォーマンスの向上
