Integrate Okta with your third-party MDM provider

Configure your MDM provider to manage Okta Verify and to install it on end user devices.

Before you begin

At minimum, your MDM must support managed app configuration. For best results, integrate with an MDM that can:

  • Use the managed app configuration to configure the key-value pair.
  • Set Okta Verify to install on end-user devices silently and automatically when they enroll in your chosen MDM.

    • If you are configuring your MDM to deploy Okta Verify to Android devices, make sure that Okta Verify is installed in the work profile of the device.

Start this task

This task describes MDM integration in general and provides some configuration tips for some MDM solutions. For specifics instructions for your chosen MDM solution, please see their documentation.

  1. Configure your MDM provider to manage Okta Verify and to install it on end user devices if it isn't installed already.
  2. Configure the key/value pair through your MDM provider's managed app configuration as described in their documentation:
    • Domain: Enter the URL of your Okta org
    • Key: managementHint
    • Value: Use the Secret Key value that you saved in Configure Device Management for mobile devices.

      • The Key/Value pair is case sensitive.

    Configuration tips

    Below are tips for configuring the key/value pair in select MDM applications. As MDM configurations may change without notice, Okta recommends that you always consult your MDM's documentation for the most up-to-date information.

    VMware

    Android

    For how to add, assign, and manage Okta Verify with Workspace ONE UEM, perform the procedures as described in the following Workspace ONE UEM documentation:

    Add Assignments and Exclusions to your Android Applications

    Make sure to configure these settings as shown:

    • App Delivery Method: Automatic
    • Managed Access: Enable

    iOS

    In Add Application

    • Platform: Apple iOS
    • Source: Search App Store
    • Name: Enter the name of the app. A search finds the app after you click Next.
    • Details: Keep the defaults, then click Save & Assign

    In Assignment

    • Distribution:
      • Name: Enter a name.
      • Assignment Groups: Specify a group(s).
      • App Delivery Method: Auto
    • Restrictions
      • Make App MDM Managed if User Installed: Enable
    • Application Configuration:
      • Managed Access: Enable
      • Send Configuration: Enable
      • Click +Add and configure settings:
        • Configuration Key: managementHint
        • Value Type: String
        • Configuration Value: Enter the Secret Key that you generated in the previous procedure STEP 2 – Enable Management Attestation for your org.

    Microsoft Intune (Android only)

    For how to manage Okta Verify with Microsoft Intune for Android devices, perform the procedures as described in the Microsoft Intune document Add app configuration policies for managed Android Enterprise devices.

    • Device enrollment type: Managed devices
    • Associated App: Okta Verify
    • Configuration settings format: Use configuration designer
    • Username (string): Enter your username for your Okta org
