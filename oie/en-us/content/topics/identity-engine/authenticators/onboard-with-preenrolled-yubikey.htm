Onboard users with pre-enrolled YubiKey

Limited Early Access release

This configuration ensures that new users sign in with a phishing-resistant hardware security key from the day they join your organization. This phishing-resistant onboarding configuration is created using the FIDO2 (WebAuthn) authenticator and Okta policies. The setup ensures that the users can only use WebAuthn when signing in to Okta or accessing your company resources. Part of this configuration is automated using a pre-designed Okta Workflows template.

When a new user is added to the specified group in Okta, Yubico receives a YubiKey shipment request for them through Okta Workflows. After the request is fulfilled and the credential is activated in Okta, a YubiKey PIN is generated and emailed to the user. Later, when you activate the user in Okta, they receive a welcome email with sign-in information. Users can then use this information and the PIN on their first day to sign in securely using YubiKey.

Prerequisites

Onboarding with pre-enrolled YubiKey is available for app access only. Ensure that you've enabled the following Okta features:

  • Adaptive MFA
  • Okta Workflows
  • Okta Workflows template for Pre-enrolled YubiKey. Contact Okta Support for this template.

YubiKeys can only be shipped to US addresses. You need the following from Yubico:

  • YubiEnterprise Subscription
  • YubiEnterprise Delivery
  • Your custom product IDs and subscription IDs for YubiKey 5 NFC and YubiKey 5C NFC. Contact Yubico Support for these values.

Configure phishing-resistant onboarding flow

This procedure includes the following tasks:

  1. Create groups for YubiKey users
  2. Configure FIDO2 (WebAuthn) as authenticator
  3. Configure Okta policies
  4. Set up Yubico-Okta flow in Okta Workflows

Create groups for YubiKey users

Create two empty groups for the YubiKey users:

  1. In the Admin Console, go to DirectoryGroups.

  2. Click Add group.

  3. Create two groups, and name them YubiKey 5 NFC and YubiKey 5C NFC.

    Don't change the group names. These names are included in the Okta Workflows templates, therefore they must be used verbatim.

  4. Click Save.

Don't add any users to these groups yet. You'll add the users later when onboarding them.

Configure FIDO2 (WebAuthn) as authenticator

  1. In the Admin Console, go to SecurityAuthenticators.
  2. On the Setup tab, click Add Authenticator.
  3. Click Add on the FIDO2 (WebAuthn) tile.
  4. Set User verification to Preferred.
  5. Click Add.

Add another phishing-resistant authenticator as a backup (like Okta FastPass), so that users can access their Okta account if they lose their YubiKey. See Okta FastPass.

Configure Okta policies

Add these Okta policies to ensure that the users authenticate with YubiKey only.

Global session policy

Create a global session policy so that users sign in to Okta with YubiKey.

  1. In the Admin Console, go to SecurityGlobal Session Policy.

  2. Enter a policy name and description.

  3. Assign it to the YubiKey user groups: YubiKey 5 NFC and YubiKey 5C NFC.

  4. Click Create policy and add rule. The Add Rule window opens.

  5. Enter a Rule name and configure the rule with the following options:

    Field Option
    Establish the user session with Any factor used to meet the Authentication Policy requirements
    Multifactor authentication (MFA) Required
    Users will be prompted for MFA At every sign in

  6. Click Create rule.

  7. Move this policy to the top of the list (priority 1).

Authenticator enrollment policy

Create an authenticator enrollment policy to disable all authenticators except FIDO2 (WebAuthn) for the YubiKey group users.

  1. In the Admin Console, go to SecurityAuthenticators.

  2. Open the Enrollment tab. Click Add policy. Enter a policy name and description.

  3. Assign it to the YubiKey user groups: YubiKey 5 NFC and YubiKey 5C NFC.

  4. Set the following options:

    Field Option
    FIDO2 (WebAuthn) Required
    Allowed authenticators Any WebAuthn authenticators

    Okta Verify

    Required or Optional
    All other eligible authenticators Disabled

  5. Click Create policy.The Add Rule window opens.

  6. Enter a Rule name and configure the rule with the following options:

    Field Option
    User is accessing

    Okta

    Applications: Any application that supports MFA enrollment

    Enrollment is

    Allowed if required authenticators are missing

  7. Click Create rule.

  8. Move this policy to the top of the list (priority 1).

Authentication policy

Add an authentication policy and assign it to the Okta End-User Dashboard and other apps the YubiKey users will access. This ensures they use only YubiKey when accessing the dashboard and apps.

  1. In the Admin Console, go to SecurityAuthentication Policies.

  2. Click Add policy. Enter a policy name and description.
  3. On the Rules tab, click Add rule. The Add Rule window opens.

  4. Enter a Rule name and configure the rule with the following options:

    Field Option
    User's group membership includes At least one of the following groups: YubiKey 5 NFC, YubiKey 5C NFC
    User must authenticate with Any 2 factor types
    Possession factor constraints are Phishing resistant
  5. Click Save. Ensure this rule is at Priority 1.
  6. On the Applications tab, click Add app.
  7. Add the Okta Dashboard app to the policy. Search for other apps you want to assign to the YubiKey users and add them to the policy.
  8. Click Close.

Password policy

Create a password policy to disable self-service password change, password reset, and account unlock for YubiKey users.

  1. In the Admin Console, go to SecurityAuthenticators.

  2. On the Setup tab, find the Password authenticator, and click ActionsEdit. The Password page opens.
  3. Click Add new password policy. Enter a policy name and description.

  4. Assign it to the YubiKey user groups: YubiKey 5 NFC and YubiKey 5C NFC.

  5. Keep the default options. Click Create policy. The Add Rule window opens.

  6. Enter a Rule name and configure the rule with the following options:

    Field Option
    Users can perform self-service

    Clear all options for this condition:

    Password change

    Password reset

    Unlock account

  7. Click Create rule.

  8. Move this policy to the top of the list (priority 1).

Set up Yubico-Okta flow in Okta Workflows

After you configure the policies in the Admin Console, go to WorkflowOkta Workflows Console to set up a flow between your Yubico and Okta orgs.

Create a connection from the current Okta org

  1. In the Okta Workflows Console, go to ConnectionsNew Connection. All available connectors appear.

  2. Select the Okta connector.

  3. In the New Connection window, enter the Connection Nickname. This is the display name you want to appear in your list of connections.

  4. Enter the following values:

    Field Option
    Domain Enter your Okta org domain without https://

    For example, company.okta.com

    If your org uses a custom domain, enter the custom domain.

    Client ID and Client Secret

    		 To find these values, follow these steps:

    1. In the Admin Console, go to ApplicationsApplications.

    2. Go to Okta Workflows OAuthSign on tab.

    3. Copy the values for Client ID and Client Secret and paste them in the New Connection window.

  5. Click Create.

The new connection appears in the Connections list.

Create a connection from the Yubico org

  1. Generate an API token in your Yubico org. Go to your YubiEnterprise Console ProfileGenerate new API token.
  2. Make a copy of the token and store it in a secure location.

  3. In the Okta Workflows Console, go to ConnectionsNew Connection. All available connectors appear.

  4. Select the Yubico connector.

  5. In the New Connection window, enter the Connection Nickname. This is the display name you want to appear in your list of connections.

  6. In API Secret, paste the API token from Yubico.

  7. Click Create.

The new connection appears in the Connections list.

Set up the Okta Workflows template for pre-enrolled YubiKey

You receive the flow template in an email sent by Okta Support.

  1. Go to Okta Workflows ConsoleFlows.
  2. In Folders, create a folder.
  3. Click the three-dot menu for the folder where you want to import this template and click Import.
  4. Upload the flow template from your computer and click OK.

The flow template for pre-enrolled YubiKey is added to your Okta Workflows.

Activate Okta and Yubico connection in the flows

The imported template folder includes three Workflows: Create shipment, Call enrollment API, and Call activate API.

For each flow, activate the Okta and Yubico connections:

  1. Open the flow and find the cards with an Okta or Yubico connection.
  2. Click Choose connection.
  3. Select your Okta org or Yubico org. Click Save. A green check mark appears next to Okta and Yubico, indicating the connection is successfully established.
  4. Repeat for all other cards and the flows in the template.
  5. In the folder, turn on each flow by toggling the on/off switch.

Update Create shipment flow

Update the Create shipment flow with your Yubico product IDs.

  1. Open the Create shipment flow.

  2. In the Edit Conditions card, open the dropdown menu and update the following fields.

    Contact Yubico Support for these values.

    Condition Field Option
    If product_id Your custom product ID for the YubiKey 5 NFC
    inventory_product_id Your subscription ID
    Else if product_id Your custom product ID for the YubiKey 5C NFC
    inventory_product_id Your subscription ID

  3. Click Save.

You've now configured the flow for onboarding users with phishing-resistant YubiKey.

Onboard new users

After you’ve set up the phishing-resistant onboarding flow, you can onboard new users. This procedure includes the following tasks:

  1. Create users in Okta
  2. Request pre-enrolled YubiKey
  3. Activate users in Okta

Create users in Okta

  1. In the Admin Console, go to DirectoryPeople.

  2. Click Add person.

  3. Enter the user details: First name, Last name, Username, Primary email, and Secondary email. Don’t assign the user to a group.

    The YubiKey PIN and welcome email are sent to the user’s secondary email. Make sure they have access to this email before activation, for example, a personal email.

  4. In the Activation dropdown menu, select Activate later. This creates the user in the Staged status.
  5. Click Save. Or click Save and Add Another to add more users.
  6. On the People page, go to the StagedUserProfile Edit.

  7. Enter these user details: Primary phone, Street address, City, State, Zip code, Country code, Organization. These details are required for shipping YubiKey.

  8. Click Save.

Request pre-enrolled YubiKey

When you add a user to the YubiKey group, the YubiKey template in Okta Workflows runs automatically. Yubico receives a request to ship a pre-enrolled YubiKey.

  1. In the Admin Console, go to DirectoryPeople.

  2. Find the Staged user and click their name. Their profile opens.
  3. On the Profile page, go to Groups. Add the user to either the YubiKey 5C NFC or the YubiKey 5 NFC group. This depends on the type of YubiKey you want to ship to them.

After the request is fulfilled and the credential is activated in Okta, a YubiKey PIN is generated and sent to the user’s secondary email address.

Activate users in Okta

When it’s time to activate the user, check the Yubico Console to ensure that a YubiKey was delivered to them. Then, change their status in Okta.

  1. In the Admin Console, go to DirectoryPeople.

  2. Next to the user, click Activate in the Status column, then in the Activate Person dialog box, click Activate User.

A welcome email containing the sign-in URL is sent to the user’s secondary email address.

You can customize the content of this email in CustomizationsBrandsyour brandEmailsUser Bootstrapped Activationemail template.

End-user experience

After the user receives both emails, they use the URL to sign in to Okta. After entering their username, they're prompted to insert their YubiKey. They enter the PIN that they received in the email, and they finish setting up their YubiKey. When they finish the setup, they're authenticated into Okta.

Advise users to add another phishing-resistant authenticator as a backup (for example Okta FastPass) so they can access their Okta account if they lose their YubiKey.
