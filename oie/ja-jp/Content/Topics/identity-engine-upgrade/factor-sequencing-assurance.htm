要素シーケンシングから保証モデルへアップグレードする

Okta Classic Engineでは、要素シーケンシングにより、ユーザーが標準パスワードの代わりに一連の構成された多要素認証（MFA）要素を使用して認証できます。Okta Identity Engineでは、要素シーケンシングが保証ベースのサインオンポリシー設定に置き換えられ、認証ポリシーを構成する際の柔軟性が高まっています。これにより、汎用的なソリューションを適用する代わりに、あるアプリにはパスワードなしの認証を構成し、別のアプリにはMFAを構成することができるようになりました。

orgにアクティブな要素シーケンシングチェーンがあるかどうかを判断する

  1. 管理コンソールで、［Security（セキュリティ）］ > ［Authentication（認証）］に移動します。

  2. ［Sign On（サインオン）］タブをクリックします。

  3. Okta Sign-on Policyを選択します。

  4. リスト内のいずれかのルールの鉛筆アイコンをクリックします。

  5. ［Edit Rule（ルールを編集）］ダイアログで、［Authentication（認証）］の選択内容を確認します。

    • ［Password / Any IdP（パスワード/任意のIdP）］または［Password / Any IdP + Any factor（パスワード/任意のIdP + 任意の要素）］：アクティブな要素シーケンシングチェーンはありません。「Okta Identity Engineにアップグレードして識別子先行フローを維持する」に進みます。

    • ［Factor Sequence（要素シーケンス）］：アクティブな要素シーケンスチェーンがあります。「要素シーケンスをOkta Classic EngineからOkta Identity Engineの保証モデルにアップグレードする」に進みます。

Okta Identity Engineにアップグレードして識別子先行フローを維持する

アクティブなチェーンはありませんが、要素シーケンシング機能によりユーザーに識別子先行のエクスペリエンスが提供されます。アップグレードする前に、要素シーケンシングを無効にするようOktaアカウントチームに依頼してください。

  1. アップグレード後、［Directories（ディレクトリ）］>［Groups（グループ）］に移動します。

  2. 空のグループを作成し、Testという名前を付けます。

  3. Okta Admin Consoleで、［Security（セキュリティ）］>［Global Session Policy（グローバルセッションポリシー）］に移動します。

  1. グローバルセッションポリシーを作成し、作成したグループに割り当てます。

  2. ［Global Session Policy（グローバルセッションポリシー）］ページで、作成したポリシーを選択し、［Add Rule（ルールを追加）］をクリックします。

  1. And［Primary factor is（プライマリ要素）］セクションで、［Password / IDP / any factor allowed by app sign on rules（パスワード/IDP/アプリのサインオンルールで許可された任意の要素）］オプションを選択します。

  2. ［Create Rule（ルールを作成）］をクリックします。

要素シーケンスをOkta Classic EngineからOkta Identity Engineの保証モデルにアップグレードする

Oktaでは、本番環境に変更をデプロイする前に、本番環境を反映したプレビュー環境でこの手順をテストすることをお勧めします。

  1. 新しい保証ベースのモデルがニーズを満たしているかどうかを評価します。

  2. Okta Identity Engineで、Okta Classic Engineの要素シーケンシングルールで提供されているものと同様のセキュリティの結果と保護を提供する、新しい認証ポリシーを設計します。

  3. アップグレードを実行するために必要な時間の間、本番環境で現在のOkta Classic Engineの要素シーケンシングルールをオフにできるかどうかを判断します。

  4. 要素シーケンシングを使用するすべてのOkta Sign-on Policyルールを削除します。または、それらを参照用に保持するには、［Factor Sequence（要素シーケンス）］の代わりに［Password / Any IDP（パスワード/任意のIDP）］または［Password / IDP + Any factor（パスワード/IDP + 任意の要素）］を選択します。

  5. Oktaアカウントチームに連絡して、要素シーケンシングを無効にするよう依頼します。

  6. Oktaアカウントチームに、orgをOkta Identity Engineにアップグレードするよう依頼します。

  7. すべてのアプリの認証ポリシーを構成してアクティブにします。

  8. シナリオをテストします。
