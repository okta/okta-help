Oktaユーザー・プロファイルにカスタム属性を追加する

カスタム属性をOktaユーザー・プロファイルに追加して、基本属性では利用できない追加の属性を定義します。

空のままのユーザー属性値フィールドは""または空の文字列として処理され、プロファイルに""として保存されます。search=(profile.<propertyName> pr)を使用すると、空の文字列を持つユーザー・プロファイルはNULLではなく値を含むため返されます。属性をNULLとして設定するには、Okta APIを使用する必要があります。「Okta開発者用ドキュメント」を参照してください。

  1. 管理コンソールで、[ディレクトリー] > に移動します [プロファイル・エディター]
  2. [フィルター]リストで[Okta]をクリックします。
  3. [Oktaユーザー]（デフォルト）の[プロファイル]をクリックします。[プロファイル]が使用できない場合は、[ユーザー]（デフォルト）をクリックします。
  4. [属性を追加]をクリックします。
  5. 以下のフィールドに入力します。
    • [データ型]：次のデータ型のいずれかを選択します。
      • [文字列]：0個以上のUnicode文字（文字、数字、句読点）のチェーン
      • [数値]：Javaの64ビットDouble形式の浮動小数点10進数。「プラットフォーム仕様」をご覧ください。
      • [ブール値]：true、false、またはnullのデータ値を格納します
      • [整数]：64ビットJavaのLong形式の整数
      • [文字列配列]：文字列の順次コレクション
      • [数値配列]：数値の順次コレクション
      • [整数配列]：整数の順次コレクション
      • [国コード]：ユーザーの出身国を表すコード
      • [言語コード]：ユーザーの言語を表すコード
      • [リンクされたオブジェクト]：別の属性に対するユーザーの関係を表すコード
    • [表示名]： UIに表示される人間が読み取れるラベル
    • [変数名]：マッピングで参照可能な属性の名前
    • [説明]：属性の説明
  6. 任意：以下のフィールドに入力します。
    • [列挙型]： 列挙された値のリストを定義するには、このチェック・ボックスをオンにします。ブール値を除くすべてのデータ型をサポートします。
      • [属性メンバー]： 表示名と値を入力します。たとえば、小、中、大などです。
      • [属性の長さ]： 値フィールドの最小および最大の長さを入力します。
    • [必須属性]：属性が必須であることを示すには、このチェック・ボックスをオンにします。
    • [機密データ]： このオプションを選択するにはOkta Super管理者権限が必要です。チェック・ボックスをオンにして属性を機密として識別し、ユーザーとすべてのOkta管理者に対してデータを非表示にします。

    Oktaでは、アプリケーションのユーザー名に使用される属性を読み取り専用または非表示にすることを推奨しています。

  7. [保存]または[保存してほかにも追加]をクリックすると、別のカスタム属性が追加されます。
