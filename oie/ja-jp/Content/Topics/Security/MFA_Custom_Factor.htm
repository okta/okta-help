カスタムIdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>

IDプロバイダー（IdP）要素認証により、管理者は信頼できるOIDCまたはSAML IDプロバイダーによる認証を追加の検証として有効にできます。構成すると、エンドユーザーには追加検証用に信頼できるIdPを使用するオプションが表示され、検証に使用するIdPにリダイレクトされます。この検証により、認証がパスワード以外の<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>（Okta Verifyなど）に置き換えられます。

IdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>が有効になり、<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">MFA</MadCap:conditionalText>登録ポリシーに追加されると、Oktaにサインインするユーザーはそれを使用してIDを確認できます。エンドユーザーは、認証のためにIDプロバイダーに転送され、検証が成功するとOktaにリダイレクトされます。

この機能を使用すると、次のことが可能になります。

  • 既存のSAMLベースまたはOIDCベースのIdP認証用にカスタムIdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>を追加する。
  • Okta Admin Consoleからカスタム<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>を有効または無効にする。
  • カスタム<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>プロバイダーとして使用する既存のSAML 2.0 IdPまたはOIDC IdPをリンクする。

開始する前に

  • Oktaへの管理者アクセスは、目的のカスタム<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>を登録して構成するために必要です。
  • 既存のIDプロバイダーを追加のステップアップ認証プロバイダーとして使用するには、そのプロバイダーが使用可能である必要があります。

SAMLおよびOIDCのクレーム

Oktaで想定されているSAMLおよびOIDCのクレームは次のとおりです。

  • SAMLレスポンスの場合、subjectNameIdクレームがOktaユーザー名にマッピングされます。
  • OIDCレスポンスの場合、preferred_usernameクレームがOktaユーザー名にマッピングされます。

カスタムIdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>の構成

カスタムIdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>を設定するには、主に次の2つの手順があります。

  1. MFA用にIdPを追加する。
  2. IdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>を有効にする。

ステップ1：MFA用にIdPを追加する

  1. MFA用にSAML IDプロバイダーを作成する方法の詳細については、「IDプロバイダー」を参照してください。このワークフローは、［Identify Providers（IDプロバイダー）］>［Configure Inbound SAML（インバウンドSAMLの構成）］>［Workflow（ワークフロー）］>［Part 1 – Add a SAML Identity Provider（パート1 - SAML IDプロバイダーの追加）］にあります。
  2. IdPの用途を「 係数のみ」として、IdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>を作成します。JIT設定はサポートされていないため、「SSOのみ」と設定されたIdPはカスタムIdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>には使用できません。
  3. 構成が完了したら、Oktaコンソールで［Security（セキュリティ）］>［Identity Providers（IDプロバイダー）］の順に進み、IDプロバイダーを追加します。

OpenID Connect

  • OpenID Connectに関する一般的な情報については、「汎用OpenID Connect」を参照してください。
  • OIDC IDプロバイダーの設定方法については、「汎用OpenID Connect IDプロバイダー」を参照してください。
  • 構成が完了したら、Oktaコンソールで［Security（セキュリティ）］>［Identity Providers（IDプロバイダー）］の順に進み、IDプロバイダーを追加します。

ステップ2：カスタムIdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>を有効にする

  1. Okta Admin Consoleで、［Security（セキュリティ）］>［Authenticators（オーセンティケーター）］に移動します。
  2. <MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">［Add Authenticator（オーセンティケーターを追加）］をクリックします。</MadCap:conditionalText>
  3. <MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">カスタムIdPの下で、［Add（追加）］をクリックします。</MadCap:conditionalText>
  4. メニューから［Identity Provider（IDプロバイダー）］を選択します。選択するIDプロバイダーは事前に構成しておく必要があることに注意してください。
  5. IDプロバイダーを追加したら、［Save（保存）］をクリックして設定を保存します。
  6. カスタム<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>のステータスを［Active（アクティブ）］に設定し、エンドユーザーに対して有効にするか、［Inactive（非アクティブ）］に設定して無効にします。

カスタム<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>がアクティブになったら、［Authenticator（オーセンティケーター）］> ［Enrollment（登録）］に移動し、orgの<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">MFA</MadCap:conditionalText>登録ポリシーにIdP要素を追加します。

エンドユーザーエクスペリエンス

  • 管理者がカスタム<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>を追加して有効にすると、エンドユーザーは、次回サインインしたときにカスタム認証を設定するように求められます。
  • エンドユーザーの<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>の設定が正常に完了すると、［Settings（設定）］>［Extra Verification（追加認証）］の下に構成済み<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>として表示されます。
  • エンドユーザーが<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>の使用をトリガーすると、<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>は5分後にタイムアウトになります。その後、再度<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>を使用してトリガーする必要があります。

制限事項

  • カスタムIdP<MadCap:conditionalText data-mc-conditions="MultiProdPubish.OIE">オーセンティケーター</MadCap:conditionalText>では、IDプロバイダーとしてのMicrosoft Azure Active Directory（AD）の使用はサポートされていません。Microsoft Azure ADをIDプロバイダーとして使用するには、「Azure Active DirectoryをIDプロバイダーにする」を参照してください。
