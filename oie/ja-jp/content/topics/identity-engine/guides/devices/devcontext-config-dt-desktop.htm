デスクトップデバイスでDevice Trustを構成する

macOSおよびWindowsコンピューターでDevice Trustをデプロイする方法を説明します。

Device Trustを利用するアプリレベルのポリシーを作成する前に、OktaとEMMソリューションを構成する必要があります。

注：Device Trustを使用してmacOSユーザーのパスワードエクスペリエンス（Okta FastPass）を有効にする場合は、EMMソリューションでApple拡張機能SSOを構成する必要があります。「SafariおよびmacOSデバイスで管理されているネイティブアプリ向けに拡張機能SSOを構成する」を参照してください。

認証局を選択する

WindowsおよびmacOSのデバイス管理には、対象デバイスにクライアント証明書を発行できる認証局（CA）が必要です。Device Trustでは、これらのクライアント証明書が使用されて、デバイスが管理対象かどうかが判断されます。アプリケーションサインオンポリシーでは、アプリケーションへのアクセス権が付与または拒否されたり、デバイスの状態に基づいて追加の認証要素を求めるプロンプトが表示されたりします。

OktaをCAとして使用するか、独自の既存CAがすでにある場合はそれを使用することができます。

デバイスコンテキスト向けに独自の認証局を提供する

独自の認証局（CA）を使用するには、Oktaから提供されたクライアント証明書を対象デバイスに配布するために、EMMソリューションと統合されたPKIインフラストラクチャーをご使用の環境に用意する必要があります。EMMソフトウェアでは、証明書の配布に加えて、有効期限が切れる前の証明書の更新と、デバイスが管理対象でなくなった際のEMMサーバーおよび管理対象デバイスからの証明書の取り消しも行います。

既存のEMMソリューションによって管理されるデバイスに加えて、Oktaでは、既存のActive Directory証明書サービス（ADCS）インフラストラクチャーによってデプロイされた証明書を持つデバイスを管理できます。この場合、デバイスには、Oktaでセットアップされたものと同じCAからデプロイされた証明書が必要です。

この手順を開始する

タスク1：SCEPペイロードを構成する

タスク2：Oktaで管理証明を構成して、証明書をアップロードする

タスク1：SCEPペイロードを構成する

SCEPプロファイルが［DEVICE（デバイス）］レベルではなく、［USER（ユーザー）］レベルをターゲットにしていることを確認してください。これにより、証明書がログインキーチェーンにデプロイされて、Okta Verifyにアクセスできるようになります。SCEPポリシーにはユーザーコンテキストが必要です。各ユーザーが別々のorgに属している場合に限り、複数のユーザーが同一のデバイスを使用できます。登録されたユーザーはEMMソリューションによって管理され、証明書を所有している必要があります。

次の設定を使用してSCEPペイロードを構成します。

キー タイプ 値
KeyUsage 整数 ［signing（署名）］に設定して、Oktaサーバーから送信されたNonceにOkta Verifyで署名できるようにします。
AllowAllAppAccess ブール値 ［true（真）］に設定すると、Okta Verifyでユーザーにサインインを求めずにリクエストに署名できるようになります。それ以外の場合、ユーザーはOkta Verifyによるキーへのアクセスを許可するよう求められます。
KeysExtractable ブール値 別のデバイスに簡単にコピーできないように、［false（偽）］に設定します。

タスク2：Oktaで管理証明を構成して、証明書をアップロードする

  1. Okta Admin Consoleで、［Security（セキュリティ）］>［Device integrations（デバイス統合）］に移動します。
  2. ［Endpoint Management（エンドポイント管理）］ページで、［Add platform（プラットフォームを追加）］をクリックします。
  3. ［Desktop (Windows and macOS only)（デスクトップ（WindowsおよびmacOSのみ））］を選択して、［Next（次へ）］をクリックします。
  4. ［Certiﬁcate authority（認証局）］で、［Use my own certiﬁcate authority（独自の認証局を使用する）］を選択し、［Save（保存）］をクリックします。
  5. ［Save（保存）］をクリックします。
  6. ［Certiﬁcate authority（認証局）］ページで、［Add certiﬁcate authority（認証局を追加）］をクリックします。
  7. ［Add certiﬁcate authority（認証局を追加）］ダイアログボックスで、クライアント証明書の発行に使用する中間CAを参照します。該当する発行者が複数の場合は、すべてを一度にアップロードしてください。
    • 注：Oktaでは、PKCS#7、PKCS#12、PFX証明書の形式がサポートされていません。
    • 証明書は自動的にアップロードされます。正常にアップロードされるとメッセージが表示されます。詳細を表示するには、［View root certiﬁcate chain details（ルート証明書チェーンの詳細を表示）］をクリックします。
  8. ［Close（閉じる）］をクリックします。

OktaをCAとして使用する

EMMソリューションでは、Simple Certiﬁcate Enrollment Protocol（SCEP）が使用されて、管理対象デバイスに証明書が発行されます。OktaをCAとして構成する場合は、SCEPチャレンジタイプを静的、動的、または委任に設定できます。一部のEMMソリューション（Jamf Proなど）は複数のチャレンジタイプをサポートしていますが、1つのみをサポートしているものもあります。OktaをCAとして構成する前に、EMMアプリがサポートまたは推奨するチャレンジタイプを決定します。

macOSデバイスでJamf Proを使用した静的SCEPチャレンジを使用する

  1. 管理証明を構成し、SCEP URLを生成する
  2. 静的SCEPプロファイルを作成する

管理証明を構成し、SCEP URLと秘密鍵を生成する

注：この手順は、Apple SCEP EMMペイロードのプッシュをサポートするすべてのEMMソリューションに適用されます。

  1. Okta Admin Consoleで、［Security（セキュリティ）］>［Device integrations（デバイス統合）］に移動します。
  2. ［Endpoint management（エンドポイント管理）］ページで、［Add platform（プラットフォームを追加）］をクリックします。

    注：同じタイプのプラットフォームに複数の構成を追加する場合は、「デバイスの既知の問題」を参照してください。

  3. ［Desktop (Windows and macOS only)（デスクトップ（WindowsおよびmacOSのみ））］を選択して、［Next（次へ）］をクリックします。
  4. ［Add device management platform（デバイス管理プラットフォームを追加）］ページで、次の設定を構成します。
    • ［Certiﬁcate authority（認証局）］：［Use Okta as certiﬁcate authority（Oktaを認証局として使用する）］を選択します。
    • ［SCEP URL challenge type（SCEP URLチャレンジタイプ）］：［Static SCEP URL（静的SCEP URL）］を選択します。
    • ［Generate（生成）］をクリックします。
    • ［SCEP URL］：値をコピーして保存します。この値は後で必要になります。
    • ［Secret key（秘密鍵）］：値をコピーして保存します。この値は後で必要になります

      • 注：SCEP URLと秘密鍵は安全な場所に保存してください。Okta Admin Consoleに表示されるのは今回だけです。

  5. ［Save（保存）］をクリックします。

静的SCEPプロファイルを作成する

  1. Jamf Proで、［Computers（コンピューター）］>［Conﬁguration Proﬁles（構成プロファイル）］に移動します。
  2. ［+New（+新規作成）］をクリックします。
  3. ［Options（オプション）］>［General（一般）］に移動します。
  4. ［General proﬁle（一般プロファイル）］ページで、次の値を入力します。
    • ［Name（名前）］：プロファイルの名前を入力します。
    • ［Description（説明）］：任意。プロファイルの説明を入力します。
    • ［Level（レベル）］：［User Level（ユーザーレベル）］を選択します。
  5. ［Options（オプション）］>［SCEP］に移動します。
  6. ［Conﬁgure（構成）］をクリックします。
  7. ［SCEP proﬁle（SCEPプロファイル）］ページで、次の値を入力して［Save（保存）］をクリックします。
    • ［URL］：上記のステップ6bで保存したSCEP URLを入力します。
    • ［Name（名前）］：SCEPプロファイルの名前を入力します。
    • ［Subject（件名）］：件名を入力します。

      証明書がOktaへのデバイス管理シグナルとして使用されることを示す名前を選択してください。ベストプラクティスとして、Jamf Proが提供するプロファイル変数を含めてデバイスID（UDID）を含めることもできます。サポートされている変数のリストについては、Jamf Proのドキュメント「Payload Variables for ComputerConﬁguration Proﬁles」を参照してください。

    • ［Challenge type（チャレンジタイプ）］：静的
    • ［Challenge（チャレンジ）］：ステップ6eで生成した秘密鍵をコピーして貼り付けます。
    • ［Verify Challenge（チャレンジの確認）］：秘密鍵を再度コピーして貼り付けます。
    • ［Key Size（キーサイズ）］：2048を選択します。
    • ［Use as digital signature（デジタル署名として使用する）］：このオプションを選択します。
    • ［Allow export from keychain（キーチェーンからのエクスポートを許可する）］：このオプションは選択しないままにします。
    • ［Allow all apps access（すべてのアプリのアクセスを許可する）］：このオプションを選択します。

macOSデバイスでJamf Proを使用した動的SCEPチャレンジを使用する

  1. 管理証明を構成し、SCEP URLを生成する
  2. Jamf Proで動的SCEPプロファイルを作成する
  3. Okta CAがデバイスにインストールされていることを確認する

管理証明を構成し、SCEP URLを生成する

注：この手順は、Apple SCEP EMMペイロードのプッシュをサポートするすべてのEMMソリューションに適用されます。

  1. Okta Admin Consoleで、［Security（セキュリティ）］>［Device integrations（デバイス統合）］に移動します。
  2. ［Endpoint management（エンドポイント管理）］ページで、［Add platform（プラットフォームを追加）］をクリックします。

    注：同じタイプのプラットフォームに複数の構成を追加する場合は、「デバイスの既知の問題」を参照してください。

  3. ［Desktop (Windows and macOS only)（デスクトップ（WindowsおよびmacOSのみ））］を選択して、［Next（次へ）］をクリックします。
  4. ［Next（次へ）］をクリックします。
  5. ［Add Device Management Platform（デバイス管理プラットフォームを追加）］ページで、次の設定を構成します。
    • ［Certiﬁcate authority（認証局）］：［Use Okta as certiﬁcate authority（Oktaを認証局として使用する）］を選択します。
    • ［SCEP URL challenge type（SCEP URLチャレンジタイプ）］：［Static SCEP URL（静的SCEP URL）］を選択し、［Generic（汎用）］をクリックします。
    • ［Generate（生成）］をクリックします。
    • ［SCEP URL］：値をコピーして保存します。この値は後で必要になります。
    • ［Challenge URL（チャレンジURL）］：値をコピーして保存します。この値は後で必要になります。
    • ［Username（ユーザー名）］：値をコピーして保存します。この値は後で必要になります。
    • ［Password（パスワード）］：パスワードを表示するには、［Show password（パスワードを表示）］をクリックします。値をコピーして保存します。この値は後で必要になります。

      注：パスワードは安全な場所に保存してください。Okta Admin Consoleに表示されるのは今回だけです。

  6. ［Save（保存）］をクリックします。

Jamf Proで動的SCEPプロファイルを作成する

  1. Jamf Proで、［Computers（コンピューター）］>［Conﬁguration Proﬁles（構成プロファイル）］に移動します。
  2. ［+New（+新規作成）］をクリックします。
  3. ［Options（オプション）］>［General（一般）］に移動します。
  4. ［General proﬁle（一般プロファイル）］ページで、次の情報を入力します。
    • ［Name（名前）］：プロファイルの名前を入力します。
    • ［Description（説明）］：任意。プロファイルの説明を入力します。
    • ［Level（レベル）］：［User Level（ユーザーレベル）］を選択します。
  5. ［Options（オプション）］>［SCEP］に移動します。
  6. ［Conﬁgure（構成）］をクリックします。
  7. ［SCEP proﬁle（SCEPプロファイル）］ページで、次の情報を入力します。
    • ［URL］：上記のステップ5で保存したSCEP URLを入力します。
    • ［Name（名前）］：SCEPプロファイルの名前を入力します。
    • ［Subject（件名）］：件名を入力します。

      証明書がOktaへのデバイス管理シグナルとして使用されることを示す名前を選択することをお勧めします。ベストプラクティスとして、Jamf Proが提供するプロファイル変数を含めてデバイスID（UDID）を含めることもできます。サポートされている変数のリストについては、Jamf Proのドキュメント「Payload Variables for Computer Conﬁguration Proﬁles」を参照してください。

    • ［Challenge type（チャレンジタイプ）］：［Dynamic-Microsoft CA（動的-Microsoft CA）］を選択します。
      • ［URL To SCEP Admin（SCEP管理者へのURL）］：上記のステップ5で保存したチャレンジURLを入力します。
      • ［Username（ユーザー名）］：上記のステップ5で保存したユーザー名を入力します。
      • ［Password（パスワード）］：上記のステップ5で保存したパスワードを入力します。
      • ［Verify Password（パスワードの確認）］：保存したパスワードを再度入力します。
    • ［Key Size（キーサイズ）］：2048を選択し、［Use as digital signature（デジタル署名として使用する）］を選択します。
    • ［Allow export from keychain（キーチェーンからのエクスポートを許可する）］：選択しないままにします。セキュリティプラクティスとして、証明書にエクスポート不可のマークを付けることをお勧めします。
    • ［Allow all apps access（すべてのアプリのアクセスを許可する）］：このオプションを選択します。
  8. ［Save（保存）］をクリックします。
  9. プロファイルをデプロイするターゲットを構成します。
    1. ［Conﬁguration Proﬁles（構成プロファイル）］をクリックします。
    2. 適用する構成プロファイルの名前をクリックします。
    3. ［Scope（スコープ）］タブをクリックします。
    4. ［Edit（編集）］をクリックします。
    5. ［+Add（+追加）］をクリックします。
    6. 必要なデプロイ対象を見つけて、［Add（追加）］をクリックします。
  10. ［Save（保存）］をクリックします。

Okta CAがデバイスにインストールされていることを確認する

Jamf Proによって管理されるmacOSデバイスに、SCEPプロファイルがインストールされていることを確認します。

  1. ［System Preference（システム環境設定）］>［Proﬁles（プロファイル）］に移動します。
  2. 動的SCEPプロファイルがインストールされていることを確認します。
  3. ［Keychain（キーチェーン）］>［Login（ログイン）］を開きます。
  4. クライアント証明書と関連する秘密鍵が存在することを確認します。

Workspace ONEを使用して、WindowsのCAとしてOktaを構成する

  1. Oktaでx509証明書をダウンロードする
  2. Oktaで管理証明を構成し、SCEP URLと秘密鍵を生成する
  3. Workspace ONEで、静的SCEPプロファイルを作成する
  4. Workspace ONEで、証明書テンプレートを追加/編集する
  5. Workspace ONEで、Okta中間CAをデバイスの中間ストアにデプロイするためのデバイスプロファイルを定義する
  6. Workspace ONEで、Okta CAが発行したクライアント証明書を管理証明のためにデバイスの個人用ストアにデプロイするユーザープロファイルを定義する
  7. Windowsコンピューターで、証明書のインストールを確認する

Oktaでx509証明書をダウンロードする

Oktaからダウンロードするx509証明書は、組織の中間証明書です。

  1. Okta Admin Consoleで、［Security（セキュリティ）］>［Device integrations（デバイス統合）］>［Certificate authority（認証局）］に移動します。
  2. ［Okta CA］認証局の［Actions（アクション）］列で、［Download x509 certiﬁcate（x509証明書をダウンロード）］アイコンをクリックします。

この証明書は、後でWorkspace ONEにアップロードします。

Oktaで管理証明を構成し、SCEP URLと秘密鍵を生成する

  1. Okta Admin Consoleで、［Security（セキュリティ）］>［Device integrations（デバイス統合）］に移動します。
  2. ［Endpoint management（エンドポイント管理）］ページで、［Add platform（プラットフォームを追加）］をクリックします。

    注：同じタイプのプラットフォームに複数の構成を追加する場合は、「デバイスの既知の問題」を参照してください。

  3. ［Desktop (Windows and macOS only)（デスクトップ（WindowsおよびmacOSのみ））］を選択して、［Next（次へ）］をクリックします。
  4. ［Add device management platform（デバイス管理プラットフォームを追加）］ページで、次の設定を構成します。
    1. 認証局として、［Use Okta as certiﬁcate authority（Oktaを認証局として使用する）］を選択します。
    2. SCEPチャレンジタイプとして、［Static SCEP URL（静的SCEP URL）］を選択します。
    3. ［Generate（生成）］をクリックします。
    4. Okta SCEPのURLと秘密鍵をコピーして保存します。これらは、「静的SCEPプロファイルを作成する 」のフェーズでWorkspace ONEで貼り付けます。注：SCEP URLと秘密鍵は安全な場所に保存してください。Okta Admin Consoleに表示されるのは今回だけです。
  5. ［Save（保存）］をクリックします。

Workspace ONEで、静的SCEPプロファイルを作成する

  1. 管理者としてWorkspace ONEにサインインします。
  2. Workspace ONEで、［DEVICES（デバイス）］（左側のリボンバー）をクリックします。
  3. ［Certiﬁcates（証明書）］>［Certiﬁcate Authorities（認証局）］をクリックします。
  4. ［+ADD（+追加）］をクリックします。
  5. ［Certiﬁcate Authority -Add/Edit（認証局 - 追加/編集）］ページで、次の値を入力します。
    • ［Name（名前）］：CAの名前を入力します。
    • ［Description（説明）］：任意。CAの説明を入力します。
    • ［Authority type（権限のタイプ）］：［Generic SCEP（汎用SCEP）］を選択します。
    • ［SCEP Provider（SCEPプロバイダー）］：基本項目は自動的に入力され、変更できません。
    • ［SCEP URL］：上記のステップ4bで生成したSCEP URLをコピーして貼り付けます。
    • ［Challenge Type（チャレンジタイプ）］：［STATIC（静的）］をクリックします。
    • ［Static Challenge（静的チャレンジ）］：上記のステップ4dで生成した秘密鍵をコピーして貼り付けます。
    • ［Conﬁrm Challenge Phrase（チャレンジフレーズの確認）］：上記のステップ4dで生成した秘密鍵をコピーして貼り付けます。
    • ［Retry Timeout（再試行タイムアウト）］：デフォルト値の30を受け入れます。
    • ［Max Retries When Pending（保留中の最大再試行回数）］：デフォルト値の5のままにするか、権限の保留中にシステムが許可する再試行回数を別途指定します。

Windowsコンピューターへの証明書のインストールを確認する

  1. クライアント証明書のインストールを確認します。
    1. Windowsコンピューターで［Start（スタート）］をクリックし、certと入力してから、［Manage user certiﬁcates（ユーザー証明書の管理）］をクリックします。
    2. ［Personal（個人）］>［Certiﬁcates（証明書）］を確認します。
  2. 認証局を確認します。
    1. Windowsコンピューターで［Start（スタート）］をクリックし、certと入力してから、［Manage user certiﬁcates（ユーザー証明書の管理）］をクリックします。
    2. ［Intermediate Certiﬁcate Authority（中間認証局）］>［Certiﬁcates（証明書）］を確認します。
    3. ［Issued To（発行先）］で、［Organization Intermediate Authority（組織中間機関）］を探してダブルクリックします。
    4. 「発行者：組織ルート機関」を参照してください。
  3. SCEP証明書のインストールとフローが正常に行われたことを確認します。
    1. Windowsコンピューターで、［Start（スタート）］をクリックし、Eventと入力して、［Event Viewer（イベントビューアー）］をクリックします。
    2. ［Applications and Service Logs（アプリケーションとサービスログ）］>［Microsoft］>［Windows］>［DeviceManagement-Enterprise］>［Admin］を確認します。
    3. ［General（一般）］タブで、以下を見つけます。
      • SCEP: Certiﬁcate installed successfully.
      • SCEP: Certiﬁcate request generated successfully
