Integrate Jira with

The Jira integration allows organizations to create Jira tickets based on requests processed through . supports both Jira Service Management and Jira Software.

Before you begin

  • Configure [%=okta-feature-names.Access Requests%]

Start this task

  1. From the Access Requests Console, go to Settings Integrations .
  2. Under Jira, click Connect.
  3. From the approval window, click Accept.
  4. In the Connect to Jira window, configure default settings.
    1. Identify an existing Access Request team.
    2. Select a Jira project to assign to new requests.
    3. Select a Jira issue type to assign to new requests.
  5. Click Update Connection.

When you integrate Jira with , your projects within Jira are synced with . You can view the Jira projects resource in the Settings Resources tab. You can also create a configuration list for Jira. See Create a resource list.

Set up automated actions

To use automated actions, such as creating a Jira issue for a Request Type, you must make the resource available to a team using either of the following methods.

Method one

Create a configuration list from the Jira projects resource. While creating the resource list, specify teams that can access the resource list. See Create a resource list.

Method two

Assign the Jira projects resource directly to your teams by performing the following steps:

To use automated actions, such as creating a Jira issue for a Request Type, perform the following steps:

  1. From the Console, go to the Settings Resources tab.

  2. Select Jira projects.

  3. Click Manage Access.

  4. Set the toggle to on position for teams that should be able to access the resource.
