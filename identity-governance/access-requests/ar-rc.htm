Resource Catalog

The Resource Catalog feature improves requesters' experience by making the process of requesting access intuitive and user-friendly.

It integrates the End-User Dashboard with so that requesters can see the apps that they have access to and submit access requests directly from their dashboard. The apps that you made requestable are visible to your requesters in the resource catalog when they click + Add apps from their dashboard. They can see the app’s icon, the groups associated with the app, and the group description (if available). They can also search the catalog for the required app.

Requesters can select an app to view the groups they can request access to for the app. If you’ve made the app requestable, requesters also see the default app access option. After they choose an option and click Next, they may be redirected to the web app and may have to enter some information based on the request type you created for the app or group.

Use Okta groups with Resource Catalog

Use your groups to provide access to apps or specific permissions within an app from Access Requests. This feature makes it easy to make an app or group requestable:

  • You don’t need to create and maintain configuration lists for each resource, because you can use the existing relationships between the app and group in .

  • You can build simpler request types that grant access to only one Okta resource.

  • You can use the app’s metadata such as icon, group name, and group description that’s already defined in .

Limitations

Consider the following limitations for the Beta release:

  • An app or a group associated with the app is requestable only if you’ve referenced the app or group in a request type.

  • Only groups assigned to a single app can be made requestable. This means that you can’t use groups that aren’t assigned to an app or are assigned to more than one app.

  • A requestable group isn't available as an option for the app if the group isn’t assigned to an app or is assigned to more than one app.

  • Entitlement values and bundles aren't supported.

  • You must create a request type for each group to make it requestable.

  • The resource catalog may take 5–10 minutes to display a newly added resource.

  • Requesters can't submit requests using Slack, Microsoft Teams, or email after you enable the Resource Catalog feature. However, requests can still be approved or discussed using Slack, Microsoft Teams, and email in addition to the web app.

  • The existing app self-service workflow from the End-User Dashboard is unavailable if your org has the Resource Catalog feature enabled.

  • The self-service option, located on the Assignments tab of the app's profile page, doesn’t have any impact on the app’s availability on the End-User Dashboard.
