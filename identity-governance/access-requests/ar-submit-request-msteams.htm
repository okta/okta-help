Submit requests from

Users can submit requests directly fromif you have integrated with and you have one or more active Request Types. They can manage the entire lifecycle of a request through a chat. Any messages and actions within are also reflected in the Access Requests web app.

By default, all users can view requests. You can mark a submitted request as private or you can update your team settings to mark all incoming requests for the team as private.

Users can submit requests from using natural language:

  1. Open a chat that includes the bot. Alternatively, locate the app in .
  2. Enter a request using natural language.
  3. Review the response from the bot and verify that the information is correct.
  4. Click Continue.
  5. Enter the required information in the dialog.
  6. Click Submit to create the request. Any changes or updates to the request are immediately reflected in the same chat.

After users interact with the bot or app once, they automatically receive a notification when an action is assigned to them or they’re mentioned on a request.

Users can cancel a submitted request using the web app if a team member or approver hasn't taken any action on it. They receive a cancellation notification based on the notification preferences that they’ve set up.
