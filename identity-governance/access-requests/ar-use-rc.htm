Use Resource Catalog

Resource Catalog makes requesting access easy for requesters by allowing them to request resources from the End-User Dashboard. It also makes the configuring request types easy for admins.

This topic uses an example to show how the Resource Catalog feature improves efficiency and makes the request process more intuitive. Requester refers to a user who submits a request. Approver refers to a user who has tasks assigned to them in a request type. Request assignee refers to a user responsible for managing the request and who is also a member of the team that owns the request type.

Resources are limited to applications and groups for the beta release.

How work today

As an IT admin of an enterprise company that uses Okta, you’ve created groups in Okta and assigned those groups to your apps. These assignments govern the group members’ access to the apps.

Users need the ability to request access to resources like apps and groups. You need to control who can request certain resources and what approvals must be gathered.

To meet these requirements, you had to configure , and create teams, configuration lists, and request types (similar to this sample request type) that reference the configuration lists.

The configuration lists in let you recreate the relationships between apps and groups. You also create complex request types with conditional logic to meet your requirements. Requesters can submit requests to access apps and groups from Access Requests and they can also add apps from their dashboard.

After you enable the feature

Enable the Resource Catalog feature for your org to make apps and groups requestable from the dashboard. This feature improves efficiency by allowing you to use the existing relationship between the app and the group, app icon, group name, and group description that you defined in Okta.

  • The catalog may take a 5–10 minutes to display a newly added resource.
  • If you have multiple request types associated with the same option in an app, displays the first request type for the requester to enter information.
  • Admins and users are automatically assigned to the Okta Access Request Resource Catalog app.

Make a group requestable

Follow these steps when you create a request type.

  1. In the console, use the Add user to a group action to specify the group directly instead of referencing a configuration list.

  2. Enable it to run automatically.

  3. Select a group that is associated with a single app in Okta.

Requesters see this group (and its description, if available) as an option for that app. If this is the first time you’ve made a group requestable for an app, requesters see a new app with its icon in the catalog of their dashboard.

Create a request type for each group that you want to make requestable. Other groups that you make requestable using request types appear as options for the app.

Make an app requestable

Follow these steps when you create a request type.

  1. In the console, use the Assign individual app to user action to specify the app directly instead of a configuration list.

  2. Enable it to run automatically.

  3. Select an app.

Requesters can select the Default application access option in the app to request access to the app. When you first make an app or its associated groups requestable, requesters see a new app with its icon in the catalog of their dashboard.

Requesters can request an app or a group associated with the app only if the following conditions are true:

  • You’ve referenced the app or group in a request type.

  • The requester is part of the audience that you specified.
