Create a sample Request Type

Request Types are heavily dependent on the configuration and capabilities of each organization. The following process gives a simple example of how a team can structure a Request Type to assign a user to Jira. The steps are similar for other applications as well. For details, see Create an [%=okta-feature-names.Access Requests%] Request Type.

Before you begin

  • Create an [%=okta-feature-names.Access Requests%] team
  • Push [%=okta-variables.ProductName%] groups to [%=okta-feature-names.Access Requests%]

Start this task

  1. Open the Admin Console.
  2. On the left panel, click Access Requests.
  3. Click Create Request Type.
  4. From the editor window, configure the general settings.
    SettingAction
    NameEnter a name.
    Description Enter a description.
    TeamSelect the IT team.
    Audience Select a specific group allowed to submit requests using this Request Type or make it available to everyone within your org.
    Mark as done automatically Enable to close the request automatically after all questions and tasks are complete.
  5. Add questions.
    QuestionAction
    Justification
    1. Under Text, enter Why do you need access?.
    2. Under Type, select Text.
    3. Under Assigned to, select Requester.
  6. Add tasks.
    TaskAction
    Route the request to a manager for approval
    1. Under Text, enter Manager approval.
    2. Under Type, select Approval task.
    3. Under Assigned to, select Requester’s manager.

    Route the request to an IT group for approval

    1. Under Text, enter IT approval.
    2. Under Type, select Approval task.
    3. Under Assigned to, select An Okta group and select a group pushed from .
    4. Go to the Logic tab.
    5. Select Only show this task if.
    6. Select Manager approval.
    7. Select is approved.
    Assign the app to the requester
    1. Under Text, enter Assign Jira.
    2. Under Type, select Assign individual app to user.
    3. Enable Run automatically.
    4. Under Email address, select Requester email.
    5. Under Select the application, select A configuration list item.
    6. Under Applications, select Jira.
    7. Go to the Logic tab.
    8. Select Only show this task if.
    9. Select Manager approval.
    10. Select is approved.
    11. Click Add a Condition.
    12. Select IT approval.
    13. Select is approved.
  7. Click Publish.

The system creates the Request Type.
