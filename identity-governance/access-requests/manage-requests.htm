Manage requests

After a request is submitted, it is managed by a request assignee. A request assignee is usually an Access Request admin or a member of the team that owns the request type used by the request. They can update request details like request type, team, assignee, followers, and tags. They can also view and update the open tasks assigned to the approvers.

Manage requests from the Access Requests console

Request assignees can manage requests using the following steps:

  1. Request assignees go to the Okta Access Requests console.

  2. They select Inbox and go to the Open tab to view requests.

  3. They select a request and review the request details.

  4. They update the request details or open tasks that are assigned to the approvers, if required. They can’t modify tasks that the approvers have already completed.
